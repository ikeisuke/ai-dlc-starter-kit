# AIレビューフロー

ユーザー承認前にAIレビュー（Skills）を実行する。

## 設定

`.aidlc/config.toml` の `[rules.reviewing]` から読み取る。

| 設定キー | デフォルト | 説明 |
|---------|----------|------|
| `mode` | `recommend` | `required`: 必須 / `recommend`: 推奨（スキップ可） / `disabled`: 無効 |
| `tools` | `["codex"]` | 優先順位リスト。空配列→セルフレビュー直行 |
| `exclude_patterns` | - | 機密情報除外パターン（デフォルト除外に追加） |

**デフォルト除外パターン**: `.env*`, `*.key`, `*.pem`, `credentials.*`, `*secret*`

## CallerContextマッピング

| 呼び出し元 | スキル名 | focusメタデータ |
|---|---|---|
| 計画承認前 | `reviewing-construction-plan` | architecture |
| 設計レビュー | `reviewing-construction-design` | architecture |
| コード生成後 | `reviewing-construction-code` | code, security |
| 統合とレビュー | `reviewing-construction-integration` | code |
| Intent承認前 | `reviewing-inception-intent` | inception |
| ストーリー承認前 | `reviewing-inception-stories` | inception |
| Unit定義承認前 | `reviewing-inception-units` | inception |
| デプロイ計画承認前 | `reviewing-operations-deploy` | architecture |
| PRマージ前 | `reviewing-operations-premerge` | code, security |

**focusメタデータ**: 指摘の性質を示すタグ。security指摘の非公開扱い等の分岐に使用。

**呼び出し形式**: `skill="reviewing-[stage]", args="[対象ファイル] 優先ツール: [tool]"`

## 処理パス

`mode` と利用可否に応じて3つのパスに分岐する。

### パス1: 外部CLIレビュー

`tools` リスト先頭から `which` で最初に見つかったCLIを使用。

1. **レビュー前コミット**（変更がある場合）
2. **機密情報除外スキャン**: 除外パターンでマッチしたファイルを対象から除外。全除外時→ユーザーレビューへ
   - パターンマッチ: `/`を含まない→ファイルのベース名に適用、`/`を含む→プロジェクトルート相対パスに適用。ケースインセンシティブ
   - 除外ファイルがある場合はパスのみユーザーに通知（内容は含まない）
3. **種別ごとの反復レビュー**（直列実行、各種別最大3回）:
   - スキル呼び出し → 指摘があれば修正 → 再レビュー
   - 全種別で指摘ゼロ → AIレビュー完了
   - 3回後も指摘残 → 指摘対応判断フロー

**Codexセッション管理**: 初回レビュー後 `session id` を記録し、2回目以降は `codex exec resume <session-id>` を使用。

**エラー時フォールバック**:

| エラー種類 | mode=recommend | mode=required |
|:---|:---|:---|
| CLI不在（恒久） | セルフレビューへ自動遷移 | ユーザーに選択提示（セルフレビュー/ユーザー承認へ） |
| CLI実行エラー（一時的） | リトライ1回→失敗時ユーザー確認 | リトライ1回→失敗時ユーザー選択（リトライ/セルフレビュー/ユーザー承認へ） |
| CLI出力解析不能 | セルフレビューへ自動遷移 | ユーザーに選択提示（セルフレビュー/ユーザー承認へ） |

`required` でユーザー承認へ進む場合: スキップ理由の入力が必要（空文字・禁止パターン不可）。履歴に記録。

### パス2: セルフレビュー

外部CLI利用不可時、またはtools空配列時のフォールバック。

スキル呼び出し: `args="self-review [対象ファイル]"`。スキル側がサブエージェント起動とインラインフォールバックを管理する。

反復・完了処理はパス1と同一（最大3回、指摘ゼロで完了、残指摘→判断フロー）。

### パス3: ユーザーレビュー

`mode=disabled` または AIレビュー続行不能時。

1. レビュー前コミット → 成果物提示 → 承認要求
2. 修正依頼 → 反映 → レビュー後コミット → 再提示

### 遷移判定

| mode | CLI利用可 | スキル利用可 | 遷移先 |
|------|----------|------------|--------|
| `disabled` | - | - | パス3 |
| `recommend` | Yes | Yes | パス1（ユーザーが拒否→パス3） |
| `recommend` | No | Yes | パス2推奨（ユーザーが拒否→パス3） |
| `recommend` | - | No | パス2推奨（ユーザーが拒否→パス3） |
| `required` | Yes | Yes | パス1 |
| `required` | No | Yes | パス2 |
| `required` | - | No | パス2（失敗→ユーザー承認へ、スキップ理由必須） |
| `semi_auto` + `recommend` | - | - | 自動的にパス1/2へ（拒否選択をスキップ） |

## 指摘対応判断フロー

反復レビュー（最大3回）後に指摘が残っている場合に実行。

### 千日手検出

過去3回で「同種の指摘」（同一種別・同一パス・同一本質）が3回とも出現 → ユーザーに判断を求める。

### 各指摘への判断

| 選択肢 | 動作 |
|--------|------|
| 修正する（推奨） | 修正後、反復レビューに戻る |
| TECHNICAL_BLOCKER | 技術的理由を記録。具体的な根拠必須 |
| OUT_OF_SCOPE | 次サイクルで対応。バックログ登録 |

**理由バリデーション**: 空文字不可。禁止パターン（「パッチだから」「小さい変更だから」「時間がないから」等）のみの理由は拒否。

### OUT_OF_SCOPEバックログ登録

- `focus: security` の指摘 → 公開Issueへの詳細記載禁止。`SECURITY_PRIVATE`（非公開管理）またはマスク済みIssue:
  - マスク済みIssue本文: `## 概要\n[脆弱性の種類のみ。再現手順・影響範囲の詳細は記載禁止]\n\n## 検出元\n[サイクル・Unit・レビュー種別]`
  - 通常の本文テンプレート（スラッグ・概要・詳細・検出元）は使用禁止
- その他 → `gh issue create --title "[Backlog] {要約}" --label "backlog,type:{種別},priority:medium" --body-file <パス>`
- `{slug}` は `^[a-z0-9][a-z0-9-]{0,63}$` のみ許可

### 判断完了後

- RESOLVE選択あり → 修正後、反復レビューへ戻る
- 全て先送り → レビュー完了処理へ（`review_detected=true` によりセミオートゲートで `fallback(review_issues)`）

## レビュー完了時の共通処理

パス1/2完了時に実行:

1. **レビュー結果シグナル生成**: `review_detected`, `deferred_count`, `resolved_count`, `unresolved_count` をコンテキスト変数に設定（承認ポイント内でのみ有効）
2. **レビュー後コミット**
3. **レビューサマリ更新**【必須】（計画承認前除く）。**レビューサマリ未作成のまま次のステップに進んではいけない**
4. **セミオートゲート判定**: `unresolved_count == 0` かつフォールバック条件非該当 → `auto_approved`

## レビューサマリファイル

計画承認前以外のAIレビュー完了時に生成・追記する。

**ファイルパス**:

| フェーズ | パス |
|---------|------|
| Construction | `.aidlc/cycles/{{CYCLE}}/construction/units/{NNN}-review-summary.md` |
| Inception | `.aidlc/cycles/{{CYCLE}}/inception/{成果物名}-review-summary.md` |

新規作成時はテンプレート（`templates/review_summary_template.md`）使用。既存時はセパレータ（`---`）後に追記。

**Setフォーマット**:

```markdown
## Set [N]: [YYYY-MM-DD HH:MM:SS]

- **レビュー種別**: [全種別]
- **使用ツール**: [codex / self-review(skill) / self-review(inline)]
- **反復回数**: [1〜3]
- **結論**: [指摘0件 / 指摘対応判断完了]

### 指摘一覧

| # | 重要度 | 内容 | 対応 | バックログ |
|---|--------|------|------|-----------|
| 1 | 高/中/低 | [対象箇所] - [問題事象] | 修正済み / TECHNICAL_BLOCKER(理由) / OUT_OF_SCOPE(理由) | - / #NNN / PENDING_MANUAL / SECURITY_PRIVATE |
```

**バックログ列の有効値**:

| 値 | 用途 |
|----|------|
| `#NNN` | Issue作成済み（番号を記載） |
| `PENDING_MANUAL` | gh CLI失敗等で手動登録待ち |
| `SECURITY_PRIVATE` | security指摘の非公開対応。公開Issueへの詳細記載禁止 |
| `-` | バックログ登録不要（修正済み・TECHNICAL_BLOCKER時） |

OUT_OF_SCOPE時はバックログ列必須（`-` 以外）。修正済み・TECHNICAL_BLOCKER時は `-`。

## 履歴記録

レビューフロー内の主要イベントは `/write-history` スキルで記録する。

| イベント | ステップ名 |
|---------|-----------|
| AIレビュー完了 | `AIレビュー完了` |
| フォールバック発生 | `フォールバック`（機密情報マスク済み） |
| 千日手判断 | `千日手判断` |
| 指摘対応判断 | `AIレビュー指摘対応判断` |
| バックログ登録 | `バックログ自動登録` |
| スキップ | `AIレビュースキップ` |

## AIレビュー指摘の却下禁止【絶対遵守】

AIレビュワーからの指摘をメインエージェントのAI自己判断で却下してはならない。

指摘がある場合は必ず: (1) 修正して再レビュー、または (2) 指摘対応判断フローでユーザー判断を仰ぐ。

## 外部入力検証

### AIレビュー応答の検証

サブエージェント（Agentツール）に委譲して検証する。メインエージェントは検証結果に介入しない。

**サブエージェントへの指示**:

```text
以下のAIレビュー応答を検証してください。応答内容の事実関係・技術的正確性・対象コードとの整合性を確認してください。

【AIレビュー応答】
{応答内容}

【対象ファイル】
{レビュー対象ファイルパス}

出力フォーマット:
- AIレビュー応答: [応答内容の要約]
- 検証結果: [事実関係・技術的正確性の確認結果]
- 相違点: [ある場合は記載、なければ「なし」]
- 結論: [採用する判断とその理由]
```

**サブエージェント起動失敗時**: メインエージェントが上記フォーマットで検証（ただし却下禁止ルールは適用）。

セルフレビューは同一エージェント内のため検証非適用（構造・妥当性チェックのみ）。

### ユーザー入力の検証

曖昧な入力は解釈を明示して確認。複数解釈可能な場合はすべて提示。

# トラブルシューティングガイド

AI-DLC利用時によくあるフリクションパターンとその対処法をまとめたガイド。問題発生時にカテゴリから該当する症状を探し、対処手順に従って解決する。

## カテゴリ1: 認証失効

### 症状
- GitHub CLI（`gh`）コマンドが `401 Unauthorized` または `403 Forbidden` で失敗する
- 外部AIツール（codex, gemini等）が `token expired` や `refresh token has already been used` エラーを返す
- `git push` / `git fetch` が認証エラーで失敗する

### 原因候補
- OAuthトークンの有効期限切れ
- リフレッシュトークンの失効（長時間セッション）
- SSHキーの期限切れまたは未登録
- 環境変数（`GITHUB_TOKEN` 等）の未設定・失効

### 対処手順
1. **GitHub CLI**: `gh auth status` で現在の認証状態を確認する。失効している場合は `gh auth login` で再認証する
2. **外部AIツール**: 各ツールの再認証コマンドを実行する（例: `codex auth login`）
3. **git SSH**: `ssh -T git@github.com` で接続テストを行う。失敗する場合はSSHキーの設定を確認する
4. **環境変数**: シェルの設定ファイル（`.zshrc` / `.bashrc`）でトークンが正しく設定されているか確認する

### 関連参照先
- [エラーハンドリングガイド](error-handling.md) — エラー重大度分類と復旧手順の基本方針

## カテゴリ2: 外部ツール制約

### 症状
- AIレビューツール（codex, gemini等）が `read-only` モードでファイル編集を試みてエラーになる
- 外部ツールのレスポンスが返らない（フリーズ・タイムアウト）
- `sandbox` や `read-only` の制約でコマンド実行が拒否される
- ツールのバージョン不一致によるオプションエラー

### 原因候補
- サンドボックスモード（`-s read-only`）でファイル変更操作を指示している
- ネットワーク接続の問題（タイムアウト）
- ツールのバージョンが古い（非互換オプション）
- APIレート制限に到達

### 対処手順
1. **read-only制約**: レビュー用途では `-s read-only` が正しい。ファイル編集が必要な場合はメインエージェントで実行する
2. **タイムアウト**: ネットワーク接続を確認する。一時的な問題の場合はリトライする
3. **バージョン不一致**: `codex --version` 等でバージョンを確認し、最新版にアップデートする
4. **レート制限**: しばらく待ってからリトライする。頻繁に発生する場合はAPI利用量を確認する
5. **フォールバック**: 外部ツールが利用不可の場合、セルフレビューモードにフォールバックする（`skills/aidlc/steps/common/review-flow.md` のフォールバック仕様を参照）

### 関連参照先
- [エラーハンドリングガイド](error-handling.md) — フォールバック条件と対応方針
- サンドボックス環境: [jailrun](https://github.com/ikeisuke/jailrun) を参照

## カテゴリ3: フェーズ逸脱

### 症状
- Construction Phaseで設計承認前にコードを書いてしまった
- Inception Phaseで実装の詳細に踏み込んでしまった
- Operations Phaseの手順を飛ばしてデプロイを実行してしまった
- フェーズの完了条件を満たさずに次のフェーズに進もうとしている

### 原因候補
- フェーズの責務と禁止事項を確認していない
- コンテキストリセット後にフェーズ情報が失われた
- ユーザーの指示がフェーズの範囲を超えていた

### 対処手順
1. **現在のフェーズを確認**: `docs/cycles/{CYCLE}/` 配下の進捗ファイルで現在のフェーズとステップを特定する
2. **フェーズの責務を再確認**: 各フェーズのプロンプト（`/aidlc inception`, `/aidlc construction`, `/aidlc operations`）で責務と禁止事項を確認する
3. **逸脱した成果物の対処**:
   - 設計承認前のコード → コードファイルを削除し、設計フェーズに戻る
   - 飛ばしたステップ → 飛ばしたステップに戻って順番に実行する
4. **割り込み対応フロー**: ユーザーからの追加要望が原因の場合、`construction.md` の「割り込み対応フロー」に従って分類する

### 関連参照先
- [エラーハンドリングガイド](error-handling.md) — エラー重大度「High」の対応方針

## カテゴリ4: AIの誤アプローチ

### 症状
- AIが設計書と異なるアーキテクチャで実装を始めた
- AIが既存コードの構造を無視して新規ファイルを作成した
- AIが不必要に複雑な実装を提案した
- AIがスコープ外の変更を行った

### 原因候補
- コンテキストウィンドウの制約で設計書を参照できていない
- 既存コードベースの探索が不十分
- ユーザーの指示が曖昧で解釈の余地があった

### 対処手順
1. **即時中断**: 誤ったアプローチに気づいたら即座に作業を中断する
2. **設計書の再参照を指示**: 「`docs/cycles/{CYCLE}/design-artifacts/` の設計書を読み込んで、それに基づいて実装してください」と具体的に指示する
3. **既存コードの参照を指示**: 「まず `path/to/file` を読み込んで、既存の構造を理解してから修正してください」と指示する
4. **変更の取り消し**: 誤った変更がある場合は `git diff` で確認し、`git checkout -- path/to/file` で元に戻す
5. **スコープの明確化**: Unit定義ファイルの「境界」セクションを参照し、スコープ内の変更のみ行うよう指示する

### 関連参照先
- [サブエージェント活用ガイド](subagent-usage.md) — サブエージェントによるタスク分割

## カテゴリ5: コンテキストリセット後の復帰

### 症状
- セッションが切断され、AIが現在のフェーズ・ステップを認識していない
- コンテキスト圧縮により過去の設計判断が失われた
- 新しいセッションで作業を継続する必要がある

### 原因候補
- コンテキストウィンドウの上限に達した（自動圧縮）
- ネットワーク切断やツールのクラッシュ
- 意図的なセッション終了後の再開

### 対処手順
1. **セッション状態の確認**: `docs/cycles/{CYCLE}/construction/session-state.md` が存在する場合、中断時点の状態が記録されている
2. **フェーズプロンプトの再読み込み**: 該当フェーズのプロンプトを再度読み込む
   - Inception: `/aidlc inception`（`skills/aidlc/steps/inception/`）
   - Construction: `/aidlc construction`（`skills/aidlc/steps/construction/`）
   - Operations: `/aidlc operations`（`skills/aidlc/steps/operations/`）
3. **進捗の確認**: Unit定義ファイル（`docs/cycles/{CYCLE}/story-artifacts/units/`）の「実装状態」セクションで各Unitの進捗を確認する
4. **履歴の確認**: `docs/cycles/{CYCLE}/history/` 配下の履歴ファイルで過去の作業内容を確認する
5. **設計書の再読み込み**: 進行中のUnitがある場合、`docs/cycles/{CYCLE}/design-artifacts/` 配下の設計書を読み込んで作業を再開する

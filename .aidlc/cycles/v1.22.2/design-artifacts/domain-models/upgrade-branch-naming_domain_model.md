# ドメインモデル: アップグレード用ブランチ名の改善

## 概要

`upgrade/vX.X.X` プレフィックスをブランチ命名規則に追加し、`post-merge-sync.sh`の削除対象パターンを拡張する。

**重要**: このドキュメントでは**コードは書かず**、構造と責務の定義のみを行います。

## 関数責務表

### post-merge-sync.sh

| 項目 | 内容 |
|------|------|
| **現状の責務** | マージ済み`cycle/*`ブランチの検出・削除（ローカル+リモート） |
| **変更後の責務** | マージ済み`cycle/*`と`upgrade/*`ブランチの検出・削除（ローカル+リモート） |
| **変更箇所** | ブランチ列挙パターン（ローカル・リモート各1箇所）、ヘルプ、メッセージ |

## 影響範囲

| コンポーネント | 影響 | 変更要否 |
|-------------|------|---------|
| `bin/post-merge-sync.sh` | パターン拡張 | 必要 |
| `prompts/setup-prompt.md` | ブランチ命名案内追加 | 必要 |
| `docs/cycles/rules.md` | 運用フロー記載更新 | 必要 |
| `prompts/package/skills/aidlc-setup/SKILL.md` | 既に`upgrade/vX.X.X`使用 | 不要（確認のみ） |
| `prompts/package/prompts/AGENTS.md` | サイクル判定は`cycle/vX.X.X`のみ | 不要（upgradeブランチはサイクル判定対象外、意図的設計） |

## 設計前提

- **マージ先ブランチ**: アップグレードブランチのマージ先は常に`main`（またはサイクルブランチの元ブランチ）。`post-merge-sync.sh`は`--merged main`を安全フィルタとして使用し、未マージブランチは削除されない。
- **サイクル判定との関係**: AGENTS.mdのサイクル判定（`cycle/vX.X.X`パターン）は変更しない。`upgrade/vX.X.X`ブランチはサイクル判定対象外であり、これは意図的な設計（アップグレードブランチでサイクルを開始しない）。

## ユビキタス言語

- **サイクルブランチ**: `cycle/vX.X.X` 形式のブランチ。開発サイクル用。
- **アップグレードブランチ**: `upgrade/vX.X.X` 形式のブランチ。AI-DLCバージョンアップ用。
- **マージ済みブランチ**: mainにマージ完了したブランチ（cycle + upgrade両方含む）。

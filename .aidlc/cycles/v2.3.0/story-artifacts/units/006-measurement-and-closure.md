# Unit: 削減目標達成の計測レポートと #519 クローズ判断

## 概要

全 Unit（001-005）の実装完了後に、各フェーズの初回ロード tok 数を実測してレポート化し、#519 コンテキスト圧縮プロジェクトのクローズ判断を行う。

## 含まれるユーザーストーリー

- ストーリー8: 削減目標達成の計測レポート
- ストーリー9: #519 クローズ判断と未達時運営タスク

## 責務

### 計測レポート作成

- v2.2.3 ベースライン値を同一計測コマンドで再計測し、記録する（Inception / Construction / Operations の3フェーズ）
- v2.3.0 実装後の各フェーズ初回ロードを tiktoken (cl100k_base) で計測する
- 差分（tok 数、削減率 %）を算出する
- 計測に使用したファイルリスト（インデックス化後の初回ロードに含まれるファイル）を記録する
- `.aidlc/cycles/v2.3.0/measurement-report.md` として成果物化する
- 目標達成判定: Inception 15,000 tok 以下、Construction / Operations 現状維持を明示する
- ステップファイル内 boilerplate 削減の達成状況（Intent で「自動解消扱い」としたもの）を併記する

### #519 クローズ判断と運営タスク

- 計測レポートを元に #519 Issue にクローズ判断コメントを追加する
- **達成時**: #519 のステータスラベルを `status:done` に更新し、Issue をクローズする
- **未達時**: 未達項目（目標 tok 数未達、未適用の施策、異常系未対応など）をバックログ Issue として登録する（`backlog` + 適切な type ラベル）
- 本サイクルで実装された主要変更点（案D、#553 解決、Tier 2 施策）を `CHANGELOG.md` に記載する

## 境界

- **含まない**: インデックス化自体の実装（Unit 001-004）
- **含まない**: Tier 2 施策の実装（Unit 005）
- **含まない**: MCP / ツールベース検索の設計・実装（Intent で次サイクル切り出し済み）
- **含まない**: Operations Phase の通常リリース作業（ブランチマージ、タグ作成等は Operations Phase で対応）

## 非機能要件（NFR）

- **正確性**: 計測値は同一コマンド・同一環境で再現可能
- **可視性**: 達成状況が定量的に把握できる形式でレポート化
- **ガバナンス**: クローズ判断は計測データに基づき、主観を排除する

## 技術的考慮事項

- **計測コマンドの統一**: ユーザーストーリーの「検証用サンプルの定義」に記載された計測コマンドを使用する
- **ベースライン再計測**: v2.2.3 のタグ（コミット `d88b0074`）をチェックアウトせず、本サイクルブランチ上で参照するだけにする（git show 等）
- **ファイルリストの決定方法**: インデックス化後の初回ロードに含まれるファイルは Unit 001-004 の実装結果から抽出する
- **未達時の扱い**: 目標未達でも本 Unit は完了とし、未達項目をバックログ化することでサイクルを閉じる（#519 のクローズは達成時のみ）

# Unit: Inception フェーズインデックスのパイロット実装

## 概要

案D（インデックス集約型プログレッシブロード）の基盤を Inception Phase で先行実装する。全フェーズ共通のインデックスファイル構造・集約ルール・参照方式を本 Unit で確立し、以降の Unit の土台を作る。

## 含まれるユーザーストーリー

- ストーリー1: Inception フェーズインデックスによる初回ロード削減

## 責務

- `steps/inception/` 配下にフェーズインデックスファイル（`index.md`）を新規作成する
- フェーズインデックスファイルの構造仕様（セクション構成、見出しレベル、必須項目）を確立する
- Inception Phase の全ステップ（01-setup.md 〜 05-completion.md）の目次・概要・分岐ロジック・現在位置判定チェックポイントをインデックスに集約する
- 既存の Inception ステップファイルから、インデックスに集約された分岐・判定の重複記述を削除し、詳細手順に特化させる
- `SKILL.md` の共通初期化フロー（ステップ4: フェーズステップ読み込み）を更新し、Inception Phase 開始時に読み込む対象をインデックスファイルのみにする
- Inception 初回ロードが 15,000 tok 以下であることを実測で確認する
- 既存 Inception フローの回帰を検証サンプル A で確認する

## 境界

- **含まない**: Construction / Operations フェーズのインデックス化（Unit 003 / 004 で対応）
- **含まない**: 汎用復帰判定基盤の実装（Unit 002 で対応、本 Unit ではインデックスファイルに「判定チェックポイントが置ける領域」を用意するのみ）
- **含まない**: Tier 2 施策（Unit 005 で対応）
- **含まない**: `compaction.md` の現在位置判定テーブル廃止（Unit 002 で対応）

## 非機能要件（NFR）

- **パフォーマンス**: Inception 初回ロード 15,000 tok 以下（tiktoken 計測）
- **セキュリティ**: 該当なし（ドキュメント変更のみ）
- **スケーラビリティ**: 他フェーズにも展開可能な汎用構造を採用する
- **可用性**: 既存 Inception フローの動作を破壊しない（検証サンプル A で確認）

## 技術的考慮事項

- **インデックスファイルの構造設計**: 他フェーズに展開することを前提に、フェーズ名を除いた共通セクション構成を採用する（例: 「目次」「分岐ロジック」「現在位置判定チェックポイント」）
- **不変ルールとの整合**: `SKILL.md` の「ステップファイルの読み込みは省略不可」ルールに抵触しないよう、インデックスファイルを「フェーズステップ読み込み」の対象として再定義する
- **Inception ステップファイルの分離**: 既存の `01-setup.md` 〜 `05-completion.md` は「詳細手順」として残し、インデックスから必要時に参照する形にする
- **複雑度判定（エクスプレス）**: 本 Unit は SKILL.md/ステップファイル編集 + tok 計測 + 回帰検証を含むため、見積もりは中規模。エクスプレス適格性は不適格（要慎重実装）

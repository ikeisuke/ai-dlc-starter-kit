# Unit: ステップファイル内スクリプトパスのプラグインモデル対応

## 概要
ステップファイル内のスクリプト参照パス `skills/aidlc/scripts/` をプラグインモデルで動作するよう修正する。

## 含まれるユーザーストーリー
- ストーリー 9: スクリプトパス解決のプラグインモデル対応

## 責務
- SKILL.md にスクリプトパス解決ルールを追加（`scripts/` パスを `steps/` と同様にスキルベースディレクトリから解決）
- 全ステップファイルの `skills/aidlc/scripts/` を `scripts/` に一括置換
- squash-unit スキルの `skills/aidlc/scripts/` 参照も同様に更新

## 境界
- スクリプト自体の内容変更は行わない
- スクリプト間の相互参照（bootstrap.sh内等）は対象外

## 技術的考慮事項
- Claude Codeのスキルシステムではベースディレクトリが `.claude/skills/aidlc` と報告されるが、実際のファイルは `skills/aidlc/` にある（既知のバグ）
- AIがステップファイル読み込み時に解決したパスを基にスクリプトの絶対パスを構築する必要がある

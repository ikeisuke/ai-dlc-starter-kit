# Unit 002: SKILL.md参照混同修正 - 計画

## 概要

メタ開発時（`STARTER_KIT_DEV`）に、AIエージェントがプラグインキャッシュのSKILL.mdとリポジトリの`skills/aidlc/SKILL.md`を混同する問題を修正する。Inceptionセットアップの「デプロイ済みファイル確認」ロジックで、環境判定との依存順序を整理し、メタ開発環境の参照先ポリシーを明確化する。

## 変更対象ファイル

- `skills/aidlc/steps/inception/01-setup.md` - ステップ3-4の責務整理と参照先ポリシー定義

## 実装計画

### 問題分析

- ステップ3「デプロイ済みファイル確認」で `skills/aidlc/SKILL.md` の存在を確認している
- メタ開発リポジトリ（`STARTER_KIT_DEV`）では、リポジトリ内に `skills/aidlc/SKILL.md` が常に存在するため、プラグインがインストールされていなくても「存在する」と判定される
- ステップ4でリポジトリ判定（`STARTER_KIT_DEV` / `USER_PROJECT`）が行われるが、ステップ3はその前に実行される
- 環境判定に依存するロジックが環境判定前のステップに存在し、責務分離が不適切

### 修正方針

ステップ3を環境非依存の存在確認のみに限定し、環境依存の参照先ポリシー定義をステップ4（環境判定）以降に移す:

1. **ステップ3（存在確認のみ）**: `skills/aidlc/SKILL.md` の存在を確認する責務のみ。環境依存の判断は行わない
2. **ステップ4（環境判定）に参照先ポリシーを追加**: `STARTER_KIT_DEV` / `USER_PROJECT` 判定後に、環境ごとの参照先ポリシーを定義する
   - `STARTER_KIT_DEV`: ステップ3の存在確認はプラグインのインストール状態を保証しない。スキル内リソースの編集対象はリポジトリ内の `skills/aidlc/`。`~/.claude/plugins/` 配下は編集候補・更新対象・差分比較対象から除外
   - `USER_PROJECT`: 従来通り。ステップ3の存在確認がプラグインデプロイの有無を示す

- [x] `USER_PROJECT` の既存動作が不変である
- [x] `~/.claude/plugins/` を編集候補・更新対象・差分比較対象にしないことが明記される

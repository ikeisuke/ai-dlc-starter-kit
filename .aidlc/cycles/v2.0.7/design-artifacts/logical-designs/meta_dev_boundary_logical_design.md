# 論理設計: メタ開発境界ルール策定

## 概要

メタ開発時のファイル参照境界ルールの記述構造、配置場所、およ��既存ファイルの監査・修正手順を設計する。

**重要**: この論理設計では**コードは書かず**、構造とインターフェース定義のみを行います。

## アーキテクチャパターン

**ポリシーパターン**: ルール定義（ポリシー）��適用（監査・修正）を分離。ルール定義は `.aidlc/rules.md` に集約し、適用は手動監査で実施する。

## コンポーネント構成

```text
.aidlc/rules.md
└── メタ開発の意識セクション
    ├── 既存ルール（変更なし）
    └── 【新規追加】ファイル参照境界ルール
        ├── 許可パス一覧（操作種別付き）
        ├── デフォルト禁止（代表的な違反例）
        └── 例外リスト（操作種別・適用範囲付き）
```

### コンポーネント詳細

#### ファイル参照境界ルールセクション（`.aidlc/rules.md` 内の新規セクション）

- **責務**: AIエージェントが参照・作成してよいパスの判定基準を提供
- **配置**: 「メタ開発の意識」セクションの直後（同じ `---` 区切り内）
- **対象読者**: AIエージェント（プロンプトとして読み込まれる）

#### 許可パス一覧

ホワイトリスト形式。以下のカテゴリで構成:

| カテゴリ | パスパターン | アクセス方法 | 許可操作 | 説明 |
|---------|-------------|-------------|---------|------|
| AIDLC設定 | `.aidlc/config.toml`, `.aidlc/config.local.toml` | 直接参照 | read | プロジェクト設定（編集はsetupスキル経由） |
| サイクルデータ | `.aidlc/cycles/{{CYCLE}}/**` | 直接参照 | read, write, create | 現サイクルの成果物・履歴 |
| AIDLC追加ルール | `.aidlc/rules.md` | 直接参照 | read, write | プロジェクト固有ルール |
| スキル内リソース | `templates/**`, `scripts/**`, `steps/**` | スキルベースディレクトリ相対 | read, execute | SKILL.mdのパス解決ルールに従う |
| エージェント設定 | `CLAUDE.md`, `AGENTS.md` | 直接参照 | read | エージェント設定ファイル |

#### デフォルト禁止（代表的な違反例）

許可パス一覧・例外リストに該当しないパスは全て禁止。代表的な違反例:

| パスパターン | 禁止理由 |
|-------------|---------|
| `skills/**`（絶対パスまたはプロジェクトルート相対パス） | メタ開発外プロジェクトには存在しない。スキル内リソースはスキルベースディレクトリ相対パスで参照すべき |
| `prompts/**` | スターターキット本体。直接参照は非メタ開発プロジェクトで破綻する（例外あり） |
| `docs/aidlc/**`（直接編集） | rsyncコピー先。Operations Phaseのrsyncで上書きされる |
| 任意のプロジェクトファイルの直接作成 | AIDLC管理外のファイル��AIエージェントが勝手に作成すべきでない |

#### 例外リスト

| exception_id | 対象パス | 許可操作 | 理由 | 適用範囲 | 注意事項 |
|-------------|---------|---------|------|---------|---------|
| META-001 | `prompts/package/**` | read, write, create | メタ開発時のスターターキット本体参照（rsync元） | メタ開発リポジトリのみ | `docs/aidlc/` ではなく必ずこちらを編集 |
| META-002 | `docs/aidlc/**` | read | rsyncコピー先の参照（読み取りのみ許可） | メタ開発リポジトリのみ | 直接編集（write/create）は禁止 |
| META-003 | `bin/**` | execute | プロジェクト固有スクリプト | カスタムワークフロー実行時 | `.aidlc/rules.md` で明示的に指定されたコマンドのみ |

## 処理フロー概要

### 監査フロー

1. **対象ファイル収集**: `skills/aidlc/steps/` 配下の全ファイル（30ファイル）+ `SKILL.md` を列挙
2. **パス参照抽出**: 各ファイルからファイルパスの参照を検索（パターン: バッククォート内のパス、コードブロック内のパス）
3. **ルール照合**: 抽出した各パス参照を許可/禁止/例外ルールに照合
4. **違反レポート**: 禁止パスに該当する参照を一覧化（ファイル名、行番号、現在の参照、推奨修正）
5. **修正**: 違反箇所を修正（主に「スキルベースディレクトリ相対パス」へ��変換）

### 修正パターン

| 違反パターン | 修正方法 |
|-------------|---------|
| `skills/aidlc/templates/xxx.md` → | `templates/xxx.md`（スキルベースディレクトリ相対） |
| `skills/aidlc/scripts/xxx.sh` → | `scripts/xxx.sh`（スキルベースディレクトリ相対） |
| `skills/aidlc/steps/xxx.md` → | `steps/xxx.md`（スキルベースディレクトリ相対） |
| プロジェクトファイルへの直接参照 → | 適切な間接参照に変換、または不要なら削除 |

## Unit 004との境界インターフェース

### 本Unit（003）が提供するもの

- `.aidlc/rules.md` のファイル参照境界ルールセクション（セクション名・構造が確定）
- 監査済み・修正済みのステップファイル群

### Unit 004が前提とするもの

- 本Unitで追加されたルールセクションの存在
- ステップファイルのパス参照が修正済みであること
- Unit 004の責務は「フェーズプロンプトからrules.mdへの参照整備」であり、rules.md自体のルール追加は本Unitで完了

## 実装上の注意事項

- ルール記述はAIエージェントが読み込むプロンプトとして機能するため、明確かつ簡潔に記述する
- 既存の「メタ開発の意識」セクションの内容と矛盾しないよう整合性を確保する
- SKILL.mdの「パス解決」ルール（`steps/` および `scripts/` はスキルベースディレクトリからの相対パス）との整合性を維持する

# 論理設計: ブランチ切り替え後のrules.md再読み込み

## 変更方針

### inception.md への追加

ステップ11の末尾（11-4の後、ステップ12の前）に「11-5. rules.md再読み込み」を追加。

### 追加するコンテンツ

```markdown
#### 11-5. rules.md再読み込み【ブランチ切り替え時】

ステップ11でブランチ切り替えが発生した場合（setup-branch.sh実行 または git checkout実行）、
切り替え先ブランチの `docs/cycles/rules.md` を再読み込みする。

**条件**: 上記のブランチ切り替え経路を通った場合のみ実行
**スキップ**: サイクルブランチに既にいた場合、または「現在のブランチで続行」を選択した場合

**手順**:
1. `docs/cycles/rules.md` の存在を確認
2. 存在する場合: ファイルを読み込み、ステップ5で読み込んだ内容を更新
3. 存在しない場合: スキップ（通知なし）
```

## 設計判断

- ステップ5の既存ロジックは変更しない（境界条件）
- 再読み込みは「上書き」方式（差分マージではない）
- ファイル不存在時は無言でスキップ（新規サイクルの場合、rules.mdがまだ存在しない可能性がある）

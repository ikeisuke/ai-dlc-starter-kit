# ドメインモデル: ブランチ切り替え後のrules.md再読み込み

## 概要

inception.mdのステップ11でブランチ切り替えが発生した場合、rules.mdの内容が変わる可能性があるため再読み込みが必要。

## ドメイン構造

### 対象エンティティ

- `rules.md`: プロジェクト固有の追加ルール（`docs/cycles/rules.md`）
- `inception.md` ステップ11: ブランチ確認・切り替えステップ

### 判定ロジック

ブランチ切り替えが発生する経路:
1. setup-branch.shの実行（main/masterからのブランチ作成 or worktree作成）
2. git checkoutによる既存サイクルブランチへの切り替え

ブランチ切り替えが発生しない経路:
1. 既にサイクルブランチにいる場合（次のステップへ直行）
2. 「現在のブランチで続行する」を選択した場合

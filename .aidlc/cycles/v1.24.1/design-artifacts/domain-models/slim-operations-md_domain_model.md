# ドメインモデル: operations.mdのスリム化

## 概要

運用引き継ぎ情報ファイル（operations.md）とそのテンプレートの構造を整理し、冗長な情報を削減する。

## ドメイン構造

### operations.md の責務

運用引き継ぎのための**運用固有情報**の保持が唯一の責務。

**必須セクション**（削除不可）:

| セクション | 責務 |
|-----------|------|
| デプロイ方針 | リリース手順・バージョニング・ロールバック・CI/CDフロー要点 |
| 既知の問題・注意点 | 運用上の既知問題とワークアラウンド |
| メタ開発特有のOperations Phase手順 | AI-DLCスターターキット固有の追加手順（プロジェクト拡張） |

**削除対象セクション**（他ソースで取得可能）:

| セクション | 代替ソース |
|-----------|-----------|
| 目的・使い方 | operations.mdプロンプト |
| プロジェクト概要 | aidlc.toml、README |
| CI/CD設定方針（詳細） | .github/workflows/ |
| 監視設定 | 不要（「なし」） |
| アップデート方法 | setup-prompt.md |
| 更新履歴 | git log |

### テンプレートの責務

新規プロジェクトへの初期スキャフォールド提供。

**共通必須セクション**（テンプレート・実ファイル共通）: デプロイ方針、既知の問題・注意点

**プロジェクト拡張セクション**（実ファイルのみ、テンプレートには含めない）: CI/CDフロー要点、メタ開発特有のOperations Phase手順

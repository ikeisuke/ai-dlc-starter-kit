# Unit: マージ後worktree同期自動化

## 概要
PRマージ後の親リポジトリのmain pull、worktreeのdetached HEAD化、マージ済みサイクルブランチ削除を自動化するスクリプトを作成する。

## 含まれるユーザーストーリー
- ストーリー 7: マージ後のworktree同期自動化 (#211)

## 関連Issue
- #211

## 責務
- `bin/post-merge-sync.sh` の新規作成
- 親リポジトリで `git pull origin main` を実行
- worktreeをdetached HEAD状態にする
- マージ済みの `cycle/` プレフィックス付きブランチをローカル・リモートから削除
- `--dry-run` オプションで実際の操作を行わず実行予定を表示する機能
- `docs/cycles/rules.md` の運用手順を更新

## 境界
- 共通プロンプト（`prompts/package/`）には含めない（リポジトリ固有）
- worktreeの作成・削除機能は含まない
- `cycle/` プレフィックス以外のブランチは操作対象外

## 非機能要件（NFR）
- **パフォーマンス**: 該当なし

- **スケーラビリティ**: 該当なし
- **可用性**: 該当なし

## 技術的考慮事項
- `git worktree list` で親リポジトリのパスを自動検出
- `--yes` オプションでリモートブランチ削除の確認をスキップ可能
- 各処理の失敗時にエラーメッセージと従来の手動手順を表示し中断
- `git branch -d`（マージ済み確認付き）でローカル削除、`git push origin --delete` でリモート削除

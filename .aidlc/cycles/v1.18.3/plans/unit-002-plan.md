# Unit 002 計画: CI Bashコードブロック コマンド置換検出

## 概要

プロンプト内のBashコードブロックで `$()` およびバッククォートによるコマンド置換が使用されていないことをCIで自動検出する。

## 変更対象ファイル

- `bin/check-bash-substitution.sh`（新規作成）
- `.github/workflows/pr-check.yml`（ジョブ追加）

## 実装計画

### ファイル1: `bin/check-bash-substitution.sh`（新規作成）

`bin/check-size.sh` のパターンに倣い、以下の構成で作成:

- **検出対象**: `prompts/package/prompts/**/*.md` 内の ` ```bash ` ～ ` ``` ` ブロック
- **検出パターン**: `$()` および `` ` `` （バッククォート）
- **出力**: 違反検出時にファイル名と行番号を報告
- **終了コード**: 0=合格, 1=違反検出, 2=スクリプトエラー

#### 検出ロジック

1. `find` で対象ファイルを列挙
2. `awk` で状態機械的にBashコードブロックの開始/終了を追跡
3. ブロック内で `$()` またはバッククォートを検出
4. フェンス判定: 先頭0-3スペース + 3つ以上のバッククォートまたはチルダ + `bash`（開始）、同数のフェンス文字（終了）

#### バッククォート検出の注意点

- Bashコードブロック内の内容はMarkdownとして解釈されないため、ブロック内のバッククォートは全てコマンド置換として扱う
- フェンス自体のバッククォート（` ``` `）はフェンス判定で除外済み

### ファイル2: `.github/workflows/pr-check.yml`（ジョブ追加）

- `bash-substitution-check` ジョブを追加
- `ubuntu-latest` で `bin/check-bash-substitution.sh` を実行

# ドメインモデル: validate_cycle() Git ref安全性修正

## 概要

validate_cycle()関数にGit ref安全性チェックを追加し、ブランチ名として無効なサイクル名を拒否する。

**重要**: このドメインモデル設計では**コードは書かず**、構造と責務の定義のみを行います。

## エンティティ（Entity）

### CycleValidator（サイクル名バリデータ）

- **属性**: validate_cycle()関数（既存）
- **振る舞い**:
  - 既存チェック: 空文字、パストラバーサル、空白、制御文字、先頭スラッシュ、正規表現
  - 新規追加チェック:
    - Git ref安全性: `git check-ref-format --branch "cycle/${cycle}"` で検証
    - `.lock`接尾辞: セグメント末尾が`.lock`で終わる場合に拒否
    - 末尾ドット: セグメント末尾がドットで終わる場合に拒否

## 値オブジェクト（Value Object）

### ValidationRule（バリデーションルール）

- **既存ルール**: 空文字/パストラバーサル/空白/制御文字/先頭スラッシュ/正規表現
- **新規ルール**:
  - `git_ref_format` - Git refフォーマット準拠チェック
  - `lock_suffix` - .lock接尾辞拒否
  - `trailing_dot` - 末尾ドット拒否

## チェック順序の設計

```text
1. 空文字チェック（既存）
2. パストラバーサル防止（既存）
3. 空白チェック（既存）
4. 制御文字チェック（既存）
5. 先頭スラッシュ防止（既存）
6. 形式チェック（正規表現、既存）
7. ★ 末尾ドットチェック（新規）
8. ★ .lock接尾辞チェック（新規）
9. ★ git check-ref-format チェック（新規、最終防衛線）
```

**注意**: git check-ref-formatを最後に配置する理由:
- 軽量なパターンマッチを先に実行し、明確なエラーメッセージを返す
- git check-ref-formatは包括的だが、エラー原因の特定が困難なため最終チェックとする

## ユビキタス言語

- **Git ref**: Gitの参照名（ブランチ名・タグ名等）
- **末尾ドット**: セグメントの最後がドット文字（`.`）で終わるパターン
- **.lock接尾辞**: Gitの予約パターンで、ロックファイルとの競合を防ぐ

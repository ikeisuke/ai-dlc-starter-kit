# Intent（開発意図）

## プロジェクト名
AI-DLC Starter Kit

## 開発の目的
AI-DLCフレームワークの堅牢性と開発体験を改善する。具体的には、Claude Code Insightsレポートで検出されたフリクション（操作順序の誤り、テスト失敗時の手動介入、外部ツール制約の未文書化）への対応、validate_cycle()のGit ref安全性修正、および古いバージョンからのアップグレードパスの堅牢化を実施する。

## ターゲットユーザー
AI-DLCスターターキットを使用して開発を行うAIエージェントおよび開発者

## ビジネス価値
- AIエージェントのフリクション削減による開発サイクルの高速化
- テスト失敗時の自動修正によるユーザー介入の削減
- 古いバージョンからのアップグレードが確実に成功することで、ユーザーの離脱を防止
- 外部ツール制約の明文化によるトラブルシューティング時間の短縮

## 成功基準
- commit-flow.mdに操作順序ルールセクションが追加され、禁止される操作順序（例: コミット前のPR作成、スカッシュ前のUnit完了マーク）が明文化される。AIエージェントが順序違反を検知した場合、正しい手順を提示する
- Construction Phase Step 6でテスト失敗時に自動修正ループが最大3回実行される（成功時は即終了、3回失敗後はユーザーに判断を求める）
- review-flow.mdに外部レビューツールの既知制約と対処法が記載される
- validate_cycle()がGit refとして無効なサイクル名（末尾ドット、.lock接尾辞）を拒否する
- setup-prompt.mdおよびaidlc-setup.shが、古いバージョンの`docs/aidlc/bin/`にスクリプトが存在しない環境でもエラーなく動作する（完了の定義: アップグレード実行後に`docs/aidlc/`配下のファイル同期が正常終了し、`docs/aidlc.toml`のバージョンが更新されること）

## 期限とマイルストーン
特になし

## 制約事項
- プロンプト・テンプレートの編集は `prompts/package/` で行うこと（`docs/aidlc/` は直接編集禁止）
- `$()` およびバッククォート（`` ` ``）によるコマンド置換はBashコマンド実行で使用しない（`.sh`スクリプト内部は例外）
- 後方互換性を維持すること（古いバージョンからのアップグレードを壊さない）
- validate_cycle()への拒否条件追加は、既存の正当なサイクル名（末尾ドットや`.lock`を含まない名前）に影響しないことを確認する

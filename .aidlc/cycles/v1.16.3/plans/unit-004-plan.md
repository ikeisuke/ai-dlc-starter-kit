# Unit 004 計画: セルフレビューフォールバック

## 概要

外部AIレビューツール（Skills）が利用できない場合に、実行中のエージェント自身がセルフレビューを実施するフォールバック機能を `review-flow.md` に追加する。

## 変更対象ファイル

- `prompts/package/prompts/common/review-flow.md` - セルフレビューフォールバックの追加

## 実装計画

### 遷移モデル

既存フローにセルフレビューを組み込む遷移:

```text
ステップ3（利用可否チェック）
  ├─ 全種別が利用可能 → ステップ4 → ステップ5（AIレビューフロー）
  ├─ 一部のみ利用可能 → （既存ロジック: 利用可能な種別のみでステップ4へ）
  └─ すべて利用不可 → ステップ6（AIレビュー不可時）【変更箇所】
                          ├─ セルフレビューを実施 → ステップ5.5（セルフレビューフロー）【新設】
                          ├─ 人間承認へ進む（既存）
                          └─ 処理を中断（既存）
```

**設計判断**: セルフレビューの起動はステップ6（AIレビュー不可時）に集約する。ステップ3の遷移先は変更しない（シンプルさの維持）。

### 1. ステップ6（AIレビュー不可時）にセルフレビュー選択肢を追加

現在のステップ6は「人間レビューへ進む / 処理を中断」の2択。ここにセルフレビューの選択肢を追加する。

**modeごとの動作**:
- `mode = "required"`: セルフレビューを最優先選択肢として提示。セルフレビューは「必須AIレビュー充足」として扱う（根拠: エージェント自身による構造的レビューであり、品質は外部ツールに劣るが無レビューより改善されるため）。履歴に「セルフレビューによる充足」を明記する。
- `mode = "recommend"`: セルフレビューを推奨選択肢として提示

### 2. セルフレビューフローの新設（ステップ5.5）

ステップ6からの遷移先として「セルフレビューフロー」セクションを新設する。

**実行手順**:
1. レビュー前コミット（`commit-flow.md` に従う。既存ステップ5と同じ扱い）
2. 対象の成果物を自ら読み込み、レビュー観点に基づいてレビューを実施
3. 指摘を出力フォーマットに従って記録
4. 指摘がある場合は修正を反映
5. 反復レビュー（最大3回、既存ステップ5と同じルール）
6. 3回後も指摘が残る場合: 既存の「指摘対応判断フロー」へ遷移（ステップ5と同じ判断フローを適用）
7. レビュー後コミット（`commit-flow.md` に従う）
8. 修正後の成果物を人間に提示し承認を求める

**レビュー観点**: 呼び出し元のレビュー種別（code / architecture / security / inception）に応じた観点をそのまま適用する。各種別の観点はレビュースキルのSKILL.mdに定義されている内容を参照する。

**出力フォーマット**（既存の指摘対応判断フローに投入可能な形式）:

```text
【セルフレビュー結果】
レビュー種別: {code | architecture | security | inception}

指摘 #1
- 重要度: {高 | 中 | 低}
- 内容: {指摘内容の要約}
- 推奨修正: {修正方法の提案}
- 判定: {未対応 | 対応済}

指摘 #2
...

合計: {N}件（高: {n}件 / 中: {n}件 / 低: {n}件）
```

指摘がゼロの場合:

```text
【セルフレビュー結果】指摘0件
レビュー種別: {種別}
```

**注意**: セルフレビューは同一エージェントが実施するため、「外部入力検証ルール」の「AIレビュー応答の検証」は適用しない（自己の判断を自ら検証することは無意味であるため）。

### 3. 履歴記録フォーマットの定義

セルフレビューであることを明記するフォーマットを定義する。

**変更内容**:
- レビューツールに「self-review」を記録
- セルフレビュー固有の注記（「外部ツールに比べて品質が劣る可能性があるため、重要な変更では外部レビューを推奨」）を履歴に含める

**履歴記録テンプレート**:

```bash
docs/aidlc/bin/write-history.sh \
    --cycle {{CYCLE}} \
    --phase {{PHASE}} \
    --unit {N} \
    --unit-name "[Unit名]" \
    --unit-slug "[unit-slug]" \
    --step "AIレビュー完了" \
    --content "【AIレビュー完了（セルフレビュー）】指摘0件
【対象タイミング】{呼び出し元のステップ名}
【対象成果物】{成果物名}
【レビュー種別】{実行した全種別}
【レビューツール】self-review
【注記】セルフレビューによる実施。外部レビューツール利用不可のため、エージェント自身がレビューを実施。"
```

**注意**: `--unit`, `--unit-name`, `--unit-slug` はconstructionフェーズの場合のみ指定。inceptionフェーズではこれらの引数を省略する（既存ステップ5の履歴記録ルールと同じ）。

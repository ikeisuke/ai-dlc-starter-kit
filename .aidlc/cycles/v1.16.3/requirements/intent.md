# Intent（開発意図）

## プロジェクト名
AI-DLC Starter Kit v1.16.3 - 設定基盤・セッション管理・レビューフローの改善

## 開発の目的
AI-DLCの設定基盤とセッション管理の堅牢性を向上させ、レビューフローの柔軟性を改善する。具体的には、設定読み取りのデフォルト値統一と一括取得対応、セッションリセット時の情報継続性改善、ブランチ作成方式の設定固定化、レビューツール未設定時のセルフレビューフォールバック対応を行う。

## ターゲットユーザー
AI-DLCスターターキットを利用する開発者およびAIエージェント

## ビジネス価値
- 設定読み取りエラーの削減とフェーズ開始時の処理効率向上
- セッション間の情報継続性向上による作業再開のスムーズ化
- 毎サイクル同じブランチ方式を選択する手間の削減
- レビューツール未設定環境でもレビュー品質を一定水準で維持
- コンテキストリセット指示の確実な提示によるセッション品質の維持

## 成功基準
- read-config.sh が defaults.toml からフォールバックし、--default なしで呼び出しても終了コード0を返し正常値を出力する
- read-config.sh に --keys オプションで複数キーを渡すと、key:value 形式で各行に結果を出力する
- Unit完了時およびPhase完了時のリセット告知に、完了作業の要約・リポジトリ状態・次のアクションを含む3行程度のサマリが表示される
- `[rules.branch].mode` に `branch` または `worktree` を設定すると、ステップ7でユーザーへの質問なしに自動でブランチ/worktreeが作成される
- 外部レビューツール（Skills）が利用不可の場合、実行中のエージェント自身がセルフレビューを実施し、レビュー結果が出力される
- Unit完了時およびPhase完了時に、プロンプトで定義されたコンテキストリセットメッセージが100%提示される（ユーザーからの明示的な連続実行指示がある場合を除く）

## スコープ外（除外事項）
- read-config.sh の TOML パーサー自体の変更や dasel 以外のパーサーへの移行
- セミオートモード (#164) や GitHub Projects連携 (#31) の実装
- 外部レビューツール（Codex, Claude CLI, Gemini）自体のインストール・設定の自動化
- コンテキストリセット後のセッション自動再開

## 期限とマイルストーン
v1.16.3 タグ作成・リリースまでに完了

## 制約事項
- スターターキット共通の変更は `prompts/package/` 配下を編集する（メタ開発ルール）
- プロジェクト固有の変更は `docs/cycles/rules.md`, `AGENTS.md` 等を直接編集する
- 後方互換性を維持する（既存設定が未設定の場合は現行動作をデフォルトとする）
- パッチリリースのため破壊的変更は含めない

# 論理設計: セルフレビューフォールバック

## 概要

`review-flow.md` にセルフレビューフォールバック機能を追加するための変更仕様。プロンプトファイルの変更のみであり、アプリケーションコードの実装は発生しない。

**重要**: この論理設計では**コードは書かず**、プロンプト変更の構成と仕様のみを定義する。

## 変更対象

- `prompts/package/prompts/common/review-flow.md`

## 変更箇所と仕様

### 変更0: 設定確認セクションの更新

**現在の仕様**（review-flow.md L9）:
- `mode = "required"`: AIレビュー必須（AIレビューツールが利用不可の場合、ユーザー承認により例外的に人間レビューへ移行可能。承認は履歴に記録する）

**変更後の仕様**:
- `mode = "required"`: AIレビュー必須（外部AIレビューツールまたはセルフレビューにより充足。いずれも利用不可の場合、ユーザー承認により例外的に人間レビューへ移行可能。承認は履歴に記録する）

### 変更1: ステップ6（AIレビュー不可時）の選択肢追加

**現在の仕様**（review-flow.md L342-363）:

- `mode = "required"`: 「はい - 人間承認へ進む / いいえ - 処理を中断」の2択
- `mode = "recommend"`: 自動的にステップ7へ

**変更後の仕様**:

- `mode = "required"`:
  ```text
  【警告】AIレビューが必須設定ですが、AIレビューツール（Skills）が利用できません。

  どのように対応しますか？
  1. セルフレビューを実施する（推奨） - エージェント自身がレビューを実施
  2. 人間承認へ進む - レビュースキップを履歴に記録
  3. 処理を中断
  ```
  - 選択肢1 → ステップ5.5（セルフレビューフロー）へ。セルフレビューは「必須AIレビュー充足」として扱う
  - 選択肢2 → 既存の処理（履歴記録後、ステップ7へ）
  - 選択肢3 → 既存の処理（中断）

- `mode = "recommend"`:
  ```text
  【レビュー推奨】AIレビューツール（Skills）が利用できませんが、セルフレビューを実施できます。
  品質向上のため、セルフレビューの実施を推奨します。
  セルフレビューを実施しますか？
  1. はい - セルフレビューを実施する（推奨）
  2. いいえ - 人間レビューへ進む
  ```
  - 選択肢1 → ステップ5.5（セルフレビューフロー）へ
  - 選択肢2 → ステップ7（人間レビューフロー）へ

### 変更2: ステップ5.5（セルフレビューフロー）の新設

ステップ5（AIレビューフロー）とステップ6（AIレビュー不可時）の間に新設。

**セクション構成**:

```markdown
5.5. **セルフレビューフロー**（Skills利用不可時のフォールバック）:

   **注意**: セルフレビューは外部ツールに比べて品質が劣る可能性があります。
   重要な変更では、可能な限り外部AIレビューツールの利用を推奨します。

   - **レビュー前コミット**: `common/commit-flow.md` の「レビュー前コミット」手順に従う

   - **セルフレビュー実行手順**:
     1. レビュー種別（code / architecture / security / inception）に応じた観点を確認
        - 各種別の観点は対応するレビュースキルのSKILL.mdを参照（注: Skillの実行は不可でもSKILL.mdファイルはディスク上に存在するため参照可能）
     2. 対象の成果物を読み込む
     3. 観点に基づいてレビューを実施し、出力フォーマットに従って記録
     4. 指摘がある場合は修正を反映
     5. 反復レビュー（最大3回）: 修正後に再度セルフレビューを実施
     6. 全種別で指摘がゼロになった場合:
        - 【AIレビュー完了（セルフレビュー）】メッセージを出力
        - 履歴に記録
        - レビュー後コミット → 人間への承認提示へ
     7. 3回後も指摘が残る場合: 「指摘対応判断フロー」（ステップ5と同じ）を実行

   - **出力フォーマット**:
     [出力フォーマット定義]

   - **履歴記録**:
     [セルフレビュー用の履歴テンプレート]

   - **外部入力検証の非適用**:
     セルフレビューは同一エージェントが実施するため、
     「外部入力検証ルール」の「AIレビュー応答の検証」は適用しない。

   - **レビュー後コミット**: `common/commit-flow.md` の「レビュー反映コミット」手順に従う
   - 修正後の成果物を人間に提示
   - 人間の承認を求める
```

### 変更3: セルフレビュー出力フォーマット

指摘ありの場合:

```text
【セルフレビュー結果】
レビュー種別: {code | architecture | security | inception}

指摘 #1
- 重要度: {高 | 中 | 低}
- 内容: {指摘内容の要約}
- 推奨修正: {修正方法の提案}
- 判定: {未対応 | 対応済}

合計: {N}件（高: {n}件 / 中: {n}件 / 低: {n}件）
```

**指摘対応判断フローとの接続ルール**:
- 判断フローへの入力は `判定: 未対応` の指摘のみ（対応済は除外）
- 反復レビュー時、同一指摘IDの判定を `未対応 → 対応済` に更新する
- 3回後の収束判定: `判定: 未対応` の指摘が残っている場合に指摘対応判断フローを起動

指摘なしの場合:

```text
【セルフレビュー結果】指摘0件
レビュー種別: {種別}
```

### 変更4: 履歴記録テンプレート

セルフレビュー完了時:

```bash
docs/aidlc/bin/write-history.sh \
    --cycle {{CYCLE}} \
    --phase {{PHASE}} \
    --unit {N} \
    --unit-name "[Unit名]" \
    --unit-slug "[unit-slug]" \
    --step "AIレビュー完了" \
    --content "【AIレビュー完了（セルフレビュー）】指摘0件
【対象タイミング】{呼び出し元のステップ名}
【対象成果物】{成果物名}
【レビュー種別】{実行した全種別}
【レビューツール】self-review
【注記】セルフレビューによる実施。外部レビューツール利用不可のため、エージェント自身がレビューを実施。"
```

- `--unit`, `--unit-name`, `--unit-slug`: constructionフェーズの場合のみ指定。inceptionフェーズではこれらの引数を省略する。

## 非機能要件（NFR）への対応

Unit定義より全項目「該当なし」。

## 実装上の注意事項

- **メタ開発の意識**: `prompts/package/prompts/common/review-flow.md` を編集すること（`docs/aidlc/prompts/common/review-flow.md` ではない）
- 既存のステップ番号を変更しない（5.5として挿入）
- 既存のフロー（ステップ5、ステップ6、ステップ7）のロジックは保持する

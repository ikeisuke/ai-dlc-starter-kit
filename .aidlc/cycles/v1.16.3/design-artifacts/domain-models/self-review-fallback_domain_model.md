# ドメインモデル: セルフレビューフォールバック

## 概要

AIレビューフロー内に「セルフレビュー」という新しいレビュー実行方式を追加する。外部AIレビューツール（Skills）が利用不可の場合に、実行中のエージェント自身がフォールバックとしてレビューを担当する概念モデル。

**重要**: このUnitはプロンプトファイル（`review-flow.md`）の変更のみであり、アプリケーションコードの実装は発生しない。ここではフロー内の概念と責務を定義する。

## 概念モデル

### レビュー実行方式（ReviewExecutor）

レビューの実行方式を表す概念:

| 方式 | 説明 | 起動条件 |
|------|------|----------|
| skill | 外部AIレビューツール（Codex, Claude CLI, Gemini等）によるレビュー | Skillが利用可能 |
| self-review | 実行中のエージェント自身によるレビュー | ステップ6（AIレビュー不可時）に到達し、ユーザーがセルフレビューを選択（全種別不可 or 一部不可+承認拒否のいずれでも提示） |
| human | 人間によるレビュー | mode=disabled、またはAIレビュー不可でスキップ選択時 |

### セルフレビューの位置づけ

- **必須AIレビュー充足性**: `mode = "required"` の場合、セルフレビューは必須AIレビューを充足するものとして扱う
- **品質の注記**: セルフレビューは外部ツールに比べて品質が劣る可能性があるため、履歴に明記する
- **観点の一致**: セルフレビューでも外部レビューと同じレビュー観点（code / architecture / security / inception）を適用する

### フロー内の遷移

```text
ステップ6（AIレビュー不可時）
  ├─ [1] セルフレビューを実施 → ステップ5.5（セルフレビューフロー）
  │     ├─ 反復レビュー（最大3回）
  │     ├─ 3回後も指摘残 → 指摘対応判断フロー（既存）
  │     └─ 指摘0件 → レビュー後コミット → 人間への承認提示
  ├─ [2] 人間承認へ進む → ステップ7（人間レビューフロー）
  └─ [3] 処理を中断（mode=requiredのみ）
```

## ユビキタス言語

- **セルフレビュー**: 外部AIレビューツールを使わず、実行中のエージェント自身が成果物をレビューすること
- **レビュー実行方式**: レビューを誰（何）が担当するかを示す区分（skill / self-review / human）
- **必須AIレビュー充足**: mode=required設定において、AIレビューの要件を満たすとみなされること

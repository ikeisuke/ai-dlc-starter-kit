# Intent（開発意図）

## プロジェクト名
ai-dlc-starter-kit

## 開発の目的
AI-DLCスターターキットのスキル管理を一般化する。具体的には、スキルのマーケットプレイス方式対応、AI-DLC固有スキルと汎用ツールスキルの分離、jjサポートの外部化、およびマルチプラットフォーム対応状況の棚卸しを行う。

## ターゲットユーザー
AI-DLCスターターキットの利用者および開発者

## 用語定義

| 用語 | 定義 |
|------|------|
| マーケットプレイス方式 | `.claude-plugin/marketplace.json` で管理されるスキルカタログからインストールする配信方式。`claude-skills` リポジトリが参照実装 |
| 埋め込み方式 | `prompts/package/skills/` にスキルを同梱し、セットアップ時に `docs/aidlc/skills/` → `.claude/skills/` のシンボリックリンクで配信する従来方式 |
| 一般化 | AI-DLC固有のスキル配信を、マーケットプレイス方式と埋め込み方式の両方に対応させること |
| 名前空間分離 | AI-DLC固有スキル（例: `aidlc:setup`）と汎用ツールスキル（例: `tools:session-title`）をプレフィックスで区別すること |
| リネーム対象 | `upgrading-aidlc` → `aidlc-setup`（セットアップとアップグレードを統合した名称。名前空間導入後は `aidlc:setup` としても呼び出し可能） |

## ビジネス価値
- スキルの配信方式を柔軟化し、ユーザーが必要なスキルを選択的にインストールできるようにする
- スターターキット本体をスリム化し、保守性を向上させる
- AI-DLC固有機能と汎用ツールの境界を明確にし、再利用性を高める
- AIエージェント非依存性を確認・強化し、幅広い環境での利用を可能にする

## 成功基準
- マーケットプレイス方式の `marketplace.json` が定義され、`/plugin marketplace add` / `/plugin install` でスキルがインストール可能である
- 既存の埋め込み方式（`prompts/package/skills/`）でセットアップしたプロジェクトが、変更なく動作する（確認項目: セットアップ成功、主要スキル呼び出し成功、既存プロジェクト更新フロー成功）
- `upgrading-aidlc` が `aidlc-setup` にリネームされ、既存のスキル参照箇所が更新されている
- jj関連コード（`prompts/package/skills/versioning-with-jj/`、各スクリプトのjj分岐）がスターターキット本体から削除されている
- マルチプラットフォーム対応の現状整理ドキュメント（対応状況マトリクス、ギャップ分析、次期サイクルへの提案）が作成されている

## 期限とマイルストーン
Unit単位で管理（AI-DLCのConstruction Phaseで段階的に完了）

## 制約事項
- 既存の埋め込み方式を破壊しない（後方互換性の維持）
- `prompts/package/` がスターターキット本体であるメタ開発構造を維持
- `docs/aidlc/` は直接編集禁止（`prompts/package/` を編集）
- #281（マルチプラットフォーム対応）は調査・棚卸しが主スコープで、大規模実装は含まない

## 既存機能への影響分析

| 変更内容 | 破壊的変更 | 移行手順 |
|---------|-----------|---------|
| マーケットプレイス方式の追加 | なし（新規機能追加） | 既存ユーザーは影響なし。オプトイン |
| `upgrading-aidlc` → `aidlc-setup` リネーム | あり（スキル名変更） | CLAUDE.md/AGENTS.md のスキル参照を更新。旧名 `upgrading-aidlc` は本リリースで削除（v1.19.0で非推奨化済みのため互換期間不要） |
| jjサポート削除 | あり（非推奨機能の削除） | v1.19.0で非推奨化済み。skillsリポジトリへの移行手順を提供 |
| 名前空間分離 | なし（プレフィックス追加は既存名にも互換対応） | 段階的にプレフィックス付き名前に移行 |

## スコープ

### 含まれるもの
- スキルのマーケットプレイス方式対応の設計と実装（#292）
- `upgrading-aidlc` → `aidlc-setup` へのリネーム（#292）
- AI-DLC固有スキルと汎用ツールスキルの名前空間分離（#292）
- jjサポートのスターターキット本体からの削除とskillsリポジトリへの移動（#276）
- マルチプラットフォーム対応の現状調査と方針策定（#281）
  - 成果物: 現状整理ドキュメント（対応状況マトリクス、ギャップ分析、次期サイクルへの提案）
  - 実装作業は含まない

### 含まれないもの
- マルチプラットフォーム対応の実装作業（調査結果に基づき別サイクルで対応）
- 既存のレビュースキル（reviewing-*）の機能変更
- AI-DLC のフェーズフロー自体の変更

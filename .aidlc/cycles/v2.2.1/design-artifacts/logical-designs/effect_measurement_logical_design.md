# 論理設計: 効果計測・検証

## 概要
Wave 2施策の効果を計測・判定し、結果をbaseline.mdの事後計測セクションに記録する手順を定義する。

**重要**: この論理設計では**コードは書かず**、コンポーネント構成とインターフェース定義のみを行います。

## アーキテクチャパターン
パイプライン（計測 → 集計 → 比較 → 判定 → 記録）。各ステップは前ステップの出力を入力として受け取る線形フロー。

## コンポーネント構成

### コンポーネント詳細

#### A. 計測コンポーネント
- **責務**: 現在の共通初期化フローで実際にロードされるファイルに対し `wc -c` でバイト数を取得
- **依存**: なし（ファイルシステムのみ）
- **入力**: 現在のSKILL.mdの共通初期化フローで定義されたファイルリスト（Wave 2で構成変更があった場合、旧ファイルは0B/削除扱い、新ファイルは追加計上）
- **出力**: ファイル別バイト数一覧
- **エラー時**: ファイル未存在の場合は対象ファイルを警告出力し、計測結果を0Bとして記録（削除されたファイルの検知）

#### B. 集計コンポーネント
- **責務**: ファイル別バイト数をフェーズ別（共通/ステップ/合計）に集計
- **依存**: A（計測コンポーネント）
- **入力**: ファイル別バイト数一覧
- **出力**: フェーズ別集計テーブル（Inception/Construction/Operations）

#### C. 比較コンポーネント
- **責務**: ベースライン値と事後計測値の差分を算出
- **依存**: B（集計コンポーネント）、baseline.md（参照入力、不変）
- **入力**: フェーズ別集計テーブル + ベースラインのフェーズ別合計
- **出力**: フェーズ別差分テーブル（削減量バイト数）

#### D. 判定コンポーネント
- **責務**: 成功基準（フェーズ別に12,500B以上削減）の達成判定
- **依存**: C（比較コンポーネント）
- **入力**: フェーズ別差分テーブル
- **出力**: 判定結果（全フェーズ達成/一部達成/未達成 + フェーズ別根拠）
- **エラー時**: 差分データ不完全の場合は判定不能として警告出力

#### E. 記録コンポーネント
- **責務**: 計測結果・判定結果をbaseline.mdの事後計測セクションに追記
- **依存**: B, C, D
- **入力**: フェーズ別集計、差分、判定結果（各コンポーネントの出力を集約）
- **出力**: baseline.md更新（事後計測セクション追記のみ。既存セクションは不変の参照データとして読み取り専用）
- **注**: baseline.mdは「参照データ（既存セクション）」と「レポート出力（事後計測セクション）」の責務をセクション単位で分離。同一ファイル内だがセクション境界で入出力を分離している

#### F. 統合検証コンポーネント（スモークテスト・事前検証）
- **責務**: 全フェーズの共通初期化フローが正常完了することを確認（圧縮変更による破壊がないことのスモークテスト）
- **依存**: なし（主パイプラインの前段として実行。失敗時は計測を中止して原因調査を促す）
- **出力**: 検証結果（OK/NG）

## 処理フロー概要

### ステップ0: 統合検証（事前検証・スモークテスト）

1. 共通初期化で読み込まれるファイルがすべて存在し、構文的に有効であることを確認
2. 失敗時は計測を中止し原因調査を促す

### ステップ1-6: 計測・判定フロー

1. 現在のSKILL.mdの共通初期化フローから対象ファイルリストを抽出（ベースラインのリストと差分がある場合は構成変更として記録）
2. 各ファイルに対し `wc -c` を実行しバイト数を取得（未存在ファイルは警告+0B記録）
3. フェーズ別に共通/ステップ/合計を集計
4. baseline.mdのベースライン値と比較しフェーズ別差分を算出
5. 成功基準（フェーズ別12,500B以上削減）を判定
6. baseline.mdの事後計測セクションに結果を追記

## データモデル概要

### ファイル形式（baseline.md 事後計測セクション）
- **形式**: Markdown テーブル
- **内容**: ベースラインと同一フォーマットの計測テーブル + 差分サマリ + 判定結果

## 非機能要件（NFR）への対応
Unit定義より全項目「該当なし」。

- baseline.mdの既存セクション（ベースライン計測結果）は編集しない
- ファイルパスはスキルベースディレクトリ（`/Users/keisuke/.claude/plugins/marketplaces/ai-dlc-starter-kit/skills/aidlc/`）を基準に解決

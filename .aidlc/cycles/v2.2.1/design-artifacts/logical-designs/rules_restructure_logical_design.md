# 論理設計: rules.md 3階層分割

## 概要
rules.md と agents-rules.md を3ファイル（rules-core/automation/reference）に再構成する際のファイル構成・セクション配置・参照パス更新の詳細設計。

**重要**: この論理設計では**コードは書かず**、コンポーネント構成とインターフェース定義のみを行います。

## アーキテクチャパターン

**階層化参照パターン**: 常時ロード層（core）と必要時参照層（automation, reference）の2層構成。フェーズステップファイルは必要なルールファイルのみを参照し、不要なコンテキスト消費を回避する。

## コンポーネント構成

### ファイル構成

```text
steps/common/
├── rules-core.md          (常時ロード: ~9,700B)
├── rules-automation.md    (必要時参照: ~2,813B)
└── rules-reference.md     (必要時参照: ~1,512B)
```

- **依存**: なし

**統合詳細 - 質問セクション (#5)**:

rules.md「質問と実行の判断基準」をメインとし、agents-rules.md「質問と深掘り」のサブセクション（質問の目的、深掘りのテクニック、情報提示ルール）を追加サブセクションとして配置。rules.md側の「質問フロー」「質問不要でも確認が必要な場面」「不明点の記録」は維持。agents-rules.md側の「注意事項」は「質問フロー」と重複するため削除。

**統合詳細 - バックログセクション (#9)**:

rules.md「改善提案のバックログ登録ルール」をメインとし、agents-rules.md「バックログ管理」「即時実装優先ルール」を統合。「適用場面の違い」テーブル（即時実装 vs バックログ登録）を維持。

## インターフェース設計

### SKILL.md ステップ1の変更

**変更前**:
```
1. `steps/common/agents-rules.md` — エージェントルール
2. `steps/common/rules.md` — 共通開発ルール
```

**変更後**:
```
1. `steps/common/rules-core.md` — 共通開発ルール
```

### AGENTS.md の参照変更

**変更前**: `@steps/common/agents-rules.md を参照してください。`
**変更後**: `@steps/common/rules-core.md を参照してください。`

### ステップファイルからの参照インターフェース

各ステップファイルは以下の形式で新ファイルを参照する:

- `rules-core.md` の「[セクション名]」を参照 → 常時ロード済みのため追加読み込み不要
- `rules-automation.md` の「セミオートゲート仕様」を参照 → 必要時に読み込み
- `rules-reference.md` の「Depth Level仕様」を参照 → 必要時に読み込み

## 処理フロー概要

### 実装手順

1. rules-core.md を新規作成（rules.md + agents-rules.md のセクションを統合配置）
2. rules-automation.md を新規作成（セミオートゲート + エクスプレス移動）
3. rules-reference.md を新規作成（Depth Level + 設定仕様移動）
4. SKILL.md ステップ1のロード指示を更新
5. AGENTS.md の参照を更新
6. 全ステップファイルの参照パスを更新（grep検索→個別対応）
7. サイズ検証（3ファイル合計 < 14,732B）
8. 内容等価性検証（分割前後で漏れなし）
9. 元ファイル（rules.md, agents-rules.md）を削除（検証完了後）

## 非機能要件（NFR）への対応

### パフォーマンス
- **要件**: 3ファイル合計 < 14,732B、常時ロード ~9,700B
- **対応策**: 重複セクションの統合によるサイズ削減、不要セクションのオンデマンド化

## 実装上の注意事項

- `.aidlc/rules.md`（プロジェクト固有ルール）への参照は変更対象外
- `common/rules.md` の参照と `.aidlc/rules.md` の参照を混同しないこと
- セミオートゲート仕様のフォールバック条件テーブルの内容は移動のみ、変更禁止
- スコープ保護ルールの内容は変更禁止

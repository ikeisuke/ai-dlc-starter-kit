# ドメインモデル: review-flow圧縮

## 概要

review-flow.mdの遷移判定テーブルと冗長説明文を圧縮するパターンを定義する。

## 圧縮パターン一覧

### RCP-001: 遷移判定テーブル → if/else形式

- **対象**: L79-91（8行テーブル + ヘッダ3行）
- **圧縮前**: Markdownテーブル（ヘッダ+区切り+8データ行 = 約650B）
- **圧縮後**: if/else条件分岐リスト。「遷移先」と「拒否可否」の2軸を保持
- **推定削減**: ~300B

### RCP-002: パス1説明文の圧縮

- **対象**: L39-62（パス1: 外部CLIレビュー）
- **圧縮前**: 番号付きリスト + エラー時フォールバックテーブル（約1,200B）
- **圧縮後**: 手順を圧縮、エラーテーブルの説明文を最小化
- **推定削減**: ~400B

### RCP-003: パス2/3説明文の圧縮

- **対象**: L64-78（パス2: セルフレビュー、パス3: ユーザーレビュー）
- **圧縮前**: 各パスの説明文（約500B）
- **圧縮後**: 最小テキスト
- **推定削減**: ~200B

### RCP-004: レビューサマリファイル説明の圧縮

- **対象**: L160-184（レビューサマリファイルセクション）
- **圧縮前**: テーブル + 説明文（約900B）
- **圧縮後**: テーブル維持、説明文を圧縮
- **推定削減**: ~200B

### RCP-005: CallerContextマッピング説明文の圧縮

- **対象**: L17-33
- **圧縮前**: テーブル + 3行の説明文（約300B）
- **圧縮後**: テーブル維持、説明文を1行に圧縮
- **推定削減**: ~100B

## 推定合計削減量: ~1,200B

目標: 11,950B → ~10,750B

- 指摘対応判断フローセクション（L92-149）: 変更禁止

# Intent（開発意図）

## プロジェクト名
AI-DLCスキル コンテキストサイズ圧縮 Wave 2

## 開発の目的
v2.2.0（Wave 1）で実施した低リスク・高効果の圧縮施策に続き、Wave 2として中リスク・中効果の構造変更を実施する。初回ロードサイズをベースライン再計測値から12.5KB以上削減し（Wave 1後の推定値~40KBから~30KB程度を想定）、セッション中盤でのcompactionリスクを低減する。

## ターゲットユーザー
AI-DLCスターターキットを利用する開発者（AIエージェント経由でスキルを実行するユーザー）

## ビジネス価値
- コンテキストウィンドウの消費を削減し、長時間セッションでのcompaction頻度を下げる
- compactionによる指示欠落・品質劣化のリスクを低減する
- ファイル構造の整理によりメンテナビリティを向上させる

## 成功基準
- ベースライン再計測後の値を起点に、初回ロードサイズが12.5KB以上削減されること（目標値はベースライン計測後に確定。Issue #519のWave 2推定: 12.5-26.5KB削減）
- 全フェーズ（Inception/Construction/Operations）のステップ実行が正常動作すること（下記「影響範囲と検証方法」で確認）
- 既存のAIレビュー・セミオートゲート・compaction復帰が劣化しないこと（下記「影響範囲と検証方法」で確認）

## スコープ

### 含まれるもの
- **ベースライン再計測**: Wave 1実施後の現状バイト数を計測（トークン数は参考値、バイト数を主指標とする）
- **S18: rules.md 3階層分割**: 現在の `steps/common/rules.md` を以下の3ファイルに分割
  - `rules-core.md`（常時ロード、全フェーズ共通初期化時に読み込み）: 質問判断基準、承認プロセス、スコープ保護ルール
  - `rules-automation.md`（`automation_mode=semi_auto` 時のみロード）: セミオートゲート仕様、エクスプレスモード仕様
  - `rules-reference.md`（各ステップファイルから必要時に参照）: Depth Level詳細テーブル、設定仕様リファレンス
- **S6: compaction.md二重ロード解消**: SKILL.mdの不変ルール内のcompaction.md参照を削除し、session-continuity.md（コンパクション復帰判定時）に統一
- **S3: agents-rules.md統合**: rules.mdとの40%重複を解消し統合
- **S17: フェーズ完了処理の共通化**: inception/construction/operationsの完了処理の共通部分を抽出
- **S10: review-flow圧縮**: 遷移判定8パターンの簡略化、サブエージェント指示テンプレートの外部化
- **効果計測**: Wave 2実施後のバイト数を再計測し、ベースラインとの差分で削減効果を定量評価

### 含まれないもの
- Wave 3（ステッププログレッシブロード、SKILL.md/CLAUDE.md重複排除等）のアーキテクチャ変更
- MCPベース検索やHooks連携による動的注入などのアーキテクチャ代替案
- session-management.md重複排除（メンテナビリティ施策、ランタイム効果なし）
- 新機能の追加

## 期限とマイルストーン
- ベースライン計測 → Wave 2各戦略の実装 → 効果計測・検証

## 制約事項
- Issue #519の品質劣化リスクマトリクスに記載されたセクション（compaction.mdのautomation_mode復元手順、review-flow.mdの指摘対応判断フロー・却下禁止ルール、preflight.mdのblocker判定、rules.mdのセミオートゲートフォールバック条件・スコープ保護ルール）は変更禁止またはリスクを十分評価した上で変更
- メタ開発の意識: ツール側（skills/aidlc/）の変更であり、スキル実行時の動作に影響する

## 影響範囲と検証方法

| 変更施策 | 影響を受ける既存機能 | 検証方法 |
|---------|-------------------|---------|
| S18: rules.md 3階層分割 | セミオートゲート判定、エクスプレスモード、Depth Level分岐 | 各フェーズのステップ実行でrules参照が正しく解決されることを確認 |
| S6: compaction.md二重ロード解消 | compaction復帰フロー（automation_mode復元） | compaction.mdの内容が1回のみロードされ、復帰手順が正常動作することを確認 |
| S3: agents-rules.md統合 | 全フェーズの共通初期化（agents-rules.md読み込み） | 統合後のrules.mdに元の指示が全て含まれることを差分確認 |
| S17: フェーズ完了処理共通化 | Inception/Construction/Operations各完了処理（履歴記録、squash、コミット、コンテキストリセット） | 各フェーズの完了処理が共通化後も同一の動作をすることを確認 |
| S10: review-flow圧縮 | AIレビューフロー（遷移判定、指摘対応判断、セルフレビュー） | review-flow.mdの全遷移パターンが圧縮後も同一の結果を返すことを確認 |

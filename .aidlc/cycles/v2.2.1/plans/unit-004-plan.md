# Unit 004: フェーズ完了処理の共通化 - 実装計画

## 背景

3フェーズのcompletion.md（Inception: 11,135B、Construction: 8,276B、Operations: 10,311B、合計29,722B）に共通ステップが重複している。

## AIレビュー指摘への対応

codexレビューで以下の指摘を受け、設計方針を修正:

1. **凝集度の問題（高）**: 5つの共通ステップを1ファイルに集約すると凝集度が低い。OperationsにはSquash/コミットがなく、フェーズごとに分岐条件が大きく異なる
2. **先読み指示の増加（高）**: `【次のアクション】` 参照指示の増加がv2.2.0の先読み指示廃止方針と衝突
3. **責務の二重化（中）**: コンテキストリセットは既存 `context-reset.md` と二重化

## 修正後の設計方針

### 方針転換: 新規共通ファイル作成ではなくインライン圧縮

`completion-common.md` の新規作成は行わない。代わりに各フェーズのcompletion.md内で以下のアプローチで圧縮する:

### 圧縮対象の分析

| ステップ | 現状 | 圧縮方針 |
|---------|------|---------|
| 履歴記録 | 各フェーズで1-2行 | 圧縮不要（既に最小） |
| Squash + Gitコミット | Inception/Constructionで各約600B重複 | `commit-flow.md` への委譲を明確化し、squash結果分岐の説明文を圧縮 |
| 完了サマリ出力 | 各フェーズで固有テンプレート | フォーマット説明の冗長テキストを圧縮（テンプレート自体は維持） |
| コンテキストリセット提示 | Inception/Constructionで約800B重複 | セミオートゲート判定の重複説明を圧縮。既存 `context-reset.md` への参照を統一 |
| エクスプレスモード完了処理 | Inception内で約2,600B | squash/コミットのエクスプレス版を本体と統合し重複排除 |

### 具体的な圧縮施策

1. **Squash + Gitコミット（Inception/Construction共通）**:
   - 現状: 各フェーズに squash 結果分岐（`squash:success` → コミットスキップ / `squash:skipped` → コミット実行 / `squash:error` → リカバリ後コミット）の同一テキストが重複
   - 圧縮: commit-flow.md への `【次のアクション】` 参照は既存例外として維持。分岐結果の説明を最小テキストに圧縮

2. **コンテキストリセット提示（Inception/Construction共通）**:
   - 現状: セミオートゲート判定の説明文が各フェーズで重複
   - 圧縮: 共通の判定ロジック説明を1行に圧縮し、`context-reset.md` への参照を明示

3. **エクスプレスモード完了処理（Inception内）**:
   - 現状: squash/コミットの手順が通常完了処理と重複（~600B）
   - 圧縮: 「完了時の必須作業」のステップ番号参照で重複排除

4. **完了サマリ出力（全フェーズ）**:
   - 現状: 各フェーズに「AIがコンテキスト情報から動的に生成」等の同一説明文が重複
   - 圧縮: 説明文を最小化、テンプレートのみ維持

# ドメインモデル: バリデーション共通化とサイクルID緩和

## ドメイン概要

サイクル名のバリデーションロジックを共通ライブラリに集約し、非SemVer形式のカスタム名を許可するように緩和する。

## エンティティ / 値オブジェクト

### CycleName（値オブジェクト）

サイクル名を表す文字列。以下の3形式を許可する。

| 形式 | パターン | 例 |
|------|---------|-----|
| SemVer | `v<MAJOR>.<MINOR>.<PATCH>` | `v1.21.2` |
| 名前付きSemVer | `<name>/v<MAJOR>.<MINOR>.<PATCH>` | `waf/v1.0.0` |
| カスタム名 | `<custom>` | `feature-auth`, `2026-03` |

**許可ルール**:
- 使用可能文字: 英小文字、数字、ハイフン、ドット、スラッシュ（区切りとして1回のみ）
- 先頭文字: 英小文字または数字または `v`
- 最小長: 1文字（空文字は不可）

**拒否ルール**:
- 空文字列
- `..` を含む（パストラバーサル防止）
- 先頭が `/`（絶対パス防止）
- 連続スラッシュ `//`
- スラッシュが2つ以上（多重ネスト防止）
- スペースを含む
- 制御文字を含む
- 大文字を含む名前プレフィックス（SemVerのprerelease部分は除く）

### validate_cycle 関数

**責務**: CycleNameの妥当性を検証する

**インターフェース**:
```
validate_cycle(cycle_name: string) -> exit_code (0: valid, 1: invalid)
```

**副作用**: なし（stderr出力なし、ファイルI/Oなし）

## ドメインサービス

### validate.sh（共通ライブラリ）

**責務**: バリデーション関数群を提供する共通ライブラリ

**配置**: `prompts/package/lib/validate.sh`

**利用方法**: 呼び出し元スクリプトが `source` で読み込む

**パス解決方式**: 各スクリプトの `SCRIPT_DIR` から相対パスで解決
```bash
source "${SCRIPT_DIR}/../lib/validate.sh"
```

**設計制約**:
- ライブラリは関数定義のみ。トップレベルで実行されるコードは含まない
- `set -euo pipefail` は含まない（呼び出し元が設定）
- シバン行 `#!/usr/bin/env bash` は含む（直接実行の防止のため `sourced check` は不要だが、慣例として含める）

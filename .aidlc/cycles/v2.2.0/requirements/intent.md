# Intent（開発意図）

## プロジェクト名
AI-DLC Starter Kit

## 開発の目的
AI-DLCスキルの初回ロード量が大きく（推定80KB超）、コンテキストウィンドウを圧迫している問題を解決する。Wave 1（低リスク・高効果）の5戦略を実施し、初回ロードを約半減（~40KB）させる。あわせて、設定キーの不整合修正やAIエージェントの判断時情報提示ルールの追加など、小規模な改善も実施する。

## ターゲットユーザー
AI-DLC Starter Kitの利用者（AIエージェントを活用して開発を行う開発者）

## ビジネス価値
- コンテキストウィンドウの消費削減により、セッション中盤でのコンパクションリスクを低減
- 指示欠落による品質劣化を防止し、AIエージェントの応答品質を維持
- 判断時の情報提示改善により、ユーザーの意思決定負荷を軽減

## 成功基準
- 初回ロードサイズが現状の80KBから40KB以下に削減される
  - **計測対象**: SKILL.md起動時にReadツールで読み込まれる全ファイルの合計（SKILL.md + 共通ステップ + フェーズステップ）
  - **計測単位**: バイト数（`wc -c` で計測）
  - **計測手順**: フェーズ別にロード対象ファイルを `cat` で結合し `wc -c` で合計バイト数を取得。Before/Afterを比較。初回ロード時点で実際にReadツールで読み込まれるファイルのみを含む（`guides/` や `templates/` に外部化したファイルは必要時参照のため除外）
  - **基準**: Inception Phase初回ロード（最大ケース）で40KB以下（目標値。未達でも全実装完了ならサイクル完了可、ただし成功判定は40KB以下達成時のみ）
- 既存の全フェーズのステップ実行が正常動作する
  - 外部化したテンプレート・共通基盤ファイルの参照が正しく解決される
  - 設定キー変更後の既存 `config.toml` との互換性が維持される
  - Reviewingスキル9種がそれぞれ正常に呼び出し可能である
- `rules.linting` 設定キーの不整合が解消される
- defaults.tomlの先頭コメントが現行仕様と一致する
- 判断を求める際の情報提示ルールがプロンプトに追加される

## 含まれるもの

### 主目的: コンテキストサイズ圧縮（必須）
1. **S7/S13: Reviewingスキル共通基盤抽出** — 9スキルの共通ボイラープレートを `reviewing-common-base.md` に抽出（20-25KB削減）
2. **S5+: 01-setup.mdバージョンテーブル外部化** — バージョン比較ロジックを `guides/version-check.md` に移動（5-7KB削減）
3. **S14: 条件分岐ロジックのスクリプト化** — preflight.md, 01-setup.mdの冗長な条件分岐をテーブル圧縮・スクリプト委譲（5-8KB削減）
4. **S15: インラインテンプレート外部化** — 05-completion.md, review-flow.mdのテンプレートを `templates/` に移動（3-5KB削減）
5. **S16: 先読み指示パターン廃止** — 各ステップファイルの「今すぐ読み込んで」パターンを削除（1-2KB削減）

### 付随改善（必須だが主目的未達でもサイクル完了可）
6. **#530: rules.linting設定キー不整合修正** — template/defaults/docs間のキー名を `rules.linting.enabled` に統一
7. **#531: defaults.toml先頭コメント更新** — 現行仕様（`.aidlc/config.toml` ベース）に修正
8. **#534: 判断時の情報提示ルール追加** — `agents-rules.md` の「質問と深掘り」セクションにルールを追加

## 含まれないもの
- Wave 2（rules.md分割、compaction.md二重ロード解消等の構造変更）
- Wave 3（プログレッシブロード等のアーキテクチャ変更）
- トークナイザーによる厳密な計測（バイト数ベースの概算で十分）
- reviewing-*/session-management.mdの一元化（メンテナビリティ施策、ランタイム効果なし）

## 期限とマイルストーン
なし（品質優先）

## 制約事項
- 品質劣化リスクの高いセクション（compaction.md automation_mode復元手順、review-flow.md指摘対応判断フロー、preflight.md blocker判定ロジック等）は変更しない
- 既存のAI-DLCフロー（全フェーズのステップ実行）が正常動作することを確認する
- 不変ルール（ステップファイル読み込み必須等）に抵触しないよう設計する

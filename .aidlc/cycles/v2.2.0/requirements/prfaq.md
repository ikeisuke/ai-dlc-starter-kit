# PRFAQ: AI-DLC Starter Kit v2.2.0

## Press Release（プレスリリース）

**見出し**: AI-DLC Starter Kit、コンテキストサイズを半減しセッション品質を大幅改善

**副見出し**: スキルファイルの初回ロード量を107KBから40KB以下に削減し、長時間セッションでの応答品質劣化を防止

**発表日**: 2026-04-05

**本文**:

AI-DLC Starter Kitのスキルファイル群は、フェーズ開始時に100KB超のプロンプトテキストを読み込んでおり、コンテキストウィンドウの大部分を消費していました。これにより、セッション中盤でのコンパクション（自動要約）が頻発し、重要な指示が欠落する品質問題が生じていました。

v2.2.0では、Wave 1（低リスク・高効果）の5戦略を実施します。Reviewingスキル9種の共通ボイラープレートを共通基盤に抽出し、バージョンチェックロジックを外部ガイドに移動、プリフライトチェックの冗長な説明をテーブル圧縮、インラインテンプレートを外部化、先読み指示パターンを廃止します。

「長いInceptionセッションでも、最後までレビュー指示が正しく実行されるようになりました。以前はセッション後半でAIが手順を飛ばすことがありましたが、コンテキスト消費が減ったおかげで安定しています。」（想定ユーザーの声）

今後はWave 2（構造変更）、Wave 3（アーキテクチャ変更）を通じて、さらなるコンテキスト最適化を進める予定です。

## FAQ（よくある質問）

### Q1: 既存のconfig.toml設定は変更が必要ですか？
A: いいえ。旧キー `markdown_lint` を使用している場合もフォールバックにより自動的に `enabled` として読み取られます。既存設定はそのまま動作します。

### Q2: Wave 1だけで十分な効果がありますか？
A: Wave 1の5戦略で34-47KB（バイト数ベース）の削減が見込まれ、初回ロードを約半減できます。多くのユースケースでコンパクション頻度が大幅に低下するため、Wave 1のみでも実用上十分な効果があります。

### Q3: Reviewingスキルの共通基盤化で、スキルの動作は変わりますか？
A: 動作は変わりません。共通セクション（実行コマンド・セッション継続・セルフレビューモード等）を共有ファイルに抽出するだけで、各スキル固有の「レビュー観点」はそのまま維持されます。

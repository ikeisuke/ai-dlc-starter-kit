# 意思決定記録 - v2.2.0

## DR-001: Waveスコープの選定

- **ステップ**: Intent明確化
- **日時**: 2026-04-05

### 背景

#519のIssueにはWave 1-3の3段階の圧縮戦略が定義されていた。サイクルのスコープをどこまで含めるか決定が必要だった。

### 選択肢

| # | 選択肢 | メリット | デメリット |
|---|--------|---------|-----------|
| 1 | Wave 1のみ | 低リスク・高効果（34-47KB削減）、工数が限定的 | Wave 2-3の効果は得られない |
| 2 | Wave 1+2 | 46.5-73.5KB削減、構造改善も含む | 中リスクの変更が入り、テスト範囲が広がる |
| 3 | Wave 1+2+3 | 最大削減（60.5-99.5KB）| 高リスクのアーキテクチャ変更を含む |

### 決定

**Wave 1のみ** を採用

### トレードオフと判断根拠

- **得たもの**: 低リスクで目標（40KB以下）を達成できる見込み、サイクル期間の短縮
- **犠牲にしたもの**: さらなる圧縮効果（Wave 2-3の追加削減分）
- **判断根拠**: Wave 1だけで初回ロードを約半減できる見込みがあり、品質優先のトレードオフスライダーに合致。Wave 2-3は別サイクルで実施可能

## DR-002: 付随Issueのスコープ包含

- **ステップ**: サイクルバージョン決定
- **日時**: 2026-04-05

### 背景

#519（コンテキスト圧縮）に加えて、#530（linting設定不整合）、#531（defaults.tomlコメント）、#534（情報提示ルール）をサイクルに含めるか判断が必要だった。

### 選択肢

| # | 選択肢 | メリット | デメリット |
|---|--------|---------|-----------|
| 1 | #519のみ | スコープが最小、集中できる | 小規模な既知バグが残る |
| 2 | #519 + #530 + #531 + #534 | 小規模修正をまとめて解消 | スコープが若干広がる |

### 決定

**#519 + #530 + #531 + #534** を採用

### トレードオフと判断根拠

- **得たもの**: 既知のバグ/改善を一括解消、リリース回数の削減
- **犠牲にしたもの**: サイクルスコープの若干の拡大
- **判断根拠**: #530/#531/#534はいずれも小規模（1-2ファイル）で主目的に影響しない。ユーザーの「まとめてやろう」という判断を尊重

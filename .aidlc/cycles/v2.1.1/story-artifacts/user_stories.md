# ユーザーストーリー

## Epic 1: レビューフロー信頼性向上

### ストーリー 1: レビューサマリの確実な生成 (#495)
**優先順位**: Must-have

As a AI-DLCユーザー
I want to AIレビュー完了時にreview-summary.mdが必ず生成される
So that レビュー指摘と対応判断の履歴を追跡できる

**受け入れ基準**:
- [ ] AIレビュー完了後、対応するreview-summary.mdファイルが生成されている
- [ ] review-flow.mdのレビュー完了時共通処理でサマリ更新が必須ステップとして明記されている
- [ ] 計画承認前レビューではサマリを作成しない（既存仕様通り）
- [ ] セルフレビューモードでもサマリが生成される

**技術的考慮事項**:
review-flow.mdの「レビューステップ完了時の共通処理」セクション内のサマリ更新指示を強化する。

### ストーリー 2: タスクリスト駆動による工程漏れ防止 (#493)
**優先順位**: Must-have

As a AI-DLCユーザー
I want to 各フェーズの作業開始時にタスクリスト（チェックリスト）が生成され、レビュー工程を含む全ステップが追跡される
So that レビュー工程のスキップや完了処理の漏れを防止できる

**受け入れ基準**:
- [ ] Inception/Construction/Operationsの各ステップファイルで、レビュー工程を含むタスク管理指示が明示されている
- [ ] Construction Phaseのステップファイル（02-design.md, 03-implementation.md）でAIレビュー実施がタスクとして明記されている
- [ ] Inception Phaseの完了処理ステップ（05-completion.md）で意思決定記録・履歴記録がタスクとして明記されている
- [ ] 既存のステップファイル構造を大幅に変更しない（指示の追加・強化のみ）

**技術的考慮事項**:
全フェーズの大規模リファクタリングは行わず、ステップファイルへのタスク管理指示の強化に限定する。意思決定記録の保証はストーリー3で対応し、本ストーリーでは工程追跡の仕組みに集中する。

## Epic 2: スコープ管理の強化

### ストーリー 4: スコープ縮小時のユーザー確認必須化 (#497)
**優先順位**: Must-have

As a AI-DLCユーザー
I want to レビュー指摘への対応でスコープが縮小される場合に必ず確認を求められる
So that 要件が暗黙的に狭められることを防止できる

**受け入れ基準**:
- [ ] rules.mdにスコープ縮小時のユーザー確認必須ルールが追加されている
- [ ] スコープ縮小を伴う提案時はユーザー確認なしに先へ進まないことが明記されている
- [ ] スコープ縮小を伴わない修正では追加確認を要求しないことが明記されている
- [ ] automation_modeに関わらず、スコープ変更時はユーザー確認が必須と明記されている
- [ ] ルール違反時の行動指針（レッドフラグ）が定義されている

**技術的考慮事項**:
rules.mdの「Overconfidence Prevention原則」セクション付近にスコープ保護ルールを追加する。

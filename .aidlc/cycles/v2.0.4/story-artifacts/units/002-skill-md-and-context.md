# Unit: SKILL.md パス解決 + 追加コンテキスト対応

## 概要
`/aidlc` オーケストレーターの SKILL.md を更新し、`steps/` パスがプラグインベースディレクトリから正しく解決されるようにする。また、`/aidlc <action> <追加コンテキスト>` で追加テキストをオーケストレーターに渡せるようにする。

## 含まれるユーザーストーリー
- ストーリー 2: ステップファイルのパス解決（SKILL.md修正部分）
- ストーリー 3: 追加コンテキスト渡し

## 責務
- SKILL.md 内の `steps/` パス参照をプラグイン内相対パスに修正
- ARGUMENTS パーシングロジックの更新（action + 追加コンテキストの分離）
- 追加コンテキストをオーケストレーター内でコンテキスト変数として保持する仕組み

## 境界
- ステップファイル自体の内容変更は最小限（パス参照の更新のみ）
- 新機能の追加は行わない

## 依存関係

### 依存する Unit
- Unit 001: プラグインディレクトリ構造構築（依存理由: steps/ ファイルの配置が完了している必要がある）

### 外部依存
- Claude Code の SKILL.md ARGUMENTS 仕様

## 非機能要件（NFR）
- 該当なし

## 技術的考慮事項
- ARGUMENTS の先頭トークンを action、残りを追加コンテキストとしてパースする
- 追加コンテキストが空の場合は従来通りの動作

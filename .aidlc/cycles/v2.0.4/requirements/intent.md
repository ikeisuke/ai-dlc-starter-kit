# Intent（開発意図）

## プロジェクト名
ai-dlc-starter-kit

## 開発の目的
AI-DLCスターターキットをClaude Codeプラグインリポジトリに構造変換する。現在の「aidlc-setupでファイルをプロジェクトにコピーする」方式を廃止し、`claude install` でインストールできるプラグイン形式に完全移行する。併せて、v1から残存しているパス関連の不整合を解消する。

## ターゲットユーザー
AI-DLCを使用するClaude Codeユーザー

## ビジネス価値
- インストール・アップデートが `claude install` / `claude update` で完結する
- プロジェクト側にコピーされるファイルがなくなり、メンテナンスが容易になる
- `/aidlc` スキルのパス解決エラーが解消される
- SKILL.md の `steps/` 参照がプラグイン内で自己完結する

## 含まれるもの
1. **プラグインリポジトリ構造への変換**: `skills/` ディレクトリにオーケストレーター含む全スキルを配置
2. **`/aidlc` 追加指示サポート**: `/aidlc <action> <追加コンテキスト>` で action 以降のテキストを追加コンテキストとしてオーケストレーターに渡せるようにする
3. **SKILL.md の steps/ パス解決修正**: プラグインのベースディレクトリからの相対パスで正しく解決されるようにする
4. **aidlc-setup コピー方式の廃止**: rsync同期・マニフェスト管理を削除
5. **#429**: セットアップ時のスターターキットパス判定をスキル化に合わせて改善
6. **#430**: defaults.tomlのパス設定がv1のまま残っている問題の修正
7. **#431**: v1のrsyncコピー処理の残存を削除
8. **#433**: 同期マニフェストにレビュースキル・marketplace.jsonを追加（同期方式変更に伴う対応）

## 含まれないもの
- #432 初回バージョン提案を3択にする（次サイクルへ）
- 既存プロジェクトのv1→v2移行スクリプトの変更
- full_autoモード等の新機能追加

## 成功基準
- `claude install` でこのリポジトリからプラグインがインストールできる
- `/aidlc inception` で steps/ ファイルが正しく読み込まれる（パス解決エラーなし）
- `/aidlc inception 追加コンテキスト` で追加テキストがオーケストレーターに渡される
- aidlc-setup のコピー関連処理が削除されている
- defaults.toml のパス設定が v2 の構造と整合している

## 期限とマイルストーン
1サイクル内で完了

- プラグインリポジトリの構造は Claude Code のプラグイン仕様に準拠する必要がある
- 既存の ikeisuke-skills マーケットプレイスの実績を参考にする

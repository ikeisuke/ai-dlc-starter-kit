# Unit: 部分対応Issue自動判別

## 概要

PR Closes記載時に部分対応Issueを自動判別し、Closes/Relatesを区別する機能を追加する。Unit定義ファイルの「関連Issue」セクションに部分対応記法を導入し、`get-related-issues` の出力とPR Closes確認ステップを改善する。

## 含まれるユーザーストーリー

- ストーリー 3: PR Closes記載時の部分対応Issue自動判別

## 責務

- Unit定義テンプレートの「関連Issue」セクションに部分対応記法の説明追加
- `pr-ops.sh` の `get-related-issues` コマンドにCloses/Relates区別出力を追加
- `operations-release.md` のPR Closes確認ステップに自動検証追加
- PR本文テンプレートのCloses/Relates記載対応

## 境界

- 既存Unitの自動修正は行わない
- 過去サイクル成果物への注記追加は行わない
- PR本文の自動編集機能は含まない（検証と提案のみ）

## 技術的考慮事項

- `get-related-issues` の出力形式変更は呼び出し側への影響あり（後方互換性の維持が必要）
- 正規表現で `#NNN（部分対応）` パターンを検出
- 注記なしの `#NNN` は既存互換で「完全対応」扱い

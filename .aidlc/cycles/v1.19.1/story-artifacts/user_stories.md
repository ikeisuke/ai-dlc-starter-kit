## Epic: レビューフロー品質改善

### ストーリー 1: レビュースキルの外部ツール優先（#285）

**優先順位**: Must-have

As a AI-DLCを使用する開発者
I want to レビュースキルが外部ツール（codex等）利用可能時に外部ツールを優先して使用する
So that 外部AIによる高品質なレビューを確実に受けられる

**受け入れ基準**:

- [ ] `review-flow.md` の「優先ツール決定・利用可否チェック」セクションで、スキル存在確認に加えて外部CLIの可用性確認（PATH上の存在チェック）が行われる
- [ ] 外部CLI（codex/claude/gemini）がPATH上に存在する場合、ステップ5（外部AIレビュー）が実行される
- [ ] 外部CLIが全て利用不可の場合のみ、ステップ6（セルフレビュー提案）にフォールバックする
- [ ] ステップ5内でスキル実行が失敗した場合（CLIタイムアウト・実行エラー等）、エラー理由を表示した上でステップ6へ遷移するフォールバック処理が定義されている
- [ ] 3つのレビュースキルのSKILL.mdの`compatibility`行にセルフレビューモードでは外部CLI不要である旨が追記されている

**技術的考慮事項**:

- `prompts/package/prompts/common/review-flow.md` と `.claude/skills/reviewing-*/SKILL.md` の両方を変更
- 既存のセミオートゲート判定フローとの整合性を維持
- フロー改修と各スキルSKILL.md修正は依存関係あり（フロー改修が先行）

### ストーリー 3: 改善提案のバックログ登録ルール（#289）

**優先順位**: Must-have

As a AI-DLCを使用する開発者
I want to AIが「次のサイクルで対応」「将来的に改善」等の提案をする際に、必ずバックログに記録される
So that 口頭提案がセッション終了で消えることなく追跡可能になる

**受け入れ基準**:

- [ ] `rules.md` の「コード品質基準」セクション直前に「改善提案のバックログ登録ルール」セクションが追加されている
- [ ] 「次のサイクル」「将来的に」「改善の余地がある」等の改善提案時にバックログ登録が必須とされている
- [ ] バックログに記録せずに口頭で提案だけすることが明示的に禁止されている
- [ ] `backlog_mode` に応じた登録方法が記載されている:
  - `issue`: GitHub Issue作成。`gh` CLI未利用時はファイルベース（`docs/cycles/backlog/`）にフォールバック
  - `issue-only`: GitHub Issue作成。`gh` CLI未利用時はフォールバック不可、ユーザーに手動Issue作成を要求
  - `git` / `git-only`: ローカルファイル作成（`docs/cycles/backlog/`）

**技術的考慮事項**:

- `rules.md` の「コード品質基準」セクション直前に新セクションとして追加

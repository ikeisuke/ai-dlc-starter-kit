# ドメインモデル: preflight.md圧縮

## 概要

preflight.mdの構成要素を「仕様層」と「説明層」に分類し、説明層のみを圧縮対象とすることで機能を保ちつつトークン数を削減する。

## 不変条件

1. **判定ロジックと公開契約は変更しない**（全手順共通。手順1-3のblocker判定を含む）
   - 判定ロジック: severity判定、フォールバック条件分岐、派生解決の順序
   - 公開契約: コンテキスト変数名、結果セクション構造、警告メッセージ契約
2. **説明層のみ変更可**: 判定ロジック・公開契約に該当しない補足文・注記・例示
3. **デフォルト値の正本は defaults.toml**: preflight.md側はデフォルト値を参照情報として記載しない（defaults.tomlに正本を委譲）
4. **圧縮後トークン数 ≤ 2,268**（v2.2.2比 800tok以上削減）

## 層分類の基準

### 仕様層（変更禁止）

| カテゴリ | 具体例 |
|---------|--------|
| 判定テーブル | severity判定、オプションチェック条件テーブル |
| 入出力形式 | バッチモード出力形式、結果提示フォーマットのセクション構造 |
| 解決順序本体 | 派生ロジックの番号付きステップ（1→2→3の順序と条件分岐） |
| フォールバック条件 | 旧キー参照、自動導出テーブル |
| 警告テンプレート | エラー・警告メッセージの `{変数}` を含むテンプレート文字列 |
| コンテキスト変数名 | `gh_status`, `depth_level`, `automation_mode` 等の全変数名 |

### 説明層（圧縮対象）

| カテゴリ | 具体例 |
|---------|--------|
| 重複説明 | 「出力から以下を抽出し〜」（テーブルで自明） |
| 冗長注記 | 手順2の注記（手順5に記載あり）、手順5の評価順注記 |
| 出力例 | 手順4のkey:value形式サンプル |
| 見出し装飾 | セクション前の説明前文 |
| デフォルト値列 | defaults.tomlに正本があるため重複 |
| 互換エイリアス | v2.2.3時点で不要 |
| 実行順序まとめ | 手順見出しで自明な繰り返し |

## ユビキタス言語

- **仕様層**: 判定ロジック・公開契約を含む変更禁止要素
- **説明層**: 仕様の理解を補助する補足文・注記・例示
- **公開契約**: コンテキスト変数名、結果セクション構造等、下流フェーズとの接点
- **blocker判定**: 環境チェック（手順1-3）でフェーズ続行可否を決定するロジック

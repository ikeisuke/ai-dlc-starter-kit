# ドメインモデル: 設定値に応じた条件ロードスキップ

## 概要

フェーズステップファイル内の共通ファイル読み込み指示に条件分岐を付与し、設定値に応じて不要なロードをスキップする。

## 不変条件

1. フェーズステップファイル自体のロードは省略不可
2. steps/common/配下のファイル内容は変更しない
3. 未定義値・不正値では安全側にフォールバック（ロードする）
4. review-flow.mdは常にロードする（disabled時パス3維持）

## 条件分岐ルール

### review-flow.md関連

| 条件 | 動作 |
|------|------|
| `review_mode=disabled` | review-flow.mdはロードするが、パス3（ユーザーレビュー）に直行する旨を注記 |
| `review_mode` が disabled以外 / 未定義 / 不正値 | 通常通りreview-flow.mdに従う |

### rules-automation.md関連

| 参照種別 | 条件化対象 | 理由 |
|---------|----------|------|
| セミオートゲート判定 | 対象（manual時スキップ） | manual時はユーザー承認固定のため不要 |
| エクスプレスモード仕様 | 除外（常時ロード） | manual時でもexpress起動時に必要 |
| 複雑度判定 | 除外（常時ロード） | express判定で必要 |
| フォールバック条件 | 除外（常時ロード） | エラー時のユーザー確認で必要 |
| コンパクション復帰 | 除外（常時ロード） | 設定再確認で必要 |

## review-flow-reference.mdの扱い

- **分類**: review-flow.mdに委譲（除外）
- **理由**: review-flow.mdの内部分割参照であり、review-flow.md内の遷移で必要時にのみロードされる。フェーズステップファイルからの直接参照はないため、独立した条件化対象ではない

## 上位成果物との差分解消

Unit定義の責務記述「review_mode=disabled時のreview-flow.md/review-flow-reference.mdスキップ条件記述」は、計画レビューにより「review-flow.mdは常時ロード、disabled時はパス3直行を注記」に方針変更された。Unit定義の責務は設計書の方針を正本とし、「disabled時のパス3直行条件注記」と読み替える。

## ユビキタス言語

- **条件ロード**: 設定値に基づくファイル読み込みのスキップ/実行判定
- **セミオートゲート判定**: automation_mode=semi_auto時の自動承認判定ロジック
- **パス3**: review-flow.mdのdisabled時経路（ユーザーレビュー）

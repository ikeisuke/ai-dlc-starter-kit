# 論理設計: preflight.md圧縮

## 概要

preflight.mdの各手順から説明層要素を特定し、仕様層を維持しつつ圧縮する具体的な変更箇所を定義する。

## アーキテクチャパターン

ドキュメントリファクタリング。ファイル構造（手順1-7 + 実行順序まとめ）は維持し、各手順内の冗長要素のみ削除・圧縮する。

#### 派生ロジック分解（手順4の仕様/説明境界）

**history_level**:

| 要素 | 分類 | 編集可否 |
|------|------|---------|
| 解決順序（1.明示→2.旧キー→3.自動導出）| 仕様層 | 不可 |
| フォールバックコマンド `scripts/read-config.sh rules.history.level` | 仕様層 | 不可 |
| 自動導出テーブル（depth_level→history_level） | 仕様層 | 不可 |
| 「上記バッチ取得後〜」の前置き説明 | 説明層 | 除去可 |
| 「（明示オーバーライド）」等の括弧内注釈 | 説明層 | 除去可 |

**markdown_lint**:

| 要素 | 分類 | 編集可否 |
|------|------|---------|
| 解決順序（1.enabled確認→2.旧キー→3.デフォルト） | 仕様層 | 不可 |
| フォールバックコマンド `scripts/read-config.sh rules.linting.markdown_lint` | 仕様層 | 不可 |
| 「上記バッチ取得後〜」の前置き説明 | 説明層 | 除去可 |

**merge_method**:

| 要素 | 分類 | 編集可否 |
|------|------|---------|
| 有効値リスト（merge/squash/rebase/ask） | 仕様層 | 不可 |
| 警告テンプレート | 仕様層 | 不可 |
| 「上記バッチ取得後〜」の前置き説明 | 説明層 | 除去可 |

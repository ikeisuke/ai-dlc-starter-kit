# 実行計画: Unit 004 SKILL.md構造整理

## 概要

SKILL.mdのセクション構造を責務単位で再編し、文書としての可読性を向上させる。

## 不変条件

1. **手順文・条件分岐・コマンド例・テーブルの意味変更は行わない**
2. **500行以内の制限を維持**
3. **必須セクション全てを維持**: 非AIDLCプロジェクトガード、不変ルール、質問と実行の判断基準、承認プロセス、AskUserQuestion使用ルール、ARGUMENTSパーシング、引数ルーティング、共通初期化フロー、制約事項
4. **セクション依存方向を保全**: 主実行線（前提ガード → 実行判断・対話規約 → 引数処理 → 実行フロー → 補助フロー）の順序維持。横断制約（不変ルール・制約事項）とユーティリティ（独立処理）の配置も保全

## 対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| `skills/aidlc/SKILL.md` | セクション構造の整理・整形 |

## 変更方針

### 1. 責務単位での章立て設計

現状: H1(1) → H2(16) → H3(6)。H2が16個ありフラットで、責務の関係性が不明確。

責務グループに基づいてH2を再編し、関連セクションをH3に降格:

| 責務グループ（H2） | 含まれるセクション（H3化） |
|-------------------|------------------------|
| 前提ガード | 非AIDLCプロジェクトガード |
| 不変ルール | （現状維持、子セクションなし） |
| 実行判断・対話規約 | 質問と実行の判断基準、承認プロセス、AskUserQuestion使用ルール |
| 引数処理 | ARGUMENTSパーシング、引数ルーティング |
| 実行フロー | 共通初期化フロー、Expressモード |
| 補助フロー | ワークフロー共通ステップ、独立フロー委譲 |
| ユーティリティ | ヘルプ表示、バージョン表示 |
| 制約事項 | （現状維持） |

### 2. 冗長な記述の簡潔化

- 複数行にわたる説明で一文にまとめられるものを短縮
- 重複する注記の統合

### 3. #549ハーネス互換性への配慮

#549はハーネスがSKILL.md前半セクション（L18-L94）を間引く問題。本Unitは構造整理が主目的であり、ハーネスバグ自体の修正は対象外。ただし構造変更時に以下を配慮する:

- セクションの抽出単位（H2境界）が変わるため、間引きパターンが変化する可能性がある
- 整形後に実際のスキル呼び出しで欠落が改善/悪化していないか確認する

## セクション依存マップ

【主実行線】

# Intent（開発意図）

## プロジェクト名
ai-dlc-starter-kit

## 開発の目的
バージョン管理の比較モデルを2点間（リモート vs ローカル設定）から三角モデル（リモート / インストール済みスキル / ローカル設定）に拡張し、アップグレードフローの信頼性を向上させる。併せてUnit完了時のコンテキストリセット提示に対応内容サマリを追加し、セッション間の情報伝達を改善する。また、全フェーズでセットアップ完了直後からタスク・進捗管理を開始し、初動の作業漏れを防止する。

## ターゲットユーザー
AI-DLCスターターキットの利用者（外部プロジェクト開発者）およびスターターキット自体の開発者（メタ開発）

## ビジネス価値
- バージョン不整合の早期検知により、古いスキルや設定での作業を防止する
- Unit完了時の対応内容サマリにより、セッション間のコンテキスト引き継ぎが改善され、作業の可視性が向上する
- aidlc-setup同期カスタムワークフローの廃止により、責務境界が明確化する
- セットアップ直後からのタスク管理により、レビュー漏れ・手順スキップを防止する

## 成功基準
- 三角モデルの全比較パターン（下記参照）が正しく動作し、各パターンで定義されたアクションが提示される
- Unit完了時のコンテキストリセット提示に対応内容サマリ（必須項目: 下記参照）が含まれる
- `.aidlc/rules.md` の「aidlc-setup同期」カスタムワークフローが廃止される
- starter_kit_version更新の信頼性が向上する（アップグレード案内に確認手順を明示）
- 全フェーズでセットアップ完了直後にタスクリストが作成される。Inception Phaseではprogress.mdもセットアップ直後に作成される

### 三角モデル比較パターンと期待アクション

| パターン | 状態 | 期待アクション |
|---------|------|--------------|
| 全一致 | remote = skill = local | アクションなし（最新） |
| リモートのみ新しい | remote > skill = local | スキル更新を促す（プラグイン再インストール） |
| スキルのみ古い | remote = local > skill | スキル更新を促す |
| ローカルのみ古い | remote = skill > local | `/aidlc setup`（アップグレードモード）を促す |
| ローカルのみ進んでいる | local > remote = skill | 警告表示（設定が先行、スキルが古い可能性） |
| 複数不一致 | 上記以外 | 各差分を表示し、スキル更新→ローカル設定更新の順にアクションを提示 |
| 比較不能 | いずれかの取得失敗 | 取得できた情報のみで比較、不能箇所は警告表示して続行 |

### 対応内容サマリの必須項目

| 項目 | 説明 | 必須/任意 |
|------|------|----------|
| 実施内容 | 主要な変更の箇条書き（関連Issue番号付き） | 必須 |
| 変更対象 | 変更したファイル・モジュールの概要 | 必須 |
| 未対応事項 | スコープ外としたレビュー指摘や残課題 | 必須（なければ「なし」） |
| 次回の着手点 | 次セッションで最初に行うべきこと | 必須 |

- 情報源: AIがUnit作業中のコンテキストから生成。推測で補完しない
- 情報不足時のフォールバック: 「（コンテキスト情報不足のため省略）」と明記

## 既存機能への影響分析

### 後方互換方針
- **既存v2プロジェクト**: バージョン比較ロジックの変更はInception Phase ステップ6内のみ。ステップ6はユーザープロジェクト向けの機能（STARTER_KIT_DEVではスキップ）のため、スターターキット開発リポジトリには影響なし
- **aidlc-setup同期廃止**: `.aidlc/rules.md`のカスタムワークフローセクションの変更のみ。メタ開発リポジトリ固有の設定であり、外部プロジェクトの`.aidlc/rules.md`には存在しない
- **タスク管理タイミング変更**: ステップファイルの指示位置変更のみ。既存の`task-management.md`テンプレート自体は変更しない

### フェイルセーフ
- バージョン情報の取得失敗時: 取得できた情報のみで比較し、不能箇所は警告を表示して続行（フェーズをブロックしない）
- インストール済みスキルのバージョン取得失敗時: 従来の2点間比較（リモート vs ローカル設定）にフォールバック

## 期限とマイルストーン

### 必須成果物（今サイクルで完了）
- 三角モデル比較ロジックの実装とステップ6への組み込み
- 対応内容サマリのフォーマット定義とステップファイルへの反映
- aidlc-setup同期カスタムワークフローの廃止

### 持ち越し可能事項
- なし（全項目を今サイクルで完了する）

## 制約事項
- メタ開発リポジトリ固有の変更（aidlc-setup同期廃止）を含むため、外部プロジェクトへの影響がないことを確認する
- バージョン比較ロジックはプラグイン構成（v2.0.5以降）を前提とする
- 対応内容サマリはAIのコンテキスト内情報から生成する（外部ソース不要）

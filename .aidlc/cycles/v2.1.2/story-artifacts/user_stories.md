# ユーザーストーリー

## Epic: アップグレードフロー信頼性向上 (#503)

### ストーリー 1: 三角モデルバージョン比較
**優先順位**: Must-have

As a AI-DLCスターターキット利用者
I want to リモート・スキル・ローカル設定の3点でバージョン比較される
So that バージョン不整合を早期に検知し、適切なアクションを案内される

**受け入れ基準**:
- [ ] Inception Phase ステップ6で、リモート・スキル・ローカル設定の3点バージョンを取得する
- [ ] 全一致（remote = skill = local）の場合、「アクションなし（最新）」と表示される
- [ ] リモートのみ新しい（remote > skill = local）場合、スキル更新を促すメッセージが表示される
- [ ] スキルのみ古い（remote = local > skill）場合、スキル更新を促すメッセージが表示される
- [ ] ローカルのみ古い（remote = skill > local）場合、`/aidlc setup`の実行を促すメッセージが表示される
- [ ] ローカルのみ進んでいる（local > remote = skill）場合、警告メッセージが表示される
- [ ] 複数不一致（上記以外）の場合、各差分を表示し、スキル更新→ローカル設定更新の順にアクションを提示する
- [ ] いずれかの取得に失敗した場合、取得できた情報のみで比較し、警告を表示して続行する
- [ ] スキルバージョン取得失敗時は従来の2点間比較にフォールバックする

**技術的考慮事項**:
- スキルバージョンの取得にはスキルベースディレクトリの `version.txt` を使用
- 比較ロジックはプロンプト内でAIが実行（スクリプト不要）

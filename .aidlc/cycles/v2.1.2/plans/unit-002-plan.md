# Unit 002 計画: 対応内容サマリ表示

## 概要
全フェーズのcompletion.mdのコンテキストリセット提示に「対応内容」セクションを追加する。

## 変更対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| `skills/aidlc/steps/construction/04-completion.md` | Unit完了時・全Unit完了時のコンテキストリセットに対応内容サマリ追加 |
| `skills/aidlc/steps/inception/05-completion.md` | Inception完了時のコンテキストリセットに対応内容サマリ追加 |
| `skills/aidlc/steps/operations/04-completion.md` | サイクル完了時のコンテキストリセットに対応内容サマリ追加 |

## 実装計画

各completion.mdのコンテキストリセット提示フォーマットに「対応内容」セクションを追加:

```markdown
**対応内容**:
- **実施内容**: [主要な変更の箇条書き（関連Issue番号付き）]
- **変更対象**: [変更したファイル・モジュールの概要]
- **未対応事項**: [スコープ外としたレビュー指摘や残課題。なければ「なし」]
- **次回の着手点**: [次セッションで最初に行うべきこと]
```

- 情報源: AIのコンテキスト内情報から生成
- 情報不足時: 該当項目に「（コンテキスト情報不足のため省略）」の固定文言
- 情報源に存在しないIssue番号・ファイル名・作業内容を出力しない

## 完了条件チェックリスト
- [ ] Construction Phase（Unit完了時）のコンテキストリセットに対応内容サマリが含まれる
- [ ] Construction Phase（全Unit完了時）のコンテキストリセットに対応内容サマリが含まれる
- [ ] Inception Phase完了時のコンテキストリセットに対応内容サマリが含まれる
- [ ] Operations Phase完了時のコンテキストリセットに対応内容サマリが含まれる
- [ ] 必須4項目（実施内容・変更対象・未対応事項・次回の着手点）が定義されている

# ユーザーストーリー - v1.18.0

## 共通NFR: セミオートモード履歴記録仕様

セミオートモードで自動処理を行った場合、以下の形式で履歴に記録する:
- **自動承認時**: `【セミオート】AIレビュー合格により自動承認 - [対象ステップ名]`
- **自動遷移時**: `【セミオート】自動遷移 - [遷移元] → [遷移先]`
- **フォールバック時**: `【セミオート・フォールバック】[理由] - ユーザー確認へ移行`

## Epic 1: セミオートモード（#164）

### ストーリー 1: セミオートモードの有効化設定
**優先順位**: Must-have

As a AI-DLC開発者
I want to `docs/aidlc.toml` でセミオートモードを有効化できる
So that AIレビュー合格時に手動承認を省略して開発サイクルを高速化できる

**受け入れ基準**:
- [ ] `docs/aidlc.toml` に `[rules.semi_auto]` セクションが追加されている
- [ ] `enabled = true` でセミオートモードが有効化される
- [ ] `enabled = false`（デフォルト）で以下の完全互換が保証される:
  - 全ての承認プロンプトが表示される（スキップされない）
  - フェーズ・Unit遷移時にコンテキストリセット提示が行われる
  - 履歴出力にセミオート関連の記録が含まれない
- [ ] `docs/aidlc.toml.local` で個人設定として上書きできる

**技術的考慮事項**:
- `prompts/package/` 内のテンプレート・プロンプトを修正
- `read-config.sh` で設定値を取得

---

### ストーリー 2: AIレビュー合格時の自動承認
**優先順位**: Must-have

As a AI-DLC開発者
I want to AIレビューで指摘0件のとき自動的に次ステップへ進む
So that レビュー合格した成果物に対して手動で「OK」と入力する手間を省ける

**受け入れ基準**:
- [ ] AIレビュー指摘0件時、ユーザー承認プロンプトをスキップして次ステップへ自動遷移する
- [ ] 計画承認、設計レビュー承認、コードレビュー承認がAIレビュー合格（指摘0件）で自動承認される
- [ ] AIレビューで1件以上の指摘が残った場合（重要度問わず）、通常フローに戻りユーザーに確認を求める
- [ ] 自動承認した旨が共通NFRの履歴記録仕様に従って記録される

**技術的考慮事項**:
- `review-flow.md` のユーザーレビューフロー分岐にセミオート判定を追加
- 各フェーズプロンプトの承認ポイントにセミオート分岐を追加
- 自動承認の判定条件は「指摘0件」のみ（重要度による分岐は行わない）

### ストーリー 3: フェーズ・Unit間の自動遷移
**優先順位**: Must-have

As a AI-DLC開発者
I want to フェーズ完了時やUnit完了時に自動で次へ遷移する
So that コンテキストリセット・手動指示なしに連続して開発を進められる

**受け入れ基準**:
- [ ] Inception Phase完了後、コンテキストリセット提示をスキップしてConstruction Phaseへ自動遷移する
- [ ] Construction PhaseのUnit完了後、次の実行可能Unitへ自動遷移する
- [ ] Unit選択は番号順（Unit定義ファイルの連番 = 実行優先度）で自動選択する
- [ ] Construction Phase全Unit完了後、Operations Phaseへ自動遷移する
- [ ] 未完了Unitがあるが実行可能Unitが0件の場合（全Unitがブロック中）、ユーザーにフォールバックし、ブロック中のUnit一覧・ブロック理由・推奨アクションを実行ログに表示する
- [ ] 依存関係の循環が検出された場合、ユーザーにフォールバックし、循環に含まれるUnit一覧を実行ログに表示する
- [ ] 自動遷移した旨が共通NFRの履歴記録仕様に従って記録される

**技術的考慮事項**:
- 各フェーズ完了時のコンテキストリセット提示をセミオート時スキップ
- Unit選択は番号の昇順（番号は依存関係に基づく実行順序を表す、inception.md定義済み）

### ストーリー 4: ユーザー確認フォールバック
**優先順位**: Must-have

As a AI-DLC開発者
I want to セミオートモードでも不明点や問題がある場合はユーザーに確認を求める
So that 品質を担保しつつ自動化のメリットを享受できる

**受け入れ基準**:
- [ ] 必須入力が欠落している場合、ユーザーに確認を求める
- [ ] 指示に矛盾がある場合、ユーザーに確認を求める
- [ ] AIレビューで1件以上の指摘が残った場合、ユーザーに確認を求める
- [ ] 外部依存エラー（gh CLI失敗等）が発生した場合、ユーザーに確認を求める
- [ ] 未完了Unitがあるが実行可能Unitが0件の場合、ユーザーにフォールバックする
- [ ] Unit依存関係循環が検出された場合、ユーザーにフォールバックする
- [ ] フォールバック発生時は共通NFRの履歴記録仕様に従って理由が記録される

**技術的考慮事項**:
- フォールバック条件を各プロンプト内で統一的に定義

# Unit 002 計画: レビューサマリ指摘一覧詳細化

## 概要

レビューサマリファイルの指摘一覧テーブルにおける「内容」列・「対応」列の記述ガイダンスを強化し、後から問題と対応内容を把握できるようにする。

**正本関係**: `review-flow.md` が仕様の正本（記述ルール定義）、`review_summary_template.md` はその具体例。テンプレートは規約への参照整合を満たすこと。

## 変更対象ファイル

1. `prompts/package/prompts/common/review-flow.md` - 指摘一覧テーブルフォーマット定義の拡張（正本）
2. `prompts/package/templates/review_summary_template.md` - テンプレートへの記述例追加（具体例）

## 依存関係

- Unit 001（セミオートモード実装）: 完了済み。同一ファイル `review-flow.md` を変更するため、Unit 001完了後に実施。Unit 001の変更は現ブランチに反映済み。

## 記述契約（列ごとの必須フィールド）

既存テーブル列の構造は変更しない（列追加なし）。記述ガイダンスの強化で対応する。

- **「内容」列**: `[対象ファイル/箇所] [問題事象]`（例: `review-flow.md のセルフレビュー手順 - 反復回数上限が未定義`）
- **「対応」列**: `[対応種別]（[変更箇所]: [変更内容]）`（例: `修正済み（review-flow.md L640: 反復回数上限3回を明記）`）

## 実装計画

### Phase 1: 設計

1. **記述ルール定義**: 「内容」列・「対応」列の記述契約（必須要素・記述順・最小粒度・NG例）を定義
2. **設計レビュー**: AIレビュー + ユーザー承認

### Phase 2: 実装

3. **review-flow.md の修正**:
   - 「レビューサマリファイル更新手順」セクション内の指摘一覧テーブルフォーマットにガイダンス追加
   - 「内容」列: 記述契約に基づくガイダンス
   - 「対応」列: 対応種別パターンと記述契約に基づくガイダンス
   - 指摘対応判断フロー内の履歴記録フォーマットも同様に整合

4. **review_summary_template.md の修正**:
   - 記述例セクション追加（良い例・悪い例の対比）
   - review-flow.md の記述契約への参照

5. **統合とレビュー**: AIレビュー + ユーザー承認

## 完了条件チェックリスト

- [ ] `review-flow.md` の指摘一覧テーブルフォーマット定義が拡張されている（「内容」列・「対応」列の詳細化）
- [ ] `review_summary_template.md` テンプレートに記述例が追加されている
- [ ] 「内容」列に対象ファイル・問題の具体的コンテキストを記載するガイダンスが含まれている
- [ ] 「対応」列に修正内容の具体的な変更点を記載するパターンが定義されている
- [ ] テンプレートが規約（review-flow.md）への参照整合を満たしている

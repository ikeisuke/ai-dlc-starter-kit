# Unit 005 計画: Amazon AI-DLCリポジトリ調査

## 概要

Amazon公開の AI-DLC リポジトリを調査し、本プロジェクト（AI-DLC Starter Kit）との差分・共通点を整理した調査レポートを作成する。

## 調査対象

- **対象リポジトリ**: Amazon の AI-DLC 関連公開リポジトリ
  - 選定基準: GitHub で `amazon` または `aws` org から公開されている AI-DLC（AI-Driven Development Lifecycle）関連リポジトリ
  - リポジトリ特定後、URL・ブランチ・調査時点のコミットSHAをレポートに記録する
- **調査日**: 2026-03-01
- **比較対象（本プロジェクト側）**: `cycle/v1.18.0` ブランチ時点の `docs/aidlc/` 配下

## 変更対象ファイル

- `docs/cycles/v1.18.0/requirements/amazon-aidlc-report.md`（新規作成）

## 実装計画

### Phase 1: 設計（調査計画策定）

1. **ドメインモデル設計**: 調査レポートの構造定義（比較軸、評価基準、レポートフォーマット）
   - 比較表スキーマ: `axis`（比較軸）, `current_state`（本プロジェクト）, `amazon_state`（Amazon側）, `gap`（差分）, `evidence`（根拠）
   - 取り込み候補スキーマ: `priority`（P0-P2）, `effort`（S/M/L）, `impact`（1-5）
2. **論理設計**: 調査手順、情報収集方法、分析アプローチの設計
3. **設計レビュー**: 調査計画の承認

### Phase 2: 実装（調査実施・レポート作成）

4. **調査実施**: Amazon AI-DLCリポジトリの情報収集・分析
   - リポジトリ構成の把握
   - プロンプト・テンプレートの分析
   - ワークフロー・自動化の調査
   - 品質管理プロセスの調査
5. **レポート作成**: 4軸比較表と取り込み候補リストの作成
6. **統合とレビュー**: レポートのレビューと最終化

## 完了条件チェックリスト

- [ ] 4軸での比較が完了していること（プロンプト構成・構造、フェーズ・ワークフロー設計、自動化・ツール連携、品質管理・レビュープロセス）
- [ ] 取り込み候補の優先度付きリストが作成されていること
- [ ] 調査レポートが以下の構成で作成されていること:
  1. 対象リポジトリの概要（構成・目的・規模）
  2. 4軸比較表
  3. 差分・共通点の分析
  4. 取り込み候補リスト（各候補に優先度・適用難易度・期待効果を付記）

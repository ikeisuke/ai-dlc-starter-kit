# ドメインモデル: Amazon AI-DLC リポジトリ調査

## 概要

Amazon公開の AI-DLC リポジトリと本プロジェクト（AI-DLC Starter Kit）を4軸で比較分析し、取り込み候補を優先度付きでリストアップする調査レポートの構造を定義する。

**重要**: このドメインモデル設計では**コードは書かず**、レポート構造と評価基準の定義のみを行います。

## レポート構成（エンティティ相当）

### 調査レポート

調査レポートは以下の4セクションで構成される。

#### セクション1: 対象リポジトリの概要

- **リポジトリ情報**: URL, ブランチ, 調査時点のコミットSHA
- **目的**: リポジトリが目指すゴール・ユースケース
- **規模**: ファイル数、ディレクトリ構成の深さ、プロンプト数
- **技術スタック**: 使用言語、フレームワーク、ツール

#### セクション2: 4軸比較表

4つの比較軸それぞれについて以下のスキーマで比較する。

| カラム | 型 | 説明 |
|--------|-----|------|
| axis | Axis (enum) | 比較軸（下記4軸のいずれか） |
| sub_item | string | 軸内の具体的な比較項目 |
| current_state | string | 本プロジェクトの現状 |
| amazon_state | string | Amazon側の実装・設計 |
| gap | string | 差分の要約 |
| evidence | Evidence (structured) | 根拠情報（下記構造参照） |

**Axis enum 定義**:

| 値 | 表示名 |
|----|--------|
| `prompt_structure` | プロンプト構成・構造 |
| `workflow_design` | フェーズ・ワークフロー設計 |
| `automation` | 自動化・ツール連携 |
| `quality_management` | 品質管理・レビュープロセス |

**Evidence 構造**:

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| source_type | enum | 必須 | `file`（ファイル参照）/ `url`（URL参照） |
| path_or_url | string | 必須 | source_type=file: リポジトリルート基準のパス、source_type=url: URL |
| quote | string | 任意 | 該当箇所の引用テキスト（補足情報として付記） |

**4つの比較軸**:

1. **プロンプト構成・構造**: プロンプトのディレクトリ構造、ファイル分割方針、テンプレート管理方法
2. **フェーズ・ワークフロー設計**: 開発ライフサイクルのフェーズ定義、フェーズ間の遷移ルール、ワークフロー制御
3. **自動化・ツール連携**: スクリプト、CI/CD、外部ツール統合、コマンドラインツール
4. **品質管理・レビュープロセス**: コードレビュー、AIレビュー、テスト戦略、品質ゲート

#### セクション3: 差分・共通点の分析

- **共通点**: 両プロジェクトに共通するアプローチ・設計思想
- **差分**: 異なるアプローチを取っている箇所とその理由の推察
- **優位性分析**: 各差分について、どちらのアプローチが優れているかの評価

#### セクション4: 取り込み候補リスト

取り込み候補は以下のスキーマで記録する。

| カラム | 型 | 説明 |
|--------|-----|------|
| id | integer | 連番 |
| title | string | 取り込み候補の名称 |
| description | string | 概要説明 |
| priority | enum | P0（即座に取り込むべき）/ P1（次サイクルで検討）/ P2（将来検討） |
| effort | enum | S（小規模）/ M（中規模）/ L（大規模） |
| impact | integer | 1-5（5が最大） |
| related_axis | Axis (enum) | 関連する比較軸 |
| notes | string | 補足事項・適用時の注意点 |

## 評価基準（値オブジェクト相当）

### 優先度（Priority）

- **P0**: 本プロジェクトに明確な改善をもたらし、適用コストが低い
- **P1**: 改善が期待されるが、設計変更やテストが必要
- **P2**: 参考情報として保持、将来的な検討対象

### 適用難易度（Effort）

- **S**: プロンプト・テンプレートの追加修正のみで完了
- **M**: 複数ファイルの変更やスクリプト修正が必要
- **L**: アーキテクチャレベルの変更が必要

### 期待効果（Impact）

- **5**: 開発効率に大幅な改善
- **4**: 品質や安定性に明確な向上
- **3**: 開発体験の改善
- **2**: 軽微な改善
- **1**: 参考になる程度

## ユビキタス言語

- **AI-DLC**: AI-Driven Development Lifecycle。AI主導の開発ライフサイクル手法
- **比較軸（Axis）**: 2つのプロジェクトを比較するための観点カテゴリ
- **取り込み候補（Adoption Candidate）**: Amazon側のアプローチで本プロジェクトに適用可能なもの
- **ギャップ（Gap）**: 2つのプロジェクト間の差異

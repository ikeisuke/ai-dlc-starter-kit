# Unit 003 計画: Codex PRレビュー絵文字リアクション検出

## 概要

PRマージ前ゲートにCodex PRレビューの絵文字リアクション（👍/👀）検出を追加し、レビュー完了の自動判定精度を向上させる。現在のPRマージ前レビューコメント確認（`rules.md` 6.6.7相当）は、`@codex review` コメントへの返信コメントのみで判定しているが、Codexは絵文字リアクション（👍=承認/👀=確認中）で応答するパターンがあり、これを検出できていない。

## 変更対象ファイル

| ファイル | 変更種別 | 概要 |
|---------|---------|------|
| `docs/cycles/rules.md` | 修正 | PRマージ前レビューコメント確認セクションに絵文字リアクション検出ロジックを追加 |
| `prompts/package/prompts/operations-release.md` | 修正 | 7.13 PRマージ前レビューのサブステップ2に絵文字リアクション検出の統合ポイントを追加（必要に応じて） |

## 設計方針

### リアクション検出の位置

既存の「PRマージ前レビューコメント確認」（`rules.md` 6.6.7相当）のステップ3（未対応指摘の判定）に、新たな判定項目「c. 絵文字リアクション判定」を追加する。

### リアクション判定ルール

- 対象: `@codex review` を含むPRコメントの最新1件
- API: `gh api repos/{owner}/{repo}/issues/comments/{comment_id}/reactions`
- 有効なリアクション: Codexボットアカウントからの 👍 または 👀 のみ
- 👀（確認中）と👍（承認）の優先度: 👍が存在すれば承認、👀のみの場合は確認中として扱う

### フォールバック

- API失敗時: 既存のコメントベース判定にフォールバック（リアクション判定をスキップ）
- Codexボットアカウントの特定失敗時: 同上

### Codexボットアカウント

GitHub Appとしての `login` を確認し、`rules.md` に定数として記載する。

# 論理設計: Claude Code 許可設定推奨パターン策定

## 概要

ai-agent-allowlist.md セクション4.1に追加するサブセクションの構成を定義する。

## ドキュメント構成

既存のセクション4.1構成を維持し、「使い分けの指針」の後に新しいサブセクションを追加する。

```text
### 4.1 Claude Code（既存）
├── 設定ファイル（既存・維持）
├── 優先順位（既存・維持）
├── 設定例（AI-DLC推奨）（既存・維持 ← 汎用リファレンスとして残す）
├── 設定のポイント（既存・維持）
├── ワイルドカード（既存・維持）
├── 絞り込み例（既存・維持）
├── 使い分けの指針（既存・維持）
└── 【新規】AI-DLC運用時の推奨設定パターン ← 追加
    ├── なぜ相対パスを推奨するか
    ├── ミニマル推奨セット（allow）
    │   ├── AI-DLCスクリプト群
    │   ├── Git/jj操作
    │   ├── GitHub CLI
    │   └── AIレビューSkill
    ├── オプション追加（allow）
    │   ├── 外部AIツール
    │   └── ユーティリティ
    ├── deny推奨（セキュリティ）
    └── 既存設定からの移行ガイド
```

## 各サブセクションの内容定義

### 「なぜ相対パスを推奨するか」
- worktree環境での絶対パス問題
- チーム共有・テンプレート化の観点
- 具体例: 絶対パスと相対パスの比較

### 「ミニマル推奨セット」
- 純粋JSON形式の設定例（コピペ可能）
- 各エントリの用途説明はJSON外の本文・表で提供（JSONはコメント非対応のため）
- カテゴリ見出しでグループ化

### 「オプション追加」
- プロジェクト固有の追加パターン
- 各項目に「いつ必要か」の説明

### 「deny推奨」
- 機密ファイル保護のdenyパターン
- 追加推奨項目（プロジェクトに応じて）

### 「既存設定からの移行ガイド」
- 現在の設定を推奨パターンに切り替える手順
- フルパスエントリの特定と置換方法

## 非機能要件への対応

### セキュリティ
- deny設定で機密情報保護を明示
- `gh:*` のようなワイルドカードのリスク説明（`gh repo delete`等）を注記

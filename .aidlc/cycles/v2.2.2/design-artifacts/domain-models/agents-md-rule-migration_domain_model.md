# ドメインモデル: AGENTS.md活用・ルール移行

## 概要

rules-core.mdから3セクション（承認プロセス、質問と実行の判断基準、AskUserQuestion使用ルール）をAGENTS.mdに移行する。AGENTS.mdは全スキル呼び出し時に自動注入されるため、フェーズステップ読み込み前に必要なルールの早期適用を実現する。

**重要**: このドメインモデル設計では**コードは書かず**、構造と責務の定義のみを行います。

## エンティティ（Entity）

### RuleSection
- **ID**: セクション見出し文字列
- **属性**:
  - heading: String - セクション見出し（例: "承認プロセス【重要】"）
  - content: Markdown - セクション本文
  - importance: Enum(全スキル共通/フェーズ固有) - 適用範囲

## 集約（Aggregate）

### RuleSource（rules-core.md）
- **集約ルート**: rules-core.md
- **含まれる要素**: RuleSection（移行対象3セクション + 残存セクション）
- **不変条件**: 移行後も残存セクションの文脈が整合していること

### RuleTarget（AGENTS.md）
- **集約ルート**: AGENTS.md
- **含まれる要素**: 既存コンテンツ + 移行されたRuleSection
- **不変条件**: 既存コンテンツとの整合性を維持、`@rules-core.md` 参照は維持

## 移行マッピング

| 移行セクション | rules-core.md位置 | AGENTS.md挿入位置 |
|--------------|-------------------|------------------|
| 質問と実行の判断基準 | L30-76 | `@rules-core.md` 参照の前 |
| 承認プロセス | L78-83 | 同上 |
| AskUserQuestion使用ルール | L85-110 | 同上 |

## ユビキタス言語

- **自動注入**: AGENTS.mdがスキル呼び出し時にシステムプロンプトとして自動読み込みされること
- **初期ロード**: フェーズステップファイル読み込み前に利用可能な情報

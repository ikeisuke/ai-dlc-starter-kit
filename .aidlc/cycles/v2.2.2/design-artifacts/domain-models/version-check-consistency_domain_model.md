# ドメインモデル: バージョンチェック一貫性修正

## エンティティ

### バージョンチェックフロー

- **責務**: 3点（リモート、スキル、ローカル設定）のバージョン情報を取得・比較し、不一致時に警告を表示する
- **制約**: リポジトリ種別（STARTER_KIT_DEV / USER_PROJECT）による分岐を持たない。全プロジェクトで同一のComparisonModeと表示アクションを適用する

### リポジトリ種別判定

- **責務**: 参照先ポリシー（スキル内リソース編集対象、除外対象）の決定のみ
- **境界**: バージョンチェック結果の解釈・警告文面への影響を持たない

## 値オブジェクト

- **ComparisonMode**: THREE_WAY / REMOTE_LOCAL / SKILL_LOCAL / REMOTE_SKILL / SINGLE_OR_NONE（リポジトリ種別に依存しない）
- **バージョン情報**: semver文字列 + available/unavailableフラグ

## 不変条件

- バージョンチェックフローの入出力にリポジトリ種別を参照するパスが存在しない
- 同一のバージョン情報セットに対して、STARTER_KIT_DEVでもUSER_PROJECTでも同一の警告が出力される

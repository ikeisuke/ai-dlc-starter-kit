# ドメインモデル: upgrading-aidlcスキルのスクリプト化

## コンテキスト

upgrading-aidlcスキルのアップグレード処理をスクリプト化し、AIエージェントが`setup-prompt.md`を対話的に実行する現行方式から、`upgrade-aidlc.sh`による一括実行方式に移行する。

## ドメインエンティティ

### 1. UpgradeOrchestrator（upgrade-aidlc.sh）

アップグレード処理全体を統括する薄いコーディネータ。

**属性**:
- `mode`: 実行モード（`execute` | `dry-run`）
- `config_path`: aidlc.tomlのパス
- `starter_kit_root`: 解決されたスターターキットルートパス
- `force`: 強制実行フラグ
- `skip_sync`: 同期スキップフラグ
- `skip_migrate`: マイグレーションスキップフラグ

**制約**:
- 既存のサブスクリプトの内部変更は行わない（呼び出しのみ）
- 出力はkey:value形式に統一する
- サブスクリプト出力は透過的に転送する（プレフィックス二重化防止）
- 冪等性を保証する（各サブスクリプトが冪等なため、順次呼び出しで達成）

### 2. SetupTypeResolver（check-setup-type.sh - 既存）

プロジェクトのセットアップ状態を判定する。

**出力値と意味**:
| setup_type | 意味 | upgrade-aidlc.shでの扱い |
|------------|------|-------------------------|
| `upgrade:<project>:<kit>` | アップグレード可能 | 正常系：実行続行 |
| `cycle_start` | バージョン同じ | `--force`で強制実行、なければスキップ |
| `warning_newer:<project>:<kit>` | プロジェクトが新しい | 警告出力して続行 |
| `initial` | 初回セットアップ | エラー終了 |
| `migration` | 旧形式移行 | エラー終了 |
| （空値） | dasel未インストール | 警告出力、version.txtのみで判定 |

### 3. UpgradeBranch

アップグレード変更を分離するためのブランチ。

**属性**:
- `name`: `upgrade/vX.X.X`（X.X.Xはアップグレード先バージョン）
- `base`: 現在のブランチ（main, cycle/vX.X.X等）

**制約**:
- ブランチ作成はAIエージェント（SKILL.md）の責務であり、upgrade-aidlc.shの責務ではない
- upgrade-aidlc.shはファイル変更のみを行い、git操作は行わない

### 4. UpgradePR

アップグレード変更のプルリクエスト。

**属性**:
- `title`: `chore: AI-DLCをバージョンX.X.Xにアップグレード`
- `base`: UpgradeBranchのbase
- `head`: UpgradeBranch

**制約**:
- PR作成はAIエージェント（SKILL.md）の責務
- gh未認証時・ネットワークエラー時はwarning出力でスキップ

### 5. SkillDefinition（SKILL.md）

AIエージェントが読み込むスキル定義。アップグレードフローの指示書。

**責務**:
- `upgrade-aidlc.sh`の呼び出し手順の記述
- dry-run結果の提示方法
- ブランチ・PR作成フローの記述
- 利用プロジェクトモード時の環境変数設定手順

**制約**:
- Bashコードブロック内で`$()`コマンド置換を使用しない（Unit 001ルール準拠）
- スクリプトパスはvar-pattern（事前にBashで取得）で記述

## 境界コンテキスト

```
┌─────────────────────────────────────────────────┐
│ SKILL.md（AIエージェントフロー制御）             │
│  - 環境変数設定（利用プロジェクトモード時）       │
│  - upgrade-aidlc.sh呼び出し                      │
│  - ブランチ作成・コミット・プッシュ・PR作成       │
│  - ユーザーへの結果提示                          │
└──────────────────┬──────────────────────────────┘
                   │ 呼び出し
┌──────────────────▼──────────────────────────────┐
│ upgrade-aidlc.sh（オーケストレーション）          │
│  - 自前パス解決（BASH_SOURCE判定）               │
│  - 種別判定・バージョン取得                       │
│  - サブスクリプトの順次呼び出し                   │
│  - 結果集約・出力                                │
│  - 前提: カレントディレクトリ = プロジェクトルート │
└──────┬───────┬───────┬───────────────────────────┘
       │       │       │
  ┌────▼──┐┌───▼──┐┌──▼────┐
  │check  ││sync  ││migrate│  ← 既存スクリプト
  │-type  ││-pkg  ││-config│    （変更なし）
  └───────┘└──────┘└───────┘
```

**注**: `setup-ai-tools.sh`は同期後のパス（`docs/aidlc/bin/`）から実行する。同期によって最新版に更新された後に実行するため、意図的にスターターキットルート基点ではなくプロジェクトルート基点で参照する。

## 責務分離ルール

| 層 | 責務 | git操作 | ファイル変更 |
|----|------|---------|-------------|
| SKILL.md | フロー制御、ユーザーインタラクション | あり | なし（upgrade-aidlc.sh経由） |
| upgrade-aidlc.sh | サブスクリプト呼び出し、結果集約 | なし | 直接変更: バージョン更新のみ（他はサブスクリプト経由） |
| サブスクリプト群 | 個別処理の実行 | なし | 各自の責務範囲 |

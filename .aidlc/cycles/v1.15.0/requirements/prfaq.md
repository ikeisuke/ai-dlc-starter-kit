# PRFAQ: AI-DLC Starter Kit v1.15.0

## Press Release（プレスリリース）

**見出し**: AI-DLC Starter Kit v1.15.0 - コミット履歴の自動整理とSkills化基盤整備

**副見出し**: Unit完了時のコミット自動squash機能とプロンプト構造リファクタリングにより、開発ワークフローの品質を向上

**発表日**: 2026-02-14

**本文**:

AI-DLC Starter Kit v1.15.0では、開発ワークフローの2つの課題を解決します。

1つ目は、PRのコミット履歴が冗長になる問題です。AI-DLCのワークフローでは、レビュー前コミット・レビュー反映コミット・Unit完了コミットなど、1つのUnitに対して複数のコミットが作成されます。v1.15.0では、Unit完了時に中間コミットを自動的にsquash（まとめ）する機能を導入しました。git環境（`git reset --soft` 方式）とjj環境（`jj squash` 方式）の両方で動作し、インタラクティブな操作なしで安全に実行できます。

2つ目は、各AI-DLCフェーズをSkillsとして独立提供するための基盤整備です。現在のプロンプト構造を分析し、Skills化に向けた整理方針を策定。コミット処理の統合やプロンプト全体のリファクタリングを実施し、将来のSkills化をスムーズに進められる構造を整備しました。

「PRのコミット数が多すぎて読みづらかったのですが、squash機能のおかげでUnit単位で1コミットにまとまり、レビューがとても楽になりました」（AI-DLCユーザー）

今後は、整備した基盤を活用して各AI-DLCフェーズをSkills形式で提供し、KiroCLIなど多様なAIツールからの直接呼び出しを実現する予定です。

## FAQ（よくある質問）

### Q1: squash機能は必須ですか？

A: いいえ。Unit完了時にsquash実行の確認が表示されますが、スキップ可能です。スキップした場合は従来通り複数コミットが維持されます。

### Q2: jj環境でもsquashは動作しますか？

A: はい。`docs/aidlc.toml` の `[rules.jj].enabled = true` 設定時は自動的にjj用のsquash処理（`jj squash`）が選択されます。`jj undo` で復旧も可能です。

### Q3: squash後にコミット履歴を元に戻せますか？

A: はい。git環境では `git reflog` から復旧可能、jj環境では `jj undo` で復旧できます。

### Q4: Skills化はv1.15.0で完了しますか？

A: いいえ。v1.15.0ではSkills化に向けた構造整理（分析・方針策定・リファクタリング）のみを行います。実際のSkillファイル作成は次サイクル以降で実施します。

### Q5: 既存のプロジェクトへの影響はありますか？

A: squash機能は任意で使用できるため、既存のワークフローに影響はありません。プロンプトのリファクタリングも現行の実行手順（コミットタイミング、メッセージ形式）は維持されます。

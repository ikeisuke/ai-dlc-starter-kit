# ユーザーストーリー

## Epic 1: post-merge-cleanup.shの通常ブランチ対応（#377）

### ストーリー 1: 通常ブランチでのクリーンアップ実行
**優先順位**: Must-have

As a 開発者（通常ブランチ運用）
I want to `post-merge-cleanup.sh` を通常ブランチ環境で実行できるようにしたい
So that worktree環境を使わない場合でもマージ後のクリーンアップを自動化できる

**受け入れ基準**:
- [ ] 通常ブランチで `post-merge-cleanup.sh --cycle vX.X.X` を実行すると、終了コード0で完了する
- [ ] 実行後、mainブランチが最新の状態に更新されている
- [ ] 実行後、mainブランチに切り替わっている
- [ ] マージ済みの `cycle/` および `upgrade/` プレフィックスのブランチが削除されている
- [ ] `--dry-run` オプションで実際の変更なく実行予定が表示される
- [ ] worktree環境で実行した場合、従来と同一の処理結果になる（後方互換性）
- [ ] worktree環境では従来通りの出力フォーマット（`step:`, `step_result:`, `status:` 等）が維持される

**異常系の受け入れ基準**:
- [ ] `--cycle` 未指定時: エラーメッセージ「--cycle オプションは必須です」が表示され、終了コード1で終了する
- [ ] 未コミット変更がある状態で実行時: エラーメッセージで状態を通知し、終了コード1で終了する
- [ ] `git pull` 失敗時（ネットワークエラー等）: エラーメッセージでステップと原因を表示し、終了コード1で終了する

**技術的考慮事項**:
- `step_0a()` のworktree判定ロジックを分岐に変更（エラー終了→環境フラグ設定）
- worktree固有のステップ（detached HEAD化）は通常ブランチ用の代替処理（main checkout）に差し替え
- `git -C` による親リポジトリ操作を自リポジトリ操作に変更

## Epic 2: フリクション対処トラブルシューティングガイド（#324）

### ストーリー 2: トラブルシューティングガイド作成
**優先順位**: Must-have

As a AI-DLCユーザー
I want to よくあるフリクションパターンとその対処法がドキュメント化されていてほしい
So that 問題発生時に自己解決でき、作業中断時間を短縮できる

**受け入れ基準**:
- [ ] `prompts/package/guides/troubleshooting.md` が作成されている
- [ ] 以下の5カテゴリがカバーされている（各カテゴリの対象範囲を明確化）:
  - **認証失効**: OAuth/APIトークンの期限切れ、リフレッシュトークンの再利用エラー。対象: gh CLI、codex CLI等の認証関連
  - **外部ツール制約**: ツールのread-only制限、フリーズ、接続エラー。対象: CLIツール自体の問題（AIの判断ミスは除く）
  - **フェーズ逸脱**: AIがフェーズの責務範囲外の作業を行う問題。対象: Inception中にコード実装する等のフェーズ違反
  - **AI誤アプローチ**: AIが技術的に間違った方向に進む場合の指示方法。対象: AIの判断・行動に関する問題（ツール障害は除く）
  - **コンテキストリセット復帰**: セッション中断後の作業再開手順。対象: コンパクション、明示的リセット後の状態復元
- [ ] 各カテゴリに「症状」「原因候補」「対処手順」「関連参照先」が含まれている
- [ ] ユーザーが症状から該当カテゴリを特定でき、対処後の次アクションが判断できる構成になっている
- [ ] 関連参照先が実在するファイルを指しており、参照先がない場合は代替情報が示されている
- [ ] 既存ガイド（worktree-usage.md、error-handling.md等）との重複がなく、必要に応じて相互参照されている

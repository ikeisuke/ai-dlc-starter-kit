# Unit: post-merge-cleanup.sh 通常ブランチ対応

## 概要
`post-merge-cleanup.sh` を拡張し、worktree環境だけでなく通常ブランチ（非worktree環境）でもマージ後のクリーンアップ処理を実行可能にする。

## 含まれるユーザーストーリー
- ストーリー 1: 通常ブランチでのクリーンアップ実行

## 関連Issue
- #377

## 責務
- worktree/通常ブランチの環境自動判定
- 通常ブランチ用のクリーンアップフロー実行（main pull、main checkout、ブランチ削除）
- 既存worktreeフローの後方互換性維持

## 境界
- 新規スクリプトの作成は行わない（既存スクリプトの拡張のみ）
- `post-merge-sync.sh` 等のラッパースクリプトの変更は行わない
- rules.md のワークフロー記述の更新は行わない（必要に応じてバックログ登録）

## 依存関係

### 依存する Unit
- なし

### 外部依存
- git CLI
- bash（標準Unixユーティリティ）

## 非機能要件（NFR）
- **パフォーマンス**: 既存のworktreeフローと同等の実行速度
- **セキュリティ**: 該当なし
- **スケーラビリティ**: 該当なし
- **可用性**: 該当なし

## 技術的考慮事項
- `step_0a()` のworktree判定を環境フラグ方式に変更（`is_worktree` 変数）
- 通常ブランチではステップ3（detached HEAD化）を `git checkout main` に差し替え
- 通常ブランチではステップ1（main pull）を `git pull origin main`（自リポジトリ）に変更
- 出力フォーマットは既存の構造化形式を維持

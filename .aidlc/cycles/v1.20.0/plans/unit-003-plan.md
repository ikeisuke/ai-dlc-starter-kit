# Unit 003 計画書 - Inceptionプロンプト名前付きサイクル統合

## 概要

`prompts/package/prompts/inception.md` に名前付きサイクルのフロー分岐を追加する。

## 変更対象

- `prompts/package/prompts/inception.md` （1ファイルのみ）

## 変更箇所と詳細

### 1. ステップ5.5（サイクルモード確認）の拡張

**現状**: `cycle_mode` の読み取りとバリデーションのみ（L269-293）。末尾に「Unit 003で実装する」注記あり（L293）。

**変更**: 注記（L293）を削除し、以下のモード別分岐ロジックを追加。

#### mode=named の場合

```text
【名前付きサイクルモード】
サイクル名を入力してください（半角英数字・ハイフンのみ、例: waf, auth-system）。
```

- バリデーション: `^[a-z0-9][a-z0-9-]{0,63}$` + 禁止名チェック（後述）
- 失敗時: エラーメッセージ表示 + 再入力
- 成功時: `cycle_name` をコンテキスト変数として保持

#### mode=ask の場合

AskUserQuestion機能で選択肢を提示:

```text
サイクルの種類を選択してください：
1. 名前付きサイクル（例: waf/v1.0.0 - 機能テーマごとに分類）
2. 通常サイクル（例: v1.0.0 - 従来形式）(推奨)
```

- 「1」選択: `mode=named` と同じ名前入力フローへ
- 「2」選択: `mode=default` と同じフローへ
- AskUserQuestion使用のため、無効入力の再入力ループは機能側で担保される

#### mode=default の場合

変更なし（既存動作維持）。現行の `suggested_*` + カスタム名入力 + `all_cycles` 重複チェックフローを完全維持する。

### 2. ステップ6（サイクルバージョン決定）の拡張

**現状**: `suggest-version.sh` を実行してバージョン提案（L295-381）。

**変更**:

#### 6-1. コンテキスト表示の拡張

名前付きサイクル時、直近サイクル表示（L339-358）で名前付きサイクルのディレクトリをスキャンする。

```bash
# 名前付きの場合（cycle_nameが設定されている場合）
ls -d docs/cycles/${cycle_name}/v*/ 2>/dev/null | sort -V | tail -3
# 名前なしの場合（従来）
ls -d docs/cycles/v*/ 2>/dev/null | sort -V | tail -3
```

#### 6-2. バージョン提案（名前付き時のsuggest-version.sh制限対応）

**問題**: ステップ5.5後、まだ `main` ブランチ上にいるため、`suggest-version.sh` は名前付き系列を認識できない（ブランチ名から名前を検出する仕組みのため）。

**対応**: `mode=named` 時は `suggest-version.sh` の出力を**部分的に**利用する:

1. `suggest-version.sh` を実行して `all_cycles` と `suggested_*` を取得
2. AIが `all_cycles` から `${cycle_name}/v*` にマッチする既存バージョンを抽出
3. **既存バージョンがある場合**: 抽出した最新バージョンを基準にpatch/minor/majorを再計算して提案
4. **既存バージョンがない場合**: `v1.0.0` を提案（名前付きサイクルの初回）
5. `suggested_*`（名前なし系列基準）はそのまま使用しない

`mode=default` 時は `suggest-version.sh` の出力をそのまま使用（従来フロー完全維持）。

#### 6-3. `{{CYCLE}}` 変数の組み立て

- `cycle_name` が設定されている場合: `{{CYCLE}}` = `${cycle_name}/${version}`（例: `waf/v1.0.0`）
- `cycle_name` が未設定の場合: `{{CYCLE}}` = `${version}`（例: `v1.0.0`）

#### 重複チェック

`all_cycles` をカンマ分割してトークン単位の完全一致で判定:
- `cycle_name` が設定されている場合: `${cycle_name}/${version}` の完全一致で検索（例: `waf/v1.0.0`）
- `cycle_name` が未設定の場合: `${version}` の完全一致で検索（例: `v1.0.0`）
- 部分一致は使用しない

### 3. 継続プロンプト例の更新

**L78**: 継続プロンプト例 `サイクル vX.X.X` を `サイクル {{CYCLE}}` に変更。名前付き形式（`waf/v1.0.0`）と従来形式（`v1.0.0`）の両方に対応する表記にする。

### 4. ステップ7以降の `{{CYCLE}}` 変数

**重要**: `{{CYCLE}}` は名前付きの場合 `waf/v1.0.0` 形式（名前/バージョン）になる。

以下のスクリプト呼び出しは Unit 002 で名前付き対応済みのため、`{{CYCLE}}` をそのまま渡せば正常動作する:
- `setup-branch.sh {{CYCLE}} branch/worktree`
- `init-cycle-dir.sh {{CYCLE}}`
- `suggest-version.sh`（ブランチ名から自動検出）

以下のパス参照も名前付き形式で正常動作する:
- `docs/cycles/{{CYCLE}}/` → `docs/cycles/waf/v1.0.0/`
- `history/inception.md` 等のサブパスは変更不要

### 5. PR作成時のタイトル

L1044-1047: `--title "サイクル {{CYCLE}}"` は名前付き時 `サイクル waf/v1.0.0` となり、そのままで問題なし。

## バリデーション仕様

### サイクル名の正規表現

```regex
^[a-z0-9][a-z0-9-]{0,63}$
```

- 先頭: 英小文字または数字
- 以降: 英小文字、数字、ハイフン
- 最大64文字
- 空文字不可

### 禁止名チェック

正規表現通過後、以下の追加チェックを実施:

1. **予約ディレクトリ名の禁止**: `backlog`, `backlog-completed` は `docs/cycles/` 直下の共通ディレクトリと衝突するため禁止
2. **`v[0-9]` 開始名の禁止**: `v1`, `v2` 等は名前なしサイクルのバージョンディレクトリ（`v1.0.0` 等）と誤認されるリスクがあるため、`^v[0-9]` で始まる名前を禁止

禁止名一覧:
- `backlog`
- `backlog-completed`
- `^v[0-9]` にマッチする名前（例: `v1`, `v2-feature`）

### エラーメッセージ

バリデーション失敗時:
```text
【エラー】サイクル名 "[入力値]" は無効です。
有効な形式: 半角英小文字・数字・ハイフンのみ（先頭は英小文字または数字、最大64文字）
例: waf, auth-system, feature-1
```

禁止名衝突時:
```text
【エラー】サイクル名 "[入力値]" は予約語のため使用できません。
別の名前を入力してください。
```

`v[0-9]` 開始時:
```text
【エラー】サイクル名 "[入力値]" は "v" + 数字で始まるため、バージョンディレクトリと混同される可能性があります。
別の名前を入力してください（例: feature-v2, ver2 など）。
```

## 後方互換性

- `mode=default` 時は既存フローと完全同一（`suggested_*` + カスタム名入力 + `all_cycles` 重複チェックを維持）
- `{{CYCLE}}` 変数が名前付き形式でも、Unit 002で対応済みのスクリプト群がそのまま処理

## テスト観点

1. `mode=default`: 従来動作が維持されること（カスタム名入力含む）
2. `mode=named`: 名前入力→バージョン選択→ディレクトリ作成→ブランチ作成の一連フロー
3. `mode=named` 初回: 名前付き系列が未存在時に `v1.0.0` が提案されること
4. `mode=named` 2回目以降: 既存バージョンから正しくpatch/minor/majorが再計算されること
5. `mode=ask`: AskUserQuestion選択肢表示→分岐→各フロー
6. バリデーション: 無効な名前、禁止名（backlog, v1等）で再入力要求
7. 重複チェック: 既存の `[name]/[version]` 完全一致でエラー、部分一致は許容
8. 継続プロンプト: 名前付き形式が正しく表示されること

# Unit 003 計画: セルフレビューでのサブエージェント活用

## 概要

review-flow.md のセルフレビューフォールバック（ステップ5.5）を、サブエージェント（コンテキスト分離）方式に変更する。現在のセルフレビューは同一エージェント内で実行されるため、コンテキストの汚染リスクがある。サブエージェントを使用することで、レビュアーとしての独立した視点を確保し、セルフレビューの品質を向上させる。

## 変更対象ファイル

| ファイル | 変更内容 |
|----------|----------|
| `prompts/package/prompts/common/review-flow.md` | ステップ5.5のセルフレビュー実行手順をサブエージェント起動方式に改訂 |

**注意**: `docs/aidlc/prompts/common/review-flow.md` は rsync コピーであるため、ソース (`prompts/package/`) を編集する。

## 実装計画

### Phase 1: 設計

このUnitはプロンプト変更のみであり、コードの実装は行わない。Unit定義に「技術的考慮事項」でサブエージェントの入出力仕様が既に定義されているため、ドメインモデル・論理設計は最小限に留め、review-flow.md の改訂内容の設計に注力する。

#### 設計方針

1. **環境別方針**:
   - **正式サポート: Claude Code のみ**。`Task` ツール（`subagent_type` 指定）によるサブエージェント起動を主方式とする
   - **その他のAIツール**: 参考情報としてサブプロセス/サブセッションでの代替手段を記載する。実行方式は未保証であり、失敗時はインラインレビューにフォールバックする
   - **サブエージェント起動失敗時**: 従来のインラインレビューにフォールバック

2. **メイン/サブエージェント責務分担**:
   - **サブエージェント**: レビュー実行のみ（対象ファイルの読み取り、観点に基づくレビュー、結果出力）
   - **メインエージェント**: 反復制御（最大3回）、指摘対応判断フロー接続、履歴記録、ユーザーへの提示

3. **サブエージェント入力仕様**:
   - `target_files[]`: レビュー対象ファイルパス一覧
   - `review_type`: レビュー種別（code / architecture / security / inception）
   - `rubric_ref`: レビュー観点（対応するレビュースキルの SKILL.md パス）

4. **サブエージェント出力フォーマット**:
   - `status`: `ok`（レビュー完了）/ `failed`（レビュー実行失敗）
   - `findings[]`: 指摘一覧。各要素の必須フィールド: `{id: string, severity: 高|中|低, content: string, recommendation: string, judgement: 未対応|対応済}`。空配列 `[]` = 指摘0件
   - **`status=failed` 時**: フォールバック（インラインレビュー）に移行

5. **Claude Code固有の実装詳細**:
   - `Task` ツール（`subagent_type` 指定）を使用
   - サブエージェントにはReadツール等の読み取り専用操作のみ推奨

### Phase 2: 実装

`prompts/package/prompts/common/review-flow.md` のステップ5.5を改訂:

1. セルフレビュー実行手順をサブエージェント方式に変更
2. サブエージェントへの指示テンプレートを追加
3. フォールバック処理（インラインレビュー）を追加
4. 履歴記録フォーマットの更新（レビューツール欄に `self-review(subagent)` / `self-review(inline)` を記載）

# ドメインモデル: PRマージ前のリモート同期確認ステップ追加

## 概要

PRマージ前に未pushコミットがないことを確認するステップのロジックフローを定義する。push漏れによるデータ損失リスクを防止する。

**重要**: このドメインモデル設計では**コードは書かず**、構造と責務の定義のみを行います。実装はImplementation Phase（コード生成ステップ）で行います。

## ドメインサービス

### リモート同期確認サービス

- **責務**: ローカルの未pushコミットを検出し、ユーザーに対応を促す
- **操作**:
  - リモート名解決
  - リモート最新化（fetch）
  - 未pushコミット検出
  - 結果に応じたユーザーへのメッセージ出力

## 処理フロー

### メインフロー: リモート同期確認

```text
[開始]
  │
  ├─ リモート名解決
  │    ├─ upstream設定あり → upstream のリモート名を取得（例: origin, upstream）
  │    └─ upstream未設定 → origin にフォールバック
  │
  ├─ git fetch {remote} 実行
  │    ├─ 成功 → 次へ
  │    └─ 失敗 → エラーメッセージ表示、マージ停止
  │
  ├─ リモート追跡ブランチ解決
  │    ├─ @{u} が存在 → @{u} を使用
  │    └─ @{u} が未設定 → {remote}/{branch} にフォールバック
  │         ├─ {remote}/{branch} が存在 → {remote}/{branch} を使用
  │         └─ {remote}/{branch} も不存在 → マージ停止 + 手動push確認を促す
  │
  ├─ git log {remote_ref}..HEAD で未pushコミット検出
  │    ├─ コマンド失敗（終了コード非0）→ エラーメッセージ表示、マージ停止
  │    ├─ 空（差分なし）→ 次のステップ（6.7 PRマージ）へ進む
  │    └─ 非空（未pushコミットあり）→ 警告メッセージ + push実行を促す + マージ停止（再実行必須）
  │
[終了]
```

### エッジケース

| ケース | 対応 |
|--------|------|
| upstream未設定 | リモート名を `origin` にフォールバック |
| `git fetch` 失敗（ネットワークエラー等） | エラー表示、マージを停止、ユーザーに対処を提示 |
| upstream (`@{u}`) 未設定 | `{remote}/{branch}` にフォールバック |
| `{remote}/{branch}` も存在しない | マージを停止し、手動でpush状態を確認してから進むよう案内 |
| `git log` コマンド失敗（終了コード非0） | エラー表示、マージを停止 |
| detached HEAD | 通常Operations PhaseではブランチベースのためNot Applicable |

## ユビキタス言語

- **未pushコミット**: ローカルには存在するがリモートにpushされていないコミット
- **upstream tracking branch**: `git push -u` 等で設定されたリモート追跡ブランチ
- **リモート同期**: ローカルとリモートのコミット履歴が一致している状態
- **リモート名**: pushやfetchの対象となるリモートリポジトリ名（通常は `origin`）

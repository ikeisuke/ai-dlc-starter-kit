# Unit: aidlc-migrate での個人好みキー移動提案

## 概要

`aidlc-migrate` 実行時に project `.aidlc/config.toml` から「個人好み」7 キー（user_stories.md ストーリー 1 の正規定義に従う）を検出し、各キーごとに `~/.aidlc/config.toml`（user-global）への移動を提案する。dry-run 対応・「そのまま残す」選択時の非破壊動作・全件移動 (yes-to-all) / 全件残す (no-to-all) の対話遷移をサポートする。

## 含まれるユーザーストーリー

- ストーリー 3: aidlc-migrate で個人好みキーの移動提案

## 責務

- 既存 `.aidlc/config.toml` から「個人好み」7 キーを検出する判定ロジックの追加
- 各キーごとに `AskUserQuestion` で 4 択（移動 / 残す / 全件移動 (yes-to-all) / 全件残す (no-to-all)）を提示
- 対話遷移規則: 最初のキーで yes-to-all / no-to-all が選択された時点で、残りのキーは無質問で同一適用される（追加対話なし）。「移動」/「残す」は次キーで再度 4 択
- 移動選択時: project から削除＋user-global に追記（dasel または read/write-config.sh の組み合わせ）
- dry-run モードで差分のみ表示し書き込みを行わない
- 「残す」選択時: 警告ログのみで非破壊

## 境界

- defaults.toml への既定値追加は Unit 001 で完了
- ウィザード新規 setup 時の案内は Unit 002 で対応
- 「再提案抑制 / 次回も提案する」のオプション化は v2.5.0 では未実装（Q4: Construction 設計時に詳細化）

## 依存関係

### 依存する Unit

- Unit 001（前提: defaults.toml 集約 / 4 階層マージで残存値が無害化されている）
- Unit 002（推奨: ウィザード案内と migrate 提案の文言整合のため、文言テンプレートを共有）

### 外部依存

- `dasel` v3+（TOML 編集）

## 非機能要件（NFR）

- **非破壊性**: 「残す」選択時に `.aidlc/config.toml` を変更しない
- **冪等性**: 既に移動済みのキーは再検出しない
- **dry-run 完全性**: dry-run の差分が実際の変更と完全に一致する

## 技術的考慮事項

- `~/.aidlc/config.toml` が存在しない場合は新規作成する（最小ヘッダコメント付き）
- 同一キーが user-global に既に存在する場合は「上書きしますか?」を追加確認
- 配列値（`rules.reviewing.tools`）は完全置換（マージしない、4 階層マージ仕様維持）

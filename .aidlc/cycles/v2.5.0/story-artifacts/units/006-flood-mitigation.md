# Unit: 氾濫緩和（重複検出 + サイクル毎上限）

## 概要

Unit 005 が定義した `mirror` フロー（下書き生成 → AskUserQuestion 承認 → 起票）に対する**後付けフィルタ**として、同一サイクル内 retrospective で重複する `skill_caused=true` 項目を検出し、最初の 1 件のみ Issue 下書き候補に通す。さらに `feedback_max_per_cycle`（デフォルト 3）を超える場合は上限超過分をローカル記録のみとし、対話を抑制する。重複検出スコープは「同一サイクル内」に限定（過去サイクル横断は v2.6.x 以降）。

## 含まれるユーザーストーリー

- ストーリー 7: 氾濫緩和（重複検出 + サイクル毎上限）

## 責務

- `defaults.toml` に `[rules.retrospective] feedback_max_per_cycle = 3` 追加
- 重複検出: 同一サイクル retrospective 内で「skill 引用箇所文字列の正規化一致」または「タイトル類似度しきい値超過」の項目をグルーピング
- 重複グループ内で最初の 1 件のみ Issue 下書き候補とし、残りは「重複候補（最初の項目に統合）」と retrospective.md に記録
- 上限ガード: サイクル内 `skill_caused=true` 項目数が `feedback_max_per_cycle` を超えた時点で、超過分は AskUserQuestion 表示を抑制してログのみ
- 重複検出と上限ガードのユニットテスト
- **Unit 005 への組み込み点**: Unit 005 の「retrospective.md パーサー出力 → 下書き生成」の間に本 Unit のフィルタ関数を差し込む。Unit 005 の API（`build_drafts(parsed_items)` 想定）に対し、本 Unit は `filter_items(parsed_items) -> deduplicated_capped_items` を提供し、Unit 005 が呼び出す形にする

## 境界

- /aidlc-feedback 連動本体は Unit 005 が責務（本 Unit は Unit 005 のフローに**後付けフィルタ**として挿入される）
- 過去サイクル横断の AI 類似判定は v2.6.x 以降（本 Unit は同一サイクル内に限定）
- 優先度フィルタ（自動起票候補 / Mirror 推奨の振り分け）は v2.5.0 スコープ外

## 依存関係

### 依存する Unit

- Unit 004（前提: retrospective テンプレートと `skill_caused` フラグが定義されている）
- Unit 005（前提: 下書き生成 → AskUserQuestion → 起票 のフローが実装されている。本 Unit はその「前段」に挿入される）

### 外部依存

- なし（タイトル類似度は単純な文字列正規化 + Jaccard 係数または編集距離で実装）

## 非機能要件（NFR）

- **AI 推論非依存**: 類似判定に LLM 推論を使わない（実装複雑度を抑える）
- **テスタビリティ**: 重複検出ロジックと上限ガードロジックを純粋関数として切り出し、ユニットテストでカバレッジ 100% を目指す
- **設定上書き互換**: user-global で `feedback_max_per_cycle` を上書きすると即時反映される

## 技術的考慮事項

- 類似度しきい値: タイトル正規化後の Jaccard 係数 0.7 以上、または編集距離が短い側の長さの 30% 以下（候補値、テストで確定）
- 「skill 引用箇所」一致は完全一致 + 正規化（前後空白・全角半角揃え）
- 上限ガード時は履歴に「上限超過のためスキップ: count=N, max=M」を残す

# PRFAQ: AI-DLC Starter Kit v2.0.3

## Press Release（プレスリリース）

**見出し**: AI-DLC Starter Kit v2.0.3 - v2基盤のクリーンアップと品質改善

**副見出し**: 廃止機能の削除、ドキュメント整備、テスト追加でv2基盤の保守性と信頼性を向上

**発表日**: 2026-03-28

**本文**:

v2.0.0〜v2.0.2で構築したv2基盤には、廃止予定のLite版ルーティングやローカルバックログの仕組みが残存していました。また、Kiro設定のドキュメントと実装に矛盾があり、移行スクリプトのテストも不足していました。

v2.0.3では、これらの技術的負債を一掃します。Lite版ルーティングとローカルバックログを正式に廃止し、バックログ管理をGitHub Issueに一本化。Construction Phaseのバックログチェックも効率化し、ドキュメントの矛盾を解消。さらにv1→v2移行スクリプトのE2Eテストを追加し、移行フローの信頼性を担保します。

利用者はよりシンプルで一貫性のある開発体験を得られ、開発者はメンテナンスコストの削減と品質基盤の強化を実現できます。

## FAQ（よくある質問）

### Q1: ローカルバックログ（git/git-onlyモード）を使っている場合はどうなりますか？
A: 旧設定は自動的に`issue`モードにフォールバックし、警告メッセージが表示されます。既存のローカルバックログデータは自動削除されないため、手動でGitHub Issueに移行できます。

### Q2: `/aidlc lite inception` のようなLiteコマンドはどうなりますか？
A: 廃止メッセージが表示され、対応する通常コマンド（`/aidlc inception`等）の使用が案内されます。

### Q3: v1→v2移行スクリプトに変更はありますか？
A: スクリプト本体の変更はありません。E2Eテストが追加されるのみです。

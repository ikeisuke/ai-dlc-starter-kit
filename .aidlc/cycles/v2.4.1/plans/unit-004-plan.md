3. **ikeisuke/norigoro の事例 reference** を 1〜2 行で記述（再現手順までは書かない）

### Phase 2（実装）

`skills/aidlc-setup/steps/01-detect.md` の「早期判定」セクション内、3 条件リスト直後に新規見出し「独立チェックの実装指針【必須】」を挿入。

挿入内容（論理設計で確定後の最終案。Markdown コードフェンスのネストは外側 4 バッククォートで実装し Markdownlint 警告を回避）:

````markdown
#### 独立チェックの実装指針【必須】

「早期判定」の 3 条件は **必ず独立に評価** すること。3 条件チェックの場面で `&&` / `||` での短絡評価は禁止する（`set -e` 環境のフォールバックなど他用途への適用は対象外）。

**正しい例**:

```bash
# 各条件・各サブパスを個別に test し、結果を別々の変数に保存
test -f .aidlc/config.toml; aidlc_v2_present=$?
test -f docs/aidlc.toml; v1_main_present=$?
test -f docs/aidlc/project.toml; v1_project_present=$?
# 集約は事後判定で行う（v1 残骸はいずれか一方が存在すれば true）
```

**禁止例（誤判定の原因）**:

```bash
# &&/|| で短絡評価すると後続条件が評価されない場合がある
test -f .aidlc/config.toml && echo v2 || test -f docs/aidlc.toml && echo v1
```

**事例**: ikeisuke/norigoro で `.aidlc/` 不在時に短絡評価により v1 残骸の検出が漏れた事故が発生（#600）。各条件・各サブパスを独立に評価することで本事故を再発防止する。
````

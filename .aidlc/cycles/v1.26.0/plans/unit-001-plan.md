# Unit 001: バックログモード別フォーマットバリデーション - 計画

## 概要
バックログモード（git, issue, git-only, issue-only）ごとの許可・禁止される成果物形式を排他的バリデーションルールとして `rules.md` に追加する。

## 変更対象ファイル
- `prompts/package/prompts/common/rules.md` — バックログセクションに排他テーブルとバリデーションルールを追加

## 実装計画

### Phase 1: 設計
`prompts/package/prompts/common/rules.md` の「改善提案のバックログ登録ルール」セクションの後に、新セクション「バックログモード別フォーマットバリデーション」を追加。

#### 追加内容
1. **排他テーブル**: 各モードで許可/禁止される保存先を明示
2. **バリデーションルール**: バックログ保存前にモードを確認する手順
3. **違反時の期待動作**: 警告メッセージと正しい形式での保存し直し

### Phase 2: 実装
`rules.md` にプロンプトテキストを追加（コード変更なし）。

## 完了条件チェックリスト
- [ ] `git-only` モード時、ローカルファイルのみ許可・Issue作成禁止がルール化されている
- [ ] `issue-only` モード時、GitHub Issueのみ許可・ローカルファイル作成禁止がルール化されている
- [ ] `git` モード時、ローカルファイル優先・Issue作成も許可がルール化されている
- [ ] `issue` モード時、GitHub Issue優先・ローカルファイル作成も許可がルール化されている
- [ ] バックログ保存前にモード確認し、禁止形式での保存を行わない旨が明記されている
- [ ] 禁止形式での保存を試みた場合のバリデーションエラーメッセージと期待動作が定義されている
- [ ] バリデーションルールは `prompts/package/prompts/common/rules.md` のバックログセクションに追加されている

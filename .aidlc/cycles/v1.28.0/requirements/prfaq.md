# PRFAQ: AI-DLC Starter Kit v1.28.0

## Press Release（プレスリリース）

**見出し**: AI-DLC v1.28.0 — アーキテクチャ品質ガードと意思決定トレーサビリティを実現

**副見出し**: プロジェクトのアーキテクチャスタイルを設定で宣言し、AIレビューで違反を自動検出。さらにInception Phaseの意思決定プロセスを構造的に記録可能に。

**発表日**: 2026年3月

**本文**:

AI-DLCスターターキットは、AIを開発プロセスの中心に据えた開発方法論の実践キットです。v1.28.0では、サイクルを重ねるごとに品質が維持・向上する体制を構築するため、3つの新機能を導入します。

**アーキテクチャスタイル宣言**: `docs/aidlc.toml` に `[rules.architecture]` セクションを追加することで、プロジェクトのレイヤー構成や依存方向ルールを宣言できるようになりました。reviewing-architectureスキルがこの設定を参照し、レビュー時にスタイル違反を自動で指摘します。

**意思決定プロセスの記録**: Inception Phaseで複数の選択肢から意思決定した際に、選択理由と却下理由を構造的に記録する機能を追加しました。後続サイクルで過去の判断を追跡でき、「なぜこの判断をしたか」が失われません。

**AIDLC固有レビュー観点の強化**: reviewing-inceptionスキルにIntent-Unit整合性チェックと意思決定記録の充足性チェックを追加。Inception Phase成果物の品質をAIDLC固有の基準で体系的に評価できます。

「サイクルを重ねてもアーキテクチャが崩れず、過去の判断根拠を参照できる。これがAI駆動開発の持続可能性を支える基盤です」

今後は、Construction/Operations Phase向けのフェーズ別レビュースキルや、品質メトリクスの自動計測・追跡機能の追加を予定しています。

## FAQ（よくある質問）

### Q1: アーキテクチャスタイルを設定しないとどうなりますか？
A: 設定しなくても既存の動作に影響はありません。`defaults.toml` のデフォルト値が適用され、reviewing-architectureスキルは従来の汎用レビューのみを実行します。

### Q2: 意思決定記録はどこに保存されますか？
A: `docs/cycles/{{CYCLE}}/inception/decisions.md` に保存されます。サイクルの成果物としてGit管理され、後続サイクルで参照可能です。

### Q3: 意思決定記録は自動で作成されますか？
A: AIが複数選択肢を提示してユーザーが選択した場面で自動的に記録されます。単一選択肢のみの場合はスキップされるため、過度な記録で作業効率が低下することはありません。

### Q4: フェーズ別レビュースキルはConstruction/Operations Phaseでも使えますか？
A: v1.28.0ではInception Phaseのみ対応です。Construction/Operations Phase向けは観点定義ドキュメントを先行整備し、スキル実装は後続サイクルで対応予定です。

# ドメインモデル: フェーズ別レビュー観点定義ドキュメント

## 概要
各フェーズのAIDLC固有レビュー観点を「チェック対象」「チェック項目」「重要度」の3列テーブル形式で定義するガイドドキュメントの構造設計。

## 構造

### Inception Phase
- Intent-Unit整合性チェック（Unit 003 で実装済み）
- 意思決定記録の充足性チェック（Unit 003 で実装済み）
- 既存の汎用観点（Intent品質、ユーザーストーリー品質、Unit定義品質）

### Construction Phase（後続サイクル向け設計基盤）
- 設計-実装整合性チェック
- テストカバレッジ充足性チェック
- コミット粒度の適切性チェック

### Operations Phase（後続サイクル向け設計基盤）
- リリース準備完了チェック
- ドキュメント更新の完全性チェック
- バックログ処理の完了チェック

# Unit 004 計画: セルフレビュースキル統合

## 概要

review-flow.mdのステップ5.5（セルフレビューフォールバック）を、サブエージェント直接起動からレビュースキル呼び出し方式に変更する。各レビュースキルのSKILL.mdにself-reviewモードの手順を追加する。

## 変更対象ファイル

1. `prompts/package/prompts/common/review-flow.md` - ステップ5.5の変更
2. `prompts/package/skills/reviewing-code/SKILL.md` - self-reviewモード追加
3. `prompts/package/skills/reviewing-architecture/SKILL.md` - self-reviewモード追加
4. `prompts/package/skills/reviewing-security/SKILL.md` - self-reviewモード追加
5. `prompts/package/skills/reviewing-inception/SKILL.md` - self-reviewモード追加

## 実装計画

### 変更1: 各SKILL.mdにセルフレビューモードセクション追加

4つのレビュースキルのSKILL.mdに `## セルフレビューモード` セクションを追加する。

**追加位置**: 各SKILL.mdの末尾（「セッション継続」セクションの後）

**追加内容（4スキル共通構造）**:

```markdown
## セルフレビューモード

引数の先頭トークンが `self-review` の場合、このモードで実行する。
引数の残り部分はレビュー対象ファイルパス（半角スペース区切り）。

### 手順

1. 引数の先頭トークン `self-review` を除去し、残りをレビュー対象ファイルパスとして取得する
2. 上記「レビュー観点」セクションの基準に基づいてレビューを実行する
3. レビュー結果はreview-flow.mdのセルフレビュー出力フォーマットに準拠して返す

### 実行方式

- **サブエージェント方式（推奨）**: Taskツールで `subagent_type: "general-purpose"` を起動し、以下の指示テンプレートを渡す
- **インライン方式（フォールバック）**: サブエージェント起動失敗時、メインエージェント自身がレビューを実施する。フォールバック発生時はその旨を結果に含める

### サブエージェントへの指示テンプレート

（review-flow.mdから移動したテンプレート。レビュー観点部分は本SKILL.mdの「レビュー観点」セクション内容を埋め込む）

### 制約

- ファイルの編集・コマンド実行・外部通信は行わない（読み取り専用）
- セルフレビューは外部ツールに比べて品質が劣る可能性がある
```

**スキルごとの差分**: レビュー観点セクションの参照先が異なるのみ（各SKILL.mdの既存「レビュー観点」セクションをそのまま使用）。指示テンプレートは4スキル共通。

### 変更2: review-flow.md ステップ5.5の変更

**現在の構造**（L572-737）:
- 5.5. セルフレビューフロー（Skills利用不可時のフォールバック）
  - 共通準備（SKILL.mdから観点読み取り）
  - サブエージェント方式（指示テンプレート含む）
  - インライン方式（フォールバック）
  - 反復・完了処理
  - 出力フォーマット
  - 外部入力検証の非適用ルール

**変更後の構造**:
- 5.5. **セルフレビューフロー**（外部AIレビューツール不可時のフロー）
  - レビュー前コミット（変更なし）
  - **レビュー実行方式の判定**（変更: 3段階に変更）
    - A) スキル呼び出し方式（推奨）: 対応するレビュースキルを `self-review` モードで呼び出す
    - B) インライン方式（スキル呼び出し失敗時のフォールバック）: SKILL.mdの観点を読み取って自身でレビュー
    - C) 簡易インライン方式（SKILL.mdも取得不可時の最終フォールバック）: review-flow.md内の種別説明を観点として使用
  - **セルフレビュー実行手順**（変更: スキル呼び出しに簡素化）
    - スキル呼び出し方式: Skill ツールで呼び出し。指示テンプレートはSKILL.md側の責務
    - インラインフォールバック: 共通準備の観点取得手順を残す（SKILL.md読み取り + 不在時の代替観点）
  - 反復・完了処理（変更なし）
  - **出力フォーマット**（変更なし、review-flow.md側がSoT）
  - 外部入力検証の非適用（変更なし）

**スキル呼び出しの具体的な記述**:

```text
レビュー種別に対応するレビュースキルを self-review モードで呼び出す:
- code → reviewing-code
- architecture → reviewing-architecture
- security → reviewing-security
- inception → reviewing-inception

呼び出し方法:
  Skill ツール呼び出し:
    skill: "reviewing-{種別}"
    args: "self-review {target_files を半角スペース区切り}"

引数パース規則:
- 先頭トークン（最初の半角スペースまで）が `self-review` であればセルフレビューモード
- 先頭トークンが `self-review` でない場合は通常モード（外部ツール実行）
- `self-review` 以降の文字列をレビュー対象ファイルパスとして扱う（半角スペース区切り）
- 空白を含むファイルパスは非対応（本プロジェクトのパス規約上、空白パスは存在しない）

スキル呼び出し失敗時: インライン方式にフォールバック（履歴に理由を記録）
```

### 変更の範囲整理

**review-flow.mdから削除/移動するもの**:
- 指示テンプレート（L613-642）→ 各SKILL.mdに移動
- サブエージェント方式の詳細手順（L598-648）→ スキル呼び出しに簡素化

**review-flow.mdに残すもの**:
- レビュー前コミット指示
- スキル呼び出し方式の記述（新規）
- インラインフォールバックの記述（観点取得手順含む。SKILL.md読み取り + 不在時の代替観点）
- 反復・完了処理（手順5-9）
- 出力フォーマット（SoT: review-flow.md側が唯一の定義元）
- 外部入力検証の非適用ルール
- レビュー後コミット・ユーザー承認

**各SKILL.mdに追加するもの**:
- セルフレビューモードセクション（手順・テンプレート・制約）
- 出力フォーマットはreview-flow.mdを参照（「review-flow.mdのセルフレビュー出力フォーマットに準拠」と記述）

## 設計方針

1. **SoT原則**:
   - レビュー観点: 各SKILL.mdが唯一の定義元。review-flow.mdは参照のみ
   - 出力フォーマット: review-flow.mdが唯一の定義元。SKILL.mdは「準拠」と参照のみ
2. **DRY原則**: 指示テンプレートは4スキル共通だが、各SKILL.mdに同一内容を配置する（スキル単独で完結するため）。共通部分の変更時は4ファイル同時更新が必要
3. **依存方向**: 実装依存（呼び出し方向）はreview-flow.md → SKILL（一方向）。仕様参照はSKILL→review-flow.md出力フォーマット（双方向許容）。ただしSKILL.md側からのステップ番号・内部実装への依存は禁止
4. **フォールバック階層**: スキル呼び出し → インラインセルフレビュー（SKILL.md観点参照） → 簡易インラインセルフレビュー（review-flow.md内基準）の3段階
5. **後方互換**: サブエージェント方式の「読み取り専用」制約はSKILL.md側に移動して維持

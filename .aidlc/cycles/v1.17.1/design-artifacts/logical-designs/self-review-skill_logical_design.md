# 論理設計: セルフレビュースキル統合

## 概要

セルフレビューフロー（review-flow.md ステップ5.5）をスキル呼び出し方式に変更し、4つのSKILL.mdにself-reviewモードを追加する論理設計。

## 変更1: review-flow.md ステップ5.5

```
5.5. セルフレビューフロー（Skills利用不可時のフォールバック）
  ├── レビュー前コミット
  ├── レビュー実行方式の判定
  │   ├── サブエージェント方式（推奨）
  │   │   ├── Claude Code: Task ツール + general-purpose
  │   │   ├── その他AI: サブプロセス

  ├── セルフレビュー実行手順
  │   ├── 共通準備（SKILL.md観点読み取り + 不在時代替）
  │   ├── サブエージェント方式（指示テンプレート含む）

### 具体的な変更箇所

#### 見出し変更

- 旧: `5.5. **セルフレビューフロー**（Skills利用不可時のフォールバック）`
- 新: `5.5. **セルフレビューフロー**（外部AIレビューツール不可時のフロー）`

#### 「レビュー実行方式の判定」セクション書き換え

現在の内容（サブエージェント方式 / その他AIツール / 失敗時フォールバック）を以下に置換:

```markdown
**レビュー実行方式の判定**:

セルフレビューは以下の優先順位で実行方式を決定する:

- **A) スキル呼び出し方式（推奨）**: 対応するレビュースキルを `self-review` モードで呼び出す。スキル側がサブエージェント起動・インラインフォールバックを管理する
- **B) インライン方式（スキル呼び出し失敗時のフォールバック）**: スキル呼び出しが失敗した場合、SKILL.mdの「レビュー観点」を読み取り、メインエージェント自身がレビューを実施する
- **C) 簡易インライン方式（SKILL.mdも取得不可時の最終フォールバック）**: SKILL.mdが見つからない場合、review-flow.md内のレビュー種別説明を観点として使用しレビューを実施する

**注意**: A)方式では、サブエージェント管理（起動・結果受信・インラインフォールバック）はスキル側の責務。メインエージェントはスキルからの結果を受け取り、反復制御・修正判断・履歴記録を行う。
```

#### 「セルフレビュー実行手順」セクション書き換え

現在の「共通準備」「サブエージェント方式」「インライン方式」の3サブセクションを以下に置換:

```markdown
#### A) スキル呼び出し方式（推奨）

1. レビュー種別に対応するレビュースキルを self-review モードで呼び出す:

   Skill ツール呼び出し:
     skill: "reviewing-{種別}"
     args: "self-review {target_files を半角スペース区切り}"

   引数パース規則:
   - 先頭トークン `self-review` でモード判定
   - 空白を含むファイルパスは非対応

2. スキルからの結果を受信し判定する:
   - **指摘なし**: 指摘0件として処理（反復・完了処理の手順6へ）
   - **指摘あり**: 各指摘から「重要度」「内容」「推奨修正」の3項目を解析。3項目が揃わない指摘は無効として除外する
   - **フォールバック条件**（以下のいずれかに該当した場合、B)方式にフォールバック。理由を履歴に記録する）:
     - スキル呼び出し失敗（スキルが見つからない、Skillツール自体の実行エラー）
     - 結果解析不能（全指摘が無効、応答が空、形式不一致）

   **フォーマットの階層**:
   - スキル返却フォーマット（内部I/F）: 3項目（重要度・内容・推奨修正）+ 合計行。サブエージェントテンプレートの出力形式
   - 最終出力フォーマット: review-flow.mdの「出力フォーマット」セクション（SoT）。メインエージェントが `判定` 項目・`【セルフレビュー結果】` ヘッダを付与する

#### B) インライン方式（スキル呼び出し失敗時のフォールバック）

1. レビュー種別に応じた観点を確認:
   - 対応するレビュースキルのSKILL.mdを読み取り、「レビュー観点」セクションの内容を `review_criteria` として保持する
   - SKILL.mdが見つからない場合: C)方式にフォールバック
2. `review_criteria` の観点に基づいて対象ファイルをレビューし、出力フォーマットに従って記録する

#### C) 簡易インライン方式（最終フォールバック）

1. review-flow.md内の該当種別の説明（ステップ5のレビュー種別一覧等）を観点として使用する
2. 上記観点に基づいてレビューを実施し、出力フォーマットに従って記録する
```

#### 削除するもの

- 現在の「共通準備（方式共通）」サブセクション全体（手順1-2）→ B)方式の手順1に統合
- 現在の「サブエージェント方式（推奨）」サブセクション全体（手順3-4）→ A)方式に置換
- 現在の「インライン方式（フォールバック）」サブセクション（手順3'）→ B)方式に置換
- Claude Code / その他AIツール の起動方法詳細 → スキル側の責務
- 指示テンプレート（````text ブロック全体）→ SKILL.mdに移動

#### 変更しないもの

- 「反復・完了処理（方式共通）」（手順5-9）: そのまま維持
- 出力フォーマット: そのまま維持（SoT）
- 「外部入力検証の非適用」: そのまま維持
- レビュー後コミット・ユーザー承認: そのまま維持
- 「指摘対応判断フローとの接続ルール」: そのまま維持

## 変更2: 各SKILL.mdのセルフレビューモード追加

### 追加位置

各SKILL.mdの末尾（「セッション継続」セクションの後、リファレンスリンクの後）

### 追加内容（4スキル共通）

```markdown
## セルフレビューモード

引数の先頭トークンが `self-review` の場合、このモードで実行する。
引数の残り部分はレビュー対象ファイルパス（半角スペース区切り）。空白を含むファイルパスは非対応。

### 手順

1. 引数の先頭トークン `self-review` を除去し、残りをレビュー対象ファイルパスとして取得する
2. 上記「レビュー観点」セクションの基準に基づいてレビューを実行する
3. レビュー結果は呼び出し元のフロー（review-flow.md）で定義された出力フォーマットに準拠して返す

### 実行方式

セルフレビューモードでは外部CLI（codex/claude/gemini）は使用しない。以下の方式でレビューを実行する:

- **サブエージェント方式（推奨）**: Taskツールで `subagent_type: "general-purpose"` を起動し、以下の指示テンプレートを渡す
- **インライン方式（フォールバック）**: サブエージェント起動失敗時（Taskツール利用不可含む）、メインエージェント自身がレビューを実施する。フォールバック発生時はその旨を結果に含める

### サブエージェントへの指示テンプレート

````text
以下のファイルをレビューしてください。
あなたの役割は読み取り専用のレビュアーです。ファイルの読み取りと評価のみを行い、ファイルの編集・コマンド実行・外部通信は行わないでください。

**レビュー種別**: {review_type}

**対象ファイル**:
{target_files を改行区切りで列挙}

**レビュー観点**:
{本SKILL.mdの「レビュー観点」セクション内容}

**出力フォーマット**:
レビュー結果を以下のフォーマットで出力してください。

指摘がある場合:

指摘 #1
- 重要度: {高 | 中 | 低}
- 内容: {指摘内容の要約}
- 推奨修正: {修正方法の提案}

指摘 #2
...

合計: {N}件（高: {n}件 / 中: {n}件 / 低: {n}件）

指摘がない場合:
指摘0件
````

### 制約

- ファイルの編集・コマンド実行・外部通信は行わない（読み取り専用）
- セルフレビューは外部ツールに比べて品質が劣る可能性がある
```

### スキルごとの差分

指示テンプレート内の `{本SKILL.mdの「レビュー観点」セクション内容}` 部分が各スキル固有。実行時に各SKILL.mdの既存「レビュー観点」セクション全体を埋め込む。

それ以外の構造・テンプレート・制約は4スキルで完全に同一。

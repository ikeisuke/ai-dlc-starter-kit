# ドメインモデル: Unit 001 pr-ops 空配列展開 bug 修正

## 概要

`pr-ops.sh get-related-issues` サブコマンドが提供する「Unit 定義ファイル群 → 関連 Issue 番号集合」変換のドメイン責務を明確化する。本 Unit は bash スクリプトの局所修正のため、エンティティ/集約等の DDD 構造は採用せず、純粋関数としてのドメインモデルのみを定義する。

**重要**: このドメインモデル設計では**コードは書かず**、構造と責務の定義のみを行う。実装は Phase 2 で行う。

## ドメイン責務

### `cmd_get_related_issues`（決定的変換関数）

- **責務**: サイクル配下の全 Unit 定義ファイルから「関連 Issue」セクションを走査し、Issue 番号集合を 3 つの分類（`closes` / `relates` / `all`）で出力する
- **入力**:
  - サイクル名（`v2.4.0` 等、関数引数）
  - そのサイクル配下 Unit 定義ファイル集合の内容（`${AIDLC_CYCLES}/${cycle}/story-artifacts/units/*.md` の `## 関連Issue` セクション）
- **出力**: 標準出力に 3 行（`issues:<csv>` / `closes:<csv>` / `relates:<csv>`）、空集合の場合は `none` フォールバック
- **依存**: filesystem read（Unit 定義ファイルの読み取り）。書き込み・ネットワーク・状態変更は含まない
- **冪等性**: ファイル集合の内容が同一なら、同じ引数に対して常に同じ出力を返す（決定的）
- **概念上の純粋性**: shell 関数としては filesystem read 依存を持つが、概念上は「Unit 定義集合 → Issue 集合」の純粋な決定的変換として位置付ける

## 入力境界（Unit 定義「関連Issue」セクションの記法）

Unit 定義ファイルの `## 関連Issue` セクションに記載される行のうち、本関数が認識する記法は以下:

| 記法 | 分類 | 例 |
|------|------|-----|
| `- #NNN` | `closes_list` | `- #588` |
| `- #NNN（部分対応）` | `relates_list` | `- #597（部分対応）` |
| `Closes #NNN` 等の自由記述 | `closes_list`（緩い実装上 match するが本テストの境界外） | `Closes #123` |
| その他（`#` を含まない行、空行） | 無視 | `補足: ...` |

**正規表現**（`pr-ops.sh:227-231`）:
- `\#([0-9]+)（部分対応）` → `relates_list` に追加
- `\#([0-9]+)` → `closes_list` に追加（部分対応マッチ後の elif）

**設計上の境界**: 本 Unit のテストでは Unit 定義の標準記法（`- #NNN` / `- #NNN（部分対応）`）のみを入力境界として扱う。他の自由記述（`Closes #NNN` 等）は緩い実装上動作するが、サポート対象として明示しない（仕様変更時の自由度を残す）。

## ユビキタス言語

- **closes_list**: PR マージで自動 close すべき Issue（完全対応）の集合
- **relates_list**: PR と関連はあるが close しない Issue（部分対応）の集合
- **all_list**: `closes_list` と `relates_list` の連結結果（配列としては重複除去前、後段の `sort -u` で csv 化時に重複除去される）。後方互換 `issues:` 行のソース
- **none フォールバック**: 集合が空の場合に csv の代わりに出力する文字列（既存契約）

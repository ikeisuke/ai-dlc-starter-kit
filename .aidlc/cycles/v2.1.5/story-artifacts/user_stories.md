# ユーザーストーリー

## Epic: バグ修正と運用品質向上

### ストーリー 1: aidlc-setupバージョン比較の正常動作 (#516)
**優先順位**: Must-have

As a AI-DLCスターターキット利用者
I want to `/aidlc setup` 実行時にバージョン比較が正常に動作する
So that アップグレードが必要かどうかを正確に判断できる

**受け入れ基準**:
- [ ] `/aidlc setup` 実行時のバージョン比較ステップが正常完了し、ローカルバージョンと比較対象バージョンを用いてアップグレード要否が判定できること
- [ ] ローカルバージョン（`starter_kit_version`）が正常に取得できること
- [ ] スキル間依存ルール（他スキルの内部実装への依存禁止）に違反しないこと
- [ ] （検証補足）`read-config.sh` 呼び出し時に Exit code 127 が発生しないこと

**技術的考慮事項**:
- `aidlc-setup`スキルの`scripts/`に`read-config.sh`が存在しないことが原因
- 対応方法はスキル間依存ルールを遵守する形で検討（Construction Phaseで設計）

### ストーリー 2: post-merge-sync.shの出力ステータス明確化 (#515)
**優先順位**: Must-have

As a AI-DLCスターターキット開発者
I want to post-merge-sync.shの出力で成功と警告を明確に区別できる
So that 補助操作の失敗を見落とさない

**受け入れ基準**:
- [ ] 補助操作（ブランチ削除等）が全て成功した場合に `status:success` が出力されること
- [ ] 補助操作に1件以上の失敗がある場合に `status:warning` が出力されること（exit 0のまま）
- [ ] 旧形式 `status:success(warn:N)` が出力されないこと
- [ ] 既存の正常系フロー（警告0件）の動作が変わらないこと

**技術的考慮事項**:
- 出力をパースする既存スクリプトは確認されていない
- `post-merge-cleanup.sh`が既に`status:success|warning|error`形式を使用しており、一貫性向上

### ストーリー 3: config.toml欠落キー検出・追記候補提示 (#517)
**優先順位**: Must-have

As a AI-DLCスターターキット利用者
I want to `/aidlc setup` 実行時にconfig.tomlに欠落しているキーを検出・提示してもらいたい
So that バージョンアップで追加された新しい設定の存在に気づき、必要に応じて設定できる

**受け入れ基準（検出・提示）**:
- [ ] `/aidlc setup` のアップグレードフロー実行時にdefaults.tomlとconfig.tomlのキー差分が検出されること
- [ ] 欠落キーがリスト形式で追記候補として提示されること
- [ ] 欠落キーが0件の場合は「欠落キーなし」と表示されること
- [ ] defaults.tomlが存在しない場合はスキップされること（エラーにならない）

**受け入れ基準（確認後の追記）**:
- [ ] ユーザー確認後にのみconfig.tomlに追記されること（自動更新はしない）
- [ ] 既存のconfig.toml設定値が上書きされないこと

**技術的考慮事項**:
- defaults.tomlのリーフキー（値を持つ最終キー）単位で判定
- daselを利用してTOML解析（既存のread-config.shと同様の技術スタック）
- コメント・書式の保持は保証対象外（daselによる追記のため）

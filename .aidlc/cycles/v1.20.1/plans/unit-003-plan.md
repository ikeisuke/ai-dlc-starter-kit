# Unit 003 計画: コンテンツバリデーション

## 概要

reviewing-codeスキルのSKILL.mdにASCII図・Mermaid図のバリデーション観点を追加する。

## 変更対象ファイル

- `prompts/package/skills/reviewing-code/SKILL.md`（レビュー観点セクションへの新規カテゴリ追加）

## 現状分析

### 現行レビュー観点（4カテゴリ）

1. 可読性（命名規則、関数長、ネスト深さ、コメント、意図の明確さ）
2. 保守性（SRP、DRY、疎結合・高凝集、拡張性、依存関係）
3. パフォーマンス（計算量、不要な再計算、メモリリーク、I/O効率、N+1問題）
4. テスト品質（カバレッジ、境界値、テスト名、独立性、アサーション）

### 不足している観点

- ASCII図（罫線図）の構造的整合性バリデーション
- Mermaid図の構文・構造バリデーション

## 実装計画

### Phase 1: 設計

1. ドメインモデル・論理設計は不要（SKILL.mdのレビュー観点セクションへの追加のみのため）

### Phase 2: 実装

2. **コード生成**: SKILL.mdのレビュー観点セクションに2カテゴリを追加
3. **テスト**: 追加された観点の網羅性を確認
4. **統合とレビュー**: AIレビュー実施

### 追加カテゴリ仕様

#### 適用条件

ASCII図バリデーション・Mermaid図バリデーションは、対象ファイルにASCII図またはMermaidコードブロック（` ```mermaid `）が含まれる場合にのみ適用する。図が存在しないファイルに対してはこれらのカテゴリは評価しない。

#### カテゴリ5: ASCII図バリデーション

以下の観点を記載する:

- **罫線接続**: ボックス間の接続線（`─`, `│`, `┌`, `┐`, `└`, `┘`, `├`, `┤`, `┬`, `┴`, `┼`, `|`, `-`, `+`）が途切れず接続されているか
- **ラベル対応**: 図中のラベル（ボックス内テキスト、矢印ラベル）が本文の用語・定義と一致しているか
- **交差線ゼロ原則**: 接続線の交差を避けるレイアウトになっているか。交差が不可避な場合は交差理由がコメントまたは本文で説明されているか
- **整列・余白**: ボックスやラベルの水平・垂直方向の整列が一貫しているか

#### カテゴリ6: Mermaid図バリデーション

##### 対象6種別

| 種別 | 宣言キーワード |
|------|-------------|
| flowchart | `flowchart` / `graph` |
| sequenceDiagram | `sequenceDiagram` |
| classDiagram | `classDiagram` |
| stateDiagram | `stateDiagram-v2` / `stateDiagram` |
| erDiagram | `erDiagram` |
| gantt | `gantt` |

##### レビュー観点

- **構文準拠**: Mermaid公式構文に準拠しているか（宣言キーワード、ノード定義、エッジ記法）
- **ノードID/ラベル重複**: 同一図内でノードIDまたはラベルが意図せず重複していないか
- **方向指定**: flowchartで方向（`TB`, `TD`, `LR`, `RL`, `BT`）が明示されているか
- **参加者定義**: sequenceDiagramで`participant`/`actor`が事前定義されているか
- **クラス関係記法**: classDiagramの継承（`<|--`）、実装（`<|..`）、関連（`-->`）等の関係記法が正しいか
- **状態遷移定義**: stateDiagramの遷移（`-->`）、開始状態（`[*]`）、終了状態（`[*]`）が適切に定義されているか
- **関係記法**: erDiagramのカーディナリティ記法（`||--o{`等）が正しいか
- **日付フォーマット**: ganttの`dateFormat`が指定されているか

##### 構文検証対象外の種別

**包括ルール**: 上記対象6種別以外のMermaid図種別はすべて構文検証の対象外とする。対象外種別の既知の例を以下に示すが、このリストは網羅的である必要はなく、新しいMermaid種別が追加された場合も包括ルールにより自動的に対象外となる。

既知の対象外種別: `pie`, `mindmap`, `timeline`, `journey`, `quadrantChart`, `gitGraph`, `c4Context`/`c4Container`/`c4Component`/`c4Deployment`, `block-beta`, `sankey-beta`, `xychart-beta`, `packet-beta`, `kanban`, `architecture-beta`

対象外種別については、Mermaid構文の検証は行わないが、図としての可読性（ラベルの明確さ、本文との整合性）は可読性カテゴリの一般ルールで確認する。

### 追加位置

既存の「テスト品質」カテゴリの後に、「ASCII図バリデーション」「Mermaid図バリデーション」の順で追加する。

### 後方互換性

- 既存4カテゴリは変更しない（新規カテゴリの追加のみ）
- スキルのメタデータ（name, description, allowed-tools等）は変更しない
- セルフレビューモードの動作は変更しない（レビュー観点セクションの拡張のみ）

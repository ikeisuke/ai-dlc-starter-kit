# Unit 007 実装計画: progress.md更新タイミング修正

## 概要

Operations Phaseのステップ6でprogress.mdの更新がPRに含まれるよう、更新タイミングを修正する。

## 問題点

現在のoperations.mdでは、ステップ6の最後（6.7 PRマージ後）に「ステップ完了時: progress.mdでステップ6を「完了」に更新」と記載されている。この配置では、progress.mdの更新がPRマージの**後**に行われるため、main ブランチにマージされるPRにprogress.mdの更新が含まれない。

## 設計判断

**用語の明確化**: progress.mdの「完了」状態を「PR準備完了」と解釈する。6.4.5でprogress.mdを更新し、その時点で「PR準備完了」となる。6.6 PR Ready化、6.6.5 コミット漏れ確認、6.7 PRマージはPR準備完了後の「レビュー・マージ作業」として位置付ける。

**理由**: マージはレビュワーの責務であり、AIの責務は「PRを準備するまで」。progress.mdに記録すべきは「AIの責務範囲内でのPR準備完了」である。

**スコープ外**: Construction Phaseでも同様の問題が発生する可能性があるが、Unit 007のスコープ外としてバックログに登録する。

## 変更対象ファイル

- `prompts/package/prompts/operations.md`

## 実装計画

### 修正内容

1. **サブステップ一覧に「6.4.5 progress.md更新」を追加**
   - 6.4 Markdownlint実行と6.5 Gitコミットの間に挿入

2. **新規セクション「#### 6.4.5 progress.md更新」を追加**
   - 6.4 Markdownlint実行セクションの後に追加
   - `docs/cycles/{{CYCLE}}/operations/progress.md` をPR準備完了（progress.mdの状態は「完了」）に更新し、完了日を記録
   - 「PR準備完了」の意味を注記

3. **6.7 PRマージセクションの最後にある「ステップ完了時」記述を削除**
   - 816行付近の記述を削除（6.4.5に移動するため不要）

4. **「完了時の確認」セクションの更新**
   - 839-848行付近の「## 完了時の確認【重要】」セクションを更新
   - progress.mdでの「完了」は「PR準備完了」を意味することを明記
   - 6.6-6.7はPR準備完了後のレビュー・マージ作業と注記

5. **6.6 PR Ready化セクションへの注記追加**
   - 「PR Ready化後はバグ修正や追加要件がない限りブランチに変更を加えない」ことを明示
   - これにより「PR準備完了 = 6.4.5時点」の定義が明確になる

**サブステップ一覧（569-579行付近）**:

```markdown

**注意**: progress.mdでの「完了」は「PR準備完了」を意味します。この更新をGitコミット（6.5）に含めることで、PRに正確な状態が反映されます。6.6以降はPR準備完了後のレビュー・マージ作業です。
```

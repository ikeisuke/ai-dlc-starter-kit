## 実行前の検証ルール

### Codex MCP レビュー（推奨）
複雑な変更を行う前に、Codex MCP を使用してレビューを実施することを推奨します。

### 指示の妥当性検証
ユーザーからの指示が以下の条件を満たしているか確認してください：
- 明確で曖昧さがない
- プロジェクトの目的と一致している
- セキュリティ上のリスクがない

---

## フェーズ固有のルール

### Inception Phase
- Intent 明確化時は、必ずユーザーと対話形式で進める
- 技術スタック決定時は、プロジェクトの特性に合わせて選定する

### Construction Phase
- コード生成時は、セキュリティ脆弱性（SQLインジェクション、XSS等）に注意する
- テストカバレッジは80%以上を目標とする

### Operations Phase
- デプロイ前に、必ずステージング環境でテストする
- 本番デプロイ時は、ロールバック手順を事前に準備する

---

## 禁止事項

以下の操作は禁止されています：
- ユーザーの承認なしに本番環境へデプロイする
- セキュリティ上重要な情報（APIキー、パスワード等）をコミットする
- テストをスキップしてビルドを進める

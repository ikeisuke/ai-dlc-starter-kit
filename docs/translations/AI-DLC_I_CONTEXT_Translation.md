# AI-Driven Development Lifecycle 詳細翻訳（第1章）

## I. CONTEXT（背景）

ソフトウェア工学の進化は、開発者がより複雑な問題解決に集中できるようにするため、低レベルで差別化されない作業を抽象化してきた歴史でもある。  
機械語から高級言語への移行、APIやライブラリの登場は、いずれも開発者の生産性を飛躍的に高めてきた。  

現在、その進化は**大規模言語モデル（LLM）**の統合によって新たな段階に入った。AIは自然言語による会話的インターフェースを通じて、コード生成、バグ検出、テスト生成といった開発タスクを支援している。これが「AI支援（AI-Assisted）」の時代であり、AIが人間の行う細かな作業を強化する段階である。  

しかしAIの応用範囲は、コード生成を超えて急速に広がっている。要件の具体化、計画立案、タスク分解、設計、そして開発者とのリアルタイム協働にまで拡大しつつある。これにより、AIが開発プロセス全体を能動的に調整する「**AI主導（AI-Driven）**」の時代が始まりつつある。  

ところが、既存のソフトウェア開発手法──たとえばSDLCやアジャイルなど──は、人間中心かつ長期的なプロセスを前提に設計されている。これらは手作業に依存し、固定的な役割分担を持つため、AIのスピードや柔軟性、さらには自律的行動（エージェント的能力）を十分に活かせない。AIをこれら旧来の方法に「後付け」するだけでは、AIの潜在力を封じ、むしろ時代遅れの非効率を強化してしまう。  

AIの変革力を真に引き出すには、ソフトウェア開発ライフサイクルそのものを**再構想（reimagine）**する必要がある。AIを中心的な協働者として位置づけ、ワークフロー・役割・反復サイクルを再設計することで、より迅速な意思決定、シームレスなタスク実行、継続的な適応を実現する。  

この文書では、そうした再構想を実現するための**AIネイティブな方法論「AI-Driven Development Lifecycle（AI-DLC）」**を定義し、次世代のソフトウェア工学における基盤を提示する。

# Architecture Decision Records (ADR)

プロジェクト全体に影響する技術的決定を記録します。

## ADRとは

重要な技術的決定とその背景・理由を記録するドキュメントです。
将来のメンバーが「なぜこの決定をしたのか」を理解できるようにします。

## フォーマット

```markdown
# XXXX: タイトル

## ステータス
提案中 | 承認 | 非推奨 | 置換（YYYY）

## コンテキスト
決定が必要になった背景・状況

## 決定
採用した決定内容

## 結果
決定による影響（良い面・悪い面）
```

## 命名規則

- ファイル名: `XXXX-<slug>.md`
  - 例: `0001-use-jj-for-version-control.md`
- 連番は4桁ゼロ埋め
- slugはケバブケース

## 運用ルール

1. **作成タイミング**: プロジェクト全体に影響する技術的決定時
2. **レビュー**: PR/Issue経由で議論
3. **更新**: ステータス変更時のみ（内容は原則変更しない）
4. **対象**: プロジェクト全体に影響する恒久的な決定のみ

## サイクル固有決定との違い

| 決定の種類 | 保存先 |
|------------|--------|
| プロジェクト全体の恒久決定 | ここ（docs/shared/adr/） |
| サイクル固有の設計決定 | docs/cycles/vX.X.X/design-artifacts/ |
| 実験的・暫定的決定 | docs/cycles/vX.X.X/plans/ |

暫定決定が恒久化する場合は、ADRとしてここに昇格させます。

# サイクル横断ドキュメント

サイクルに依存しない共通資料を管理するディレクトリです。

## ディレクトリ構成

| ディレクトリ | 内容 |
|-------------|------|
| [adr/](./adr/) | Architecture Decision Records（技術的決定の記録） |
| [specs/](./specs/) | 技術仕様（長期的な設計方針） |
| [runbooks/](./runbooks/) | 運用手順書 |

## 何をここに置くか

### サイクル横断ドキュメントの判断基準

以下に該当する場合はこのディレクトリに配置:

1. **複数サイクルで参照される**: 特定サイクルに限定されない
2. **長期的な決定**: アーキテクチャ選択、技術スタック決定
3. **運用手順**: サイクルに依存しない定常運用

### 判断フロー

```text
Q1: 次のサイクルでも参照される？
  No → サイクル固有（docs/cycles/vX.X.X/）
  Yes → Q2へ

Q2: プロジェクト全体の方針・決定？
  No → サイクル固有（ただし参照リンクを設置）
  Yes → Q3へ

Q3: 恒久的な決定？
  No → サイクル固有（将来ADRに昇格検討）
  Yes → サイクル横断（docs/shared/）
```

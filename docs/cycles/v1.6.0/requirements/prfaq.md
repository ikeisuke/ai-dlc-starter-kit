# PRFAQ: AI-DLC Starter Kit v1.6.0

## Press Release（プレスリリース）

**見出し**: AI-DLC Starter Kit v1.6.0 - 開発ワークフローの改善とリリース管理の整備

**副見出し**: セットアップの簡略化、Claude Code機能の活用、バージョン管理の強化を実現

**発表日**: 2026年1月（予定）

**本文**:

### 背景

AI-DLCスターターキットは、AIを開発プロセスの中心に据えた新しい開発方法論の実践キットです。これまでのバージョンで基本的なワークフローは確立されていましたが、開発者からはセットアップ手順の煩雑さ、バージョン履歴の追跡性、Claude Code固有機能の活用についてフィードバックが寄せられていました。

### プロダクト

v1.6.0では、以下の改善を実施しました：

1. **セットアップフローの簡略化**: サイクル開始時のブランチ作成とワークツリー作成を一括で実行できるようになりました。これにより、セットアップ手順が削減され、操作ミスも減少します。

2. **Claude Code機能の活用**: AskUserQuestion機能を活用した対話により、選択肢が明確になり、ユーザーの回答負担が軽減されます。また、AGENTS.mdを活用したプロンプト自動解決の可能性も検証しました。

3. **レビュー・コミットワークフローの改善**: レビュー前後に自動的にコミットが作成されるようになり、変更履歴の追跡性が向上しました。

4. **リリース管理の整備**: CHANGELOG.mdの導入とgitタグの運用により、バージョン履歴が明確になりました。v1.0.0以降の全バージョンにタグが付与され、Operations Phaseにタグ付け手順が追加されました。

### 顧客の声

「セットアップが簡単になって助かります。以前はブランチを作ってからワークツリーを作成する手順が煩雑でしたが、一括でできるようになりました。」

「CHANGELOGがあると、アップグレード時にどこが変わったか把握しやすいです。タグもあるので、特定バージョンへの切り替えも簡単です。」

### 今後の展開

今後のバージョンでは、Issue駆動統合やホームディレクトリへのユーザー共通設定配置など、延期されている機能の実装を検討していきます。

## FAQ（よくある質問）

### Q1: v1.5.xからアップグレードするには何をすればいいですか？
A: スターターキットの `setup-prompt.md` を読み込むことで、自動的にアップグレードが実行されます。CHANGELOG.mdで変更点を確認してください。

### Q2: 過去のバージョンにタグが付いていなかった場合はどうなりますか？
A: v1.6.0のOperations Phaseで、v1.0.0以降のタグがないバージョンすべてにタグを付与します。

### Q3: AskUserQuestionはどのような場合に使用されますか？
A: 選択肢が明確な場合（Yes/No、複数の選択肢から選ぶ場合など）に使用されます。自由回答が必要な場合はテキストでの質問が継続されます。

### Q4: AGENTS.mdによるプロンプト自動解決とは何ですか？
A: AGENTS.mdにプロンプトファイルへの参照を記載することで、AIが自動的に適切なプロンプトを読み込む機能です。これにより、プロンプトファイルの明示的な指定を省略できる可能性があります。

### Q5: CHANGELOG.mdはどの形式で書かれていますか？
A: Keep a Changelog形式（またはそれに準じた形式）で記載されています。Added、Changed、Fixed、Removedなどのカテゴリで変更内容を分類しています。

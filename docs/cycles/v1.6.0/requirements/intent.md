# Intent（開発意図）

## プロジェクト名
AI-DLC Starter Kit v1.6.0

## 開発の目的
AI-DLCスターターキットの開発ワークフロー改善とリリース管理の整備を行う。セットアップ時のブランチ・ワークツリー操作の簡略化、Claude Code固有機能の活用、リリースプロセスの明確化を実現する。

## ターゲットユーザー
- AI-DLCスターターキットを使用する開発者
- AI-DLCを活用したプロジェクトの開発チーム

## ビジネス価値
1. **開発効率の向上**: worktree/ブランチ操作の簡略化により、サイクル開始時の手順が減少
2. **ユーザー体験の改善**: AskUserQuestion機能の活用により、対話がよりインタラクティブに
3. **品質管理の強化**: レビュー前後のコミットにより変更履歴が明確化
4. **リリース管理の整備**: CHANGELOG.mdとタグ運用により、バージョン履歴が追跡可能に
5. **運用の簡略化**: AGENTS.mdを活用したプロンプト自動解決で操作手順を削減

## 成功基準
- セットアップ時にブランチ作成とワークツリー作成を一括で実行可能
- CLAUDE.mdにAskUserQuestion活用ルールが追記されている
- 各プロンプトにレビュー前後のコミット手順が追加されている
- CHANGELOG.mdが作成され、v1.0.0以降の全バージョンにタグが付与されている
- Operations Phaseにタグ付け手順が追加されている
- AGENTS.mdの記載でプロンプト指定が省略可能であることが検証されている

## 期限とマイルストーン
- Inception Phase: Intent、ユーザーストーリー、Unit定義
- Construction Phase: 各Unitの実装
- Operations Phase: CHANGELOG作成、過去タグ付与、リリース

## 制約事項
- `docs/aidlc/` は直接編集禁止（`prompts/package/` を編集）
- スターターキット開発のメタ構造を意識する
- 延期項目（Issue駆動統合、ホームディレクトリ共通設定）は今回対応しない

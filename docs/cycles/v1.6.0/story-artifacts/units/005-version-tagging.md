# Unit: バージョンタグ運用

## 概要
過去バージョンにgitタグを付与し、今後のOperations Phaseにタグ付け手順を追加する。

## 含まれるユーザーストーリー
- ストーリー7: バージョンタグ付け（過去分）
- ストーリー8: Operations Phaseへのタグ付け手順追加

## 責務
- v1.0.0以降のタグがないバージョンにgitタグを付与
- operations.mdにタグ付け手順を追加
- operations.mdにCHANGELOG.md更新手順を追加

## 境界
- タグ付けとoperations.mdの更新が対象
- CHANGELOG.mdの作成は別Unit（004）で実施済み

## 依存関係

### 依存する Unit
- Unit 004: CHANGELOG作成（依存理由: タグ付け時にCHANGELOGが整備されていることが望ましい）

### 外部依存
- なし

## 技術的考慮事項
- タグ形式: v1.x.x（セマンティックバージョニング）
- 過去タグは各バージョンのマージコミットまたは最終コミットに付与
- `git tag -a v1.x.x -m "Release v1.x.x" <commit-hash>`

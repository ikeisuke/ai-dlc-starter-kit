# ドメインモデル: skills構成変更

## 概要

AI-DLCのスキルファイル配置方式を rsync からシンボリックリンクに変更し、プロジェクト独自スキルの追加を可能にする。

## エンティティ

### スキルディレクトリ (SkillsDirectory)

- **場所**: `docs/aidlc/skills/`
- **性質**: 実ディレクトリ（シンボリックリンクではない）
- **責務**: スキルファイルのコンテナとして機能

### スターターキットスキル (StarterKitSkill)

- **種類**: codex, claude, gemini
- **配置方式**: シンボリックリンク
- **リンク先**: `[スターターキットパス]/prompts/package/skills/{skill_name}/`
- **責務**: AI-DLC標準のスキル機能を提供

### プロジェクト独自スキル (CustomSkill)

- **配置方式**: 実ディレクトリ
- **命名規則**: スターターキットのスキル名（codex, claude, gemini）以外
- **責務**: プロジェクト固有のスキル機能を提供

## 関係

```
SkillsDirectory (実ディレクトリ)
├── StarterKitSkill: codex/   → シンボリックリンク
├── StarterKitSkill: claude/  → シンボリックリンク
├── StarterKitSkill: gemini/  → シンボリックリンク
└── CustomSkill: {任意}/      → 実ディレクトリ
```

## ビジネスルール

1. **スターターキットスキルは上書き禁止**: codex, claude, gemini はシンボリックリンクとして維持
2. **プロジェクト独自スキルは自由追加**: スターターキットスキル名以外なら自由に追加可能
3. **アップグレード時の保護**: シンボリックリンク方式により、プロジェクト独自スキルは影響を受けない

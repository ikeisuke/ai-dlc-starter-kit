# Unit 006 計画: メタ開発ドキュメント整理

## 概要

prompts/package/ 内のメタ開発向け（非ユーザー向け）ドキュメントを調査し、適切な場所に分離する。

## 変更対象ファイル

### 調査対象

- `prompts/package/` 配下の全ファイル

### 変更の可能性があるファイル

- `prompts/package/` 内の開発者向けドキュメント（移動）
- `docs/development/` 内のドキュメント（追加）
- rsync除外設定（必要に応じて）

## 実装計画

### Phase 1: 設計

1. **ドメインモデル設計**
   - `prompts/package/` 配下の全ファイルをリストアップ
   - 各ファイルを「ユーザー向け」「開発者向け」に分類
   - 分類基準を明確化

2. **論理設計**
   - 移動対象ファイルの決定
   - 移動先の構造設計
   - rsync同期への影響確認

3. **設計レビュー**
   - 分類結果と移動計画をユーザーに提示
   - 承認を得る

### Phase 2: 実装

4. **コード生成（ファイル移動）**
   - 開発者向けドキュメントを docs/development/ へ移動
   - 必要に応じてrsync除外設定を更新

5. **テスト生成**
   - rsync --dry-run で同期対象を確認
   - 移動後のリンク整合性確認

6. **統合とレビュー**
   - 変更内容の最終確認
   - 実装記録作成

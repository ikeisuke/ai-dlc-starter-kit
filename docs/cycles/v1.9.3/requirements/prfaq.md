# PRFAQ: AI-DLC スターターキット v1.9.3

## Press Release（プレスリリース）

**見出し**: AI-DLC スターターキット v1.9.3 - ドキュメント修正と使いやすさ向上

**副見出し**: jjサポートの誤記修正、プレリリースバージョン対応、プロジェクト独自スキル追加対応など8件の改善

**発表日**: 2026-01

**本文**:

AI-DLC スターターキット v1.9.3 をリリースしました。本バージョンはパッチリリースとして、ドキュメントの誤記修正、設定の柔軟性向上、開発者体験の改善に焦点を当てています。

主な改善点:

1. **jj-support.md の誤記修正**: 未追跡ファイルの挙動とリモートブックマーク表記の記述を正確に修正しました。

2. **プレリリースバージョン対応**: `init-cycle-dir.sh` が `v2.0.0-alpha.4` のようなプレリリースバージョンや任意のサイクル名を受け入れるようになりました。

3. **プロジェクト独自スキル追加対応**: skills ディレクトリの構成を変更し、プロジェクト固有のスキルを追加できるようになりました。

4. **env-info.sh の拡張**: `--setup` オプションでセットアップに必要な情報を一括出力できるようになりました。

5. **メタ開発ドキュメントの整理**: 開発者向けドキュメントをユーザー配布パッケージから分離し、配布物をスリム化しました。

## FAQ（よくある質問）

### Q1: 既存のプロジェクトに影響はありますか？
A: 後方互換性を維持しています。既存のプロジェクトはそのまま動作します。

### Q2: プロジェクト独自のスキルはどのように追加しますか？
A: `docs/aidlc/skills/` ディレクトリ配下に新しいスキルディレクトリを作成してください。rsync 時に既存のカスタムスキルは保持されます。

### Q3: プレリリースバージョンでサイクルを作成する具体例は？
A: `docs/aidlc/bin/init-cycle-dir.sh v2.0.0-alpha.1` のように任意の文字列を指定できます。空文字のみ拒否されます。

### Q4: env-info.sh の --setup オプションで何が出力されますか？
A: 従来の依存ツール状態に加えて、project.name、backlog.mode、current_branch、latest_cycle が出力されます。

# PRFAQ: AI-DLC Starter Kit v1.27.4

## Press Release（プレスリリース）

**見出し**: AI-DLCワークフローの品質ガードを3方面から強化

**副見出し**: アップグレード時の警告検出、バックログ登録時のスコープ保護、セミオート承認時のレビュー担保を実現

**発表日**: 2026-03-25

**本文**:

v1.27.3のサイクルで、AI-DLCワークフローの自動化に関する3つの品質課題が発見されました。終了コード規約の変更によりaidlc-setupの警告検出が機能しなくなった問題、スコープ内の作業がバックログに外出しされた問題、そしてsemi_autoモードでAIレビューをスキップしたまま自動承認が行われた問題です。

v1.27.4では、これら3つの課題をそれぞれ独立したUnitとして対応します。aidlc-setup.shの警告検出をstdout解析方式に移行し、バックログ登録時にintent.mdのスコープと照合するガードを追加し、セミオートゲートにレビュー実施済み前提条件を導入します。

これにより、AI-DLC利用者はアップグレード・開発・レビューの各フェーズで、より信頼性の高い自動化ワークフローを利用できるようになります。

## FAQ（よくある質問）

### Q1: migrate-config.shの出力フォーマットは変更されますか？
A: いいえ。migrate-config.sh自体は変更しません。v1.27.3で既に終了コード規約に準拠済みです。aidlc-setup.sh側の検出ロジックのみを変更します。

### Q2: スコープガードはどのように判定しますか？
A: intent.mdの「含まれるもの」セクションに列挙されたIssue番号・作業項目と照合します。プロンプトベースの制約のため、AIエージェントが判定を行います。

### Q3: automation_mode=manualの場合、review_not_executed条件の影響はありますか？
A: いいえ。manualモードではすべての承認ポイントでユーザー確認が行われるため、この条件には影響されません。

# markdownlintルール有効化 - ドメインモデル設計

## 概要

markdownlintの高優先度ルール（MD009, MD034, MD040）を有効化し、既存ファイルを修正する。

## 対象ルールの定義

### MD009: 末尾スペース

- **説明**: 行末の不要な空白文字
- **現状**: 約50件の違反（主に履歴ファイル）
- **修正方法**: 末尾の空白を削除

### MD034: 裸URL

- **説明**: リンク形式になっていないURL
- **現状**: 検出なし
- **修正方法**: 対応不要

### MD040: コードブロック言語指定

- **説明**: フェンスドコードブロックに言語指定がない
- **現状**: 約300件以上の違反
- **修正方法**: 適切な言語（markdown, bash, json等）を追加

## 修正対象範囲

### 編集可能ファイル

| ディレクトリ | 編集可否 | 備考 |
|-------------|----------|------|
| `prompts/package/` | 可 | rsync元（優先的に修正） |
| `docs/cycles/` | 可 | 直接編集 |
| `.markdownlint.json` | 可 | ルール有効化 |

### 編集不可ファイル

| ディレクトリ | 理由 |
|-------------|------|
| `docs/aidlc/` | rsyncコピー先（prompts/package/を編集） |

## 修正戦略

### Phase 2で実施する作業

1. `.markdownlint.json` の修正
   - MD009, MD034, MD040 を `true` に変更

2. `prompts/package/` 配下のMarkdownファイル修正
   - コードブロックに言語指定を追加
   - 末尾スペースを削除（検出があれば）

3. `docs/cycles/` 配下のMarkdownファイル修正
   - 末尾スペースを削除
   - コードブロックに言語指定を追加

## コードブロック言語の決定基準

| 内容 | 言語指定 |
|------|----------|
| Bashコマンド | `bash` |
| Markdownテンプレート | `markdown` |
| JSONファイル | `json` |
| YAMLファイル | `yaml` |
| 出力例・テキスト | `text` |
| 設定例 | 該当形式 |
| 不明・汎用 | `text` |

## リスク

- 修正漏れの可能性（大量のファイル）
- 言語指定の誤り（内容の判断が必要）

## 検証方法

- markdownlint の実行（インストール後）
- 手動での grep 確認

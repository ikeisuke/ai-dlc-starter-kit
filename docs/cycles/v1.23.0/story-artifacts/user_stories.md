# ユーザーストーリー

## US-001: プロンプトのステップ番号整理

**ユーザーとして**、AI-DLCのフェーズプロンプトを読む際に、ステップ番号が連番で統一されていてほしい。枝番（5.5, 6.4.2等）やハイフン記法（6-1）、アルファベット記法（5a）が混在していると、全体の流れが把握しづらい。

### 受け入れ基準

- 全フェーズプロンプトのステップが整数連番で統一されている
- ハイフン記法・アルファベット記法が解消されている
- Part 1/Part 2等の構造分割が妥当か評価され、必要に応じて改善されている
- フローの動作・機能が変更前と同一である

## US-002: プロンプトの冗長記述削減

**ユーザーとして**、プロンプトが簡潔であってほしい。冗長な表現や重複記述があるとコンテキストウィンドウを無駄に消費し、AIの応答精度が下がる。

### 受け入れ基準

- 冗長な表現が簡潔化されている
- 重複記述が統合されている
- 不要なセクションや記述が削除されている
- プロンプトの行数が削減されている（具体的な数値は問わない）

## US-003: 非推奨スクリプトの削除

**開発者として**、使われていない非推奨ラッパースクリプトを削除して、コードベースをクリーンに保ちたい。

### 受け入れ基準

- validate-remote-sync.sh が削除されている
- validate-uncommitted.sh が削除されている
- これらのスクリプトへの参照がコードベースに残っていない

## US-004: READMEの改善

**新規ユーザーとして**、README.mdを読んだだけで迷わずAI-DLCを使い始められるようにしたい。現状のREADMEは1400行超でバージョンリリースノートが大半を占めており、重要な情報が埋もれている。

### 受け入れ基準

- クイックスタートが明確で、新規ユーザーが迷わず使い始められる
- バージョンリリースノートがREADMEから分離されている（CHANGELOG.mdに集約）
- リポジトリ構成が現状と一致している
- README全体が簡潔で500行以下になっている

# 論理設計: 共通プロンプトリファクタリング

## 概要

review-flow.mdとcommit-flow.mdのステップ番号連番化と冗長記述削減の具体的な変更方針を定義する。

## review-flow.md 変更設計

### メインフロー連番化

変更は見出し番号とその本文中の参照テキストの書き換えのみ。フローの分岐ロジック・条件判定・実行内容は一切変更しない。

| 変更箇所 | 変更前 | 変更後 |
|---------|--------|--------|
| 見出し | `5.5. **セルフレビューフロー**` | `6. **セルフレビューフロー**` |
| 見出し | `6. **外部AIレビューで…**` | `7. **外部AIレビューで…**` |
| 見出し | `7. **ユーザーレビューフロー**` | `8. **ユーザーレビューフロー**` |
| 本文参照 | 「ステップ5.5」(全箇所) | 「ステップ6」 |
| 本文参照 | 「ステップ6（外部AIレビューで続行不能時）」等のreview-flowトップレベル参照のみ | 「ステップ7」 |
| 本文参照 | 「ステップ7（ユーザーレビューフロー）」等のreview-flowトップレベル参照のみ | 「ステップ8」 |

**注意**:
- 「ステップ6」→「ステップ7」の変更は、review-flow.mdのトップレベル手順番号の参照のみ対象。「Phase 2 ステップ6（統合とレビュー）」等の他フェーズの文脈は変更しない
- 変更順序: 先に大きい番号から変更するか、アンカー付き表現（「ステップ6（外部AI」等）で特定して置換

**完了後検証**:

```bash
# 旧番号の残存チェック（全てゼロであること）
rg "5\.5" prompts/package/prompts/common/review-flow.md
rg "5a\." prompts/package/prompts/common/review-flow.md
rg "4'\.|5'\." prompts/package/prompts/common/review-flow.md

# 誤置換チェック（以下が維持されていること）
rg "Phase 2 ステップ6" prompts/package/prompts/common/review-flow.md
```

### 冗長記述削減方針

#### 1. AIレビュー完了時の履歴記録テンプレート共通化

**現状**: ステップ5（項目5）とステップ6（旧5.5、項目7）に同一の履歴記録テンプレートが存在。

**方針**: 「## AIレビュー完了時の共通履歴記録手順」セクションを新設し、ステップ5/6からは「AIレビュー完了時の共通履歴記録手順に従う」と参照のみにする。共通セクションには外部CLIレビューとセルフレビューの差分（メッセージ文言の違い）を条件分岐で記載。

#### 2. ステップ末尾の共通処理統合

**現状**: ステップ5末尾とステップ6（旧5.5）末尾に「レビュー前/後コミット」「レビューサマリ更新」「セミオートゲート判定」が重複。

**方針**: 「## レビューステップ完了時の共通処理」セクションを新設し、各ステップからは参照のみにする。

## commit-flow.md 変更設計

### 冗長記述削減方針

#### 1. コミット手順テンプレート共通化

**現状**: レビュー前コミット、レビュー反映コミット、各フェーズ完了コミットで mktemp → Write → commit → 削除 のパターンが6回繰り返されている。

**方針**: 「## コミット実行の共通手順」セクションを新設し、メッセージテンプレートIDと変数のみを各セクションで指定する形に簡略化。

#### 2. Squashフロー内の類似パターン統合

**現状**: Unit完了squashとInception Phase完了squashで中間コミット手順・squash実行手順がほぼ同一。

**方針**: フェーズ別の差分（コミットメッセージ内容、Unit-Numberトレーラの有無）のみを条件分岐で記載し、共通部分は1箇所に統合。

## docs/aidlc/ ミラーとの同期

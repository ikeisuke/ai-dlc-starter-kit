# Unit 001 計画: 共通プロンプトリファクタリング

## 概要

common/配下のプロンプトファイルのステップ番号整理（枝番・アルファベット記法の連番化）と冗長記述の削減を行う。フローの動作・機能は変更しない。

**編集対象**: `prompts/package/prompts/common/` を編集（`docs/aidlc/` は rsync コピーのため直接編集禁止）

## 変更対象ファイル

### 主要対象

| ファイル | 変更内容 |
|---------|---------|
| `prompts/package/prompts/common/review-flow.md` | ステップ番号整理、アルファベット記法解消、冗長記述削減 |
| `prompts/package/prompts/common/commit-flow.md` | 冗長記述削減 |

### 参照更新が必要なファイル

review-flow.mdのステップ番号変更に伴い、以下のファイル内のステップ番号参照をすべて更新:

| ファイル | 更新箇所 |
|---------|---------|
| `prompts/package/prompts/common/review-flow.md` | 本文中の全ステップ番号参照（5.5→6、6→7、7→8。見出し・分岐説明・共通手順説明含む） |
| `prompts/package/guides/subagent-usage.md` | ステップ5.5への参照（1箇所） |

**検証方法**: 変更後に以下で残存参照ゼロを確認（review-flow.md関連の参照のみ対象。inception.md等の別文脈の5.5は対象外）:

```bash
rg "5\.5" \
  prompts/package/prompts/common/review-flow.md \
  prompts/package/guides/subagent-usage.md
```

### 2. review-flow.md 冗長記述削減

- ステップ5とステップ6（旧5.5）で重複する「履歴への記録」テンプレートの共通化（共通フォーマットを1箇所に定義し、各ステップからは参照のみ）
- ステップ5とステップ6（旧5.5）末尾の重複する「セミオートゲート判定」「レビューサマリ更新」の共通化
- 「レビュー前コミット」「レビュー後コミット」の参照パターン統一

### 3. commit-flow.md 冗長記述削減

**注記**: Unit定義では枝番(2.5, 2.6, 2.7, 3.5)の連番化が記載されているが、現在のcommit-flow.mdにはこれらの枝番は存在しない（他のフェーズプロンプトのステップ番号と混同の可能性）。冗長記述の削減のみ実施。

- Unit完了squashとInception Phase完了squashの類似パターンの統合
- mktemp/Write/commit/delete手順の共通パターン化（手順が明確な場合は冗長な繰り返しを削減）

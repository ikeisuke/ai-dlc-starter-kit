# Unit: 名前付きサイクル継続確認機能の追加

## 概要

Inception Phaseのバージョン決定ステップの前に、既存の名前付きサイクルが存在する場合に継続利用を提案するフローを追加する。

## 含まれるユーザーストーリー

- ストーリー 6: 名前付きサイクル継続確認機能の追加

## 責務

- inception.mdのステップ5.5（サイクルモード確認）とステップ6（バージョン決定）の間に名前付きサイクル検出・確認ロジックを追加
- 名前付きサイクルの検出ロジック実装（`docs/cycles/` 配下の非バージョン・非予約ディレクトリ）
- ユーザー確認フローの記述（選択肢提示、遷移先の定義）
- スキップ条件の記述（0件の場合、既にnamedモードの場合）

## 境界

- suggest-version.shの修正は対象外
- cycle_modeの既存ロジック（default/named/ask）は変更しない
- ガイドドキュメントの修正（Unit 001〜005）は対象外

## 依存関係

### 依存する Unit

- なし

### 外部依存

- なし

## 非機能要件（NFR）

- 該当なし（プロンプト修正のみ）

## 技術的考慮事項

- 編集対象は `prompts/package/prompts/inception.md`
- 名前付きサイクルの判定: `docs/cycles/` 配下で `v[0-9]` で始まらず、`backlog` / `backlog-completed` でもないディレクトリ
- `cycle_mode` が `named` の場合は名前入力済みのため確認不要（スキップ）
- `cycle_mode` が `default` の場合でも名前付きサイクルが存在すれば確認を行う

Medium

## 見積もり

小規模（inception.mdへのフロー追記のみ）

## 完了条件

- inception.mdに名前付きサイクル検出・確認フローが追記されている
- 名前付きサイクル0件時のスキップ条件が記述されている
- `cycle_mode=named` 時のスキップ条件が記述されている
- 「はい」選択時に名前付きサイクルモードのバージョン提案に遷移する手順が記述されている

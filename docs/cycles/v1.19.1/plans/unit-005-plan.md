# Unit 005 計画: Error Handling基本定義

## 概要

エラーの重大度レベル定義と主要フェーズの基本復旧手順を文書化する。Amazon AIDLCの`error-handling.md`を参考にしつつ、基本レベルにとどめる。

## 変更対象ファイル

- `prompts/package/guides/error-handling.md` — 新規作成（エラーハンドリングガイド）
- `prompts/package/prompts/common/intro.md` — 関連ドキュメントセクションに参照追加

## 実装計画

### 1. error-handling.md 新規作成

`prompts/package/guides/error-handling.md` に以下の構成で作成する。ドキュメント内を「ポリシー（重大度定義・共通原則）」と「プレイブック（フェーズ別復旧手順）」に章レベルで明確に分離する。

#### 構成

1. **タイトルと概要**
   - AI-DLCにおけるエラーハンドリングの基本方針

2. **Part 1: エラーポリシー（安定部分）**

   2a. **エラー重大度レベル定義テーブル**

   | レベル | 説明 | 判断基準 | 対応方針 |
   |-------|------|---------|---------|
   | Critical | フロー続行不能 | データ損失リスク、成果物破損 | 即時停止、ユーザー通知、手動復旧 |
   | High | フロー品質に重大な影響 | レビュー未実施での完了、設計と実装の重大な乖離 | 作業中断、原因特定後に再開 |
   | Medium | フロー品質に軽微な影響 | 外部ツール一時的エラー、非クリティカルな検証失敗 | 警告表示、フォールバック実行 |
   | Low | 情報提供レベル | スタイル違反、推奨設定の未適用 | 記録のみ、続行 |

   2b. **共通原則**
   - エラー発生時は現在の状態を保全する（破壊的操作を避ける）
   - ユーザーへの通知は具体的なエラー内容と推奨アクションを含める
   - 自動復旧可能な場合はフォールバックを試行し、結果を記録する
   - **フォールバック条件**: 任意設定の欠落・外部ツール一時エラー等、Medium以下のエラーのみフォールバック許可。必須設定の欠落やデータ整合性に関わるエラー（High以上）は自動続行せず停止する
   - **フォールバック時の必須要件**: フォールバック実行時は必ず警告メッセージを出力し、履歴に記録する

   2c. **エラー通知の基本フォーマット**

   ```text
   【{レベル}】{エラー概要}
   - フェーズ: {Inception / Construction / Operations}
   - 原因: {具体的な原因}
   - 推奨アクション: {次に取るべき手順}
   ```

3. **Part 2: フェーズ別プレイブック（運用手順）**

   - **Inception Phase**:
     - サイクルディレクトリ作成失敗（High） → ディレクトリ権限確認、手動作成
     - GitHub Issue作成失敗（Medium） → gh auth status確認、手動作成案内
     - 設定ファイル読み取り失敗 → 任意設定: デフォルト値で続行（Medium、警告出力必須）、必須設定: 停止しユーザーに修正依頼（High）

   - **Construction Phase**:
     - AIレビューツール利用不可（Medium） → セルフレビューフォールバック（警告出力必須）
     - コミット/squash失敗（High） → git status確認、手動コミット案内
     - テスト失敗（High） → 失敗原因特定、修正後再実行

   - **Operations Phase**:
     - PR作成/マージ失敗（High） → gh auth確認、手動操作案内
     - post-merge-cleanup.sh失敗（High） → dry-runエラー確認、手動クリーンアップ
     - バージョンタグ付け失敗（Medium） → 既存タグ確認、手動タグ付け

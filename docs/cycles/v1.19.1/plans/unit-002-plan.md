# Unit 002 計画: レビュースキル外部ツール優先

## 概要

レビュースキルが外部ツール（codex等）利用可能時に外部ツールを優先し、セルフレビューはフォールバックとしてのみ使用されるようにする。

## 問題の根本原因

現在の `review-flow.md` のステップ3は、Skillツールの利用可否（`skill="reviewing-[type]"` が呼び出し可能か）のみを確認している。しかし、Skillが利用可能であっても、スキルが内部的に使用する外部CLIツール（codex / claude / gemini）が利用できない場合がある。

この場合、外部CLIが使えないにもかかわらずステップ5（通常のAIレビューフロー）に進み、スキルが呼び出されるが、外部CLIが見つからずエラーとなるか、self-reviewモードにフォールバックされる。結果として、ユーザーが `tools = ["codex"]` と設定しているにもかかわらず外部ツールが使われない。

## 変更対象ファイル

1. `prompts/package/prompts/common/review-flow.md` - フロー改修
2. `prompts/package/skills/reviewing-code/SKILL.md` - 互換性情報追記
3. `prompts/package/skills/reviewing-architecture/SKILL.md` - 互換性情報追記
4. `prompts/package/skills/reviewing-security/SKILL.md` - 互換性情報追記
5. `prompts/package/skills/reviewing-inception/SKILL.md` - 互換性情報追記

## 実装計画

### 1. review-flow.md のステップ3改修

**現状**: 優先ツール決定後、Skillの利用可否のみチェック

**改修後**: 優先ツール決定後、以下の3段階チェックを実施:

1. **Skill利用可否チェック**（既存）: `skill="reviewing-[type]"` が呼び出し可能か
2. **外部CLI利用可否チェック**（新規追加）: `tools` リストの各CLIがインストールされているか確認

```text
which codex >/dev/null 2>&1  # codexの場合
which claude >/dev/null 2>&1 # claudeの場合
which gemini >/dev/null 2>&1 # geminiの場合
```

3. **結果に基づく遷移判定**: 既存のmode/部分利用可能パターンとの整合性を維持

**判定ツリー**（既存のmode・部分利用パターンを保持）:

| Skill利用可否 | CLI利用可否 | 遷移先 |
|:---:|:---:|:---|
| 全種別利用可能 | 利用可能 | ステップ4 → ステップ5（外部ツール経由AIレビュー） |
| 全種別利用可能 | 利用不可 | ステップ5.5（セルフレビュー）※mode依存の確認メッセージを表示 |
| 一部種別利用可能 | 利用可能 | mode依存: required=スキップ承認要求 / recommend=警告のみ。利用可能種別のみステップ5 |
| 一部種別利用可能 | 利用不可 | mode依存: required=利用不可種別のスキップ承認要求後、利用可能種別のみステップ5.5 / recommend=警告後、利用可能種別のみステップ5.5 |
| 利用不可 | - | ステップ6（外部AIレビュー続行不能時） |

**CLI利用可否チェックの詳細**:

- `tools` リストを先頭から順に確認
- 最初にインストールされているCLIを「使用ツール」として決定
- 全CLIがインストールされていない場合: 「外部CLI利用不可」

**重要**: CLI可用性チェックで確認するのは**CLIのインストール状態**（`which` コマンド）のみ。CLI実行時エラー（タイムアウト、認証失敗等）は別途ステップ5内のフォールバックで処理する。

### 2. review-flow.md のステップ5内フォールバック処理追加

ステップ5の「種別ごとの反復レビュー」内で、**種別単位**のフォールバック処理を定義:

**エラー分類とフォールバック条件**:

| エラー種類 | 判定条件 | フォールバック動作 |
|:---|:---|:---|
| CLI不在（恒久） | `which` コマンド失敗 | 該当種別のみセルフレビューへ遷移 |
| CLI実行エラー（一時的可能性） | タイムアウト、認証失敗等 | リトライ1回 → 再失敗時はユーザーに確認（選択肢: セルフレビューへ遷移（推奨） / スキップ（mode=requiredの場合はステップ6の既存スキップフローに接続: スキップ理由入力・履歴記録が必要） / 中断） |
| CLI出力解析不能 | 応答形式不一致 | 該当種別のみセルフレビューへ遷移 |

**種別単位のフォールバック境界**:

- 種別A（code）が外部CLIで成功 → 結果を保持
- 種別B（security）が外部CLIで失敗 → 種別Bのみセルフレビューへ遷移
- 完了済み種別の再実行は行わない
- 各種別のフォールバック発生を履歴に個別記録

**履歴記録フォーマット**:

```text
【フォールバック】{種別}の外部CLIレビューからセルフレビューへ遷移
【エラー種類】{CLI不在 / CLI実行エラー / 出力解析不能}
【詳細】{エラー詳細}
```

### 3. 4つのSKILL.mdに互換性情報セクションを追記

各SKILL.md（reviewing-code, reviewing-architecture, reviewing-security, reviewing-inception）の「セルフレビューモード」セクションの直前に「外部ツールとの関係」セクションを追加:

- セルフレビューは外部CLIが利用不可時のフォールバックであること
- 外部CLIが利用可能な場合は外部ツールが常に優先されること
- 呼び出し元（review-flow.md）がモード決定の責務を持つこと（スキル自身はモード判定しない）

# 論理設計: レビュースキル外部ツール優先

## 概要

review-flow.mdのステップ3に外部CLI可用性チェックを追加し、ステップ5に種別単位のフォールバック処理を追加する。また4つのSKILL.mdに外部ツールとの関係性を明記する。

## コンポーネント構成

### 変更対象1: review-flow.md ステップ3

**挿入位置**: 現在のステップ3「優先ツール決定・利用可否チェック」内、Skill利用可否チェック後・遷移判定前

**追加内容**: 外部CLI可用性チェックサブステップ

```text
3. **優先ツール決定・利用可否チェック**:
   - `tools` リストの先頭を優先ツールとする（デフォルト: `codex`）
   - レビュー種別リストの**全種別**について `skill="reviewing-[type]"` が利用可能か個別に確認
   - **外部CLI可用性チェック**（新規追加）:
     `tools` リストを先頭から順に確認し、最初にインストールされているCLIを「使用ツール」として決定する

     ```text
     which codex >/dev/null 2>&1  # codexの場合
     which claude >/dev/null 2>&1 # claudeの場合
     which gemini >/dev/null 2>&1 # geminiの場合
     ```

     - いずれかのCLIが利用可能: `cli_available=true`、使用ツール名を記録
     - 全CLIが利用不可: `cli_available=false`
   - **遷移判定**（Skill可用性 × CLI可用性の組み合わせ）:
     - 全種別利用可能 かつ cli_available=true → ステップ4へ
     - 全種別利用可能 かつ cli_available=false:
       - mode=required: ステップ6へ遷移し、選択肢を提示（セルフレビュー推奨）。ユーザーがセルフレビューを選択した場合はステップ5.5へ
       - mode=recommend: セルフレビュー推奨メッセージを表示後、ステップ5.5へ直接遷移
     - 一部種別利用可能 かつ cli_available=true:
       - mode=required: 利用不可種別のスキップ承認要求後、利用可能種別のみステップ4へ
       - mode=recommend: 警告表示後、利用可能種別のみステップ4へ
     - 一部種別利用可能 かつ cli_available=false:
       - mode=required: 利用不可種別のスキップ承認要求後、利用可能種別のみステップ5.5へ
       - mode=recommend: 警告表示後、利用可能種別のみステップ5.5へ
     - すべて利用不可 → ステップ6へ
```

### 変更対象2: review-flow.md ステップ5

**挿入位置**: ステップ5「種別ごとの反復レビュー」内、項目1（AIレビュー実行）の直後

**追加内容**: 種別単位のフォールバック処理

```text
   - **種別単位のフォールバック処理**（新規追加）:

     スキル呼び出し時に外部CLIがエラーを返した場合、エラー種類に応じて以下の処理を行う:

     **エラー分類**:

     | エラー種類 | 判定条件 | フォールバック動作 |
     |:---|:---|:---|
     | CLI不在（恒久） | コマンド未検出（exit code 127等） | 該当種別のみセルフレビュー（ステップ5.5のA/B/C方式）へ遷移 |
     | CLI実行エラー（一時的可能性） | タイムアウト、認証失敗、ネットワークエラー等 | リトライ1回 → 再失敗時はユーザーに確認 |
     | CLI出力解析不能 | 応答が空、形式不一致 | 該当種別のみセルフレビューへ遷移 |

     **CLI実行エラー時のユーザー確認**:

     ```text
     【外部CLIエラー】{種別}レビューで外部CLIがエラーを返しました（リトライ後も失敗）。
     エラー内容: {エラー詳細}

     どのように対応しますか？
     1. セルフレビューへ遷移する（推奨）
     2. スキップする
     3. 処理を中断する
     ```

     - 選択肢1: 該当種別のみセルフレビューへ遷移
     - 選択肢2: mode=required の場合はステップ6の既存スキップフロー（スキップ理由入力・履歴記録）に接続。mode=recommend の場合はそのままスキップ
     - 選択肢3: 処理を中断

     **フォールバック発生時の履歴記録**:

     各種別のフォールバック発生時に以下を履歴に記録する:

     ```text
     【フォールバック】{種別}の外部CLIレビューからセルフレビューへ遷移
     【エラー種類】{CLI不在 / CLI実行エラー / 出力解析不能}
     【詳細】{エラー詳細}
     ```

     **完了済み種別の保護**:

     - 種別Aが外部CLIで成功した結果は保持する
     - 種別Bがフォールバックしても、種別Aの結果には影響しない
     - 履歴記録には各種別の実行方法（external_cli / self_review）を個別に記載する
```

### 変更対象3: 4つのSKILL.md

**挿入位置**: 各SKILL.mdの「セルフレビューモード」セクションの直前

**追加内容**: 「外部ツールとの関係」セクション

```markdown
## 外部ツールとの関係

このスキルは2つのモードで動作する:

1. **通常モード（外部CLI使用）**: 外部CLIツール（codex / claude / gemini）を使用してレビューを実行する。呼び出し元が `優先ツール: [tool]` を引数に含める
2. **セルフレビューモード（フォールバック）**: 外部CLIが利用不可の場合に使用する。呼び出し元が `self-review` を引数の先頭トークンに含める

**モード決定の責務**:

- モード決定は**呼び出し元（review-flow.md）の責務**。スキル自身はモード判定を行わない
- review-flow.mdがステップ3で外部CLI可用性を事前チェックし、適切なモードでスキルを呼び出す
- 外部CLIが利用可能な場合は、常に通常モード（外部CLI使用）が優先される
- セルフレビューモードは、外部CLIが利用不可の場合のフォールバックとしてのみ使用される
```

## インターフェース定義

### review-flow.md → SKILL.md の呼び出しインターフェース

| モード | 引数フォーマット | 使用条件 |
|:---|:---|:---|
| 通常モード | `args="[レビュー対象] 優先ツール: [tool]"` | ステップ3でcli_available=true と判定された場合 |
| セルフレビュー | `args="self-review [files]"` | ステップ3でcli_available=false の場合、またはステップ5実行中に恒久エラー（CLI不在）/出力解析不能が発生した場合、またはCLI実行エラー再失敗時にユーザーがセルフレビューを選択した場合 |

### CLI可用性チェック結果の伝播

- ステップ3で確認したCLI可用性はステップ5の初期判定に使用する
- ステップ5内でCLI不在エラーが発生した場合、ステップ3の結果に関わらずフォールバック処理を実行する（実行時の状態を優先）

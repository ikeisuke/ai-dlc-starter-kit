# 実装記録: Inceptionプロンプト名前付きサイクル統合

## 変更ファイル

- `prompts/package/prompts/inception.md`（1ファイルのみ）

## 変更内容

### 変更1: 継続プロンプト例の更新（L78）

`サイクル vX.X.X` → `サイクル {{CYCLE}}` に変更。名前付き形式（`waf/v1.0.0`）と従来形式（`v1.0.0`）の両方に対応。

### 変更2: ステップ5.5 モード別分岐ロジック追加（L293付近）

「Unit 003で実装する」注記を削除し、以下のモード別分岐を実装:

- **mode=named**: サイクル名入力プロンプト、3段階バリデーション（正規表現、禁止名、v[0-9]開始）、パターン別エラーメッセージ
- **mode=ask**: AskUserQuestion機能で「名前付き/通常」選択肢を提示
- **mode=default**: 変更なし（後方互換維持）

### 変更3: ステップ6-1 直近サイクル表示の拡張（L392付近）

`cycle_name` 設定有無による分岐を追加:
- 設定あり: `docs/cycles/${cycle_name}/v*/` をスキャン
- 設定なし: 従来の `docs/cycles/v*/` をスキャン

### 変更4: ステップ6-2 バージョン提案の二分岐化（L436付近）

AIの判断フローを名前付き/従来の2パスに分岐:
- **名前付き**: `all_cycles` から `${cycle_name}/v*` をフィルタ、AI側でバージョン再計算、`{{CYCLE}}` を `${cycle_name}/${version}` で組み立て
- **従来**: `branch_version` → `suggested_*` → カスタム入力の既存フロー維持

### レビュー修正: all_cycles出力例・トークンtrim

- `all_cycles` 出力例に名前付きサイクル形式（`waf/v1.0.0,waf/v1.1.0`）を追加
- カンマ分割トークンの比較前trim手順を明記

## 技術的決定

1. **suggest-version.sh のmainブランチ制限**: mainブランチ上では名前付き系列を認識できないため、AI側で `all_cycles` からフィルタ・再計算する方式を採用
2. **禁止名リスト**: `backlog`, `backlog-completed`（共有ディレクトリ衝突）、`^v[0-9]`（バージョンディレクトリ混同）
3. **後方互換性**: `mode=default` 時は既存フローと完全同一を維持

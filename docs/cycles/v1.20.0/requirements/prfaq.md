# PRFAQ: AI-DLC Starter Kit v1.20.0

## Press Release（プレスリリース）

**見出し**: AI-DLCに名前付きサイクルとsquash-unitスキルが登場

**副見出し**: 機能ドメイン別の並行開発管理とUnit完了操作のシームレス化を実現

**発表日**: 2026年3月

**本文**:

AI-DLCスターターキットの利用者は、1つのリポジトリ内で複数の独立したバージョン系列を管理したいというニーズがありました。例えば、AWSインフラ管理リポジトリでWAFとCDNを別々のラインで開発・バージョン管理する場合、従来は単一の `cycles/vX.X.X` 構造しか使えず、機能ドメイン別の管理が困難でした。

v1.20.0では「名前付きサイクル」機能を導入しました。`rules.cycle.mode` 設定で `default` / `named` / `ask` の3つのモードを選択でき、`named` モード時は `docs/cycles/waf/v1.0.0/` のような機能ドメイン別のディレクトリ構造で開発が進められます。ブランチも `cycle/waf/v1.0.0` 形式でネストされ、サイクル名とバージョンの両方が一目で識別できます。

さらに、Unit完了時のsquash操作をスキル呼び出しで実行できるようになりました。これまで `--cycle`、`--vcs`、`--base` 等の引数を手動で組み立てる必要がありましたが、スキルがコンテキストから自動解決し、dry-runでの事前確認も含めたシームレスなフローを提供します。

「複数プロダクトラインを1リポジトリで管理しているチームにとって、名前付きサイクルはまさに待望の機能です。WAF v2.1.0の開発中にCDN v1.3.0のホットフィックスが必要になっても、お互いに干渉せず作業できるようになりました。」

今後は、Construction PhaseやOperations Phaseプロンプトの名前付きサイクル完全対応や、サイクル間のクロスリファレンス機能の検討を予定しています。

## FAQ（よくある質問）

### Q1: 既存のサイクル（`cycles/vX.X.X`）はそのまま使えますか？
A: はい。`rules.cycle.mode` が未設定または `default` の場合、従来通り `docs/cycles/vX.X.X/` 形式で動作します。後方互換は完全に維持されています。

### Q2: `mode=ask` とはどのようなモードですか？
A: Inception Phase開始時に「名前付きサイクル」か「名前なしサイクル（従来形式）」かを毎回選択できるモードです。プロジェクトによって名前付きが必要な場合とそうでない場合が混在するケースに対応します。

### Q3: サイクル名にはどのような制約がありますか？
A: `^[a-z0-9][a-z0-9-]{0,63}$` の命名規則に従います。小文字英数字とハイフンのみ、先頭は英数字、最大64文字です。例: `waf`、`cdn-config`、`auth-v2`。

### Q4: squash-unitスキルを使わず従来の直接呼び出しも可能ですか？
A: はい。`commit-flow.md` にスキル呼び出しが推奨として追記されますが、従来の `squash-unit.sh` 直接呼び出しも引き続き動作します。

### Q5: 名前付きサイクルでsuggest-version.shはどう動作しますか？
A: 名前付きサイクル内のバージョン系列（例: `docs/cycles/waf/v*/`）をスキャンし、次バージョンを提案します。該当名前の既存バージョンがない場合は `v1.0.0` が提案されます。

# Unit: 名前付きサイクル設定

## 概要
`rules.cycle.mode` 設定の読み取り・バリデーション・フォールバック機能を実装する。名前付きサイクル機能の基盤となる設定レイヤー。

## 含まれるユーザーストーリー
- ストーリー 1: 名前付きサイクルの設定

## 責務
- `rules.cycle.mode` 設定キーの仕様定義: 有効値 `default` / `named` / `ask`、デフォルト値 `default`
- Inception Phaseプロンプトに設定値読み取り・バリデーション・フォールバックの指示を追加（`read-config.sh rules.cycle.mode --default "default"` で取得し、無効値の場合は警告を出力して `default` にフォールバック）
- 警告メッセージフォーマットの定義: `【警告】rules.cycle.mode の値 "..." は無効です`

## 境界
- 設定キーの仕様定義とプロンプトへのバリデーション指示の追加のみ。設定値に基づくモード別分岐ロジック（名前入力フロー等）はUnit 003の責務
- `docs/aidlc.toml` のスキーマ変更は含むが、設定GUI等は含まない

## 依存関係

### 依存する Unit
- なし

### 外部依存
- `docs/aidlc/bin/read-config.sh`（既存のTOML読み取り基盤）

## 非機能要件（NFR）
- **パフォーマンス**: read-config.sh呼び出しの追加遅延は無視可能レベル
- **セキュリティ**: 設定値のバリデーション（許可値のホワイトリスト判定）
- **スケーラビリティ**: 将来的なモード値の追加が容易な設計
- **可用性**: 設定読み取り失敗時のフォールバックにより動作が停止しない

## 技術的考慮事項
- `read-config.sh` は既に任意のキーパスを読み取り可能。新しい設定キーの追加のみ
- バリデーション（有効値チェック・警告出力・フォールバック）はプロンプト内の指示として記述。ランタイムでAIが `read-config.sh` を呼び出し、結果を検証する
- 既存の `rules.branch.mode` のバリデーションパターン（inception.md内）を参考にする

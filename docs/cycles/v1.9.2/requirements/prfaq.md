# PRFAQ: AI-DLC v1.9.2

## Press Release

**AI-DLC Starter Kit v1.9.2 リリース - 複数AIツール対応とプレリリースバージョンサポート**

AI-DLC Starter Kit チームは本日、v1.9.2をリリースしました。本リリースでは、複数のAIツールを使い分ける開発者向けの機能強化と、アルファ版・ベータ版開発を支援する機能を追加しています。

**主な新機能:**

- **AIツール自動検出**: 使用中のAIツール（Claude Code、Cursor、Cline、Windsurf等）を自動検出し、Co-Authored-By情報を正確に設定
- **複数AIサービス対応**: `ai_tools`設定でAIレビューに使用するサービスの優先順位を設定可能
- **プレリリースバージョン対応**: `v2.0.0-alpha.6`などのプレリリースバージョン形式で履歴管理が可能
- **KiroCLI Skills対応**: KiroCLI環境でもAI-DLCのスキルが利用可能

**改善:**

- **operations.md最適化**: 説明文を簡略化し、1000行以下に最適化（機能変更なし）

「複数のAIツールを日常的に使い分けていますが、これまではCo-Authored-Byが固定値になってしまい、正確な貢献記録が残せませんでした。v1.9.2の自動検出機能により、この問題が解消されます」と、ベータテスターは語っています。

## FAQ

### Q1: AIツール自動検出はどのように動作しますか？

A: AIツール（Claude Code、Cursor等）が自身を認識し、対応するCo-Authored-By値を設定します。環境変数も補助的に参照します。検出できない場合はユーザーに確認し、固定デフォルト値は使用しません。

### Q2: 既存の設定は影響を受けますか？

A: 影響を受けません。`aidlc.toml`で`ai_author`を設定している場合、その値が優先されます。自動検出は`ai_author`が未設定の場合のみ動作します。また、`ai_author_auto_detect = false`を設定することで自動検出を完全に無効化できます。

### Q3: サポートされているプレリリース形式は？

A: `vX.Y.Z-alpha.N`、`vX.Y.Z-beta.N`、`vX.Y.Z-rc.N`形式をサポートしています。ビルドメタデータ（`+build`）は現時点では非対応です。

### Q4: ai_tools設定のデフォルト値は？

A: `["codex"]`です。設定がない場合はCodexが最優先で使用されます。複数指定した場合、利用可能な最初のツールが使用されます。

### Q5: KiroCLIで使用するには？

A: KiroCLI v1.24.0以降が必要です。スキルファイルの配置方法は、KiroCLI公式ドキュメントのSkillsセクションを参照してください。

### Q6: operations.mdの最適化で削除された機能はありますか？

A: 機能は削除されていません。冗長な説明文と重複する注意書きを簡略化し、1000行以下に最適化しました。

### Q7: 自動検出を無効化できますか？

A: 2つの方法があります:
1. `aidlc.toml`で明示的に`ai_author`を設定する（設定値が優先）
2. `ai_author_auto_detect = false`を設定する（自動検出を完全に無効化）

# Unit 001 計画: サイクル名バリデーション統一

## 概要

write-history.shのサイクル名バリデーションを他のスクリプト（cycle-label.sh等）と統一する。
バージョン形式（vX.X.X）への依存を解除し、最低限のチェックのみに変更する。

## 変更対象ファイル

- `prompts/package/bin/write-history.sh`（編集対象）

## 実装計画

1. `validate_version()` 関数を `validate_cycle()` に名前変更
2. バリデーションロジックを変更：
   - 空文字を拒否
   - パス区切り文字（`/`）を拒否
   - それ以外は許容
3. エラーメッセージを適切に変更
4. 関数呼び出し箇所を更新

## 完了条件チェックリスト

- [ ] write-history.shがサイクル名として任意の文字列を受け入れる（vX.X.X形式以外も可）
- [ ] 空文字と危険な文字（/）は拒否される
- [ ] 既存のサイクル形式（v1.9.2等）は引き続き動作する

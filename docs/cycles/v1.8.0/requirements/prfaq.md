# PRFAQ: AI-DLC v1.8.0 - スクリプト化基盤構築

## Press Release（プレスリリース）

**見出し**: AI-DLC v1.8.0リリース - スクリプト化基盤でAIエージェント体験を大幅改善

**副見出し**: 複合コマンドのスクリプト化により、許可リスト運用が劇的に簡素化

**発表日**: 2026年1月（予定）

**本文**:

### 背景
AI-DLC（AI-Driven Development Lifecycle）スターターキットは、AIを開発の中心に据えた新しい開発手法を提供しています。しかし、プロンプト内の複合コマンド（`&&`, `||`, `$(...)` 等）がAIエージェントの許可リスト運用を複雑にしていました。シェル演算子を含むコマンドは、許可リストに登録しても毎回承認が必要となり、開発フローが中断されていました。

### プロダクト
AI-DLC v1.8.0では、複合コマンドを独立したスクリプトに移行しました。新たに追加された4つのスクリプト（check-deps, get-config, write-history, detect-cycle）により、プロンプト内のコードが大幅に簡素化されました。開発者は単純なスクリプト呼び出しを許可リストに登録するだけで、スムーズな開発フローを実現できます。

さらに、簡略指示の追加（「AIDLCアップデート」）、フィードバック手段の整備、プランモード活用ガイドなど、開発者体験を向上させる改善を多数実施しました。

## FAQ（よくある質問）

### Q1: スクリプト化によって何が変わりますか？
A: プロンプト内の複合コマンド（`if [...]; then ... fi` や `VAR=$(cmd)` 等）が、単純なスクリプト呼び出し（`docs/aidlc/bin/check-deps.sh`）に置き換わります。これにより、AIエージェントの許可リストに登録しやすくなり、承認プロンプトの発生が大幅に減少します。

### Q2: 既存プロジェクトへの影響はありますか？
A: 後方互換性を維持しています。スクリプトが存在しない環境では、従来通りAIが直接判断するフォールバック動作が有効です。アップグレード時にrsyncでスクリプトがコピーされるため、特別な移行作業は不要です。

### Q3: daselがインストールされていない環境でも動作しますか？
A: はい。`get-config.sh` はdasel未インストール時に空文字を返し、AIが設定ファイルを直接読み取るフォールバック動作をします。

### Q4: markdownlintを使っていないプロジェクトはどうなりますか？
A: `aidlc.toml` の `[rules.linting]` セクションで `markdown_lint = false` を設定することで、markdownlint実行をスキップできます。

### Q5: Unit数が13と多いですが、すべて実装する必要がありますか？
A: Phase分けにより並行実装が可能です。スクリプト化基盤（Unit 001-005）を優先し、パッチレベル（Unit 006-008）とUX改善（Unit 009-012）は状況に応じて調整できます。Unit 013（プロンプト最適化準備）は分析のみで、実装は次サイクルに持ち越します。

### Q6: プランモードとは何ですか？
A: Claude Codeの機能で、実装前に計画を立てる読み取り専用モードです。AI-DLCのConstruction Phase Phase1（設計）での活用を推奨しており、詳細は新設の `guides/plan-mode.md` を参照してください。

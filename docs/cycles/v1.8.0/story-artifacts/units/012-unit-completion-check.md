# Unit: Unit完了条件明確化

## 概要
Unit開始時に完了条件を提示し、終了時に確認するフローを追加する。

## 技術的考慮事項

### Unit開始時のフロー追加
1. Unit定義ファイルを読み込む
2. 「含まれるユーザーストーリー」から受け入れ基準を抽出
3. チェックリスト形式で完了条件を提示
4. ユーザーに確認後、実装開始

### Unit終了時のフロー追加
1. 開始時に提示したチェックリストを再表示
2. 各項目の達成状況を確認
3. 未達成項目がある場合は警告表示
4. すべて達成で完了マーク

### 変更対象ファイル
- `prompts/package/prompts/construction.md`
  - 「Unit完了時の必須作業」セクション前に確認フローを追加
  - Unit開始時の「実行前確認」セクションに完了条件提示を追加

### プロンプト追加例
```markdown
### Unit開始時: 完了条件の確認

Unit定義ファイルから受け入れ基準を抽出し、以下の形式で提示:

**完了条件チェックリスト**:
- [ ] 条件1
- [ ] 条件2
- [ ] ...

この条件を満たした時点でUnitを完了とします。よろしいですか？
```

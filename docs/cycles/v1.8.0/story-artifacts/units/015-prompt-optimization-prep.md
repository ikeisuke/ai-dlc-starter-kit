## 技術的考慮事項

### 分析項目
1. **行数・文字数**: 各プロンプトファイルのサイズ
2. **重複指示**: 複数プロンプトに同じ内容が記載されている箇所
3. **冗長指示**: 簡潔に書き直せる指示
4. **削除候補**: 不要になった指示、古い指示
5. **統合候補**: まとめられる指示

### 成果物
1. **分析レポート**: `docs/cycles/v1.8.0/requirements/prompt-analysis.md`
   - 各プロンプトの行数・文字数
   - 問題箇所の一覧
   - 改善提案

2. **バックログ追加**: 次サイクル向けの具体的なタスクをIssue化
   - 重複指示の統合
   - 冗長指示の簡素化
   - Operations Phaseでのサイズチェック機能追加

# Unit 001 実行計画: 環境情報一覧スクリプト

## 概要

gh, dasel, jj等の依存ツールの状態を一覧で出力するスクリプト `env-info.sh` を作成する。

## Phase 1: 設計

### ステップ1: ドメインモデル設計

**目的**: スクリプトの責務と構造を定義

**成果物**: `docs/cycles/v1.8.0/design-artifacts/domain-models/001-env-info_domain_model.md`

**設計内容**:
- 対象ツール: gh, dasel, jj, git
- 各ツールの状態判定ロジック
- 出力フォーマット仕様

### ステップ2: 論理設計

**目的**: 実装の詳細設計

**成果物**: `docs/cycles/v1.8.0/design-artifacts/logical-designs/001-env-info_logical_design.md`

**設計内容**:
- スクリプトの引数仕様
- 関数分割
- エラーハンドリング

### ステップ3: 設計レビュー

設計内容をレビューし、承認を得る。

## Phase 2: 実装

### ステップ4: コード生成

**成果物**: `prompts/package/bin/env-info.sh`

**実装内容**:
- 各ツールの存在・認証確認
- 統一フォーマットでの出力

### ステップ5: テスト生成

**成果物**: 手動テスト手順（シェルスクリプトのため）

### ステップ6: 統合とレビュー

- スクリプト実行テスト
- 各環境での動作確認
- 実装記録作成

## 完了条件

- [ ] `prompts/package/bin/env-info.sh` が作成されている
- [ ] gh, dasel, jj, git の状態を判定できる
- [ ] 統一フォーマット（`ツール名:状態`）で出力される
- [ ] オフライン環境でも動作する

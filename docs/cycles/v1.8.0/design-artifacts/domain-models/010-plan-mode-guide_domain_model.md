# ドメインモデル: プランモード活用ガイド

## 概要

Claude Codeのプランモード機能をAI-DLCワークフローで活用するためのガイドドキュメント。

## ドメイン概念

### プランモード（Plan Mode）

Claude Codeの機能で、実装前に計画を立案・レビューするためのモード。

**属性**:

- 目的: 実装前の設計レビューと手戻り削減
- トリガー: EnterPlanMode ツール
- 終了: ExitPlanMode ツール
- 制約: プランモード中は編集系ツールが使用不可

### プランファイル（Plan File）

プランモード中に作成される計画ドキュメント。

**属性**:

- 保存先: 指定されたパス
- 内容: 実装計画、ステップ、考慮事項
- 用途: ユーザー承認の対象

### AI-DLCとの関係

| AI-DLCフェーズ | プランモード活用シーン |
|----------------|------------------------|
| Construction Phase 1（設計） | ドメインモデル・論理設計の計画立案 |
| 複雑なUnit開始時 | 実装アプローチの事前検討 |

## ガイドの構成

```text
plan-mode.md
├── 1. プランモードとは
│   ├── 概要
│   └── メリット
├── 2. 使い方
│   ├── EnterPlanMode/ExitPlanMode
│   └── プランファイルの活用
├── 3. AI-DLCでの活用シーン
│   ├── Construction Phase での活用
│   └── 複雑なUnit実装前
└── 4. 注意事項
    ├── 編集制約
    └── 承認フローとの関係
```

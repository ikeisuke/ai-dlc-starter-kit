| 項目 | 値 |

| 想定読者 | AI-DLC利用者、Claude Code利用者 |

### セクション構成

#### 1. プランモードとは

**目的**: プランモードの概要と利点を説明

**内容**:

- 概要: 実装前に計画を立案するClaude Codeの機能
- メリット:
  - 設計レビューによる品質向上
  - 手戻りの削減
  - ユーザーとの認識合わせ

#### 2. 使い方

**目的**: 具体的な操作方法を説明

**内容**:

- EnterPlanMode: プランモードへの移行方法
- ExitPlanMode: プランモードからの終了方法
- プランファイル: 計画の記載場所と形式

#### 3. AI-DLCでの活用シーン

**目的**: AI-DLCワークフローでの具体的な活用方法を説明

**内容**:

- Construction Phase Phase1（設計）での活用
  - ドメインモデル設計前の計画立案
  - 論理設計前のアーキテクチャ検討
- 複雑なUnit実装前の計画立案
  - 実装アプローチの事前検討
  - リスクの洗い出し

#### 4. 注意事項

**目的**: 利用時の制約と注意点を説明

**内容**:

- プランモード中の制約
  - 編集系ツール（Edit, Write等）が使用不可
  - 読み取り・検索系ツールのみ使用可能
- AI-DLC承認フローとの関係
  - プランモードの承認 ≠ AI-DLCの設計承認
  - 両方の承認が必要

### スタイルガイド

- コード例は最小限（ツール説明のみ）
- 「実験的機能」表記は不要（標準機能のため）

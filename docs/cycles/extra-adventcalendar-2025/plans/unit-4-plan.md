# Unit-4 実装計画

## Unit情報
- **名称**: 本質と一人実践の差異考察セクション
- **依存関係**: Unit-1（AI-DLC概念説明）, Unit-3（一人AI-DLC体験談）
- **対応ユーザーストーリー**: US-4

## 目的
本来のAI-DLC（チーム開発想定）と一人実践の違いを考察し、一人でも活かせる点と限界がある点を明確にする。

## 実装内容

### 成果物
`docs/cycles/extra-adventcalendar-2025/article/section-4-essence-vs-solo.md`

### セクション構成案

1. **AI-DLCの本質的な前提**
   - チームでの役割分担を前提とした設計
   - 「会話の反転」の本来の意味（PM/開発者/QA間の対話）
   - LIFULLワークショップの例（組織的な取り組み）

2. **一人実践で活かせる点**
   - 「型」による開発プロセスの標準化
   - AIとの対話による要件の明確化
   - ドキュメント駆動開発の恩恵
   - 冪等性による中断・再開の容易さ

3. **一人実践の限界**
   - 複数視点によるレビューの欠如
   - インテント設定の責任が一人に集中
   - Inception Phaseでの「対話」が自問自答になりがち
   - チーム間の知見共有という効果が得られない

4. **一人でも工夫できる点**
   - 複数AIの活用（Claude Code + Codex）による疑似チーム
   - 時間を置いてのセルフレビュー
   - コミュニティやSNSでの知見共有

5. **まとめ：一人AI-DLCの価値**
   - 完全なAI-DLCではないが、エッセンスは活用可能
   - スターターキットという「型」の重要性

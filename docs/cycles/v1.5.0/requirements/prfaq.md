# PRFAQ: AI-DLC スターターキット v1.5.0

## Press Release（プレスリリース）

**見出し**: AI-DLC v1.5.0 - AIの自律判断リスクを低減し、より安全な開発体験を提供

**副見出し**: 予想禁止ルール、コード記述制限、外部入力検証など、品質向上ルールを追加

**発表日**: 2024年12月（予定）

**本文**:

AI-DLCスターターキットは、AIを開発プロセスの中心に据えた新しい開発方法論です。v1.5.0では、AIの自律的判断によるリスクを低減するための重要なルール強化を行いました。

**背景**

AI主導の開発では、AIが予想で方針を決めてしまったり、計画なしにコードを書いてしまうリスクがありました。また、外部入力（AI MCPの応答やユーザーの回答）をそのまま信頼して処理を進めることで、意図しない結果を招くケースがありました。

**v1.5.0の改善点**

1. **予想禁止・一問一答ルール**: AIは不明点を予想せず、質問リストを提示した後に一問ずつ確認します
2. **コード記述制限**: Construction Phase以外でのコード記述を原則禁止し、計画的な開発を促進します
3. **外部入力検証**: AI MCPやユーザー入力に対して批判的に評価し、AIの判断を明示的に提示します
4. **セットアップ改善**: サイクルセットアップの分離、グリーンフィールドセットアップの質問フロー改善

**開発者の声**

「AIが勝手に判断せず、確認してくれるようになったので安心して任せられる」
「フェーズごとの責務が明確になり、開発の見通しが良くなった」

**今後の展開**

今後も開発者体験の向上と、AI主導開発の安全性・信頼性向上に取り組んでいきます。

## FAQ（よくある質問）

### Q1: どのような変更がありますか？

A: 主に以下の変更があります:
- 予想禁止・一問一答質問ルールの追加
- Construction Phase以外でのコード記述制限
- 外部入力（AI MCP・ユーザー）の検証ルール追加
- サイクルセットアップ処理の専用プロンプト分離
- グリーンフィールドセットアップ時の質問フロー改善
- セルフアップデート処理の廃止（リポジトリ固有）

### Q2: 既存プロジェクトへの影響は？

A: 既存プロジェクトは、次回セットアップ時（setup-prompt.md実行時）にアップグレードを選択することで新ルールが適用されます。既存の成果物やサイクルには影響しません。

### Q3: メタ開発ルールとは？

A: AI-DLCスターターキット自体を開発する際の特別なルールです。`docs/aidlc/`は直接編集せず、必ず`prompts/package/`を編集します。これは、Operations Phaseでrsyncにより同期されるためです。

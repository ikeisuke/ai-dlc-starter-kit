# PRFAQ: AI-DLC Starter Kit v1.3.0

**副見出し**: 進捗管理の再設計、バージョン管理の改善、ワークフローの最適化により、より快適なAI駆動開発を実現

**本文**:

### 背景

AI-DLC Starter Kitは、AIを開発プロセスの中心に据えた新しい開発方法論の実践キットです。これまでの運用を通じて、チーム開発時のコンフリクト、バージョン管理の不整合、ワークフローの非効率といった課題が明らかになりました。v1.3.0では、これらの課題を解決し、より使いやすい開発フローを実現します。

### プロダクト

v1.3.0では以下の改善を行います：

1. **進捗管理の再設計**: progress.mdの維持・廃止・改善を検討し、複数人開発でもコンフリクトが起きにくい仕組みを実現
2. **バージョン管理の改善**: Operations Phaseでのバージョン更新が正しく動作するよう修正、初回セットアップ時のバージョン提案も改善
3. **ワークフローの最適化**: 気づき記録フロー、コンテキストリセットタイミング、バックログ確認ステップ、PRマージ後手順の明確化
4. **テンプレート・構造の改善**: Unit定義ファイルのパス管理、バックログファイル構造の改善

## FAQ（よくある質問）

### Q1: progress.mdはなくなるのですか？

A: Construction Phaseで検討し、最適な方針を決定します。選択肢として、(1) progress.md廃止（ファイル存在チェックやGit状態から自動検出）、(2) progress.md維持（構造改善）、(3) ハイブリッド方式があります。複数人開発でのコンフリクトを減らしつつ、進捗管理の利便性を維持できる方法を選択します。

### Q2: 既存のサイクルへの影響はありますか？

A: 既存サイクルへの影響は最小限に抑えます。後方互換性を考慮し、進行中のサイクルが壊れないよう配慮します。必要に応じて移行手順を提供します。

### Q3: バージョン管理の問題は具体的に何が修正されますか？

A: Operations Phaseでversion.txtを更新した際に、aidlc.tomlのstarter_kit_versionも同時に更新される手順を明確化します。これにより、次サイクルのセットアップ時にアップグレードが正しく検出されるようになります。

### Q4: 初回セットアップ時のバージョン提案はどう改善されますか？

A: package.json、pyproject.toml、Cargo.toml等のプロジェクト既存バージョン情報を調査し、それを考慮したサイクルバージョンを提案するようになります。既存プロジェクトにAI-DLCを導入する際の体験が向上します。

### Q5: ワークフロー改善の具体的な内容は？

A: 以下の4点を改善します：
1. Unit作業中に別Unitに関連する気づきを記録するフローの定義
2. コンテキストリセットの提案タイミングをUnit完了時からUnit開始時に変更
3. Construction Phase開始時のバックログ確認ステップの追加
4. PRマージ後にmainブランチを最新化する手順のドキュメント化

# Unit: Unit定義パス管理

## 概要

Unit定義ファイルのパスを予測せずに正確に特定できる仕組みを実現する。

## 含まれるユーザーストーリー

- US-8: Unit定義ファイルのパス管理改善

## 責務

- Unit定義ファイルの命名規則の統一、またはパス記録の仕組み導入
- ファイル名を予測して試す必要がない状態の実現

## 境界

- Unit定義ファイルの内容・フォーマット変更は含まない
- 他のファイルパス管理は対象外

## 依存関係

### 依存する Unit

- Unit 1: 進捗管理再設計（依存理由: progress.md廃止の場合、パス記録方法が変わる可能性がある）

### 外部依存

- なし

## 非機能要件（NFR）

- **パフォーマンス**: 該当なし
- **セキュリティ**: 該当なし
- **スケーラビリティ**: 該当なし
- **可用性**: 該当なし

## 技術的考慮事項

選択肢：
1. **progress.mdにパス列追加**: 各Unitのファイルパスを明記
2. **命名規則の統一**: `unit{番号}_{snake_case名}.md` 形式に統一
3. **Unit一覧ファイル**: 別途Unit一覧とパスを管理するファイルを作成

Unit 1の方針決定後に最適な方法を選択する。

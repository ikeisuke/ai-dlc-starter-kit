# ステップ4: Unit定義 計画

## 概要

9件のユーザーストーリーをUnitに分解する。

## Unit構成案

関連性と依存関係を考慮して、以下の5つのUnitに分割する。

### Unit 1: 進捗管理再設計
- **含むUS**: US-1（進捗管理の仕組み再設計）
- **理由**: 他のUnitに影響する基盤部分。方針決定が先に必要
- **優先度**: High
- **依存**: なし

### Unit 2: バージョン管理改善
- **含むUS**: US-2（バージョン更新修正）、US-3（初回バージョン提案改善）
- **理由**: バージョン関連をまとめる
- **優先度**: High
- **依存**: なし

### Unit 3: ワークフロー改善
- **含むUS**: US-4（気づき記録フロー）、US-5（リセットタイミング）、US-6（バックログ確認ステップ化）、US-7（PRマージ後手順）
- **理由**: プロンプト修正が主な作業でまとめて対応可能
- **優先度**: Medium
- **依存**: なし

### Unit 4: Unit定義パス管理
- **含むUS**: US-8（Unit定義ファイルのパス管理改善）
- **理由**: US-1の方針決定後に対応内容が変わる可能性
- **優先度**: Medium
- **依存**: Unit 1（progress.md廃止なら対応方法が変わる）

### Unit 5: バックログ構造改善
- **含むUS**: US-9（backlog.md構造改善）
- **理由**: 独立した作業
- **優先度**: Medium
- **依存**: なし

## 実行内容

1. 上記5つのUnitの定義ファイルを作成
2. 各Unitに依存関係を明記
3. progress.mdを更新（ステップ4完了）

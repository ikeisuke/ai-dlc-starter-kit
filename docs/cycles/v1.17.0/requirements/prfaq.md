# PRFAQ: AI-DLC Starter Kit v1.17.0

## Press Release（プレスリリース）

**見出し**: AI-DLC v1.17.0 — コミット管理とレビューフローを包括的に改善

**副見出し**: Inception squash対応、事後squash、AIレビュー完了強制、サブエージェント活用により、開発品質とレビュアビリティを向上

**発表日**: 2026-02

**本文**:

AI-DLC Starter Kit v1.17.0では、開発ワークフローの2つの重要な領域を改善します。

**コミット管理の強化**: これまでConstruction Phase（Unit完了時）のみで利用可能だったsquash統合が、Inception Phase完了時にも適用可能になりました。中間コミットが1つの `feat:` コミットにまとめられ、コミット履歴がクリーンに保たれます。また、squash-unit.shに `--retroactive` オプションが追加され、過去のUnit（HEAD以外）に対しても事後的にsquashを実行できるようになりました。GIT_SEQUENCE_EDITOR方式による非対話的rebaseで安全に実行されます。

**レビューフローの品質向上**: AIレビュー開始後に「ユーザーレビューへ進む」選択肢が除去され、AIレビューの完了が確実に保証されるようになりました。セルフレビューフォールバックではサブエージェント（コンテキスト分離）方式が採用され、バイアスの低減が実現されます。さらに、AIレビュー結果がサマリファイルとして蓄積され、PRの本文に記載されることで、人間レビュアーがAIレビューの結果を把握した上で効率的にレビューできるようになりました。Unit PRには要件・受け入れ基準・レビューサマリが直接記載され、サイクルPRにはIntent概要・受け入れ基準・全体修正概要に加え、各Unitのレビューサマリファイルへのリンクが記載されます。

**用語統一**: 「人間レビュー」から「ユーザーレビュー」への用語統一により、プロンプト全体の一貫性と可読性が向上しています。

**利用者の声**: 「PRにレビュー結果が載っているので、AIが何を確認済みかが一目で分かり、レビュー時間が大幅に短縮された」「事後squashのおかげで、squashし忘れたUnitも後から整理できるようになった」

**今後の展開**: 次サイクルではアップグレード処理のスクリプト化推進、worktreeクリーンアップスクリプト、セミオートモード対応などを予定しています。

## FAQ（よくある質問）

### Q1: Inception Phase完了時のsquashは必須ですか？
A: いいえ。`docs/aidlc.toml` の `rules.squash.enabled` が `false` の場合、通常コミットにフォールバックされます。また、squash実行前にユーザー確認が入るため、意図しないsquashは発生しません。

### Q2: `--retroactive` オプションでrebase衝突が発生した場合はどうなりますか？
A: 衝突発生時はエラーコード `squash:error:conflict` が返され、`git rebase --abort` によるリカバリ手順が提示されます。squash前後で `git diff <base>..HEAD` の一致を検証する仕組みもあり、安全性を確保しています。なお、`--retroactive` はGit運用時のみ対象であり、jj環境での事後squashは本バージョンではスコープ外です。

### Q3: AIレビューを途中で打ち切ってユーザーレビューに進めなくなるのですか？
A: AIレビュー開始前（ステップ4）と、外部AIレビューツールが続行不能な場合（ステップ6）にはユーザーレビューへのフォールバックが維持されています。制限されるのはAIレビュー実行中の中途離脱のみで、品質ゲートとしての機能を担保するための変更です。

### Q4: サブエージェントが起動できない環境ではどうなりますか？
A: サブエージェント起動失敗時は従来のインラインセルフレビューにフォールバックされます。Claude Code以外のAIツールでは、サブプロセスやサブセッションで同等のコンテキスト分離を実現する手順が記載されます。

### Q5: レビューサマリファイルはどこに保存されますか？
A: Construction Phaseでは `docs/cycles/{{CYCLE}}/construction/units/{NNN}-review-summary.md`（NNN = Unit番号、例: `001-review-summary.md`）、Inception Phaseでは `docs/cycles/{{CYCLE}}/inception/{step-name}-review-summary.md`（step-name = 成果物のステップ名、例: `intent-review-summary.md`）に保存されます。AIレビュー未実施の場合はファイルは生成されません。

### Q6: 既存のプロジェクトへの影響はありますか？
A: 既存の `squash-unit.sh` のデフォルト動作（`git reset --soft` 方式）は変更されません。`--retroactive` は明示指定時のみ有効です。レビューフロー変更もAIレビューモードが `disabled` の環境には影響しません。用語変更は表示文言のみで機能への影響はありません。

### Q7: `--retroactive` の実行前に影響範囲を確認できますか？
A: はい。`--dry-run` オプションと組み合わせることで、対象コミット一覧とrebase計画を事前に確認できます。実際のrebaseは実行されないため、安全に影響範囲を把握できます。

# Intent（開発意図）

## プロジェクト名
AI-DLC Starter Kit v1.17.0

## 開発の目的
AI-DLCのコミット管理とレビューフローを改善し、開発品質とレビュアビリティを向上させる。

具体的には以下の2つの軸で改善を行う：

1. **コミット/squash管理の強化**: Inception Phase完了時のsquashルール追加と、事後squash（過去Unit対象）への対応により、コミット履歴の整合性を改善する
2. **レビューフローの品質向上**: AIレビューの完了強制、セルフレビューのコンテキスト分離、PRへのレビュー結果開示、および用語統一により、レビュープロセスの信頼性とレビュアビリティを向上させる

## ターゲットユーザー
AI-DLCスターターキットを利用する開発者・レビュアー

## ビジネス価値
- コミット履歴がクリーンになり、変更追跡が容易になる
- AIレビューが確実に完了することで品質ゲートが機能する
- セルフレビューのバイアスが低減し、レビュー精度が向上する
- PRにレビュー結果が開示されることで、人間レビュアーの負荷軽減と観点共有が実現する
- 用語統一により、プロンプトの一貫性と可読性が向上する

## 成功基準
- Inception Phase完了時にsquashが実行でき、中間コミットが1つのfeatコミットにまとまる
- 過去のUnit（HEAD以外）に対して事後squashが実行できる
- AIレビュー開始後、指摘0件または指摘対応判断完了までレビューが続行される
- セルフレビューがサブエージェント（コンテキスト分離）で実行される
- Unit PR・サイクルPRにレビューサマリ（指摘内容・対応）が記載される
- プロンプト内の「人間レビュー」が「ユーザーレビュー」に統一される

## 期限とマイルストーン
1サイクルで完了

## 制約事項
- `docs/aidlc/` は直接編集禁止。変更は `prompts/package/` を編集する（メタ開発ルール）
- 既存のsquash-unit.shを拡張する形で実装（新規スクリプトは最小限）
- 事後squashは `GIT_SEQUENCE_EDITOR` 方式で実装（#228の設計に準拠）

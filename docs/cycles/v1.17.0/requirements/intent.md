# Intent（開発意図）

## プロジェクト名
AI-DLC Starter Kit v1.17.0

## 開発の目的
AI-DLCのコミット管理とレビューフローを改善し、開発品質とレビュアビリティを向上させる。

具体的には以下の2つの軸で改善を行う：

1. **コミット/squash管理の強化**: Inception Phase完了時のsquashルール追加と、事後squash（過去Unit対象）への対応により、コミット履歴の整合性を改善する
2. **レビューフローの品質向上**: AIレビューの完了強制、セルフレビューのコンテキスト分離、PRへのレビュー結果開示、および用語統一により、レビュープロセスの信頼性とレビュアビリティを向上させる

## ターゲットユーザー
AI-DLCスターターキットを利用する開発者・レビュアー

## ビジネス価値
- コミット履歴がクリーンになり、変更追跡が容易になる
- AIレビューが確実に完了することで品質ゲートが機能する
- セルフレビューのバイアスが低減し、レビュー精度が向上する
- PRにレビュー結果が開示されることで、人間レビュアーの負荷軽減と観点共有が実現する
- 用語統一により、プロンプトの一貫性と可読性が向上する

## 成功基準
- Inception Phase完了時に `squash-unit.sh` でsquashが実行でき、中間コミットが1つの `feat:` コミットにまとまる（`git log --oneline` で確認可能）
- 過去のUnit（HEAD以外）に対して `--retroactive` オプションで事後squashが実行でき、後続コミットが破損しない（`git log --oneline` で確認可能）
- AIレビュー開始後、review-flow.md の選択肢から「ユーザーレビューへ進む」が除去され、指摘0件または指摘対応判断フロー完了まで続行される
- セルフレビューフォールバック時に `Task` ツール等でサブエージェントが起動され、メインコンテキストと分離した状態でレビューが実行される
- AIレビュー実施時にレビューサマリファイル（`docs/cycles/{{CYCLE}}/construction/units/{NNN}-review-summary.md`）が生成され、Unit PR本文にその内容が記載される。サイクルPRではサマリファイルへのリンクが記載される
- prompts/package/ 配下の全プロンプトで「人間レビュー」「人間に承認」「人間に提示」が「ユーザーレビュー」「ユーザーに承認」「ユーザーに提示」に置換される

## 期限とマイルストーン
1サイクルで完了

## 制約事項
- `docs/aidlc/` は直接編集禁止。変更は `prompts/package/` を編集する（メタ開発ルール）
- 既存のsquash-unit.shを拡張する形で実装（新規スクリプトは最小限）
- 事後squashは `GIT_SEQUENCE_EDITOR` 方式で実装（#228の設計に準拠）

## 影響範囲

### squash-unit.sh 拡張（#234, #228）
- **影響**: `commit-flow.md`（Inception Phase完了コミット手順追加）、`inception.md`（完了時手順にsquashステップ追加）、`squash-unit.sh`（`--retroactive` オプション追加）
- **後方互換性**: 既存の `--base` 方式（reset --soft）はデフォルト動作として維持。`--retroactive` は明示指定時のみ有効
- **フォールバック**: squash失敗時は既存のエラーリカバリフロー（recovery コマンド提示）を踏襲

### レビューフロー変更（#231, #226）
- **影響**: `review-flow.md`（選択肢変更、セルフレビュー手順変更）
- **後方互換性**: AIレビューを使用しない環境（mode=disabled）には影響なし
- **フォールバック**: サブエージェント起動失敗時は従来のインラインセルフレビューにフォールバック

### PRレビュアビリティ向上（新規）
- **影響**: `commit-flow.md` または `review-flow.md`（レビューサマリファイル生成手順追加）、`operations.md`（PR作成テンプレート変更）、`construction.md`（Unit PR作成テンプレート変更）
- **後方互換性**: レビューサマリファイルが存在しない場合はPR本文にサマリセクションを省略

### 用語変更（#230）
- **影響**: `prompts/package/` 配下の全プロンプトファイル
- **後方互換性**: 表示文言の変更のみで機能への影響なし

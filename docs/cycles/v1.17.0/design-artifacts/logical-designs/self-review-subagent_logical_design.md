# 論理設計: セルフレビューでのサブエージェント活用

## 概要

review-flow.md のステップ5.5（セルフレビューフォールバック）を、サブエージェント（コンテキスト分離）方式に変更する論理設計。メインエージェントとは独立したサブエージェントがレビューを実行し、結果をメインエージェントに返す方式を定義する。

## アーキテクチャパターン

**Delegate Pattern（委譲パターン）** を採用。

- メインエージェントがレビュー実行をサブエージェントに委譲する
- 選定理由: コンテキスト分離による独立したレビュー視点の確保と、失敗時のフォールバック（インラインレビュー）を自然に表現できる

## コンポーネント構成

### レイヤー / モジュール構成

```text
review-flow.md ステップ5.5
├── レビュー実行戦略判定
│   ├── サブエージェント方式（Claude Code環境）
│   └── インラインレビュー方式（フォールバック / 非Claude環境）
├── サブエージェント起動・結果受信
│   └── 指示テンプレート
└── 結果処理（既存フローと共通）
    ├── 反復レビュー制御
    ├── 指摘対応判断フロー接続
    └── 履歴記録
```

### コンポーネント詳細

#### メインエージェント（ステップ5.5 制御部）

- **責務**: レビュー実行戦略の判定、サブエージェント起動、結果受信、反復制御、指摘対応判断フロー接続、履歴記録、ユーザーへの結果提示
- **依存**: サブエージェント（Task ツール経由）、既存の指摘対応判断フロー、write-history.sh
- **公開インターフェース**: 既存のステップ5.5 エントリポイント（変更なし）

#### サブエージェント（レビュー実行部）

- **責務**: 指定されたファイルを読み取り、メインエージェントから渡された観点に基づいてレビューを実施し、検出した指摘を統一フォーマットで返す（レビュー実行のみ。フィードバック反映・修正判断・反復制御はメインエージェントの責務）
- **依存**: Read ツール（対象ファイル読み取り）
- **公開インターフェース**: 自然言語による結果テキスト（統一フォーマット）

**委任範囲の明確化**: subagent-usage.md の「設計レビューは委任不可（メインで処理）」は、レビュー→フィードバック反映を含むプロセス全体を指す。本設計ではサブエージェントは「レビュー実行（読み取り・評価・結果出力）のみ」を担当し、フィードバック反映・修正判断はメインエージェントが行うため、ガイドとの矛盾はない。

## インターフェース設計

### サブエージェント入力仕様（メイン → サブ）

サブエージェントへの指示は自然言語テンプレートで構成される。

#### 入力パラメータ

| パラメータ | 型 | 必須 | 説明 |
|-----------|------|------|------|
| `target_files` | string[] | 必須 | レビュー対象ファイルパス一覧 |
| `review_type` | string | 必須 | レビュー種別（code / architecture / security / inception） |
| `review_criteria` | string | 必須 | レビュー観点の本文（メインエージェントが SKILL.md から読み取って展開した内容） |

**注意**: メインエージェントが事前に対応する SKILL.md を読み取り、「レビュー観点」セクションの内容を `review_criteria` として展開してからサブエージェントに渡す。サブエージェントはファイルシステム上の SKILL.md パスに依存しない。

#### 指示テンプレート

```text
以下のファイルをレビューしてください。

**レビュー種別**: {review_type}

**対象ファイル**:
{target_files を改行区切りで列挙}

**レビュー観点**:
{review_criteria（メインエージェントが SKILL.md から展開した観点本文）}

**出力フォーマット**:
レビュー結果を以下のフォーマットで出力してください。

指摘がある場合:

指摘 #1
- 重要度: {高 | 中 | 低}
- 内容: {指摘内容の要約}
- 推奨修正: {修正方法の提案}

指摘 #2
...

合計: {N}件（高: {n}件 / 中: {n}件 / 低: {n}件）

指摘がない場合:
指摘0件
```

### サブエージェント出力仕様（サブ → メイン）

#### 出力フォーマット

サブエージェントからの返却は自然言語テキスト。メインエージェントがサブエージェントの返却内容を確認し、指摘の有無を判断する:

- **指摘なし**（「指摘0件」等の表現を含む）: `status=ok`, `findings=[]`
- **指摘あり**（指摘内容が含まれる）: `status=ok`, `findings` を解析
- **サブエージェント起動失敗 / タイムアウト / 結果なし**: `status=failed`

**注意**: サブエージェントはLLMであるため、出力フォーマットの厳密な保証はない。メインエージェントは柔軟に結果を解釈し、解析不能な場合はインラインレビューにフォールバックする。

**最小受理条件**: メインエージェントがサブエージェントの出力を有効な指摘として受理する条件:

- 各指摘に `severity`（重要度）、`content`（内容）、`recommendation`（推奨修正）の3項目が含まれていること
- 3項目のいずれかが欠落している指摘は無効として除外する
- 全指摘が無効（解析不能）の場合: `status=failed` 扱いとし、インラインレビューにフォールバックする

#### findings 各要素の構造（メインエージェントが解析・管理）

サブエージェントは「今回検出した指摘」のみを返す（ステートレス）。`id` と `judgement`（未対応/対応済）の管理はメインエージェントの責務。

| フィールド | 型 | 管理者 | 説明 |
|-----------|------|--------|------|
| `id` | string | メイン | 指摘番号（メインが付与、例: `#1`） |
| `severity` | enum | サブ | 重要度（`高` / `中` / `低`） |
| `content` | string | サブ | 指摘内容 |
| `recommendation` | string | サブ | 推奨修正 |
| `judgement` | enum | メイン | 判定（`未対応` / `対応済`）- メインが反復間の状態を追跡 |

## 処理フロー概要

### セルフレビュー実行の処理フロー

**ステップ**:

1. **環境判定**: Claude Code 環境かどうかを確認
   - Claude Code: サブエージェント方式を試行
   - その他: インラインレビュー方式（従来のフォールバック）

2. **サブエージェント起動**（Claude Code の場合）:
   - Task ツールで `subagent_type="general-purpose"` を指定してサブエージェントを起動
   - メインエージェントが SKILL.md を読み取り、観点本文（`review_criteria`）を展開
   - 指示テンプレートに `target_files`, `review_type`, `review_criteria` を埋め込んで渡す

3. **結果受信・判定**:
   - サブエージェントからの結果テキストを受信
   - `status` を判定（ok / failed）
   - `status=failed` の場合: インラインレビュー方式にフォールバック

4. **結果処理**（既存フローと共通）:
   - 指摘があれば修正を反映
   - 反復レビュー（最大3回）: 修正後にサブエージェントを再起動
   - 全種別で指摘0件 → AIレビュー完了
   - 3回後も指摘あり → 指摘対応判断フローへ

5. **履歴記録**:
   - レビューツール欄: `self-review(subagent)` または `self-review(inline)`

**関与するコンポーネント**: メインエージェント、サブエージェント（Task ツール経由）、write-history.sh

### フォールバック処理フロー

```text
サブエージェント起動試行
├── 成功 → サブエージェント方式でレビュー実行
│   └── 結果受信
│       ├── 正常結果 → 結果処理へ
│       └── 異常（パース不能等） → インラインレビューへフォールバック
└── 失敗（環境不適合 / 起動エラー） → インラインレビューへフォールバック

インラインレビュー = 従来のステップ5.5 手順（メインエージェント自身がレビュー）
```

### Claude Code での起動例

```text
Task ツール呼び出し:
  subagent_type: "general-purpose"
  prompt: [指示テンプレートの内容]
  description: "セルフレビュー実行"
```

### 非 Claude Code 環境のガイダンス

プロンプト内に参考情報として記載:

```text
**他のAIツールでの代替方法**（参考情報、未保証）:
- サブプロセスやサブセッションを使用して、コンテキスト分離されたレビューを実行
- 実行方式はツール固有のドキュメントを参照
- 実行できない場合はインラインレビュー（従来方式）にフォールバック
```

## 履歴記録フォーマットの変更

### 変更点

レビューツール欄の値を更新:

| 状況 | レビューツール値 |
|------|----------------|
| サブエージェント方式で完了 | `self-review(subagent)` |
| インラインフォールバックで完了 | `self-review(inline)` |
| 従来（変更前） | `self-review` |

### 履歴記録テンプレート

```bash
docs/aidlc/bin/write-history.sh \
    --cycle {{CYCLE}} \
    --phase {{PHASE}} \
    --unit {N} \
    --unit-name "[Unit名]" \
    --unit-slug "[unit-slug]" \
    --step "AIレビュー完了" \
    --content "【AIレビュー完了（セルフレビュー）】指摘0件
【対象タイミング】{呼び出し元のステップ名}
【対象成果物】{成果物名}
【レビュー種別】{実行した全種別}
【レビューツール】{self-review(subagent) | self-review(inline)}
【注記】セルフレビューによる実施。{サブエージェント方式 | インラインフォールバック方式}で実施。"
```

## 非機能要件（NFR）への対応

該当なし（プロンプト変更のみ）

## 技術選定

- **変更対象**: `prompts/package/prompts/common/review-flow.md`（Markdown プロンプト文書）
- **正式サポート環境**: Claude Code（Task ツール使用）

## 実装上の注意事項

- review-flow.md のステップ5.5 のみ変更。ステップ5（外部AIレビュー）やステップ6, 7 は変更しない
- 既存の出力フォーマット・指摘対応判断フロー・外部入力検証ルールとの整合性を維持する
- 「外部入力検証の非適用」ルールはサブエージェント方式でも適用（同一エージェントシステム内のため）
- サブエージェントの反復レビュー時は毎回新しいサブエージェントを起動する（サブエージェントはステートレス）
- **ドキュメント間整合性**: review-flow.md ステップ5.5 と `prompts/package/guides/subagent-usage.md` の両方に「レビュー実行のみ委譲可、判断・反映はメイン責務」の但し書きを追記し、解釈を統一する

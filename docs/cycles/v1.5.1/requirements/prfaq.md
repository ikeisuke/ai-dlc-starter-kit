# PRFAQ: AI-DLC Starter Kit v1.5.1

## Press Release（プレスリリース）

**見出し**: AI-DLC Starter Kit v1.5.1 - セットアップ体験の改善とプロンプト構成の最適化

**副見出し**: プロジェクトタイプの明示的設定と、より直感的なセットアップ導線を実現

**発表日**: 2025-12

**本文**:

### 背景
AI-DLC Starter Kit は、AIを開発の中心に据えた新しい開発方法論を実践するためのツールキットです。これまでのバージョンでは、プロジェクトタイプ（CLI、Web、モバイルなど）の設定が明示的に行われておらず、Operations Phase で毎回確認が必要でした。また、セットアップ関連のプロンプトファイルが複数存在し、どれを使えばよいか分かりにくい状況がありました。

### プロダクト
v1.5.1 では以下の改善を実施しました：

1. **プロジェクトタイプ設定**: 初回セットアップ時にプロジェクトタイプを選択し、aidlc.toml に保存。Operations Phase で自動判断が可能に。

2. **セットアップ導線の明確化**: 通常サイクル開始時は `docs/aidlc/prompts/setup.md` を案内し、アップグレード時のみ `prompts/setup-prompt.md` を使用する導線を明確化。

3. **コミットメッセージの改善**: サイクル名を含めることで、Git履歴からの追跡が容易に。

4. **履歴記録タイミングの明確化**: 各フェーズで一貫したルールを適用。

### 今後の展開
セットアップ体験のさらなる簡素化と、より多くのプロジェクトタイプへの対応を検討しています。

## FAQ（よくある質問）

### Q1: 既存のプロジェクトはどうなりますか？
A: 後方互換性を維持しています。project.type が設定されていない場合は、Operations Phase 開始時に確認を促します。

### Q2: プロジェクトタイプは後から変更できますか？
A: はい、`docs/aidlc.toml` の `[project]` セクションに `type = "..."` を追加・変更することで設定できます。

### Q3: サイクル開始時に毎回アップグレード確認をスキップできますか？
A: はい、通常のサイクル開始は `docs/aidlc/prompts/setup.md` を使用してください。アップグレードが必要な場合のみ `prompts/setup-prompt.md` を使用します。

### Q4: コミットメッセージ形式は強制ですか？
A: 推奨形式です。AI がプロンプトに従って自動生成しますが、必要に応じて変更可能です。

# PRFAQ: ai-dlc-starter-kit v1.22.3

## Press Release（プレスリリース）

**見出し**: AI-DLCスターターキット v1.22.3 — スクリプト信頼性向上とAI開発プロセスの品質改善

**副見出し**: セットアップのサイレントエラー防止、バリデーションスコープ修正、直接実行優先原則の追加で開発体験を向上

**発表日**: 2026-03-16

**本文**:

AI-DLCスターターキット v1.22.3では、開発者から報告されたバグ修正とAI開発プロセスの品質改善を実施しました。

セットアップスクリプト `aidlc-setup.sh` において、`setup_claude_permissions` 関数の失敗がサイレントに無視される問題を修正しました。これにより、権限設定に失敗した場合でもユーザーに適切なエラーメッセージが表示されるようになります。

また、`check-bash-substitution.sh` バリデーションがスターターキット開発リポジトリ以外でもグローバルに適用される問題を修正し、他プロジェクトでの誤動作を防止しました。

AI開発プロセスの品質面では、Claude Code Insightsレポートで最大のフリクション源であった「Wrong Approach」パターンに対処するため、「直接実行優先原則」を共通ルールに追加しました。

さらに、Kiro環境でのAI-DLC利用を改善するため、エージェント設定ファイルを詳細なツール権限設定を含む構成にアップデートしました。

## FAQ（よくある質問）

### Q1: setup_claude_permissionsの修正は既存のセットアップに影響しますか？
A: 正常にセットアップが完了する場合は影響ありません。失敗時にのみ非ゼロexit statusが返されるようになり、エラーが適切に通知されます。

### Q2: check-bash-substitution.shの修正により、スターターキット開発リポジトリでのバリデーションは引き続き実行されますか？
A: はい。`project.name` が `ai-dlc-starter-kit` のリポジトリでは従来通りバリデーションが実行されます。それ以外のプロジェクトではスキップされます。

### Q3: 直接実行優先原則は既存のOverconfidence Prevention原則と矛盾しませんか？
A: 矛盾しません。直接実行優先は要件が明確な場合に適用され、要件が曖昧な場合は従来通り質問するOverconfidence Prevention原則が優先されます。両原則の適用条件が明文化されています。

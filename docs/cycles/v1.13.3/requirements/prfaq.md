# PRFAQ: AI-DLC Starter Kit v1.13.3

## Press Release（プレスリリース）

**見出し**: AI-DLCスターターキット v1.13.3 - 進捗管理の精度向上と企業利用の安全性強化

**副見出し**: Construction Phaseの進捗追跡を正確にし、フィードバック送信機能のオン/オフ制御を追加

**発表日**: 2026-02-08

**本文**:

AI-DLCスターターキット v1.13.3では、開発ワークフローの信頼性と企業利用時の安全性を向上させる2つの改善を実施しました。

1つ目は、Construction PhaseでUnitブランチを使用する際のprogress.md更新タイミングの修正です。これまでUnitブランチからのPRマージ後にprogress.mdが更新されていた問題を、PR準備完了時点で更新するように改善しました。Operations Phaseで検証済みのパターンをConstruction Phaseにも適用することで、PR上で正確な進捗状態が確認できるようになります。

2つ目は、フィードバック送信機能のオン/オフ設定の追加です。企業内でAI-DLCを利用する場合、フィードバック送信がパブリックリポジトリへのIssue作成を伴うため、機密情報漏洩のリスクがありました。`docs/aidlc.toml` の `[rules.feedback].enabled = false` を設定することで、フィードバック送信機能を無効化できるようになります。

「企業内でAI-DLCを導入する際、フィードバック機能が外部リポジトリにアクセスする点が懸念でした。この設定で安心して利用できます」（想定ユーザー）

今後もAI-DLCスターターキットの信頼性と安全性を継続的に改善していきます。

## FAQ（よくある質問）

### Q1: フィードバック機能を無効化すると、フィードバック自体が送れなくなりますか？
A: はい。`enabled = false` に設定すると、`aidlc feedback` コマンドは「この機能は無効化されています」と表示して処理を中断します。フィードバックを送りたい場合は、設定を `true` に戻すか、GitHub上で直接Issueを作成してください。

### Q2: progress.mdの更新タイミング変更は、Unitブランチを使わない場合にも影響しますか？
A: いいえ。この変更はUnitブランチを使用するフロー内のみに適用されます。Unitブランチを使用しない既存のワークフローには影響ありません。

### Q3: 個人設定（aidlc.toml.local）でフィードバック機能を上書きできますか？
A: はい。`docs/aidlc.toml.local` に `[rules.feedback].enabled = false` を設定することで、プロジェクト全体の設定に関わらず個人レベルで無効化できます。

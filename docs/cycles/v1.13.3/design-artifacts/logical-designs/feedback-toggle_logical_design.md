# 論理設計: フィードバック送信機能オン/オフ設定

## 概要

フィードバック送信機能の有効/無効を `docs/aidlc.toml` の設定で制御するため、AGENTS.mdのフィードバック送信セクションに設定読み込みと分岐ロジックを追加する。

**重要**: この論理設計では**コードは書かず**、コンポーネント構成とインターフェース定義のみを行います。

## アーキテクチャパターン

既存の設定読み込みパターン（`docs/aidlc/bin/read-config.sh` + プロンプト内分岐）を踏襲。新たなスクリプトやツールは追加しない。

## コンポーネント構成

### 変更対象コンポーネント

#### 1. `prompts/setup/templates/aidlc.toml.template`

- **責務**: 新規プロジェクトセットアップ時の設定テンプレート
- **変更内容**: `[rules.feedback]` セクションを追加
- **配置位置**: `[rules.linting]` と `[rules.custom]` の間

#### 2. `prompts/package/prompts/AGENTS.md`

- **責務**: AIエージェントの共通指示（フィードバック送信フローを含む）
- **変更内容**: フィードバック送信セクションの先頭に設定確認ステップを追加
- **依存**: `docs/aidlc/bin/read-config.sh`（既存スクリプト）

#### 3. `docs/aidlc.toml`

- **責務**: 現プロジェクトの設定ファイル
- **変更内容**: `[rules.feedback]` セクションを追加
- **配置位置**: `[rules.size_check]` の後、`[backlog]` の前（歴史的にテンプレートとセクション順序が異なるため）

## 処理フロー概要

### フィードバック送信フロー（変更後）

**ステップ**:

1. ユーザーが「AIDLCフィードバック」「aidlc feedback」と発言
2. **【新規】設定確認**: `docs/aidlc/bin/read-config.sh rules.feedback.enabled --default "true"` を実行
3. **【新規】分岐判定**:
   - 結果が `"false"` と完全一致 → ステップ4a（ブロック）
   - それ以外 → ステップ4b（既存フロー）
4. a. **ブロック時**: 以下のメッセージを表示して終了

   ```text
   【フィードバック送信機能 無効】
   この機能は無効化されています。
   `docs/aidlc.toml` の `[rules.feedback].enabled` を `true` に設定することで有効化できます。
   ```

4. b. **有効時**: 既存のフィードバック送信フロー（ヒアリング → Issue作成/URL案内）を実行

**関与するコンポーネント**: AGENTS.md, docs/aidlc/bin/read-config.sh

## aidlc.toml.template変更の具体的設計

### 追加する設定セクション

```toml
[rules.feedback]
# フィードバック送信機能設定（v1.13.3で追加）
# enabled: true | false
# - true: フィードバック送信機能を有効化（デフォルト）
# - false: フィードバック送信機能を無効化（企業利用時のセキュリティ対策）
enabled = true
```

### 配置位置

- **テンプレート**: `[rules.linting]` セクションの後、`[rules.custom]` セクションの前に配置する
- **現プロジェクト（`docs/aidlc.toml`）**: `[rules.size_check]` セクションの後、`[backlog]` セクションの前に配置する（歴史的にテンプレートとセクション順序が異なるため）

## 非機能要件（NFR）への対応

### セキュリティ

- **要件**: `enabled = false` 設定時にフィードバック送信が確実にブロックされること
- **対応策**: プロンプトの分岐構造で、false時はヒアリング・Issue作成・URL案内のすべてのステップをスキップし、ブロックメッセージのみ表示

## 技術選定

- **言語**: Markdown（プロンプトテキスト）、TOML（設定ファイル）
- **ツール**: `docs/aidlc/bin/read-config.sh`（既存の設定読み込みスクリプト）

## 実装上の注意事項

- `docs/aidlc/prompts/AGENTS.md` は直接編集しない（`prompts/package/` を編集し、Operations Phaseの `/aidlc-upgrade` で同期）
- ブロックメッセージには有効化方法の案内を含め、ユーザーが解決策を把握できるようにする
- 設定確認ステップは既存の手順の前に配置し、既存フローには変更を加えない

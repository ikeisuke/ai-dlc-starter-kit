# Intent（開発意図）

## プロジェクト名
AI-DLC v1.18.4 - フロー品質改善 + プロジェクト固有運用強化

## 開発の目的
AI-DLCのフロー実行時に発生している不具合・不便さを修正し、開発体験を向上させる。
加えて、スターターキット開発プロジェクト固有の運用手順を強化する。

## ターゲットユーザー
AI-DLCスターターキットを利用する開発者（自身を含む）

## ビジネス価値
- セミオートモードの遷移品質向上により、手動介入の削減
- エラー発生時のバックログ自動記録により、改善項目の取りこぼし防止
- セッションタイトル設定の代替手段により、複数セッション判別の実現
- PRマージ前のレビューコメント確認手順化により、レビュー指摘の見落とし防止

## 成功基準
- **#267**: セミオートモード（`automation_mode=semi_auto`）でConstruction Phase完了後、Operations Phaseへの遷移時にユーザー確認プロンプトが表示されず自動遷移する
- **#266**: Construction Phase中にビルド/テストエラーが発生した場合、バックログモードに応じて自動記録される
  - `issue` / `issue-only`: バックログIssueが自動作成される
  - `git` / `git-only`: `docs/cycles/backlog/` にバックログファイルが自動作成される
  - `issue` でIssue作成失敗時: `git` 方式にフォールバックする
  - `issue-only` でIssue作成失敗時: 警告を表示し手動対応を依頼する
- **#269**: Claude Code環境でセッション判別の代替手段が実装されている。具体的には、プロジェクト名・フェーズ・サイクルバージョンを含む情報（例: ステータスライン、環境変数、出力メッセージ等）が表示され、2セッション同時起動時に各セッションの作業対象サイクルを目視で判別できる
- **#268**: Operations Phaseのマージ手順にレビューコメント確認ステップが追加されており、未対応コメントがある場合にユーザーへ警告が表示される

## スコープ外（今回やらないこと）
- 対象Issue（#266〜#269）以外の機能追加・改善
- 大規模リファクタリング
- UIやCLI出力フォーマットの全面変更
- 他のAIツール（Cursor, Cline等）固有の対応

## 期限とマイルストーン
パッチリリース（v1.18.4）として2026-03-07までに完了を目標とする

## 既存機能への影響
| 影響対象 | 影響内容 | 回帰禁止条件 |
|---------|---------|-------------|
| セミオートモード（Phase遷移） | 遷移条件の修正（#267） | manual モードの遷移フローが変更されないこと |
| バックログ記録フロー | エラー時の自動記録追加（#266） | 既存の手動バックログ記録が引き続き動作すること |
| セッションタイトル設定 | 代替手段への置換（#269） | 代替手段が失敗してもフロー全体がブロックされないこと |
| Operations Phase PR運用 | レビューコメント確認ステップ追加（#268） | 既存のマージ手順が破壊されないこと |

## 制約事項
- プロンプト・テンプレートの修正は `prompts/package/` を編集すること（`docs/aidlc/` は直接編集禁止）
- #268はプロジェクト固有機能のため `docs/cycles/rules.md` への追記が中心
- Claude Code環境でのターミナル制御には制約がある

## 対応Issue
- #269: セッションタイトル設定が機能していない → 代替手段の調査・実装
- #268: Operations Phase: PRマージ前のレビューコメント確認ステップを手順化（プロジェクト固有）
- #267: セミオートモード: Construction→Operations遷移時に不要な停止が発生
- #266: 実装中エラー発生時のバックログ自動記録機能

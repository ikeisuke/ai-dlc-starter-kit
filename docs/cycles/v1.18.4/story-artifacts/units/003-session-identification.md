# Unit: セッション判別手段

## 概要
Claude Code環境でのセッション判別のため、既存のターミナルエスケープシーケンス方式を代替手段に置換する。

## 含まれるユーザーストーリー
- ストーリー 3: セッションタイトル代替手段（#269）

## 関連Issue
- #269

## 責務
- 代替手段の選定と実装（方式確定フェーズ: 候補調査・検証・決定）
- 3フェーズ（Inception/Construction/Operations）のステップ1.5/2.6の更新（実装フェーズ: 全フェーズ一括適用）
- プロジェクト名・フェーズ・サイクルバージョンの表示
- 設定失敗時のフォールバック（エラースキップ）

### 実装フェーズ
1. **方式確定**: 候補手段（statusline機能 → 環境変数 → 出力メッセージ）を調査・検証し、採用手段を決定
2. **全フェーズ適用**: 決定した手段を3フェーズのプロンプトに一括適用

## 境界
- ターミナルウィンドウタイトル設定の完全な互換性は保証しない
- 他のAIツール（Cursor, Cline等）への対応は行わない

## 依存関係

### 依存する Unit
- なし

### 外部依存
- 第1候補: Claude Code statusline機能（設定API経由でセッション情報を表示）
- 第2候補: 環境変数設定（AIDLC_SESSION等）
- 第3候補: 出力メッセージ（フェーズ開始時のコンソール出力）
- フォールバック順: 第1候補 → 第2候補 → 第3候補（上位が利用不可の場合に次候補を採用）

## 非機能要件（NFR）
- **パフォーマンス**: 該当なし
- **セキュリティ**: 該当なし
- **スケーラビリティ**: 該当なし
- **可用性**: 設定失敗時にフロー全体がブロックされないこと

## 技術的考慮事項
- 実装手段の候補: Claude Code statusline機能、環境変数設定、出力メッセージ等
- 修正対象: `prompts/package/prompts/inception.md`, `construction.md`, `operations.md` のステップ1.5/2.6
- statusline設定用スクリプトの作成が必要な場合あり
- `.claude/settings.local.json` の権限設定の更新が必要な場合あり

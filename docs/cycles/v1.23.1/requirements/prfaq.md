# PRFAQ: AI-DLC Starter Kit v1.23.1

## Press Release（プレスリリース）

**見出し**: AI-DLC Starter Kit v1.23.1 — ドキュメント改善・バグ修正・セットアップ整理パッチ

**副見出し**: v1.23.0リファクタリングの仕上げとして、ドキュメントの正確性向上、セットアップ体験の改善、将来のスキル化に向けた準備を実施

**発表日**: 2026-03

**本文**:

AI-DLC Starter Kit v1.23.1は、v1.23.0で実施したリファクタリングの続きとして、ドキュメント・バグ修正・セットアップ処理の整理を行うパッチリリースです。

commit-flow.mdのsquashパス表記が不明確であった問題を修正し、利用者が正しいファイルパスを参照できるようにしました。また、READMEに名前付きサイクル（Named Cycle）の説明を追加し、この機能の使い方が直感的にわかるようになりました。

aidlc-setup.shがstatus:success時にもexit code 1を返していたバグを修正しました。これにより、CI/CDパイプラインやスクリプトからの呼び出しで正しい終了コードが返されます。

Inception Phaseのステップ18（session-state.md復元チェック）は、セッション復元とコンパクション対応という2つの異なる機能が混在していた名残でした。セッション復元チェックを削除し、コンパクション対応はcompaction.md/session-continuity.mdで引き続き管理します。

セットアップ処理（setup-prompt.md）の責務を分類し、次サイクルでの完全スキル化（aidlc-setupへの吸収）に向けた移管計画を文書化しました。

## FAQ（よくある質問）

### Q1: セットアップ処理の完全スキル化はこのリリースに含まれますか？
A: いいえ。v1.23.1では責務分類と移管計画の文書化まで実施します。実際のスキル化は次サイクル以降で行います。

### Q2: session-state.mdの生成機能はなくなりますか？
A: コンパクション対応としてのsession-state.md生成はcompaction.md/session-continuity.mdで管理されており、引き続き動作します。削除対象はInception Phaseステップ18のセッション復元チェックのみです。

### Q3: 既存のセットアップフローに影響はありますか？
A: ありません。仕組みそのものは変更せず、exit codeバグ修正以外はドキュメント・プロンプト整理のみです。初回セットアップ・アップグレード・サイクル開始の主要シナリオは既存の動作を維持します。

### Q4: なぜパッチリリースなのですか？
A: 機能追加やフロー変更を含まず、バグ修正・ドキュメント改善・整理のみのため、パッチリリースとしています。

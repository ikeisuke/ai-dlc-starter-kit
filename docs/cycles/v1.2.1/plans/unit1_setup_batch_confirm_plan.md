# Unit 1: セットアップ情報まとめて確認 - 実装計画

## 概要
setup-init.mdのプロジェクト情報収集を改善：
- **README.md等を読み込んで情報を推測**
- **推測できた項目**: まとめて一覧表示し確認
- **推測できなかった項目**: 個別に質問

## 現状分析

### 現在の問題点
`prompts/setup-init.md` のセクション4「プロジェクト情報の収集」は一問一答形式で5回の質問が必要。

### 改善方針
1. README.md、package.json、pyproject.toml等のプロジェクトファイルを読み込み
2. プロジェクト名、概要、使用言語、フレームワーク等を自動推測
3. 推測結果をまとめて一覧表示し確認を求める
4. 推測できなかった項目のみ個別に質問

## 変更対象ファイル
- `prompts/setup-init.md` - セクション4の修正

## 変更内容

### 新しいフロー

#### Step 1: プロジェクト情報の自動収集

```markdown
### 4.1 プロジェクト情報の自動収集

以下のファイルからプロジェクト情報を推測します：
- README.md
- package.json（Node.jsプロジェクト）
- pyproject.toml / requirements.txt（Pythonプロジェクト）
- Cargo.toml（Rustプロジェクト）
- go.mod（Goプロジェクト）
- その他設定ファイル
```

#### Step 2: 推測結果のまとめ確認

```markdown
### 4.2 推測結果の確認

プロジェクトファイルを分析した結果、以下の情報を推測しました：

| 項目 | 推測値 | 情報ソース |
|------|--------|-----------|
| プロジェクト名 | [推測値] | [package.json / ディレクトリ名] |
| プロジェクト概要 | [推測値] | [README.md / package.json description] |
| 使用言語 | [推測値] | [ファイル拡張子 / 設定ファイル] |
| フレームワーク | [推測値] | [dependencies / imports] |
| 命名規則 | lowerCamelCase | デフォルト |

「上記の内容で問題ありませんか？変更したい項目があれば教えてください。」

- 「OK」「はい」→ 推測値を採用
- 変更がある → 指定された項目のみ更新
```

#### Step 3: 推測できなかった項目の個別質問

```markdown
### 4.3 追加情報の入力

推測できなかった項目については、個別に質問します。

（例）
「プロジェクト概要が推測できませんでした。1-2文で入力してください（スキップする場合は「スキップ」）」
```

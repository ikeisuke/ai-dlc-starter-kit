# PRFAQ: AI-DLC Starter Kit v1.1.0

## Press Release（プレスリリース）

**見出し**: AI-DLC Starter Kit v1.1.0 - 運用効率化と軽量サイクルで開発体験を向上

**副見出し**: Operations再利用性、Lite版サイクル、ブランチ確認、コンテキストリセット提案の4機能を追加し、AI駆動開発をさらに効率化

**発表日**: 2025年12月（予定）

**本文**:

### 背景

AI-DLC Starter Kit v1.0.1のリリース後、実際の運用を通じて以下の課題が明らかになりました：

- CI/CDや監視設定を毎サイクル新規作成する非効率性
- 軽微なバグ修正でもフル版サイクルを経由するオーバーヘッド
- 誤ったブランチで作業を開始するヒューマンエラー
- コンテキストリセットのタイミングが不明確

これらの課題を解決するため、v1.1.0では4つの新機能を導入します。

### プロダクト

**AI-DLC Starter Kit v1.1.0** は、以下の4つの新機能を提供します：

1. **Operations Phase再利用性**: CI/CD設定や監視設定をサイクル横断で再利用可能に。一度構築した運用基盤を次サイクルでも活用できます。

2. **軽量サイクル（Lite版）**: バグ修正や軽微な変更向けの簡略化されたフロー。Inception、Construction、Operationsの各フェーズを必要最小限のステップで実行できます。

3. **ブランチ確認機能**: セットアップ時に現在のGitブランチがサイクル名と一致しているか自動確認。不一致の場合は適切なブランチへの切り替えを提案します。

4. **コンテキストリセット提案機能**: フェーズ移行時やUnit完了時にコンテキストリセットを推奨し、次アクション開始用のコピペ可能なプロンプトを提示します。

### 顧客の声

- AIによるコンテキストサイズの自動検知

## FAQ（よくある質問）

### Q1: Lite版とFull版の違いは何ですか？
A: Lite版は軽微な変更（バグ修正、小機能追加）向けの簡略化されたフローです。
- **Inception**: Intent + 簡易Unit定義のみ（PRFAQ省略）
- **Construction**: 設計ステップ省略、直接実装可能
- **Operations**: 必要な場合のみ実行（スキップ可能）

Full版は従来どおり、すべてのステップを実行します。

### Q2: Operations再利用性はどのように使いますか？
A: Operations Phase開始時に、既存の共通設定（`docs/aidlc/operations/`）の有無を確認します。既存設定がある場合は「再利用/更新/新規作成」の選択肢が提示され、ユーザーが選択します。

### Q3: ブランチ確認機能は強制ですか？
A: いいえ、警告と提案のみです。ブランチ名がサイクル名と一致しない場合でも、ユーザーが承認すればそのまま続行できます。

### Q4: コンテキストリセットは必須ですか？
A: いいえ、推奨であり必須ではありません。ただし、コンテキストが膨らむとAIの性能が低下する可能性があるため、フェーズ移行時やUnit完了時にはリセットを推奨しています。

### Q5: v1.0.1からのアップグレード方法は？
A: セットアッププロンプトを再実行することで、プロンプトファイルとテンプレートが更新されます。既存のサイクル成果物（`docs/cycles/`配下）は影響を受けません。

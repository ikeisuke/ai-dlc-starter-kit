# Intent（開発意図）

## プロジェクト名
AI-DLC Starter Kit v1.1.0

## 開発の目的
AI-DLCスターターキットの実用性向上と運用効率化。v1.0.1サイクルで得られたフィードバックを反映し、より使いやすいツールキットを実現する。

## ターゲットユーザー
AI-DLCスターターキットを使用してAI駆動開発を行う開発者

## ビジネス価値
4つの機能追加による開発体験の向上：

| # | 機能 | 価値 |
|---|------|------|
| 1 | Operations Phase再利用性 | CI/CD等を一度構築したらサイクル横断で維持できる仕組みにより、重複作業を削減 |
| 2 | 軽量サイクル（Lite版） | 軽いバグ修正向けの簡略フローにより、小さな変更を素早く実施可能に |
| 3 | ブランチ確認機能 | セットアップ時にブランチ名をチェックし、誤ったブランチでの作業を防止 |
| 4 | コンテキストリセット提案機能 | フェーズ移行時・Unit完了時にリセット+呼び出しプロンプトを提示し、コンテキスト管理を支援 |

## 成功基準
- 4つの機能がすべて実装され、プロンプトに反映されていること
- 各機能が正常に動作すること
- 既存のAI-DLC基本フローが維持されていること

## 期限とマイルストーン
AI-DLCサイクルに従い、Inception → Construction → Operations の順で進行

## 制約事項
- 既存のAI-DLC基本フロー（Inception → Construction → Operations）を壊さないこと
- シンプルさを維持すること（過度な複雑化を避ける）
- プロンプトファイルの可読性を保つこと

## 不明点と質問（Inception Phase中に記録）

[Question] v1.1.0 サイクルの開発目的（Intent）は何ですか？
[Answer] バックログの4項目すべて（Operations Phase再利用性、軽量サイクル、ブランチ確認機能、コンテキストリセット提案機能）

[Question] コンテキストリセット提案機能の発動タイミングは？
[Answer] フェーズ移行時、Unit完了後に次のUnitへ進む際、長時間経過してコンテキストが膨らんだ時の3パターン

[Question] 「長時間経過してコンテキストが膨らんだ時」の検知方法は？
[Answer] AI側での自動検知は困難なため、まずはフェーズ移行時・Unit完了時の2パターンのみに絞る

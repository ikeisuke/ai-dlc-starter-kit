# Unit: ブランチ確認機能

## 概要
セットアップ時に現在のGitブランチ名がサイクル名と一致しているか確認し、不一致の場合は切り替えを提案する

## 含まれるユーザーストーリー
- ストーリー 3-1: ブランチ名の自動確認
- ストーリー 3-2: ブランチ切り替えの提案

## 責務
- セットアップ時の現在ブランチ取得
- サイクル名とブランチ名の比較
- 不一致時の警告表示と切り替え提案

## 境界
- ブランチの自動作成・切り替えは提案のみ（ユーザー承認が必要）
- ブランチ命名規則の強制は含まない（警告のみ）

## 依存関係

### 依存する Unit
なし（独立して実装可能）

### 外部依存
- Git（`git branch --show-current` コマンド）

## 非機能要件（NFR）
- **パフォーマンス**: 特になし
- **セキュリティ**: 特になし
- **スケーラビリティ**: 特になし
- **可用性**: Gitリポジトリでない場合はスキップ

## 技術的考慮事項
- `setup-prompt.md` の「実行環境の確認」セクションに追加
- 確認ロジック:
  1. `git branch --show-current` で現在のブランチを取得
  2. `CYCLE` がブランチ名に含まれているか確認
  3. 含まれていない場合、警告を表示
  4. `feature/{CYCLE}` 形式のブランチへの切り替えを提案

Medium

# Unit: コンテキストリセット提案機能

## 概要
フェーズ移行時およびUnit完了時にコンテキストリセットを推奨し、次のアクションを開始するためのプロンプトを提示する

## 含まれるユーザーストーリー
- ストーリー 4-1: フェーズ移行時のリセット提案
- ストーリー 4-2: Unit完了時のリセット提案

## 責務
- フェーズ完了時のリセット推奨メッセージ表示
- Unit完了時のリセット推奨メッセージ表示
- コピペ可能な次アクション開始プロンプトの提示

## 境界
- 自動リセットは含まない（ユーザーに提案するのみ）
- コンテキストサイズの自動検知は含まない（固定タイミングでの提案のみ）

## 依存関係

### 依存する Unit
なし（独立して実装可能）

### 外部依存
なし

## 非機能要件（NFR）
- **パフォーマンス**: 特になし
- **セキュリティ**: 特になし
- **スケーラビリティ**: 特になし
- **可用性**: 特になし

## 技術的考慮事項
- 追加する発動タイミング:
  1. `inception.md` - Inception Phase完了時 → Construction Phase開始プロンプト提示
  2. `construction.md` - Unit完了時 → 次Unit開始プロンプト提示
  3. `construction.md` - 全Unit完了時 → Operations Phase開始プロンプト提示
  4. `operations.md` - Operations Phase完了時 → 次サイクル開始プロンプト提示（既存を強化）

- 提示フォーマット:
  ```
  ---
  ## コンテキストリセット推奨

  このフェーズ/Unitが完了しました。コンテキストをリセットして次のステップを開始することを推奨します。

  **次のステップを開始するプロンプト**:
  ```markdown
  以下のファイルを読み込んで、サイクル vX.X.X の [Phase名] を開始してください：
  docs/aidlc/prompts/[phase].md
  ```
  ```

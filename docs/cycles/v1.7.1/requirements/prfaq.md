# PRFAQ: AI-DLC Starter Kit v1.7.1

## Press Release（プレスリリース）

**見出し**: AI-DLC v1.7.1 - プロンプト実行体験の改善とバックログ管理の安定化

**副見出し**: v1.7.0で導入した機能の安定化と、開発フローのスムーズ化を実現

**発表日**: 2026-01

**本文**:

AI-DLC Starter Kit v1.7.1をリリースしました。

v1.7.0で導入したIssue駆動バックログ管理において、セットアップ時にbacklogラベルが自動作成されない問題を修正しました。また、aidlc.tomlのbacklog.mode設定が正しく読み込まれない問題も解消しています。

プロンプト実行時の体験も改善しました。複合コマンド（&&, ||）をフェーズ開始時の事前確認に集約することで、許可リスト運用時の承認回数を削減。AIレビューの反復プロセスも明文化し、レビュー品質が向上しました。

さらに、iOSアプリ開発者向けにバージョン更新タイミングの改善オプション、jj利用者向けに有効化フラグを追加しました。

## FAQ（よくある質問）

### Q1: v1.7.0からのアップグレードは必要ですか？
A: Issue駆動バックログ（mode=issue）を使用している場合は推奨します。backlogラベル関連の問題が解消されます。

### Q2: 複合コマンド削減による動作変更はありますか？
A: 動作結果は同じですが、フェーズ開始時に環境確認がまとめて行われるようになります。許可リスト運用時の承認回数が減少します。

### Q3: iOSバージョン更新タイミングの変更は必須ですか？
A: オプションです。project.type=iosを設定した場合のみ、Inception Phaseでバージョン更新を提案します。従来のOperations Phase更新も引き続き利用可能です。

### Q4: jj有効化フラグを設定するとgitが使えなくなりますか？
A: いいえ。enabled=trueにするとjjコマンドが優先的に案内されますが、gitコマンドも引き続き使用可能です。デフォルトはfalse（git使用）です。

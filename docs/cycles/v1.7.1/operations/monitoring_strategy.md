# 監視・ロギング戦略

## サイクル情報

- **サイクル**: v1.7.1
- **作成日**: 2026-01-11

## プロジェクト特性

- **プロジェクトタイプ**: ドキュメント・テンプレートプロジェクト（general）
- **技術スタック**: Markdown、Claude Code
- **デプロイ先**: GitHub リポジトリ

## 監視設定

### 現状

- **監視ツール**: なし
- **理由**: ドキュメントプロジェクトのため不要

このプロジェクトは実行可能なアプリケーションではなく、ドキュメントとテンプレートの配布を目的としています。そのため、従来の意味でのアプリケーション監視は不要です。

## 代替の監視手段

### 1. GitHub Insights

- **用途**: リポジトリのトラフィック分析
- **メトリクス**: クローン数、ビジター数、参照元
- **アクセス**: GitHub リポジトリ > Insights > Traffic

### 2. Issue/PR トラッキング

- **用途**: ユーザーフィードバックの収集
- **方法**: GitHub Issues を使用

## ロギング設定

### 現状

- **ロギング**: なし
- **理由**: 実行可能なアプリケーションではないため不要

## 将来の検討事項

運用引き継ぎ（docs/cycles/operations.md）に記載:

- GitHub Insights によるトラフィック分析
- 利用状況のトラッキング

# Unit 004 計画: ブランチ切り替え後のrules.md再読み込み

## 概要

inception.mdのステップ11（ブランチ確認）でブランチ切り替えが発生した場合、ステップ5で読み込んだrules.mdが切り替え先ブランチのものと異なる可能性がある。ステップ11完了後にrules.mdを再読み込みするサブステップを追加する。

## 変更対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| `prompts/package/prompts/inception.md` | ステップ11末尾にrules.md再読み込みサブステップ（11-5）を追加 |

## 実装計画

### 変更箇所

ステップ11の末尾（11-4の後、ステップ12の前）に「11-5. rules.md再読み込み」を追加。

### ロジック

1. ステップ11でブランチ切り替えが実際に発生したかを判定（setup-branch.sh実行 or git checkout実行の経路を通った場合）
2. ブランチ切り替えが発生した場合のみ、`docs/cycles/rules.md` を再読み込み
3. ファイルが存在しない場合はスキップ

### 追加するテキスト（概要）

```
#### 11-5. rules.md再読み込み【ブランチ切り替え時】

ブランチ切り替えが発生した場合（setup-branch.sh実行またはgit checkout実行）、
ステップ5で読み込んだrules.mdが切り替え先ブランチのものと異なる可能性がある。

条件: ブランチ切り替えが発生した場合のみ
動作: docs/cycles/rules.md が存在すれば再読み込み
スキップ: ブランチ切り替えなし or ファイル不存在
```

# Unit 003 計画: スコープ外バックログ自動登録

## 概要

指摘対応判断フローで「スコープ外」（OUT_OF_SCOPE）を選択した場合に、バックログ登録を自動的に試行するステップを追加する。

## 変更対象ファイル

- `prompts/package/prompts/common/review-flow.md`（唯一の変更対象）

## 実装計画

### 変更箇所: 指摘対応判断フローのステップ5の後 (#240)

**対象箇所**: 「指摘対応判断フロー」セクション、ステップ5（先送り判断の履歴記録）の後

**追加内容（ステップ5a）**:
- OUT_OF_SCOPE判断があった指摘に対してバックログ登録を実行するステップ

#### mode判定

1. `docs/aidlc.toml` の `[rules.backlog].mode` を読み取る
2. **フォールバック**: 設定ファイル未存在・読み取りエラー・構文エラー・値未設定時は `git` として扱う

#### 登録方法の分岐

| mode | 正常時 | gh未認証時 | gh issue create失敗時 |
|------|--------|------------|----------------------|
| `git` / `git-only` | ファイル作成 | - | - |
| `issue` | Issue作成 | gitフォールバック（ファイル作成） | gitフォールバック（ファイル作成） |
| `issue-only` | Issue作成 | 警告+手動対応依頼 | 警告+手動対応依頼 |

#### gh CLI判定の3段階

1. `gh` コマンドの存在確認
2. `gh auth status` による認証状態確認
3. `gh issue create` の実行結果確認

いずれかの段階で失敗した場合、上記分岐表のフォールバック処理を実行する。

#### バックログ登録フォーマット

**Issue駆動時**:
- タイトル: `[Backlog] {指摘内容の要約}`
- ラベル: `backlog`, `type:chore`（レビュー指摘由来）, `priority:medium`（デフォルト）
- 本文: 概要、検出元（サイクル・フェーズ・レビュー種別）、指摘内容

**Git駆動時**:
- 保存先: `docs/cycles/backlog/chore-{slug}.md`
- テンプレート: `docs/aidlc/templates/backlog_item_template.md` に準拠

#### 履歴記録

write-history.sh形式でバックログ登録完了を記録する。Unit 001で確立した安全なパターン（heredoc形式`CONTENT_EOF`、シェル展開回避）を使用する。

### ステップ番号の調整

現在のフロー:
1. 指摘一覧の表示
2. 各指摘への判断要求
3. 理由入力要求
4. 理由のバリデーション
5. 先送り判断の履歴記録
6. 全指摘の判断完了後サマリ記録
7. 次のアクション決定

変更後:
- ステップ5の後に「5a. OUT_OF_SCOPEバックログ自動登録」を挿入
- 「5a」は指摘対応判断フロー内のサブステップ（トップレベルのステップ5.5「セルフレビューフロー」とは無関係）
- 既存のステップ6, 7はそのまま（番号変更なし）

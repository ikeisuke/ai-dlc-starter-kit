# Unit 002 計画: レビューフロー判定ルール改善

## 概要

review-flow.mdの3箇所を改善する:
1. 千日手検出の「同種の指摘」判定基準を明文化
2. ステップ6フォールバック承認時の理由記録を必須化（mode=required時）
3. スキップ記録をwrite-history.sh呼び出し形式に統一

## 変更対象ファイル

- `prompts/package/prompts/common/review-flow.md`（唯一の変更対象）

## 実装計画

### 変更1: 千日手検出の「同種の指摘」判定基準明文化 (#239)

**対象箇所**: 「千日手検出（フロー実行前チェック）」セクション（L238-280付近）

**追加内容**:
- 「同種の指摘」の判定基準を明文化（3条件以下、AIエージェントが一意に判定可能）
- 判定に使用する必須キー: レビュー種別 + 対象ファイル + 指摘内容の要約
- 判定手順: 3つのキーがすべて実質的に一致する指摘を「同種」とみなす
- サンプルケースを追加（同種とみなす例、みなさない例）

### 変更2: ステップ6選択肢2の理由記録必須化 (#238)

**対象箇所**: ステップ6「外部AIレビューで続行不能時」セクション（L564-586付近）

**変更内容**:
- mode=required時の選択肢2「ユーザー承認へ進む」選択時に理由入力を必須化
- 禁止パターンバリデーションは「指摘対応判断フロー」セクションの定義を唯一の参照元（SoT）とし、ステップ6では「指摘対応判断フローの禁止パターンに準ずる」と参照する
- バリデーション順序: 空チェック → 禁止パターン → 受理
- 理由記録のフォーマットを定義

### 変更3: スキップ記録のwrite-history.sh統一 (#237)

**対象箇所**: ステップ6の選択肢2の履歴記録部分（L577-584付近）

**変更内容**:
- 現在のMarkdown断片形式をwrite-history.sh呼び出し形式に置き換え
- フェーズ・Unit引数の条件分岐を他の履歴記録と同じ形式で記述

### ストーリー 2: スコープ外指摘のバックログ自動登録 (#240)
**優先順位**: Must-have

As a AI-DLC開発者
I want to AIレビューで「スコープ外」と判断した指摘を自動的にバックログに登録したい
So that 改善点の漏れが防止され、次サイクルで確実に対応できる

**受け入れ基準**:

**技術的考慮事項**:

## Epic 4: セキュリティ改善

### ストーリー 9: write-history.shのエスケープ確認 (#236)
**優先順位**: Must-have

As a AI-DLC開発者
I want to write-history.shの--contentパラメータでユーザー入力が安全に処理されることを確認したい
So that コマンドインジェクションのリスクが排除される

**受け入れ基準**:
- [ ] write-history.sh内部の--content処理で、特殊文字（`'`, `"`, `` ` ``, `$`, `;`等）を含む入力が安全に処理される
- [ ] review-flow.md内のwrite-history.sh呼び出し例が、シングルクォートのヒアドキュメント方式（`$(cat <<'EOF'...EOF)`）に統一されている
- [ ] `'; rm -rf /`等のインジェクションパターンが無効化されることが確認されている
- [ ] 通常の履歴記録（特殊文字を含まない入力）が従来通り動作する

**技術的考慮事項**:
- write-history.shはシェル引数として値を受け取るため、呼び出し側のクォーティングが防御の主な手段
- プロンプト内のコマンド例をヒアドキュメント方式に統一するのが最も実効的

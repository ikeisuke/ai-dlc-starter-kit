# Intent（開発意図）

## プロジェクト名
AI-DLC Starter Kit v1.17.1

## 開発の目的
v1.17.0開発中に発見されたレビューフロー・squash・DX・セキュリティの改善点を一括対応する。

具体的には以下の4つの軸で改善を行う：

1. **レビューフロー改善**: セルフレビューのスキル統合、スコープ外指摘のバックログ自動登録、千日手検出ルールの明確化、ステップ6フォールバックの承認強化、スキップ記録のwrite-history.sh統一により、レビュープロセスの一貫性と信頼性を向上させる
2. **squash管理改善**: retroactiveモードのコミットフォーマット依存リスクを軽減し、事後squashの信頼性を向上させる
3. **DX改善**: $()を含むコマンドのスクリプトファイル化、Operations PhaseのIssueクローズ確認の二重確認解消、アップグレード処理のスクリプト化により、開発者体験を改善する
4. **セキュリティ改善**: write-history.shの--contentパラメータにおけるユーザー入力のエスケープを確認・対策し、コマンドインジェクションのリスクを排除する

## ターゲットユーザー
AI-DLCスターターキットを利用する開発者

## ビジネス価値
- レビュースキルにセルフレビュー機能を統合することで、レビュー実行ロジックが一元管理され保守性が向上する
- スコープ外指摘のバックログ自動登録により、改善点の漏れが防止される
- 千日手検出の判定基準が明確化され、AIエージェントの判断のブレが低減する
- $()コマンドのスクリプト化により、Claude Codeでの実行時に毎回の許可確認が不要になり自動化が促進される
- Issueクローズの二重確認が解消され、Operations Phaseの作業効率が向上する
- write-history.shのエスケープ対策により、セキュリティリスクが排除される
- アップグレード処理のスクリプト化により、AIのコンテキスト消費削減と処理の信頼性向上が実現する

## 成功基準
- レビュースキル（reviewing-code/architecture/security/inception）のSKILL.mdにself-reviewモードの手順が追加され、review-flow.mdのステップ5.5からスキル呼び出しでセルフレビューが実行される
- 指摘対応判断フローで「スコープ外」を選択した場合、バックログ登録（backlog.mode設定に従いIssue作成またはファイル作成）が必須ステップとして実行され、登録完了が履歴に記録される
- 千日手検出の「同種の指摘」判定基準がreview-flow.mdに明文化されており、AIエージェントが一意に判定可能な条件で記述されている（判定条件の具体的な定義はConstruction Phaseで設計決定）
- mode=required時のステップ6「ユーザー承認へ進む」選択時に、理由・対象成果物の記録がwrite-history.sh形式で必須化されている
- ステップ6のスキップ記録がMarkdown断片形式からwrite-history.sh呼び出し形式に統一されている
- squash-unit.sh retroactiveモードに対策（gitトレーラー方式、ハッシュ直接指定、ドライラン必須化のいずれか。方式はConstruction Phaseで設計決定）が実装され、`--dry-run --retroactive`で対象コミット一覧が正しく表示される
- `prompts/package/` 配下のプロンプトで使用されている`$()`を含むコマンド（`$(ghq root)`等）がスクリプトファイル化され、Claude Codeの許可プロンプトなしで実行可能になっている
- Operations Phaseのステップ5（Issueクローズ確認）で、PRのClosesセクションに含まれるIssueを`gh pr view`で取得し、自動クローズ対象として手動確認をスキップする
- write-history.shの--contentパラメータで、ユーザー入力（先送り理由等）がシングルクォートのヒアドキュメント方式で安全に渡され、`'; rm -rf /`等のインジェクションパターンが無効化される
- アップグレード処理の設定マイグレーション（aidlc.tomlセクション追加）とrsync同期（ドライラン→確認→実行）がそれぞれスクリプト化され、`docs/aidlc/bin/`配下から実行可能になっている

## 期限とマイルストーン
1サイクルで完了

## 制約事項
- `docs/aidlc/` は直接編集禁止。変更は `prompts/package/` を編集する（メタ開発ルール）
- 既存のスクリプト・プロンプトを拡張する形で実装（新規ファイルは最小限）
- レビュースキルへの統合はSKILL.mdの変更が中心（スキル実行基盤の変更は含まない）

## 影響範囲

**注**: 以下のパスは最終的な反映先を示す。実装は `prompts/package/` 配下のソースファイルを編集し、Operations Phaseのrsyncで `docs/aidlc/` に反映する（メタ開発ルール）。スクリプト（`bin/`配下）も同様に `prompts/package/bin/` を編集する。

### レビューフロー改善（#241, #240, #239, #238, #237）
- **影響**: `review-flow.md`（判断フロー・千日手検出・ステップ6変更）、レビュースキルのSKILL.md（self-reviewモード追加）
- **後方互換性**: AIレビューを使用しない環境（mode=disabled）には影響なし
- **回帰確認**: mode=disabled時にレビューステップが完全にスキップされること、mode=recommend時にステップ4の「いいえ」選択肢が残っていること
- **フォールバック**: スキル呼び出し失敗時は従来のインラインセルフレビューにフォールバック

### squash管理改善（#244）
- **影響**: `squash-unit.sh`（フォーマット依存リスク軽減）、`commit-flow.md`（手順更新の可能性）
- **後方互換性**: 既存のsquash動作（`--base`指定によるreset --soft方式）は維持
- **回帰確認**: `--retroactive`なしの通常squashが従来通り動作すること、`--dry-run`出力フォーマットが変わらないこと
- **フォールバック**: 新方式失敗時は既存のパターンマッチにフォールバック

### DX改善（#243, #242, #233）
- **影響**: 新規スクリプトファイル（`prompts/package/bin/` → `docs/aidlc/bin/` 配下）、`operations.md`（Issueクローズ手順変更）、`setup-prompt.md`（スクリプト化された手順への参照変更）
- **後方互換性**: 既存のコマンドライン手順も引き続き動作
- **回帰確認**: 各スクリプトの出力と終了コードがスクリプト化前のコマンドと同等であること（Operations PhaseのIssueクローズは`gh pr view`の出力で自動クローズ対象が正しく判別されること）

### セキュリティ改善（#236）
- **影響**: `write-history.sh`（入力エスケープ処理追加の可能性）、`review-flow.md`（コマンド例のヒアドキュメント方式統一）
- **後方互換性**: 既存の履歴記録フォーマットに変更なし
- **回帰確認**: 通常の履歴記録（特殊文字を含まない入力）が従来通り動作すること

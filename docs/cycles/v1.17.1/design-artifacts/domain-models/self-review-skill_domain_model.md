# ドメインモデル: セルフレビュースキル統合

## 概要

セルフレビューフロー（ステップ5.5）をサブエージェント直接起動からレビュースキル呼び出し方式に変更するための概念モデル。

## エンティティ

### セルフレビューフロー（review-flow.md ステップ5.5）

セルフレビュー実行の制御を担当するフロー。

**属性**:
- レビュー種別リスト: 実行対象のレビュー種別（code, architecture, security, inception）
- 対象ファイルパス: レビュー対象の成果物ファイルパス一覧
- 実行方式: スキル呼び出し / インライン / 簡易インライン

**責務**:
- レビュー実行方式の判定と分岐
- スキル呼び出しの実行（Skillツール経由）
- インラインフォールバックの実行
- 反復制御（最大3回）と完了判定
- 出力フォーマットの定義（SoT）
- 履歴記録

### レビュースキル（SKILL.md）

各レビュー種別に対応するスキル。通常モード（外部ツール実行）とセルフレビューモードの2つのモードを持つ。

**属性**:
- スキル名: reviewing-{code|architecture|security|inception}
- レビュー観点: 各種別固有の観点一覧（SoT）
- モード: 通常 / self-review

**責務（セルフレビューモード）**:
- 引数パース（先頭トークン `self-review` の判定）
- サブエージェント方式の実行（Taskツールで読み取り専用サブエージェント起動。外部CLI（codex/claude/gemini）は使用しない）
- インライン方式へのフォールバック（サブエージェント起動失敗時、Taskツール利用不可時）
- レビュー観点の提供（既存の「レビュー観点」セクション）
- 指示テンプレートの保持
- スキル返却フォーマット（3項目: 重要度・内容・推奨修正）での結果返却。最終出力フォーマット（`判定` 付与・ヘッダ付加）はメインエージェント（review-flow.md）の責務

## 関係

```
セルフレビューフロー ---[呼び出す]---> レビュースキル（self-reviewモード）
  │                                        │
  │ 出力フォーマット（SoT）                │ レビュー観点（SoT）
  │ 反復・完了制御                         │ 指示テンプレート
  │ 履歴記録                               │ サブエージェント/インライン方式選択
  │                                        │
  └── フォールバック時 ──> インラインセルフレビュー（SKILL.md観点参照）
                             │
                             └── さらにフォールバック ──> 簡易インライン（review-flow.md内基準）
```

**依存方向**:
- 実装依存（呼び出し方向）: review-flow.md → SKILL.md（一方向）。SKILL.mdはreview-flow.mdの特定ステップを参照しない
- 仕様参照: SKILL.mdは出力フォーマット仕様としてreview-flow.mdを参照する（双方向許容。ただしステップ番号や内部実装への依存は禁止）

## モード判定フロー

```
ステップ5.5開始
  │
  ├─ A) スキル呼び出し可能？
  │    └─ はい → Skill ツールで `reviewing-{種別}` を `self-review {files}` で呼び出し
  │              └─ 成功 → 結果を受信 → 反復・完了処理へ
  │              └─ 失敗 → B)へ
  │
  ├─ B) SKILL.md読み取り可能？
  │    └─ はい → SKILL.mdの観点でインラインレビュー
  │    └─ いいえ → C)へ
  │
  └─ C) review-flow.md内の種別説明を観点として簡易インラインレビュー
```

## スキル呼び出しインターフェース

```
skill: "reviewing-{種別}"
args: "self-review {file1} {file2} ..."

引数パース:
- 先頭トークン = "self-review" → セルフレビューモード
- 先頭トークン ≠ "self-review" → 通常モード
- 空白を含むファイルパスは非対応
```

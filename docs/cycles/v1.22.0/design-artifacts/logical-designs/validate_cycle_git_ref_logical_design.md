# 論理設計: validate_cycle() Git ref安全性修正

## 概要

validate_cycle()関数にGit ref安全性チェックを追加するための変更箇所と実装方針を定義する。

**重要**: この論理設計では**コードは書かず**、コンポーネント構成とインターフェース定義のみを行います。

## アーキテクチャパターン

既存関数の拡張パターン。validate_cycle()の既存チェックチェーンの末尾に新規チェックを追加する。

## インターフェース設計

### 新規チェックの仕様

#### 末尾ドットチェック

- **条件**: サイクル名のいずれかのセグメント（`/`区切り）が`.`で終わる
- **パターン**: `[[ "$cycle" == *. ]] || [[ "$cycle" == *./* ]]`
- **エラー**: `emit_error "invalid-cycle" "cycle name segment must not end with dot: ${cycle}"`
- **戻り値**: 1

#### .lock接尾辞チェック

- **条件**: サイクル名のいずれかのセグメントが`.lock`で終わる
- **パターン**: `[[ "$cycle" == *.lock ]] || [[ "$cycle" == *.lock/* ]]`
- **エラー**: `emit_error "invalid-cycle" "cycle name segment must not end with .lock: ${cycle}"`
- **戻り値**: 1

#### git check-ref-format チェック

- **条件**: `git check-ref-format --branch "cycle/${cycle}"` が非0で終了
- **エラー**: `emit_error "invalid-cycle" "cycle name is not a valid git ref: ${cycle}"`
- **戻り値**: 1
- **注意**: gitコマンドが利用できない環境では、このチェックをスキップする（パターンマッチで十分カバー）

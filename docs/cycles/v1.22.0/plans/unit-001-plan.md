# Unit 001 計画: フェーズ内操作順序の明示化

## 概要

commit-flow.mdに操作順序ルールセクションを追加し、コミット前のPR作成、スカッシュ前のUnit完了マーク等の順序違反を防止する。

## 変更対象ファイル

- `prompts/package/prompts/common/commit-flow.md` — 操作順序ルールセクションの追加

## 実装計画

1. `prompts/package/prompts/common/commit-flow.md` の「コミットポリシー」セクション配下に「操作順序ルール」セクションを新規追加
2. commit-flow.mdのスコープに合う順序制約を明示化（コミット・PR・スカッシュ関連）:
   - コミット → PR作成（コミット前のPR作成禁止）
   - スカッシュ → Unit完了マーク（squash.enabled時）
   - AIレビュー完了 → ユーザーレビュー依頼
3. Phase遷移に関する順序制約（全Unit完了 → Operations Phase移行）はcommit-flow.mdのスコープ外として、各フェーズプロンプトの既存記述を参照リンクで指示
4. 各ルールに判定主体（AI）と違反時アクション（停止→自己修正）を明記
5. 順序違反検知時の自己修正フローを記述

## 完了条件チェックリスト

- [ ] commit-flow.mdへの操作順序ルールセクション追加
- [ ] 各ルールの判定主体と違反時アクション明記
- [ ] スコープ外ルール（Phase遷移）の参照リンク追加
- [ ] 順序違反検知時の自己修正フロー記述

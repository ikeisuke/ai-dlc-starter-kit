# Unit 003 計画: 外部レビューツール制約のドキュメント化

## 概要

review-flow.mdに外部レビューツール（Codex等）の既知制約と対処法セクションを追加する。
Claude Code Insightsレポートで「Tool/Environment Constraints」フリクションが5件発生しており、
各ツールの制約や失敗時の対処法を明文化することで、レビュープロセスの堅牢性を向上させる。

## 変更対象ファイル

- `prompts/package/prompts/common/review-flow.md` — 既知制約セクション追加

## 実装計画

### Phase 1: 設計

1. **ドメインモデル設計**: 外部ツール制約の分類体系を定義
2. **論理設計**: セクション配置場所、既存フォールバック処理との関係を設計

### Phase 2: 実装

3. **コード生成**: review-flow.mdに以下を追加
   - 「外部レビューツールの既知制約と対処法」セクション
   - 各制約エントリは統一構造で記述: 適用条件 / 症状 / 対処法 / フォールバック可否
   - 対象制約:
     - Codex: read-only制約、ファイル読み取り制限、`--skip-git-repo-check`
     - 認証失効時のフォールバック手順
     - インタラクティブモードフリーズ対策
   - 配置場所: 「種別単位のフォールバック処理」セクション付近
   - **重複防止**: 既存エラー分類表の内容と重複しない。新セクションは既存表に無い「事前予防・回避策」に特化し、エラー発生後の対処は既存表へリンク参照する
4. **テスト**: markdownlintでフォーマット確認
5. **統合とレビュー**: AIレビュー実施

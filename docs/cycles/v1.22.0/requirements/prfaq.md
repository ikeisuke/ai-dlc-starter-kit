# PRFAQ: AI-DLC Starter Kit v1.22.0

## Press Release（プレスリリース）

**見出し**: AI-DLCスターターキット v1.22.0 — フリクション削減と堅牢なアップグレードパス

**副見出し**: AIエージェントの操作順序誤り・テスト失敗時の手動介入・アップグレード失敗を解消し、開発体験を大幅に改善

**発表日**: 2026-03

**本文**:

AI-DLCスターターキット v1.22.0 では、Claude Code Insightsレポートで検出されたフリクションパターンに対応し、フレームワークの堅牢性を向上させました。

コミット前のPR作成やスカッシュ前のUnit完了マークなど、操作順序の誤りによるフリクションを防止する操作順序ルールをcommit-flow.mdに追加。テスト失敗時にはAIエージェントが自動で最大3回の修正→再実行ループを試行するSelf-Healing機能を導入し、ユーザー介入を大幅に削減しました。

外部レビューツール（Codex等）の既知制約と対処法をドキュメント化し、ツール固有の問題でレビューがフリーズする事態を防止。validate_cycle()にGit ref安全性チェックを追加し、無効なブランチ名によるgitエラーを事前に防ぎます。

さらに、古いバージョンからのアップグレードパスを堅牢化。バージョン検出ロジックの修正とスクリプトパスのフォールバック機能により、v1.13.x以降からのアップグレードがエラーなく完了するようになりました。

## FAQ（よくある質問）

### Q1: Self-Healingテストループの最大回数は変更できますか？
A: v1.22.0では最大3回で固定です。リトライ回数の設定化は将来のサイクルで対応予定です（#322）。

### Q2: 古いバージョン（v1.13.x以前）からもアップグレードできますか？
A: v1.13.x以降を対象としています。それ以前のバージョンからは初回セットアップとして扱われます。

### Q3: validate_cycle()の変更で既存のサイクル名が使えなくなることはありますか？
A: 末尾ドットや.lock接尾辞を含むサイクル名のみが拒否対象です。v1.22.0、feature/v1.0.0等の通常のサイクル名には影響しません。

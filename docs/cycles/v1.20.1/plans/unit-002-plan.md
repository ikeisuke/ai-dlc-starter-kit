# Unit 002 計画: 監査ログ強化

## 概要

write-history.shの出力エントリにISO 8601タイムスタンプを付加し、トレーサビリティを向上させる。

## 変更対象ファイル

- `prompts/package/bin/write-history.sh`（タイムスタンプ形式変更）

## 現状分析

### 現行フォーマット

```bash
timestamp=$(date '+%Y-%m-%d %H:%M:%S %Z')
# 出力例: 2026-03-10 19:52:19 JST
```

### 問題点

- `%Z` はタイムゾーン略称（JST, UTC等）で、ISO 8601非準拠
- ロケール依存でパース困難

## 実装計画

### Phase 1: 設計

1. ドメインモデル・論理設計は不要（シェルスクリプトの1行変更のため）

### Phase 2: 実装

2. **コード生成**: write-history.shのタイムスタンプ取得処理を変更
3. **テスト**: 変更後のスクリプトをdry-runで動作確認
4. **統合とレビュー**: AIレビュー実施

### タイムスタンプ変更仕様

**目標フォーマット**: ISO 8601（`YYYY-MM-DDTHH:mm:ss±HH:MM`）

**実装方式**: macOS/Linux両対応のポータブル実装

```bash
# macOS: date '+%z' は +0900（コロンなし）を返すため、sedで挿入
# Linux (GNU date): date '+%z' も同様に +0900 を返す
timestamp=$(date '+%Y-%m-%dT%H:%M:%S%z' | sed 's/\([+-][0-9][0-9]\)\([0-9][0-9]\)$/\1:\2/')
```

**フェイルセーフ**: タイムスタンプ取得失敗時もISO 8601形式を維持し、固定値`1970-01-01T00:00:00+00:00`を使用する（出力スキーマの一貫性を保証）

```bash
timestamp=$(date '+%Y-%m-%dT%H:%M:%S%z' 2>/dev/null | sed 's/\([+-][0-9][0-9]\)\([0-9][0-9]\)$/\1:\2/')
if [[ -z "$timestamp" ]]; then
    timestamp="1970-01-01T00:00:00+00:00"
fi
```

### 後方互換性

- 出力フォーマットの変更（`## YYYY-MM-DD HH:MM:SS TZ` → `## YYYY-MM-DDTHH:MM:SS±HH:MM`）
- 既存の履歴ファイル内容は変更しない（新規エントリのみ新形式）
- `init-cycle-dir.sh` は本Unitのスコープ外（write-history.shのみ対象）

### 下流影響分析

履歴ファイルのタイムスタンプ形式に依存するプログラム的なパーサーは存在しない（調査済み）。タイムスタンプはMarkdown見出し（`## YYYY-...`）として使用され、既存のcontent-based grep（文字列検索）はタイムスタンプ形式に依存しない。AIエージェントおよび人間がMarkdownとして読む用途のみ。

### 検証手順

1. `--dry-run`でスクリプトの引数解析が正常に動作することを確認
2. 実際にエントリを追記し、出力されたタイムスタンプがISO 8601形式（`T`区切り、`±HH:MM`タイムゾーン）であることをgrepで確認
3. フェイルセーフ: `date`コマンドが失敗した場合に固定値`1970-01-01T00:00:00+00:00`が使用されることを確認（コードレビューで検証）
4. 既存の履歴ファイルの内容が変更されていないことをgit diffで確認

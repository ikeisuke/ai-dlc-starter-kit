# Unit: 複合コマンド廃止

## 概要

プロンプト内の複合コマンド（`&&`, `||`, `|`、変数代入）を単純化し、許可リスト運用を改善する。

## 含まれるユーザーストーリー

- ストーリー9: 複合コマンド廃止・許可リスト整備 (#34)

## 責務

- 複合コマンドの単純コマンドへの置き換え
- 変数代入パターンの削除
- AIがReadツールで直接読み取る方式への変更
- gh label関連コマンドの許可リスト追加

## 境界

- スクリプト化は別Unit（Unit 005）
- プロンプトの構造変更は最小限に抑える

## 依存関係

### 依存する Unit

- Unit 005: 設定確認スクリプト整備（依存理由: スクリプト化が先に完了していれば、複合コマンド廃止の対象が明確になる）

### 外部依存

- なし

## 非機能要件（NFR）

- **パフォーマンス**: 該当なし
- **セキュリティ**: 該当なし
- **スケーラビリティ**: 該当なし
- **可用性**: 該当なし

## 技術的考慮事項

- 条件分岐・ループはAIが判断/繰り返し実行する方式に変更
- `[ -f ... ] && echo ...` → `ls file 2>/dev/null`
- `VAR=$(cmd); echo $VAR` → 単純コマンド + AI解釈
- dasel構文の`-r toml`オプション削除（v2対応）

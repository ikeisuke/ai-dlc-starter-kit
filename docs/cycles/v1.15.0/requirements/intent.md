# Intent（開発意図）

## プロジェクト名

AI-DLC Starter Kit v1.15.0

## 開発の目的

AI-DLCのワークフロー品質向上と将来のSkills化に向けた基盤整備を行う。具体的には以下の2点を実現する：

1. **コミット履歴の整理（#187）**: フェーズ/Unit単位でコミットをまとめ（squash）、PRのコミット履歴を簡潔にする
2. **Skills化に向けた全体整理（#116）**: 各AI-DLCフェーズをSkillsとして提供するための前段階として、プロンプト構造の整理・リファクタリングを行う

## ターゲットユーザー

AI-DLC Starter Kitを利用する開発者およびAIツール利用者

## ビジネス価値

- **コミット履歴の可読性向上**: PRレビュー時にコミット履歴が簡潔になり、変更内容の把握が容易になる
- **将来のSkills化への準備**: プロンプト構造が整理されることで、次サイクル以降のSkills化がスムーズに進む

## 成功基準

- Unit完了時に中間コミット（レビュー前/反映コミット等）が1つのコミットにまとめられること
- git環境（`git reset --soft` + `git commit`）とjj環境（`jj squash`）の両方で動作すること
- `git rebase -i` などインタラクティブな操作を使用しないこと
- Skills化に向けたプロンプト構造の整理方針が確定し、リファクタリングが完了すること

## 期限とマイルストーン

- 特に期限なし（通常のサイクルペースで進行）

## 制約事項

- インタラクティブな操作（`git rebase -i` 等）は使用しない（自動化が壊れるため）
- `docs/aidlc/` は直接編集禁止（`prompts/package/` を編集する）
- #116 のSkillファイル作成自体は次サイクル以降のスコープ

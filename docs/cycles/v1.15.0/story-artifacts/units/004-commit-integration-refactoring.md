# Unit: コミット処理統合・プロンプトリファクタリング

## 概要

分散しているコミット関連のロジックを統合し、Unit 003で策定した方針に基づいてプロンプト全体のリファクタリングを実施する。Skills化に向けた構造整備の実作業Unit。

## 含まれるユーザーストーリー

- ストーリー5: コミット処理の統合
- ストーリー6: プロンプトのリファクタリング

## 関連Issue

- #116
- #187

## 責務

- コミット関連処理の `prompts/package/prompts/common/` への集約
- 各フェーズプロンプトからのコミット処理参照の統一
- squash機能の統合フローへの組み込み
- Unit 003の方針に基づくプロンプト全体のリファクタリング
- Lite版との整合性維持

## 境界

- squashスクリプト自体の実装はUnit 001で完了済み
- プロンプトへのsquash組み込みの基本部分はUnit 002で完了済み
- Skillファイル作成は次サイクル以降

## 依存関係

### 依存する Unit

- Unit 001: squashスクリプト作成（依存理由: squash機能をコミットフローに統合するため）
- Unit 003: プロンプト構造分析・方針策定（依存理由: 方針が確定しないとリファクタリングの方向性が定まらない）

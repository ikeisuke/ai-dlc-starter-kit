# Unit: squashスクリプト作成

## 概要

Unit完了時に中間コミット（レビュー前/反映コミット等）を1つのコミットにまとめるsquashスクリプトを作成する。git環境とjj環境の両方に対応する。

## 含まれるユーザーストーリー

- ストーリー1: Unit完了時のコミットまとめ（git環境）
- ストーリー2: Unit完了時のコミットまとめ（jj環境）

## 関連Issue

- #187

## 責務

- git環境用のsquashスクリプト（`git reset --soft` + `git commit` 方式）の実装
- jj環境用のsquashスクリプト（`jj squash` 方式）の実装
- 起点コミットの自動特定ロジック
- 正常系/異常系のエラーハンドリング

## 境界

- プロンプトへの組み込みはUnit 002で実施
- コミット処理の統合はUnit 004で実施
- `docs/aidlc.toml` への設定追加はUnit 002で検討

## 依存関係

### 依存する Unit

- なし

### 外部依存

- git CLI
- jj CLI（jj環境のみ）

## 非機能要件（NFR）

- **パフォーマンス**: 数十コミット程度のsquashが数秒以内に完了すること
- **セキュリティ**: 該当なし
- **スケーラビリティ**: 該当なし
- **可用性**: スクリプト実行失敗時に復旧可能であること

## 技術的考慮事項

- git用: `git reset --soft {起点コミット}` で中間コミットの変更を保持したままHEADを戻し、`git commit` で再コミット
- 起点コミットの特定: コミットメッセージのパターン（`feat:` / `chore:` + Phase/Unit完了）で検索
- jj用: `jj squash` は非インタラクティブで動作
- スクリプトの配置先: `prompts/package/bin/`
- Co-Authored-Byの引き継ぎ

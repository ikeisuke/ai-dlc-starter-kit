# バグ: アップグレード後に自動でサイクル開始フローに入ってしまう

## 概要

`setup-prompt.md` のアップグレード完了後、「新しいセッションで」サイクル開始すべきところを、自動的に `setup.md` を読み込んでサイクル開始フローに入ってしまう。

## 現象

1. ユーザーがアップグレードを選択
2. アップグレード処理が完了
3. 自動的に `setup.md` が読み込まれ、サイクルバージョン選択を求められる
4. ユーザーはまだサイクル開始を指示していない

## 原因

`setup-prompt.md` のセクション10には：

> セットアップが完了しました。**新しいセッションで**以下を実行し、サイクルを開始してください

と記載があるが、ケースB/C/Dでは「プロジェクト内の setup.md を読み込む」という指示もあり、混乱を招いている。

## 対処方針

1. `setup-prompt.md` の完了メッセージを明確化
2. アップグレードモード完了時は `setup.md` を自動読み込みしない
3. ケースB（バージョン同じ）のみ自動でサイクル開始フローへ進む

## 優先度

中 - UX の問題だが、機能的には動作する

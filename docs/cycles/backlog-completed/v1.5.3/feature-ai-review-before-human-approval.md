# 人間レビュー前にAIレビューを必須化

- **発見日**: 2025-12-27
- **発見フェーズ**: Setup
- **発見サイクル**: v1.5.3
- **優先度**: 高

## 概要

設計を多段で行う場合など、人間がレビューするタイミングでAI MCP（Claude等）が有効な場合は、必ずAIレビューを先に実行する。

## 詳細

**現状の問題**:
- 設計フェーズで複数のステップがある場合、各ステップで人間の承認を求める
- しかし、AIレビューを挟まずに直接人間に承認を求めることがある
- 人間の負担が増え、AIが検出できる問題を人間が見つける必要がある

**期待する動作**:
1. 設計ステップ完了
2. **AI MCPが有効な場合、自動的にAIレビューを実行**
3. AIレビュー結果を人間に提示
4. 人間の承認を求める

**適用タイミング**:
- ドメインモデル設計完了時
- 論理設計完了時
- アーキテクチャ設計完了時
- 計画ファイル作成完了時
- その他、人間の承認を求める全てのタイミング

## 対応案

### プロンプトへの追加ルール

```markdown
**AIレビュー優先ルール【重要】**:
人間に承認を求める前に、以下を実行する：

1. AI MCP（desktopCommander等）が有効か確認
2. 有効な場合:
   - AIレビューを実行
   - レビュー結果を修正に反映
   - レビュー済みの成果物を人間に提示
3. 無効な場合:
   - 直接人間に承認を求める

これにより、人間は高品質な成果物のみをレビューできる。
```

### 実装箇所

- `docs/aidlc/prompts/inception.md` の計画承認フロー
- `docs/aidlc/prompts/construction.md` の設計承認フロー
- その他、人間承認を求める全てのプロンプト

# markdownlintルールの段階的有効化

- **発見日**: 2026-01-04
- **発見フェーズ**: Operations
- **発見サイクル**: v1.5.3
- **優先度**: 中

## 概要

v1.5.3でCI/CDにmarkdownlintを導入したが、既存ファイルとの互換性のため多数のルールを無効化した。段階的にルールを有効化し、Markdownの品質を向上させる。

## 現状

`.markdownlint.json` で以下のルールが無効化されている:

| ルール | 内容 | 有効化優先度 |
|--------|------|-------------|
| MD003 | 見出しスタイル | 低 |
| MD007 | リストインデント | 中 |
| MD009 | 末尾スペース | 高 |
| MD012 | 連続空行 | 中 |
| MD013 | 行長 | 低（日本語対応困難） |
| MD022 | 見出し周りの空行 | 中 |
| MD024 | 重複見出し | 低（履歴ファイルで必要） |
| MD026 | 見出し末尾句読点 | 低 |
| MD029 | 順序リストプレフィックス | 中 |
| MD030 | リストマーカー後スペース | 中 |
| MD031 | コードブロック周りの空行 | 中 |
| MD032 | リスト周りの空行 | 中 |
| MD033 | インラインHTML | 低 |
| MD034 | 裸URL | 高 |
| MD036 | 強調を見出しとして使用 | 中 |
| MD040 | コードブロック言語指定 | 高 |
| MD041 | 最初の行は見出し | 低 |
| MD053 | リンク参照定義 | 低 |
| MD058 | テーブル周りの空行 | 中 |

## 対応案

### Phase 1: 高優先度ルールの有効化
1. MD009 (末尾スペース) - 既存ファイルを修正
2. MD034 (裸URL) - リンク形式に変換
3. MD040 (コードブロック言語指定) - 言語を追加

### Phase 2: 中優先度ルールの有効化
1. フォーマット関連ルールを段階的に有効化
2. 既存ファイルの修正を並行実施

### Phase 3: 残りのルールの検討
1. 低優先度ルールの有効化可否を判断
2. プロジェクト特性に合わないルールは無効のまま維持

## 備考

- 一度にすべて修正するのではなく、サイクルごとに段階的に対応
- 新規作成ファイルは最初からルールに準拠するよう心がける

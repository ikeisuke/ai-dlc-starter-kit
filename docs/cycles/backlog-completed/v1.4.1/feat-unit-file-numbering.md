# ユニット定義ファイルに実行順序番号を付与

- **発見日**: 2025-12-19
- **発見フェーズ**: Setup
- **発見サイクル**: v1.4.1
- **優先度**: 中

## 概要

ユニット定義ファイル名の先頭に0埋めした番号を付け、依存関係の実行順序を明示する。

## 詳細

現在のユニット定義ファイルは名前順でソートされるが、依存関係に基づく実行順序が不明確。ファイル名に番号を付与することで：

- 依存関係の実行順序が一目でわかる
- ファイルシステム上で自然にソートされる
- 連番の重複を禁止することで順序の曖昧さを排除

## 対応案

- ファイル名形式: `{NNN}-{unit-name}.md`（例: `001-setup-database.md`, `002-create-user-model.md`）
- 番号は依存関係の実行順序が早いものほど小さい値
- 連番の重複は許可しない
- 関連プロンプト・テンプレートを更新

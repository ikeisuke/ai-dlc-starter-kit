# レビュー前後のタイミングでコミット

- **発見日**: 2025-12-28
- **発見フェーズ**: Inception
- **発見サイクル**: v1.5.3
- **優先度**: 中

## 概要

レビュー（MCPレビューや人間レビュー）の前後のタイミングで自動的にコミットを作成する機能を追加する。

## 詳細

**現状**:
- コミットはフェーズ完了時やUnit完了時のみ
- レビュー前後の状態が記録されない

**期待する動作**:
1. レビュー依頼前に現在の状態をコミット
2. レビュー後（修正反映後）にコミット

**メリット**:
- レビュー前後の差分が明確になる
- 問題発生時のロールバックが容易
- レビュー履歴がGitに記録される

## 対応案

### コミットタイミングの追加

1. **レビュー前コミット**:
   ```
   chore: [{{CYCLE}}] レビュー前 - {成果物名}
   ```

2. **レビュー後コミット**（修正があった場合）:
   ```
   chore: [{{CYCLE}}] レビュー反映 - {成果物名}
   ```

- prompts/package/prompts/construction.md

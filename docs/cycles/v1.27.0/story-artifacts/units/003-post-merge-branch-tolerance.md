# Unit: post-merge-cleanup.shブランチ不在耐性

## 概要
post-merge-cleanup.shのstep_0aでのブランチ存在確認を警告に変更し、ローカルブランチ削除済みの状態でもクリーンアップを完走可能にする。

## 含まれるユーザーストーリー
- ストーリー 3: post-merge-cleanup.shのローカルブランチ不在時の継続実行

## 関連Issue
- #381

## 責務
- step_0aのブランチ不在時の`fatal_error`を警告に変更し、処理を続行可能にする
- ブランチ不在フラグ（`LOCAL_BRANCH_EXISTS`）をstep_0aで設定する
- step_4にブランチ不在フラグ参照を追加し、不在時はローカルブランチ削除をスキップしてログ出力する
- `--dry-run`モードでのブランチ不在時にスキップ理由をログ出力する

## 境界
- 変更対象: step_0a（警告化 + フラグ設定）、step_4（フラグ参照によるスキップ制御）
- step_5の既存ロジックは変更しない（リモートブランチ削除は独立して動作）
- 削除対象ルール（cycle/upgrade/プレフィックスのみ）は変更しない
- worktreeモードの動作は変更しない（通常ブランチモードのみ対象）

# Intent（開発意図）

## プロジェクト名
AI-DLC Starter Kit v1.27.0

## 開発の目的
開発支援ツール基盤のパーミッション管理強化とスクリプトの堅牢性向上を行う。具体的には、Kiroエージェント設定ファイル（`.kiro/agents/aidlc.json`）にAI-DLCワークフロー用の許可設定（allowed commands等）を追加し、setup-ai-tools.shのパーミッション管理ロジックを改善してワイルドカードルール存在時の重複追加を防止し、post-merge-cleanup.shのブランチ存在確認をローカルブランチ削除済みケースに対応させる。

## スコープ

### 含まれるもの
- **#385**: `.kiro/agents/aidlc.json` にClaude Code `.claude/settings.json` と同等の許可ルールを追加。setup-ai-tools.shの`setup_kiro_agent`関数を拡張して新規セットアップ時に許可設定を含める
- **#384**: setup-ai-tools.shにワイルドカードルール検出ロジックを追加し、既にワイルドカードルールが存在する場合に個別ルールの追加をスキップする
- **#381**: post-merge-cleanup.shのstep_0aのブランチ存在確認を警告に変更し、ローカルブランチ不在時もリモートブランチ削除等の後続ステップを続行可能にする

### 除外するもの（スコープ外）
- パーミッション体系全体の再設計
- Claude Code以外のエージェント設定ファイル（Kiro以外）への対応
- post-merge-cleanup.shの削除対象ルール（cycle/upgrade/プレフィックスのみ）の変更
- setup-ai-tools.shのテンプレートの外部設定ファイル化（将来検討）

## ターゲットユーザー
AI-DLCスターターキットの利用者（AIを活用した開発を行うエンジニア）

## ビジネス価値
- Kiro CLI利用者がAI-DLCワークフローのコマンドを許可プロンプトなしで実行でき、セットアップ時の手動設定が不要になる
- setup-ai-tools.shがワイルドカードルールの存在を検出し、個別ルールの重複追加を行わないため、aidlc-setup再同期時にユーザーのカスタマイズ（ルール統合等）が保持される
- post-merge-cleanup.shがローカルブランチ不在時に警告を出力して後続処理を継続し、終了コード0で正常完了するため、通常ブランチモードでの運用が安定する

## 成功基準
- `.kiro/agents/aidlc.json` に `setup_kiro_agent` 関数が許可設定を出力し、Claude Code の許可ルールと同等のコマンドが許可されること
- setup-ai-tools.shが既存のワイルドカードルール（例: `Bash(docs/aidlc/bin/*)`）を検出した場合、そのパターンに包含される個別ルールの追加をスキップすること
- post-merge-cleanup.shのstep_0aでローカルブランチが存在しない場合、警告メッセージを出力して後続ステップ（リモートブランチ削除等）を続行し、正常終了すること
- 上記すべての変更が既存の設定・運用フローの正常系を変更しないこと

## 期限とマイルストーン
特になし（通常のサイクルペースで実施）

## 制約事項
- `docs/aidlc/` は直接編集禁止（`prompts/package/` を編集すること）
- 既存のsetup-ai-tools.shの後方互換性を維持すること（既存ユーザーの設定を上書きしない）
- post-merge-cleanup.shの安全制約（cycle/upgrade/プレフィックスのみ削除対象）を維持すること
- Kiro設定追加時、既存のカスタム設定があればマージし上書きしないこと
- 各スクリプトの失敗時・スキップ時の挙動を現行運用と整合させること

# Unit 001 計画: dasel v3 予約語対応修正

## 概要

read-config.sh の get_value() 関数で、dasel v3 の予約語（`branch`, `filter` 等）によるパースエラーを修正する。v1.16.3 で実装されたブラケット記法変換の調査・検証・改善を行う。

## 調査結果（事前調査）

### 現在の実装状態

- v1.16.3 で `sed 's/\.\([^.]*\)/["\1"]/g'` によるブラケット記法変換を実装済み
- `rules.branch.mode` → `rules["branch"]["mode"]` に変換され、dasel v3.2.2 で正常動作を確認
- 全設定キー（14キー）の読み取りテスト済み → 全て成功

### 判明した問題点

1. **第1セグメント未保護**: sed正規表現はドット後のセグメントのみブラケット化する。第1セグメントが dasel v3 予約語（`branch`, `filter`）の場合、パースエラーとなる
2. **dasel v3 の制約**: `["key"]` 形式での先頭アクセスは不可（配列アクセスと解釈される）。`"key"` 形式も誤った結果を返す。ブラケット記法は第2セグメント以降でのみ有効
3. **dasel v2 互換性**: v2 環境でブラケット記法が動作するか未検証

### リスク評価

- 現在の AI-DLC 設定ではトップレベルキー（`starter_kit_version`, `project`, `paths`, `rules`）に予約語は含まれず、実害なし
- Issue #223 で提案された引用符ドット記法 `"rules"."branch"."mode"` は dasel v3.2.2 で誤った値を返すため採用不可

## get_value() 既存API契約（参照）

get_value() の入出力契約は `prompts/package/bin/read-config.sh` のコードコメントに定義済み:
- 戻り値: 0=存在, 1=不在, 2=エラー
- 標準出力: 値（存在する場合）
- 標準エラー: エラーメッセージ（エラー時）

本修正ではこのAPI契約を変更しない。キー変換ロジックの内部改善のみ行う。

## 変更対象ファイル

- `prompts/package/bin/read-config.sh` - get_value() 関数の予約語回避ロジック改善

6. **統合とレビュー**

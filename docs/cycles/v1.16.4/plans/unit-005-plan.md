# Unit 005 計画: Claude Code 許可設定推奨パターン策定

## 概要

`prompts/package/guides/ai-agent-allowlist.md` のセクション 4.1（Claude Code）を再編し、「AI-DLC運用時の推奨設定パターン」サブセクションを追加する。

現状の `.claude/settings.local.json`（2026-02-25時点で85エントリ）に以下の問題がある:
- フルパス（worktree依存）と相対パスの重複エントリ
- カテゴリ分類なく無秩序に追加されている
- 新規セットアップ時にどの設定が必要か不明確

既存4.1には設定ファイルの説明、優先順位、設定例、ワイルドカード説明が含まれる。これらを維持しつつ、推奨パターンサブセクションを統合する。

### 用語定義

本Unitでは Claude Code の `permissions` スキーマ（`allow`/`ask`/`deny`）を正準用語として使用する。Unit定義の「allowedTools」は概要レベルの表現であり、ドキュメント本文では `permissions.allow` 等の正式な用語を使用する。

## 変更対象ファイル

- `prompts/package/guides/ai-agent-allowlist.md` - セクション4.1を再編、推奨パターンサブセクション追加

## 実装計画

### Phase 1: 設計（既存4.1の差分分析 + 構成案決定）

1. **既存4.1の構成分析と再編方針の決定**
   - 既存の「設定例（AI-DLC推奨）」ブロックと推奨パターンの関係を整理
   - 追加/置換/統合の範囲を明確化
   - 出力: セクション4.1の見出し案（4.1内のサブセクション構成）

2. **推奨パターンの構成案**
   - カテゴリ分類: AI-DLCスクリプト群、Git/jj操作、GitHub CLI、外部ツール/Skill、deny対象
   - セクション3のコマンドカテゴリとの対応表
   - allow/ask/deny の使い分け方針
   - 相対パス推奨の根拠

### Phase 2: 実装

1. **ドキュメント作成**: セクション4.1を再編・追記
2. **整合性確認**: セクション3のカテゴリと4.1推奨パターンの対応を検証
3. **統合とレビュー**: AIレビュー + ユーザー承認

# Unit 007 計画: README.mdバージョン情報追加

## 概要

README.mdに不足しているバージョンセクションを追加し、全バージョンの変更内容を把握できるようにする。

## 変更対象ファイル

- `README.md`

## 不足しているバージョンセクション

CHANGELOG.mdと現在のREADME.mdを比較した結果、以下のバージョンセクションが不足:

1. **v1.10.0** - バックログ登録フロー改善、設計・実装整合性チェック機能等
2. **v1.9.3** - env-info.shオプション追加、skills構成変更等
3. **v1.9.2** - プレリリースバージョンサポート、KiroCLI Skills対応等
4. **v1.9.1** - init-cycle-dir.sh機能追加、コンテキストコンパクション対応等
5. **v1.9.0** - プロンプトモジュール化基盤、設定確認スクリプト等
6. **v1.7.4** - ツールインストール案内、サブエージェント活用ガイド等
7. **v1.7.3** - daselによるTOML読み込み対応、jj推奨設定等
8. **v1.7.2** - Claude Codeプランモード、GitHub Issueテンプレート等

## 実装計画

### Phase 1: 設計（このUnitはドキュメント更新のみのため簡略化）

設計は不要（既存のREADME.mdフォーマットに従って追記するのみ）

### Phase 2: 実装

1. README.mdの既存バージョンセクションの配置順序を確認
2. 不足バージョンをCHANGELOG.mdから抽出し、セマンティックバージョン降順で配置
3. 見出しレベル `##` で各バージョンのセクションを作成
4. 既存バージョンセクションは変更しない

### 配置位置

現在の構造:
```text

## 🔧 v1.8.1 の改善点

...
## 🔧 v1.2.0 の新機能
## 🔄 v0.1.0 から v1.0.0 への移行
```

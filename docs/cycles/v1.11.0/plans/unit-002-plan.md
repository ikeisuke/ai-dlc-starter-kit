# Unit 002 実装計画

## 概要

各フェーズ・Unitでタスク管理機能を活用する指示を追加し、タスク漏れを防止し進捗を可視化する。

## 変更対象ファイル

- `prompts/package/prompts/inception.md`
- `prompts/package/prompts/construction.md`
- `prompts/package/prompts/operations.md`

## 実装計画

### 指示文言（固定）

```text
**タスク管理機能を活用してください。**
```

### 追加箇所

#### inception.md

各ステップ見出しの直後に追加：

- `### ステップ1: Intent明確化【重要】`
- `### ステップ2: 既存コード分析`
- `### ステップ3: ユーザーストーリー作成`
- `### ステップ4: Unit定義【重要】`
- `### ステップ5: PRFAQ作成`

#### construction.md

`## フロー（1つのUnitのみ）` セクション内の各ステップ見出し直後に追加：

- `#### ステップ1: ドメインモデル設計`
- `#### ステップ2: 論理設計`
- `#### ステップ3: 設計レビュー`
- `#### ステップ4: コード生成`
- `#### ステップ5: テスト生成`
- `#### ステップ6: 統合とレビュー`

#### operations.md

各ステップ見出しの直後に追加：

- `### ステップ0: 変更確認`
- `### ステップ1: デプロイ準備【対話形式】`
- `### ステップ2: CI/CD構築【対話形式】`
- `### ステップ3: 監視・ロギング戦略【対話形式】`

## 技術的考慮事項

- 指示はツール非依存の表現（Claude Code、KiroCLI等で解釈可能）
- 既存のタスク管理関連ルール（CLAUDE.mdのTodoWriteツール活用など）との整合性を維持
- 設計省略（プロンプト修正のみのUnit）

# Unit 001: Operations Phase確認削減 - 実装計画

## 概要

Operations Phaseで変更がない場合に確認をスキップできる機能を追加し、繰り返しサイクルを高速化する。

## 変更対象ファイル

- `prompts/package/prompts/operations.md`

## 実装計画

### Phase 1: 設計

このUnitはプロンプトファイルの修正のみで、アプリケーションコードの変更はないため、設計ドキュメント（ドメインモデル・論理設計）は省略する。

**理由**: Unit定義の「技術的考慮事項」に明記されている通り、単一ファイルの修正でテキスト変更のみのため。

### Phase 2: 実装

#### ステップ4: コード生成

「## フロー」セクションの直前（「### 6. 全Unit完了確認」の後）に「### ステップ0: 変更確認」セクションを追加する。

**追加内容**:

1. 「変更したい項目はありますか？」という選択肢を提示
2. 「いいえ」選択時はステップ1-5をスキップしてステップ6へ進む
3. 「はい」選択時は従来通りの確認フローを維持

#### ステップ5: テスト生成

プロンプトファイルのテキスト変更のみのため、自動テストは不要。
手動での動作確認を実施。

#### ステップ6: 統合とレビュー

1. 変更内容のレビュー
2. 実装記録の作成

## 完了条件チェックリスト

- [ ] ステップ0冒頭に変更有無を問う選択肢を追加
- [ ] 「いいえ」選択時にステップ1-5の確認をスキップして自動進行
- [ ] 「はい」選択時は従来通りの確認フローを維持

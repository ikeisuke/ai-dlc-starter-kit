# Unit: サンドボックス環境ガイド

## 概要
AIエージェントをサンドボックス環境で安全に実行する方法を説明するガイドを作成する。

## 含まれるユーザーストーリー
- B-1: サンドボックス環境ガイド (#26)

## 責務
- sandbox-environment.mdガイドファイルの作成
- 各AIエージェント（Claude Code、Codex CLI、KiroCLI）のサンドボックス設定方法を記載
- Docker/コンテナ環境での実行例を提供
- セキュリティ上の注意事項を記載

## 境界
- 許可リスト関連の内容は含めない（Unit 005で対応）
- 各ツールの詳細な使用方法は記載しない（サンドボックス設定に特化）

## 技術的考慮事項
- 作成ファイル: `prompts/package/guides/sandbox-environment.md`
- 必須セクション:
  - 概要: サンドボックスの目的（意図しないファイル変更・コマンド実行の防止）と利点
  - Claude Code: Docker環境での実行方法、`--dangerously-skip-permissions` フラグの説明と注意事項
  - Codex CLI: `sandbox` 設定オプション（`"read-only"`, `"workspace-write"`, `"danger-full-access"`）の説明
  - KiroCLI: サンドボックス関連の設定方法（未対応の場合は「現時点で未対応」と明記）
  - Docker/コンテナ: 汎用的なコンテナ環境でのAIエージェント実行例（Dockerfile例を含む）
  - セキュリティ注意事項: サンドボックス無効化時のリスク、推奨設定

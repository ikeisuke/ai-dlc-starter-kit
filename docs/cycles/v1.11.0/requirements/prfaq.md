# PRFAQ: AI-DLC スターターキット v1.11.0

## Press Release（プレスリリース）

**見出し**: AI-DLC v1.11.0リリース - 開発サイクルの高速化とセキュリティガイド強化

**副見出し**: Operations Phaseの確認削減、タスク管理機能の活用指示、AIエージェント向けセキュリティガイドを追加

**発表日**: 2026年2月（予定）

**本文**:

### 背景

AI-DLCスターターキットは、AIを開発プロセスの中心に据えた開発方法論を実践するためのツールキットです。これまでのバージョンでユーザーから寄せられたフィードバックを基に、開発体験のさらなる向上を目指しました。

特に、繰り返しサイクルを回す際のOperations Phaseでの確認作業が冗長であること、タスク管理機能の活用が明示されていないこと、AIエージェントを安全に使用するためのガイドラインが不足していることが課題として挙げられていました。

### プロダクト

v1.11.0では以下の改善を実施しました：

1. **Operations Phase確認削減**: 変更がない場合に確認をスキップできる機能を追加。繰り返しサイクルを高速に回せるようになりました。

2. **タスク管理機能活用指示**: 各フェーズ・Unitでタスク管理機能を活用する指示を追加。Claude Code、KiroCLI等、様々なAIツールで解釈可能なツール非依存の表現を採用しました。

3. **AIレビュー完了条件明示化**: 指摘0件時に「【AIレビュー完了】指摘0件」メッセージを出力し、再レビュー漏れを防止します。

4. **セキュリティガイド整備**: サンドボックス環境での実行方法とAIエージェント向け許可リストのガイドを新設。安全にAIエージェントを活用できるようになりました。

5. **aidlc.tomlテンプレート化**: 新規セットアップ時にテンプレートから設定ファイルを生成できるようになり、セットアップが簡素化されました。

6. **README.mdバージョン情報追加**: 不足していたバージョンセクション（v1.0.1, v1.1.0, v1.7.x, v1.9.x, v1.10.0）を追加し、全バージョンの変更内容を把握できるようになりました。

## FAQ（よくある質問）

### Q1: Operations Phaseの確認スキップ機能はどのように使いますか？
A: ステップ0で「変更したい項目はありますか？」という質問が表示されます。「いいえ」を選択すると、ステップ1-5の確認をスキップして自動進行します。「はい」を選択すると従来通りの確認フローが実行されます。

### Q2: タスク管理機能の活用指示は特定のツールに依存していますか？
A: いいえ。「タスク管理機能を活用してください。」というツール非依存の表現を採用しており、Claude Code（TaskCreate/TaskUpdate）、KiroCLI等、様々なAIツールで解釈可能です。

### Q3: サンドボックス環境ガイドはどのAIエージェントに対応していますか？
A: Claude Code、Codex CLI、KiroCLIに対応しています。また、Docker/コンテナ環境での汎用的な実行例も記載しています。

### Q4: 許可リストガイドはどのAIエージェントの設定方法を記載していますか？
A: Claude Code、Cursor、Cline、Windsurf、Aiderの設定方法と、各ツール向けのコピー可能なテンプレート例を記載しています。

### Q5: 既存プロジェクトをv1.11.0にアップグレードする際、aidlc.tomlの設定は維持されますか？
A: はい。既存プロジェクトのアップグレード時は従来通りマイグレーションスクリプトで差分追加され、既存の設定値は保持されます。テンプレートからの生成は新規セットアップ時のみ使用されます。

### Q6: AIレビュー完了メッセージはどのような形式で表示されますか？
A: 指摘0件の場合のみ「【AIレビュー完了】指摘0件」形式のメッセージが出力されます。指摘がある場合は従来通り修正→再レビューのフローが動作します。

### Q7: README.mdにはどのバージョン情報が追加されますか？
A: v1.0.1, v1.1.0, v1.7.0〜v1.7.4, v1.9.0〜v1.9.3, v1.10.0のセクションが追加されます。これにより、README.mdで全バージョンの変更内容を確認できるようになります。配置順序はセマンティックバージョン降順（新しいバージョンが上）です。

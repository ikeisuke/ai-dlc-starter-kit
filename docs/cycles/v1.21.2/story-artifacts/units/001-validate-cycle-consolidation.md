# Unit: バリデーション共通化とサイクルID緩和

## 概要
サイクル名バリデーション正規表現を共通ライブラリに抽出し、非SemVer形式のカスタムサイクル名を正式サポートする。

## 含まれるユーザーストーリー
- ストーリー2: バリデーション正規表現の共通関数化 (#309)
- ストーリー1: カスタムサイクル名での履歴記録 (#312)

## 関連Issue
- #309
- #312

## 責務
- `prompts/package/lib/validate.sh` の新設と `validate_cycle` 関数の実装
- 正規表現の緩和（非SemVer名の許可）
- `write-history.sh` と `setup-branch.sh` からの共通関数呼び出し
- インラインバリデーション正規表現の削除

## 境界
- エラーメッセージ形式の統一は Unit 002 で実施
- プロンプト側のバリデーション（Inception Phase の名前入力チェック）は変更しない

## 非機能要件（NFR）
- **パフォーマンス**: `source` による読み込みオーバーヘッドが無視できること
- **セキュリティ**: バリデーション関数がパストラバーサルやインジェクションを防止すること

## 技術的考慮事項
- `source` パスは `$(cd "$(dirname "$0")" && pwd)/../lib/validate.sh` で解決
- 許可パターン: SemVer、名前付きSemVer、カスタム名すべてを許可
- 不正値（空文字、`/` のみ、制御文字等）は拒否

# Unit: バックログ登録時の不明点確認フロー

## 概要
バックログ登録時に不明点を明確にし、後から見ても意味がわかるバックログにするための確認フローを追加する。

## 含まれるユーザーストーリー
- ストーリー 9: バックログ登録時の不明点確認フロー (#101)

## 責務
- バックログ登録前の確認フロー追加
- 必須項目テンプレートの提供
- AskUserQuestionを活用した対話的登録

## 境界
- AI自動検出によるバックログ登録は対象外（ユーザー起点のみ）
- バックログの保存先（Issue/ローカルファイル）の変更は行わない

## 技術的考慮事項
- 対象: ユーザー起点のバックログ登録時のみ

**AIレビュー指摘への対応（Unit定義で特定）**:
- 対象ファイル: `prompts/package/prompts/common/` 配下に新規ファイル作成、または既存プロンプトに追記
  - 候補1: `prompts/package/guides/backlog-registration.md` を新規作成
  - 候補2: 各フェーズプロンプトのバックログ登録セクションに追記

- 確認フローの必須項目:
  1. **概要**: バックログ項目の簡潔な説明（1-2文）
  2. **背景**: なぜこの項目が必要か、発見した経緯
  3. **期待する結果**: 完了時にどうなっていれば良いか
  4. **優先度**: High / Medium / Low（任意）

- AskUserQuestion活用例:
  ```text
  【バックログ登録】

  バックログに登録する前に、以下を確認させてください。

  Q1: 概要を教えてください（何をしたいか、1-2文で）
  Q2: 背景を教えてください（なぜ必要か、発見した経緯）
  Q3: 期待する結果を教えてください（完了時にどうなっていれば良いか）
  ```

# Unit: Issue用基本ラベル移動

## 概要
Issue用の基本ラベル作成処理をセットアッププロンプト側に移動し、初期セットアップ時に基本ラベルが自動作成されるようにする。

## 含まれるユーザーストーリー
- ストーリー 3: Issue用基本ラベルの移動 (#127)

## 責務
- セットアッププロンプトへのラベル作成処理追加
- 冪等性の確保（既存ラベルのスキップ）
- GitHub CLI利用可否の確認

## 境界
- ラベルの色やdescriptionのカスタマイズは対象外
- 既存ラベルの更新は行わない

## 依存関係

### 依存する Unit
- なし

### 外部依存
- GitHub CLI (`gh`)

## 非機能要件（NFR）
- **パフォーマンス**: 該当なし
- **セキュリティ**: 該当なし
- **スケーラビリティ**: 該当なし
- **可用性**: GitHub CLI利用不可時はスキップ

## 技術的考慮事項
- 対象ファイル: `prompts/setup-prompt.md`
- GitHub CLI（`gh label create`）を使用
- 既存ラベルがある場合はエラーを無視してスキップ

**AIレビュー指摘への対応（Unit定義で特定）**:
- 作成する基本ラベル一覧:
  - `backlog` - バックログアイテム
  - `bug` - バグ報告
  - `enhancement` - 機能改善
  - `documentation` - ドキュメント
  - `question` - 質問
- ラベル作成コマンド例: `gh label create "backlog" --description "バックログアイテム" --color "0E8A16" 2>/dev/null || true`
- 注: `--force` は既存ラベル更新になるため使用しない。エラーを無視してスキップ

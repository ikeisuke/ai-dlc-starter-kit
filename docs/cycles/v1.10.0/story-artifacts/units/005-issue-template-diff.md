# Unit: Issueテンプレート差分確認機能

## 概要
Issueテンプレートのコピー前に差分を確認し、変更がある場合のみユーザーに確認を求める機能を追加する。

## 含まれるユーザーストーリー
- ストーリー 5: Issueテンプレート差分確認 (#113)

## 責務
- Issueテンプレートの差分確認ロジック追加
- 差分がない場合のスキップ処理
- 差分がある場合のユーザー確認フロー

## 境界
- Issueテンプレートの内容変更は対象外
- テンプレートのマージ処理は対象外（上書きのみ）

## 依存関係

### 依存する Unit
- Unit 004: Issue用基本ラベル移動（依存理由: 同一ファイル `prompts/setup-prompt.md` を編集するため、競合回避のため順次実施）

### 外部依存
- `diff` コマンド

## 非機能要件（NFR）
- **パフォーマンス**: 該当なし
- **セキュリティ**: 該当なし
- **スケーラビリティ**: 該当なし
- **可用性**: 該当なし

## 技術的考慮事項
- 対象ファイル: `prompts/setup-prompt.md` または専用スクリプト
- 対象: `.github/ISSUE_TEMPLATE/` 配下のファイル

**AIレビュー指摘への対応（Unit定義で特定）**:
- 差分確認の実装方法:
  1. ソースディレクトリ: `prompts/package/.github/ISSUE_TEMPLATE/`
  2. ターゲットディレクトリ: `.github/ISSUE_TEMPLATE/`
  3. `diff -q` でファイル単位の差分を確認
  4. 差分があるファイルのみ `diff` で詳細表示
- ユーザー確認フロー:
  ```text
  以下のIssueテンプレートに差分があります：
  [差分ファイル一覧]

  1. 上書きする（推奨）
  2. スキップする
  3. 差分を確認してから決める
  ```

# PRFAQ: AI-DLC v1.12.0

## Press Release（プレスリリース）

**見出し**: AI-DLC v1.12.0 - 設定の柔軟性とワークフロー効率化を実現

**副見出し**: 設定ファイルの階層化とフェーズ統合により、AI-DLCがさらに使いやすくなりました

**発表日**: 2026年2月（予定）

**本文**:

### 背景

AI-DLCは、AIを開発プロセスの中心に据えた革新的な開発方法論です。これまでも多くの開発者に利用されてきましたが、チームでの利用時に設定ファイルのコンフリクトが発生したり、新規サイクル開始時に複数のプロンプトを読み込む必要があるなど、改善の余地がありました。

### プロダクト

AI-DLC v1.12.0では、以下の新機能を導入します：

1. **設定ファイルの3階層化**: ユーザー共通設定（~/.aidlc/）、プロジェクト共有設定（docs/aidlc.toml）、プロジェクト個人設定（.local）の3階層で設定を管理できるようになりました。これにより、チーム共有の設定と個人の好みを明確に分離できます。

2. **Setup/Inception Phaseの統合**: これまで2つのプロンプトに分かれていたSetup PhaseとInception Phaseを1つに統合。新規サイクルの開始が1回のプロンプト読み込みで完了するようになりました。

3. **Codex resume機能の活用**: AIレビュー時に前回のセッションを継続できるようになり、コンテキストを保持したまま効率的なレビューが可能になりました。

4. **Dependabot PR確認のオプション化**: Dependabotを使用しているプロジェクト向けに、Inception PhaseでDependabot PRを確認する機能をオプションとして復活させました。

### 今後の展開

今後のバージョンでは、さらなるワークフローの最適化や、より多くのAIツールとの連携強化を予定しています。

## FAQ（よくある質問）

### Q1: 既存のプロジェクトでv1.12.0にアップグレードする際、設定ファイルの移行は必要ですか？
A: いいえ、既存のaidlc.tomlはそのまま使用できます。新しい.localファイルやホームディレクトリ設定は任意で追加できます。後方互換性を維持しているため、移行作業なしでアップグレードできます。

### Q2: 旧版のsetup.mdやinception.mdを読み込んだらどうなりますか？
A: 統合版のsetup-inception.mdへの誘導メッセージが表示されます。旧版のプロンプトは引き続き動作しますが、統合版の使用を推奨しています。

### Q3: 設定ファイルの優先順位はどうなっていますか？
A: ホーム設定（~/.aidlc/config.toml）→ プロジェクト設定（docs/aidlc.toml）→ ローカル設定（docs/aidlc.toml.local）の順で読み込まれ、後から読み込まれた設定が優先されます。同一キーがある場合、ローカル設定が最も優先されます。

### Q4: Lite版も同様に統合されますか？
A: はい、Lite版も同様にSetup/Inceptionが統合されます。通常版との一貫性を保ちつつ、Lite版の簡潔さを維持しています。

### Q5: Codex resume機能はどのように使いますか？
A: AIレビュー時に自動的にセッションIDが保持され、同一Unit内の2回目以降のレビューでは前回のコンテキストが引き継がれます。開発者が特別な操作をする必要はありません。

### Q6: Dependabot PR確認機能のデフォルト設定は？
A: デフォルトは無効（false）です。有効にするには、aidlc.tomlに`[inception.dependabot].enabled = true`を追加してください。

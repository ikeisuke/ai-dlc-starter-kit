# PRFAQ: AI-DLC Starter Kit v1.18.0

## Press Release（プレスリリース）

**見出し**: AI-DLC Starter Kit、AIレビュー合格時に自動で開発を進める「セミオートモード」を搭載

**副見出し**: AIレビューで品質が担保された成果物の手動承認を省略し、開発サイクルの効率を大幅に向上

**発表日**: v1.18.0リリース時

**本文**:

AI-DLCスターターキットを使用した開発では、各ステップでAIレビューとユーザー承認を繰り返して品質を確保しています。しかし、AIレビューで指摘0件となった成果物に対しても毎回手動で「OK」と入力する必要があり、開発のテンポを損なう要因となっていました。

v1.18.0では、`docs/aidlc.toml` の設定でオプトイン有効化できる「セミオートモード」を導入します。AIレビューで指摘0件の場合、ユーザー承認をスキップして自動的に次ステップ・次Unit・次フェーズへ遷移します。不明点や問題がある場合は従来どおりユーザーに確認を求めるフォールバック機構も備えており、品質を維持しつつ開発速度を向上させます。

「セミオートモードのおかげで、AIレビューが通った成果物に対する形式的な承認操作がなくなり、開発に集中できるようになりました。問題があるときだけ止まるので安心感もあります。」

また本バージョンでは、Amazon公開のAIDLCリポジトリの調査レポートを作成し、今後取り込むべきプラクティスを整理します。加えて、Issueクローズタイミングの改善、レビューサマリの詳細化、各種スクリプトのバグ修正など、日常的な開発体験の品質向上も行います。

## FAQ（よくある質問）

### Q1: セミオートモードはデフォルトで有効ですか？
A: いいえ。デフォルトは無効（`enabled = false`）です。`docs/aidlc.toml` の `[rules.semi_auto]` セクションで `enabled = true` に設定することで有効化できます。無効時は現行フローと完全に同一の動作をします。

### Q2: セミオートモードで自動承認される条件は何ですか？
A: AIレビューの指摘が0件の場合のみ自動承認されます。指摘が1件でも残っている場合は、重要度に関わらず通常フローに戻りユーザーに確認を求めます。

### Q3: セミオートモードでフェーズ間の遷移も自動化されますか？
A: はい。Inception→Construction→Operationsのフェーズ遷移、およびUnit間の遷移も自動化されます。全ステップ完了かつAIレビュー合格で次フェーズへ自動遷移し、Unit完了後は番号順で次のUnitへ自動遷移します。ブロック状態や循環依存が検出された場合はユーザーにフォールバックします。

### Q4: Amazon AIDLCリポジトリの調査結果はこのバージョンで適用されますか？
A: いいえ。v1.18.0では調査レポートの作成までです。プロンプト構成、ワークフロー設計、自動化、品質管理の4軸で比較分析を行い、取り込み候補を優先度付きでリストアップします。実際のプロンプトへの適用は次サイクル以降で行います。

### Q5: Issueクローズのタイミングはどう変わりますか？
A: これまでConstruction PhaseのUnit完了時にIssueをクローズしていましたが、v1.18.0からはPRマージ時にGitHubの `Closes #XX` 構文で自動クローズされるように変更します。これにより、マージ前にIssueが完了扱いになる不整合を防ぎます。

# 論理設計: Amazon AI-DLC リポジトリ調査

## 概要

Amazon AI-DLC リポジトリの調査手順、情報収集方法、分析アプローチを定義する。成果物は Markdown 形式の調査レポート1ファイル。

**重要**: この論理設計では**コードは書かず**、調査プロセスとレポート構造の定義のみを行います。

## アーキテクチャパターン

調査プロセスは以下の3層で構成する:

1. **情報収集層（Evidence Collection）**: リポジトリの構造・内容を収集
2. **分析層（Analysis）**: 収集した情報を本プロジェクトと比較・評価
3. **提案層（Recommendation）**: 分析結果から取り込み候補を抽出・優先度付け

## 処理フロー

### ステップ1: リポジトリ特定・概要把握

1. GitHub / Web検索で Amazon の AI-DLC 関連公開リポジトリを特定
2. リポジトリのREADME、ディレクトリ構成、主要ファイルを確認
3. リポジトリ情報（URL、ブランチ、コミットSHA、規模）を記録

**入力**: 検索キーワード（"amazon ai-dlc", "aws aidlc" 等）
**出力**: リポジトリ概要情報

### ステップ2: 4軸調査（情報収集層）

各比較軸について、Amazon側と本プロジェクト側の情報を収集する。

#### 軸1: プロンプト構成・構造

- **Amazon側**: ディレクトリ構造、ファイル命名規則、プロンプトの分割粒度、テンプレート管理方法
- **本プロジェクト側**: `docs/aidlc/prompts/` の構造を参照

#### 軸2: フェーズ・ワークフロー設計

- **Amazon側**: 開発フェーズの定義、フェーズ間遷移ルール、ゲート条件
- **本プロジェクト側**: Inception / Construction / Operations の3フェーズ構成を参照

#### 軸3: 自動化・ツール連携

- **Amazon側**: スクリプト、CI/CD統合、外部ツール連携、CLIツール
- **本プロジェクト側**: `docs/aidlc/bin/` のスクリプト群を参照

#### 軸4: 品質管理・レビュープロセス

- **Amazon側**: レビュー戦略、品質ゲート、テスト方針
- **本プロジェクト側**: `docs/aidlc/prompts/common/review-flow.md` のレビューフローを参照

### ステップ3: 比較分析（分析層）

1. 各軸について比較表を作成（ドメインモデルのスキーマに従う）
2. 共通点・差分を抽出
3. 各差分について優位性を評価

### ステップ4: 取り込み候補抽出（提案層）

1. 差分の中から本プロジェクトに適用可能な項目を抽出
2. 各候補に priority / effort / impact を付与（ドメインモデルの評価基準に従う）
3. 優先度順にソートしてリスト化

## 成果物

### ファイル形式

- **形式**: Markdown
- **配置先**: `docs/cycles/v1.18.0/requirements/amazon-aidlc-report.md`
- **構成**: ドメインモデルで定義した4セクション構成

## 情報収集方法

| 手段 | 用途 | 優先度 |
|------|------|--------|
| GitHub リポジトリ直接参照 | ファイル構成・コード内容の確認 | 最優先 |
| Web検索 | リポジトリ特定、関連記事・ドキュメント | 補助 |
| README / ドキュメント | プロジェクト概要・設計思想の把握 | 最優先 |

### リポジトリにアクセスできない場合のフォールバック

1. Web検索で関連情報（ブログ記事、発表資料等）を収集
2. 収集可能な範囲で分析を実施
3. レポートに「調査制約」セクションを追加し、アクセスできなかった情報を明記

## 非機能要件（NFR）への対応

Unit定義より全項目「該当なし」。調査タスクのため非機能要件は適用外。

## 実装上の注意事項

- Amazon側リポジトリの情報は調査日時点のスナップショットとして扱う
- 本プロジェクトとの比較は `cycle/v1.18.0` ブランチ時点の状態で行う
- リポジトリが見つからない場合やアクセスできない場合は、フォールバック手順に従う
- **部分失敗時の対応**: 特定の比較軸で十分な情報が得られない場合、レポート内に「情報不足」と明記し、収集できた範囲で分析を行う。比較不能な項目は「N/A（情報不足）」として記録する

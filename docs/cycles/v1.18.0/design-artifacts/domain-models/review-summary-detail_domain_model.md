# ドメインモデル: レビューサマリ指摘一覧詳細化

## 概要

レビューサマリファイルの指摘一覧テーブルにおける「内容」列・「対応」列の記述ルール（記述契約）を定義する。

**重要**: このドメインモデル設計では**コードは書かず**、構造と責務の定義のみを行います。

## エンティティ（Entity）

### 指摘一覧テーブル行（FindingRow）

レビューサマリファイルの1つの指摘を表す行。

- **属性**:
  - 番号(#): Integer - 指摘の連番
  - 重要度: Enum(高/中/低) - 指摘の重要度
  - 内容: FindingDescription - 問題の詳細記述（値オブジェクト）
  - 対応: ResolutionDescription - 対応内容の詳細記述（値オブジェクト）

## 値オブジェクト（Value Object）

### FindingDescription（内容列の記述構造）

指摘の「内容」を構成する情報要素。

- **属性**:
  - 対象箇所(location): String - 問題が発生したファイル・セクション・行番号等の特定情報
  - 問題事象(issue): String - 何がどう問題なのかの具体的記述
- **記述フォーマット**: `[対象箇所] - [問題事象]`（ハイフン区切り）
- **不変性**: 1つの指摘に対して必ず「どこの」「何が問題か」が明示されること
- **最小粒度**: 対象箇所は「ファイル名」レベル以上の具体性が必須。問題事象は第三者が読んで理解できる粒度

### ResolutionDescription（対応列の記述構造）

指摘への「対応」を構成する情報要素。

- **属性**:
  - 対応種別(type): Enum - 修正済み / TECHNICAL_BLOCKER / OUT_OF_SCOPE
  - 変更内容(detail): String - 対応種別に応じた詳細記述
- **記述フォーマット（対応種別別）**:
  - 修正済み: `修正済み（[変更箇所]: [変更内容]）`
  - TECHNICAL_BLOCKER: `TECHNICAL_BLOCKER（理由: [技術的理由の詳細]）`
  - OUT_OF_SCOPE: `OUT_OF_SCOPE（理由: [スコープ外である根拠]）`
- **不変性**: 1つの対応に対して必ず「対応結果（実施内容または未実施理由）」が明示されること
- **最小粒度（対応種別別）**:
  - 修正済み: 変更箇所 + 変更内容が必須
  - TECHNICAL_BLOCKER: 技術的理由 + 判断根拠が必須
  - OUT_OF_SCOPE: スコープ外である根拠が必須、対応予定は任意

## ユビキタス言語

- **対象箇所**: 指摘が該当するファイル・セクション・コード行等の位置情報
- **問題事象**: 具体的に何がどう問題なのかの記述
- **対応種別**: 指摘に対してとった対応の分類
- **変更内容**: 修正・対応の具体的な内容
- **記述契約**: 列に記載すべき必須情報要素とそのフォーマットの規約

## NG例（禁止パターン）

### 内容列のNG例

| NG | 理由 |
|-----|------|
| 影響範囲のパスと制約事項の矛盾 | 対象箇所不明、何がどう矛盾しているか不明 |
| 命名が不適切 | どのファイルのどの名前が、なぜ不適切か不明 |
| テストが不足 | どのテストが、何に対して不足しているか不明 |

### 対応列のNG例

| NG | 理由 |
|-----|------|
| 修正済み | 何をどう修正したか不明 |
| 修正済み（注記追加） | どこに何の注記を追加したか不明 |
| TECHNICAL_BLOCKER（理由: 対応不可） | 技術的理由が不明 |

# ユーザーストーリー

## Epic 1: 設定基盤リファクタ

### ストーリー 1: aidlc.toml設定キー構造の統一 (#207)
**優先順位**: Must-have

As a AI-DLCスターターキットの利用者
I want to aidlc.tomlの設定キーが一貫した階層構造で管理されている
So that 設定の理解・追加・変更が直感的に行える

**受け入れ基準**:
- [ ] `[backlog]` セクションが `[rules.backlog]` に移動されている
- [ ] `check-backlog-mode.sh` が `rules.backlog.mode` を読み取り、旧キー `backlog.mode` もフォールバックで正常読み取りできる
- [ ] `env-info.sh` が新キーで正常動作する
- [ ] `init-cycle-dir.sh` が新旧どちらのキー構造でも正常動作する
- [ ] 旧形式の `aidlc.toml`（`[backlog]`）を持つプロジェクトでエラーが発生せず、全モード（`git`/`issue`/`git-only`/`issue-only`）で既存と同じ挙動が維持される

**技術的考慮事項**:
- 直接 `backlog.mode` を読むのは `check-backlog-mode.sh` と `env-info.sh` の2箇所のみ
- `read-config.sh` 経由のフォールバック機構、またはスクリプト内で旧キー→新キーの順で試行

### ストーリー 2: read-config.shデフォルト値の集中管理 (#206)
**優先順位**: Must-have

As a AI-DLCスターターキットの開発者
I want to read-config.shのデフォルト値が一箇所で管理されている
So that デフォルト値の追加・変更時に複数ファイルを修正する必要がなくなる

**受け入れ基準**:
- [ ] デフォルト値定義ファイル（例: `defaults.toml`）が作成されている
- [ ] `read-config.sh` がデフォルト値定義ファイルを自動参照する
- [ ] 既存の `--default` 指定箇所（5回: commit-flow.md×2, rules.md×1, feedback.md×1, upgrading-aidlc/SKILL.md×1）のデフォルト値が定義ファイルに移行済み
- [ ] `--default` オプションが指定された場合は、定義ファイルより `--default` が優先される（後方互換）
- [ ] デフォルト値定義ファイルが存在しない環境でもエラーにならない

**技術的考慮事項**:
- read-config.shの3階層マージ（HOME→プロジェクト→ローカル）にデフォルト値レイヤーを追加
- デフォルト値定義ファイルは `prompts/package/` に配置し、手動rsyncまたは `sync-package.sh`（ストーリー5）で `docs/aidlc/` に配布
- ストーリー5未完了でも手動rsyncで配布可能なため、ストーリー5への依存はない

## Epic 2: Operations Phase定型処理スクリプト化

### ストーリー 3: Issueテンプレート差分確認のスクリプト化 (#205)
**優先順位**: Should-have

As a AI-DLCスターターキットの開発者
I want to ローカルとリモートのGitHub Issueテンプレートの差分を自動検出できる
So that テンプレートの更新漏れを防止できる

**受け入れ基準**:
- [ ] `check-issue-templates.sh` が作成されている
- [ ] `.github/ISSUE_TEMPLATE/` 配下のローカルファイルとリモート（mainブランチ）のファイルを比較できる
- [ ] 差分がある場合、差分のあるファイル名と差分内容のサマリを出力する
- [ ] 差分がない場合、「差分なし」と出力する
- [ ] `gh` 未インストールまたは未認証の場合、エラーメッセージを出力して終了コード1で終了する
- [ ] ネットワークエラー・API制限時に適切なエラーメッセージを出力する

**技術的考慮事項**:
- `gh api` でリモートファイル取得（ghが利用可能な場合のみ動作）
- 差分検出は `diff` コマンドを使用

---

### ストーリー 4: バージョンファイル更新のスクリプト化 (#204)
**優先順位**: Should-have

As a AI-DLCスターターキットの開発者
I want to バージョン番号の更新を1コマンドで実行できる
So that バージョン更新の手順ミスや更新漏れを防止できる

**受け入れ基準**:
- [ ] `update-version.sh` が作成されている
- [ ] サイクル番号（例: `v1.16.2`）を引数に取り、`v` プレフィックスを除去して `version.txt` を更新する
- [ ] `docs/aidlc.toml` の `starter_kit_version` を同じ値に更新する
- [ ] 更新前後の値を出力して確認できる
- [ ] `--dry-run` オプションで変更内容を表示（実際の書き込みなし）
- [ ] 引数未指定時にUsageを表示して終了コード1で終了する
- [ ] 不正フォーマット（`v` プレフィックスなし、セマンティックバージョニング非準拠等）の場合、エラーメッセージを出力する
- [ ] `version.txt` または `docs/aidlc.toml` が存在しない場合、エラーメッセージを出力する

**技術的考慮事項**:
- `sed` を使用（macOS互換の `-i ''` 形式）
- `rules.md` の手動手順を置き換える

### ストーリー 5: rsync同期処理のスクリプト化 (#203)
**優先順位**: Should-have

As a AI-DLCスターターキットの開発者
I want to prompts/package/からdocs/aidlc/へのrsync同期を1コマンドで実行できる
So that 同期忘れや同期漏れを防止できる

**受け入れ基準**:
- [ ] `sync-package.sh` が作成されている
- [ ] `prompts/package/` から `docs/aidlc/` への rsync 同期を実行する
- [ ] 除外パターンとして `.git`, `.DS_Store` を除外する
- [ ] 同期結果（追加・更新・削除されたファイル一覧）を出力する
- [ ] `--dry-run` オプションで同期内容を表示（実際の同期なし）
- [ ] `prompts/package/` または `docs/aidlc/` が存在しない場合、エラーメッセージを出力して終了コード1で終了する

**技術的考慮事項**:
- 既存の `/upgrading-aidlc` スキル内のrsyncロジックを参考にする
- スタンドアロンスクリプトとして独立動作させる（スキルからの呼び出しも可能）
- ストーリー2のデフォルト値定義ファイル配布もこのスクリプトの同期対象に含まれる

As a AI-DLCスターターキットの開発者
I want to フェーズ実行をスキル化する方式とサブエージェント化する方式の比較検討が完了している

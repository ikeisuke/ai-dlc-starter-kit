# 論理設計: Construction Phase確認の自動化

## 概要

`prompts/package/prompts/construction.md`の0.6および0.7セクションを修正し、ユーザーへの質問形式から自動確認形式に変更する。

## 変更対象ファイル

- `prompts/package/prompts/construction.md`

## 変更内容

### 0.6 AIレビュー実施確認

#### 変更前の構造

```text
### 0.6 AIレビュー実施確認【必須】
- 確認項目の説明
- **確認フォーマット**: ユーザーへの質問形式
  - 「以下のタイミングでAIレビューを実施しましたか？」
- **未実施がある場合**: 対応選択肢を提示
```

#### 変更後の構造

```text
### 0.6 AIレビュー実施確認【必須】
- 確認項目の説明
- **自動確認手順**: 履歴ファイルを読み取る手順
- **確認結果フォーマット**: 自動判定結果の出力形式
  - 実施済み / 未実施 / ファイル未検出
- **未実施の場合**: 対応選択肢を提示（変更なし）
```

#### 具体的な変更内容

1. **「確認フォーマット」セクションを削除**
   - ユーザーへの質問形式を削除

2. **「自動確認手順」セクションを追加**
   ```text
   **自動確認手順**:

   1. 履歴ファイルを確認:
   ```bash
   ls docs/cycles/{{CYCLE}}/history/construction_unit{NN}.md 2>/dev/null
   ```

   2. ファイルが存在する場合、内容を読み取り以下のパターンを検索:
      - 「AIレビュー」
      - 「レビュー反映」

   3. 確認結果を出力
   ```

3. **「確認結果フォーマット」セクションを追加**
   - ドメインモデルで定義した出力形式を記載

### 0.7 Operations引き継ぎタスク確認

#### 変更前の構造

```text
### 0.7 Operations引き継ぎタスク確認【必須】
- 確認内容の説明
- **確認メッセージ**: ユーザーへの質問形式
  - 「手動作業はありましたか？」
  - 1. はい / 2. いいえ
- **「はい」の場合**: タスク記録手順
- **「いいえ」の場合**: 次のステップへ
```

#### 変更後の構造

```text
### 0.7 Operations引き継ぎタスク確認【必須】
- 確認内容の説明
- **自動確認手順**: ディレクトリを確認する手順
- **確認結果フォーマット**: 自動判定結果の出力形式
  - タスクあり（一覧表示） / タスクなし
```

#### 具体的な変更内容

1. **「確認メッセージ」セクションを削除**
   - ユーザーへの質問形式を削除
   - 「はい/いいえ」の選択肢を削除

2. **「自動確認手順」セクションを追加**
   ```text
   **自動確認手順**:

   ```bash
   ls docs/cycles/{{CYCLE}}/operations/tasks/*.md 2>/dev/null
   ```

   ディレクトリ内の`.md`ファイルを列挙し、存在確認を行う。
   ```

3. **「確認結果フォーマット」セクションを追加**
   - ドメインモデルで定義した出力形式を記載

4. **「はい/いいえ」の分岐を削除**
   - 自動判定により分岐が不要になる
   - タスクの新規追加は別の文脈（作業中の気づき）で行われるため、完了時確認では存在確認のみ

## 削除される記述

### 0.6セクション

```text
**確認フォーマット**:

```text
【AIレビュー実施確認】

以下のタイミングでAIレビューを実施しましたか？

- [ ] 設計レビュー（Phase 1 ステップ3）
- [ ] 統合とレビュー（Phase 2 ステップ6）
```
```

### 0.7セクション

```text
**確認メッセージ**:

```text
【Operations引き継ぎタスク確認】

このUnit作業中に、Operations Phaseで実行が必要な手動作業はありましたか？

例:
- 手動でのデータベースマイグレーション
- 環境変数の追加設定
- 外部サービスの設定変更
- 手動でのデプロイ手順

1. はい - 手動作業がある（タスクを記録）
2. いいえ - 手動作業なし
```

**「はい」の場合**:
（タスク記録手順全体）

**「いいえ」の場合**: 次のステップへ進む
```

## 影響範囲

- `prompts/package/prompts/construction.md`のみ
- 他のファイルへの影響なし
- 既存の履歴ファイル形式に変更なし
- 既存のoperations/tasks/ディレクトリ構造に変更なし

## 後方互換性

- 既存の履歴ファイルは引き続き使用可能
- 既存のoperations/tasks/ファイルは引き続き使用可能
- Construction Phaseの他のステップには影響なし

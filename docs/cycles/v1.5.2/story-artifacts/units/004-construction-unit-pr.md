# Unit: Construction Phase Unit PR作成・マージ

## 概要
各Unit完了時にサイクルブランチへのPRを作成し、レビュー後にマージする機能を実装する。

## 含まれるユーザーストーリー
- ストーリー 1.2: Unit完了時のPR作成とマージ

## 責務
- Construction Phase の各Unit完了時にPR作成提案
- Unitブランチの作成・切り替え
- PRの作成（サイクルブランチへ）
- PRマージ
- Unitブランチ削除

## 境界
- レビュープロセスの自動化は含まない（ユーザーが手動レビュー）
- CI/CDとの連携は含まない

## 依存関係

### 依存する Unit
- Unit 003: Inception Phase ドラフトPR作成（依存理由: ドラフトPRが存在することが前提）

### 外部依存
- GitHub CLI（gh）
- Gitリポジトリ

## 非機能要件（NFR）
- **パフォーマンス**: PR作成・マージは各10秒以内
- **セキュリティ**: GitHub認証の確認
- **スケーラビリティ**: 複数Unitの並列PR作成に対応
- **可用性**: GitHub CLI利用不可時はスキップ

## 技術的考慮事項
- ブランチ命名規則: `cycle/vX.X.X/unit-NNN`
- PR本文にUnit定義の要約を含める
- マージ方法: squash merge推奨
- エラーハンドリング（ブランチ作成失敗、PR作成失敗等）

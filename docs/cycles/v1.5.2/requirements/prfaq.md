# PRFAQ: AI-DLC スターターキット v1.5.2

## Press Release（プレスリリース）

**見出し**: AI-DLC v1.5.2 リリース - 並行作業とセットアップの柔軟性を大幅強化

**副見出し**: ドラフトPRワークフローの導入により複数Unitを並列開発可能に。バックログ移行の自動化とセットアップ改善で開発体験が向上。

**発表日**: 2025年12月下旬（予定）

**本文**:

**[背景]**
AI-DLCスターターキットは、AIを開発の中心に据えた新しい開発手法を実践するためのツールです。これまでのバージョンでは、Construction Phaseで複数Unitを同時進行することが困難であり、1つのUnitが完了するまで次のUnitに着手できませんでした。また、スターターキットのアップグレードを選択しない場合、サイクル開始に必要なファイルが不足する問題がありました。

**[プロダクト]**
AI-DLC v1.5.2では、以下3つの機能を導入しました：

1. **ドラフトPRベースのワークフロー**
   Inception Phase完了時にドラフトPRを作成し、各UnitはサイクルブランチへのPRとして実装します。これにより、複数人または複数セッションでの並行作業が可能になり、Unit単位でのレビューと進捗可視化が実現しました。

2. **バックログ移行の自動化**
   旧形式の `backlog.md`（単一ファイル）を新形式（個別ファイル方式）に自動移行する機能を追加。完了済み項目の重複チェックも含まれており、移行ミスを防止します。

3. **セットアップ柔軟性の向上**
   アップグレードしない場合でも、コピー元（`prompts/package/prompts/setup.md`）を直接参照してサイクルを開始できるよう案内を改善しました。

## FAQ（よくある質問）

### Q1: ドラフトPRワークフローを使わない場合でも従来通り開発できますか？
A: はい、可能です。ドラフトPRワークフローはオプトイン形式で、GitHub CLIが利用できない場合や、従来のワークフローを好む場合はスキップできます。

### Q2: バックログ移行は手動で実行する必要がありますか？
A: いいえ、setup-prompt.md（アップグレード時）を実行すると、旧形式のbacklog.mdが自動的に検出され、移行処理が提案されます。ユーザーは承認するだけで移行が完了します。

### Q3: 複数人で同時に異なるUnitを開発できますか？
A: はい、ドラフトPRワークフローを使用すれば、各開発者が異なるUnitブランチ（例: `cycle/v1.5.2/unit-001`、`cycle/v1.5.2/unit-002`）で作業できます。それぞれのPRをサイクルブランチにマージすることで、コンフリクトを最小限に抑えながら並行開発が可能です。

### Q4: セットアップ改善の恩恵を受けるのはどのような場合ですか？
A: スターターキットのアップグレードを一時的にスキップしたい場合（緊急のバグ修正サイクル等）に特に有効です。従来はアップグレードしないと必要なファイルが不足しましたが、v1.5.2ではコピー元を直接参照できるため、柔軟にサイクルを開始できます。

### Q5: 既存のサイクル（v1.5.1以前）で作成したプロジェクトにも適用できますか？
A: はい、v1.5.2にアップグレードすることで、次のサイクルから新機能が利用可能になります。既存サイクルへの遡及適用は不要です。

### Q6: ドラフトPR機能を使う場合、GitHubの設定は必要ですか？
A: GitHub CLI（`gh`コマンド）がインストールされ、認証が完了している必要があります。未設定の場合は、初回実行時に設定手順が案内されます。

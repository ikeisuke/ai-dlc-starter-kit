# ドメインモデル: CI/CD構築

## 概要

GitHub ActionsによるCI/CDパイプラインの概念モデルを定義する。PRに対するMarkdownリンター自動実行の仕組みを規定。

**注**: このUnitはGitHub Actionsワークフローと設定ファイルの作成であり、実行可能なアプリケーションコードを生成するものではない。

## 概念モデル

### ワークフロー（Workflow）

GitHub Actionsで実行される自動化処理。

- **トリガー**: pull_request（mainブランチ向け）
- **ジョブ**: markdown-lint
- **実行環境**: ubuntu-latest

### リンタールール（LintRules）

Markdownファイルの品質チェックルール。

- **設定ファイル**: .markdownlint.json
- **対象**: docs/, prompts/ 配下の *.md ファイル

### チェック結果（CheckResult）

リンター実行結果。

- **success**: エラーなし
- **failure**: エラーあり（詳細を表示）

## 処理フロー

```
┌─────────────────────────────────────────────────────────────┐
│                    PR作成/更新                              │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
                   ┌───────────────────┐
                   │ ワークフロー起動   │
                   └───────────────────┘
                              │
                              ▼
                   ┌───────────────────┐
                   │ リポジトリチェックアウト │
                   └───────────────────┘
                              │
                              ▼
                   ┌───────────────────┐
                   │ Markdownリンター実行 │
                   └───────────────────┘
                              │
                    ┌─────────┴─────────┐
                    ▼                   ▼
                 成功                 失敗
                    │                   │
                    ▼                   ▼
              ✓ ステータス表示    ✗ エラー詳細表示
```

## ユビキタス言語

- **ワークフロー**: GitHub Actionsで定義される自動化処理
- **ジョブ**: ワークフロー内の実行単位
- **リンター**: コード品質をチェックするツール
- **markdownlint**: Markdown専用のリンターツール

# PRFAQ: AI-DLC スターターキット v1.5.3

## Press Release（プレスリリース）

**見出し**: AI-DLC スターターキット v1.5.3 - セットアップ体験を大幅改善

**副見出し**: macOS/zsh対応、worktree改善、AIレビュー強化で開発者体験が向上

**発表日**: 2025年（サイクル完了後）

**本文**:

### 背景
AI-DLC（AI-Driven Development Lifecycle）スターターキットは、AIを開発プロセスの中心に据えた新しい開発方法論の実践キットです。これまでのバージョンで基本機能を整備してきましたが、セットアップ時にmacOS/zsh環境でエラーが発生する、worktreeの作成先ディレクトリが意図と異なる、AIレビューが必須設定でもスキップされることがあるなど、ユーザー体験に関する課題が報告されていました。

### プロダクト
v1.5.3では、これらの課題を解決し、より快適な開発体験を提供します。

**主な改善点**:
1. **シェル互換性の向上**: macOSのデフォルトシェルであるzshでもセットアップが正常に完了するよう、スクリプトを修正しました。
2. **worktree機能の改善**: worktreeが並列ディレクトリに正しく作成されるようになり、AIによる自動作成機能も追加されました。
3. **AIレビュー機能の強化**: mode=required設定時にAIレビューが確実に実行され、人間承認前に自動的にレビューが行われるようになりました。
4. **CI/CD基盤の整備**: GitHub ActionsによるMarkdownリンターが導入され、ドキュメントの品質が自動的に担保されます。

### 顧客の声
「v1.5.3にアップグレードしてから、セットアップでつまずくことがなくなりました。特にzsh環境での動作が安定したのは大きいです。」

「AIレビューが人間の承認前に自動で実行されるようになり、レビュー依頼時の品質が向上しました。承認者の負担が大幅に減りました。」

### 今後の展開
今後は、AGENTS.mdへのAI-DLC統合、Issue駆動開発との連携、ホームディレクトリへの共通設定配置など、より高度な機能を検討していきます。

## FAQ（よくある質問）

### Q1: v1.5.2からv1.5.3へのアップグレード方法は？
A: prompts/setup-prompt.mdを読み込み、アップグレードオプションを選択してください。自動的にファイルが更新されます。

### Q2: zsh環境で以前発生していたエラーは解消されましたか？
A: はい、grep -oPコマンドをgrep -E + sedに置き換えることで、macOS標準のzsh環境でもエラーなく動作するようになりました。

### Q3: AIレビューを無効にすることはできますか？
A: はい、aidlc.tomlの[rules.mcp_review]セクションでmode = "disabled"を設定することで無効にできます。

### Q4: CI/CDでどのような品質チェックが行われますか？
A: PRに対してMarkdownリンター（markdownlint）が自動実行され、構文エラーやスタイルの問題が検出されます。

### Q5: worktreeを使わない通常のブランチ運用でも使用できますか？
A: はい、worktreeの使用はオプションです。aidlc.tomlの[rules.worktree]セクションでenabled = falseを設定すれば、worktreeの提案が表示されなくなります。

### Q6: サイクル名が自動で認識されない場合はどうすればよいですか？
A: ブランチ名がcycle/vX.Y.Z形式であれば自動認識されます。そうでない場合は、Inception Phase開始時にサイクル名の入力を求められます。

### Q7: 今回のサイクルで対応されなかった機能はありますか？
A: ホームディレクトリへのユーザー共通設定配置、Issue駆動開発統合、バックログの同名タイトル重複処理改善は、優先度の関係で今回は対応されていません。次回以降のサイクルで検討予定です。

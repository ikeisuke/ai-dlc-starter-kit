# 計画: Unit 001 - コミットハッシュ記録廃止

## 概要
Unit定義テンプレートからコミットハッシュ記録フィールドと関連注意書きを削除する。

## 変更対象ファイル
1. `prompts/package/templates/unit_definition_template.md`

## 変更内容

### 1. unit_definition_template.md
**削除する行:**
- 52行目: `- **コミット**: -`
- 54行目: `> **注意**: コミットハッシュは実際にコミットを作成した後に記録してください。短縮形（7文字）を使用します。`

**変更後の「実装状態」セクション:**
```markdown
---
## 実装状態

- **状態**: 未着手
- **開始日**: -
- **完了日**: -
- **担当**: -
```

## 影響範囲
- 新規作成されるUnit定義ファイルにはコミットフィールドが含まれなくなる
- 既存のUnit定義ファイル（v1.4.1含む）にはコミットフィールドが残るが、運用上問題なし
- git log で履歴を参照すれば、いつでもコミットを特定可能

## 設計フェーズについて
このUnitはテンプレートの軽微な修正のみであり、ドメインモデルや論理設計が不要な性質のタスクです。

**理由:**
- ビジネスロジックの設計が不要（テンプレートの文字列削除のみ）
- アーキテクチャへの影響なし
- API設計なし

**提案:** 設計フェーズをスキップし、直接実装フェーズに進むことを推奨します。

## 実装手順
1. `prompts/package/templates/unit_definition_template.md` を編集
2. コミットフィールドと注意書きを削除
3. 変更内容を確認
4. 実装記録を作成
5. Unit定義ファイルの状態を更新
6. 履歴記録
7. Gitコミット

# PRFAQ: AI-DLC Starter Kit v1.4.1

## Press Release（プレスリリース）

**見出し**: AI-DLC Starter Kit v1.4.1 - 開発体験を向上させる微調整リリース

**副見出し**: 不要なワークフローの削減、ルールの明確化、新プロジェクトタイプ対応で、より快適な AI 駆動開発を実現

**発表日**: 2025-12

**本文**:

AI-DLC Starter Kit チームは本日、バージョン 1.4.1 をリリースしました。このリリースでは、開発者からのフィードバックに基づき、開発体験を向上させるための複数の改善を行いました。

主な改善点として、コミットハッシュの手動記録という冗長なワークフローを廃止し、Unit 定義ファイルに実行順序番号を付与することで依存関係の可視性を向上させました。また、その場しのぎの対応（workaround）を行う際に本質的な対応をバックログに記録するルールを追加し、技術的負債の管理を強化しました。

さらに、コマンドラインツール（CLI）をプロジェクトタイプとして追加し、CLI ツール開発時に適切な Operations Phase 処理を受けられるようになりました。

「これらの改善により、日々の開発作業がよりスムーズになりました。特にコミットハッシュの記録廃止は、地味ですが毎回のストレスが減って嬉しいです」と、アーリーアダプターの開発者は述べています。

今後も継続的な改善を行い、AI 駆動開発のベストプラクティスを提供していきます。

## FAQ（よくある質問）

### Q1: コミットハッシュの記録を廃止したら、どうやってコミットを追跡するのですか？
A: `git log` コマンドで履歴を参照することで、必要に応じてコミットを追跡できます。ファイルに記録する必要はありません。

### Q2: Unit 定義ファイルの番号付けは既存のファイルにも適用されますか？
A: いいえ、新規作成時のルールのみです。既存のファイルはそのまま使用できます。

### Q3: workaround 時のバックログ追加ルールは必須ですか？
A: はい、技術的負債を可視化し将来的な対応を促進するため、必須としています。

### Q4: CLI プロジェクトタイプはどのように扱われますか？
A: デスクトップアプリに準じた扱いとなり、Operations Phase で配布ステップが実施されます。

### Q5: README.md のリンク辿り機能はどこまで辿りますか？
A: プロジェクト内部のドキュメントリンクのみを対象とし、外部リンクは辿りません。コンテキスト溢れ防止のため、深さや数に制限を設けています。

# ドメインモデル: CLIプロジェクトタイプ追加

## 概要

Operations Phaseのプロジェクトタイプに「cli」を追加し、配布ステップの実行条件を定義する。

**重要**: このUnitはドキュメント/テンプレートの修正のみであり、実装コードは含まない。

## ドメイン概念

### プロジェクトタイプ（ProjectType）

Operations Phaseで成果物の種類を識別する値オブジェクト。

**現在の値**:
- `ios`: iOSモバイルアプリ
- `android`: Androidモバイルアプリ
- `web`: Webアプリケーション
- `backend`: バックエンドサービス
- `general`: 汎用プロジェクト

**追加する値**:
- `desktop`: デスクトップアプリケーション（既存だが明示化）
- `cli`: コマンドラインツール（**新規追加**）

### 配布ステップ実行条件

プロジェクトタイプに応じた配布ステップ（ステップ4）の実行可否。

**実行する**:
- `ios`, `android`: App Store / Google Playへの配布
- `desktop`, `cli`: パッケージマネージャやバイナリ配布

**スキップする**:
- `web`, `backend`, `general`: 配布ステップ不要（CI/CDでデプロイ済み）

## ユビキタス言語

- **CLI（Command Line Interface）**: コマンドラインから実行するツール
- **配布（Distribution）**: エンドユーザーへの成果物の提供方法（App Store、homebrew、npm等）
- **プロジェクトタイプ**: 開発成果物の種類を示す分類

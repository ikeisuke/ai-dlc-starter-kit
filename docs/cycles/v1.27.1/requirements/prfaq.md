# PRFAQ: AI-DLC Starter Kit v1.27.1

## Press Release（プレスリリース）

**見出し**: AI-DLCスターターキット v1.27.1 — マルチリモート対応とKiroエージェント設定マージ

**副見出し**: 開発支援スクリプトの堅牢性向上とKiroエージェント設定の自動マージ機能を追加

**発表日**: 2026-03

**本文**:

AI-DLCスターターキットを使用する開発者の一部は、マルチリモート環境でpost-merge後のブランチクリーンアップが誤ったリモートに対して実行される問題や、Kiroエージェント設定ファイルのアップデートが手動でしか行えない不便さを経験していました。

v1.27.1では、post-merge-cleanup.shのリモート解決ロジックを改善し、ローカルブランチが削除済みの場合でもリモートブランチの存在を確認して正しいリモートを特定します。また、setup_kiro_agentにallowedCommandsの差分マージ機能を追加し、ユーザーカスタマイズを保持しながらテンプレートの新規コマンドを自動追加します。さらに、defaults.tomlのパス情報をドキュメントに明記し、不在時の診断メッセージを追加しました。

## FAQ（よくある質問）

### Q1: シングルリモート環境でも影響がありますか？
A: いいえ。シングルリモート環境では従来と同じ動作をします。改善はマルチリモート環境でのみ効果を発揮します。

### Q2: Kiroエージェント設定をsymlinkで管理している場合はどうなりますか？
A: symlink状態のファイルは従来通りの動作（マージなし）です。マージは実ファイル（ユーザーカスタマイズ済み）の場合のみ実行されます。

### Q3: defaults.tomlが存在しない古いプロジェクトはどうすればよいですか？
A: aidlc-setupを実行すれば自動的に同期されます。v1.27.1以降ではdefaults.toml不在時に診断メッセージが表示されるため、対処が容易になります。

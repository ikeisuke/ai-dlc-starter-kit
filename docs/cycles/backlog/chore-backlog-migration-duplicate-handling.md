# バックログ移行時の同名タイトル重複処理改善

- **発見日**: 2025-12-25
- **発見フェーズ**: Construction
- **発見サイクル**: v1.5.2
- **優先度**: 低

## 概要

backlog.md移行処理で、同名タイトルの項目が存在する場合に適切にハンドリングできない可能性がある。

## 詳細

現在の実装では、ファイル名（prefix + slug）の重複のみをチェックしている。しかし、異なる内容を持つ項目が同じタイトルを持つ場合、片方がスキップされてしまう。

例:
- `### 認証機能の改善`（延期タスク）
- `### 認証機能の改善`（次サイクル検討）

両方とも `deferred-authentication-improvement.md` または `feature-authentication-improvement.md` として生成されようとするが、片方のみが作成される。

## 対応案

1. ファイル名に連番サフィックスを追加（例: `feature-xxx-1.md`, `feature-xxx-2.md`）
2. ファイル名にハッシュを追加（例: `feature-xxx-a1b2c3.md`）
3. 移行時に警告を表示し、手動対応を促す

**推奨**: 実際の運用では稀なケースのため、警告表示（オプション3）で十分。

# スターターキット自体のアップグレードフロー明確化

- **発見日**: 2026-01-05
- **発見フェーズ**: サイクルセットアップ
- **発見サイクル**: v1.5.4 準備中
- **優先度**: 中

## 概要

setup.md のアップグレード案内が、スターターキット自体の開発リポジトリでは適切でない。

## 詳細

setup.md のステップ1では、新しいバージョンがある場合に以下を案内する:

```
アップグレードするには、スターターキットの setup-prompt.md を読み込んでください。
```

しかし、ai-dlc-starter-kit リポジトリ自体では:
- 「最新版をpullしてアップグレード」ではなく「次サイクルで変更を加えてリリース」する
- アップグレード案内は不要で、むしろ混乱を招く

## 対応案

### 案1: setup.md でスターターキット自体を検出

プロジェクト名が `ai-dlc-starter-kit` の場合、アップグレード案内をスキップする:

```bash
PROJECT_NAME=$(grep 'name\s*=' docs/aidlc.toml | head -1 | sed 's/.*"\([^"]*\)".*/\1/')
if [ "$PROJECT_NAME" = "ai-dlc-starter-kit" ]; then
  echo "スターターキット開発リポジトリのため、アップグレード案内をスキップします。"
fi
```

### 案2: aidlc.toml に開発リポジトリフラグを追加

```toml
[project]
is_starter_kit_dev = true  # スターターキット自体の開発リポジトリ
```

### 案3: ドキュメントで説明

setup.md に注記を追加して、スターターキット開発時の動作を説明する。

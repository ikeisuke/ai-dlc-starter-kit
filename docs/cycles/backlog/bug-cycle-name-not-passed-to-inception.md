# セットアップからInceptionへサイクル名が引き継がれない

- **発見日**: 2025-12-27
- **発見フェーズ**: Setup
- **発見サイクル**: v1.5.3
- **優先度**: 高

## 概要

setup.md完了後にinception.mdを読み込む際、サイクル名（例: v1.5.3）をどう伝えるかが明示されていない。

## 詳細

**現状の問題**:
- `setup.md` の完了メッセージ: 「docs/aidlc/prompts/inception.md を読み込んでください」
- `inception.md` は `{{CYCLE}}` プレースホルダーを使用
- **サイクル名の引き継ぎ方法が未定義**

**影響**:
- AIがinception.mdを読み込んだ際、どのサイクルで作業するか分からない
- ユーザーが別途サイクル名を指定する必要がある（UX悪化）

## 対応案

### 案1: setup.mdの完了メッセージを改善（推奨）

```markdown
Inception Phase を開始するには、以下のプロンプトを読み込んでください：
docs/aidlc/prompts/inception.md

サイクル: {{CYCLE}}
```

AIがこのメッセージを見れば、サイクル名を把握できる。

### 案2: inception.mdにサイクル名検出ロジックを追加

```markdown
## 最初に必ず実行すること

### 0. サイクル名の決定

サイクル名を以下の優先順位で決定：

1. ユーザーが明示的に指定した場合、その値を使用
2. 現在のブランチ名が `cycle/vX.Y.Z` 形式の場合、そこから抽出
3. `docs/cycles/` 配下の最新サイクルディレクトリを使用
4. 上記いずれも該当しない場合、ユーザーに質問
```

### 推奨

案1と案2の両方を実装する。案1は即座の改善、案2はフェールセーフとして機能する。

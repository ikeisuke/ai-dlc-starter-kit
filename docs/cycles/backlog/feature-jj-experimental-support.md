# jj（Jujutsu）の実験的サポート

- **発見日**: 2026-01-10
- **発見フェーズ**: セットアップ
- **発見サイクル**: v1.6.0
- **優先度**: 低

## 概要

Gitの代替バージョン管理システム「jj（Jujutsu）」を実験的に利用できるようにする。

## 詳細

**jjの特徴**:
- ワーキングコピーが自動コミット（stash不要）
- 匿名ブランチ（ブランチ名を考える必要がない）
- 操作の取り消しが簡単（`jj undo`）
- Git互換（既存リポジトリで使用可能）
- コンフリクトを保存可能
- 安全なrebase

**AI-DLCとの相性**:
- worktreeを多用する開発スタイルと相性が良い可能性
- サイクルごとのブランチ管理が簡素化される可能性
- 実験的な変更の取り消しが容易

## 対応案

### Phase 1: 調査・検証
1. jjのGit互換モード（colocate）での動作確認
2. AI-DLCワークフローとの互換性検証
3. worktree相当の機能確認

### Phase 2: ドキュメント整備
1. jj利用時のセットアップ手順を文書化
2. Git/jj両対応のコマンド対照表作成
3. rules.mdにjj固有の注意事項を追記

### Phase 3: プロンプト対応（オプション）
1. setup.mdにjj検出ロジック追加
2. jj利用時の代替コマンド提示

## 備考

- Gitとの共存が可能なため、段階的な導入が可能
- チーム全員がjjを使う必要はない（Gitユーザーと共存可能）
- 実験的機能として、有効化はオプトインとする

# その場しのぎ対応時のバックログ追加ルール

- **発見日**: 2025-12-14
- **発見フェーズ**: Construction
- **発見サイクル**: v1.4.0
- **優先度**: 中

## 概要

その場しのぎの対応をする際は、本質的な対応をバックログに追加することもセットにするルールを追加したい。

## 詳細

開発中に「今は時間がないから」「本質的ではないから」といった理由でworkaround（その場しのぎの対応）を行うことがある。このとき、本質的な解決策をバックログに記録せずに進めてしまうと、技術的負債が蓄積する。

workaround実施時に本質的な対応をバックログに記録することを必須化することで、技術的負債の可視化と将来的な対応を促進する。

## 対応案

- プロンプトに「workaround実施時のルール」セクションを追加
- workaroundを行う場合は以下をセットで実施：
  1. workaroundの実装
  2. 本質的な対応をバックログに記録（`chore-` または `refactor-` prefix）
  3. コード内にTODOコメント（バックログファイル名を参照）

# セットアッププロンプトパスの記録機能

- **発見日**: 2026-01-10
- **発見フェーズ**: Construction
- **発見サイクル**: v1.6.1
- **優先度**: 低

## 概要

セットアップ時に使用したセットアッププロンプトのパスを `docs/aidlc.toml` に記録し、アップグレード時に参照できるようにする。

## 詳細

現在、Operations Phase完了時のアップグレード案内では `[setup-promptのパス]` とプレースホルダーになっている。

ユーザーはセットアップ時にどのパスを使用したか覚えていない可能性があり、アップグレード時に迷う原因になる。

## 対応案

1. セットアップ時に使用したパスを `docs/aidlc.toml` の `[setup]` セクションに記録:
   ```toml
   [setup]
   prompt_path = "prompts/setup-prompt.md"  # または外部リポジトリのパス
   ```

2. Operations Phase完了時のメッセージで、このパスを参照して案内:
   ```
   **AI-DLCスターターキットをアップグレードする場合**: `prompts/setup-prompt.md` を読み込んでください。
   ```

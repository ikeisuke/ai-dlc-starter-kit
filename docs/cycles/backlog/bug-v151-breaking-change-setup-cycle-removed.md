# バグ: v1.5.1 で setup-cycle.md 削除による後方互換性の欠如

## 概要

v1.5.1 で `setup-cycle.md` が `setup.md` にリネームされた。
v1.5.0以前のプロジェクトで v1.5.1 の `setup-prompt.md` を使用し、アップグレードせずにサイクル開始しようとすると、存在しない `setup.md` を読み込もうとして失敗する。

## 状況

| バージョン | setup-cycle.md | setup.md |
|-----------|----------------|----------|
| v1.5.0以前 | あり | なし |
| v1.5.1 | なし | あり |

## 問題のシナリオ

1. v1.5.0以前のプロジェクトで `setup-prompt.md`（v1.5.1）を実行
2. ケースC「アップグレード可能」で「2. 現在のバージョンのまま続行」を選択
3. `setup-prompt.md` は `docs/aidlc/prompts/setup.md` を読み込もうとする
4. アップグレードしていないので `setup.md` は存在しない（`setup-cycle.md` のまま）
5. サイクル開始できない

## 対処方針

**スターターキット内の元ファイルを参照させる**:

`setup-prompt.md` で、プロジェクト内のファイル（`docs/aidlc/prompts/setup.md`）ではなく、スターターキット内の元ファイルを参照するよう修正:

```
# 修正前
docs/aidlc/prompts/setup.md

# 修正後
[スターターキットパス]/prompts/package/prompts/setup.md
```

これにより：
- プロジェクトがアップグレードしていなくても、スターターキット側に `setup.md` がある
- 常に最新のセットアップ手順が使われる

## 優先度

高 - アップグレードせずにサイクル開始しようとするユーザーが影響を受ける

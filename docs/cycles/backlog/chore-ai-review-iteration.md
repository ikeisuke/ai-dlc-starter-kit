# AIレビューの反復プロセス追加

- **発見日**: 2026-01-11
- **発見フェーズ**: Construction
- **発見サイクル**: v1.7.0
- **優先度**: 中

## 概要

AIレビュー（Codex MCP等）は一度だけでなく、修正後も再レビューして妥結させるプロセスを明文化する。

## 詳細

現状のAI-DLCプロンプトでは、AIレビューを実施して指摘を反映する流れは記載されているが、
「修正後に再度レビューして問題がないことを確認する」というループが明確に定義されていない。

品質向上のため、以下のフローを明文化すべき：

1. AIレビューを実施
2. 指摘事項を修正
3. 修正後に再度AIレビューを実施
4. 指摘がなくなるまで2-3を繰り返す
5. 人間レビューに進む

## 対応案

`prompts/package/prompts/construction.md` の「AIレビュー優先ルール」セクションに、
反復プロセスを追加する：

```markdown
**AIレビューフロー**:
1. AIレビューを実施
2. 指摘があれば修正を反映
3. **修正後に再度AIレビューを実施**（指摘が解消されるまで繰り返し）
4. 指摘がなくなったら人間レビューへ
```

# 論理設計: レビューフロー更新

## 概要

review-flow.mdの構造を再設計し、旧ツール別スキル呼び出しをレビュー種別ベースに変更する。また、`docs/aidlc.toml` の設定セクション名を `[rules.mcp_review]` → `[rules.reviewing]` にリネームし、関連ファイルの参照を更新する。

**重要**: この論理設計では**コードは書かず**、コンポーネント構成とインターフェース定義のみを行います。

## アーキテクチャパターン

プロンプトドキュメントのセクション構成。review-flow.mdの各セクションが処理ステップに対応する構造を維持しつつ、レビュー種別選択ステップを追加する。

### コンポーネント詳細

#### レビュー種別の決定（新規セクション）

- **責務**: 呼び出し元ステップに基づき、実行すべきレビュー種別を決定する
- **依存**: CallerContext（呼び出し元のステップ情報）
- **公開インターフェース**:
  - CallerContextマッピングテーブル
  - ユーザーへの選択提示（コンテキスト不明時）

#### tools設定と優先ツールヒント（改修セクション）

- **責務**: tools設定から優先ツールのヒントを決定し、スキル呼び出し引数に含める。**ツール選択の最終決定はスキル内部の責務**であり、このセクションは最善努力のヒントを提供するのみ
- **依存**: docs/aidlc.toml の [rules.reviewing].tools（旧 [rules.mcp_review]）
- **公開インターフェース**:
  - tools設定の読み取り方法
  - 優先ツールヒントの決定（リスト先頭を使用）
  - スキル呼び出し引数フォーマット定義

#### AIレビューツール利用可否の確認（改修セクション）

- **責務**: 新スキル（reviewing-*）の利用可否を確認する
- **依存**: Skill一覧
- **変更点**: MCPフォールバックロジックの完全削除。Skillsのみで判定する

#### ステップ5: AIレビューフロー（改修セクション）

- **責務**: レビュー種別ごとにスキルを呼び出し、反復レビューを実行する
- **依存**: レビュー種別の決定、tools設定と優先ツール
- **変更点**:
  - 種別ループの追加（複数種別時は直列実行）
  - スキル呼び出し方法の変更（skill="reviewing-[type]"）
  - 履歴記録にレビュー種別情報を含める

## 処理フロー概要

### AIレビュー実行の処理フロー

**ステップ**:

1. **mode確認**: aidlc.tomlからmode取得（既存と同一）
2. **レビュー種別決定**（新規）:
   - 呼び出し元ステップを確認
   - CallerContextマッピングテーブルからデフォルト種別を取得
   - コンテキスト不明時はユーザーに選択を求める
   - ユーザーが追加種別を指定した場合はマージ
3. **優先ツール決定**（改修）:
   - tools設定を読み取る
   - リスト先頭を優先ツールとする
4. **利用可否チェック**（改修）:
   - `skill="reviewing-[type]"` が利用可能か確認（MCPフォールバックなし）
5. **種別ごとのレビュー実行**（改修）:
   - レビュー種別リストを先頭から順に実行
   - 各種別で: `skill="reviewing-[type]", args="[対象] 優先ツール: [tool]"` を呼び出し
   - 反復レビュー（1セット最大3回）を種別ごとに独立実行
   - 全種別で指摘0件で「AIレビュー完了」
6. **履歴記録**（改修）:
   - 実行した全種別を記載（例: `【レビュー種別】code, security`）
   - 「AIレビュー完了」+「対象タイミング」の既存判定パターンは維持

### 有効なスキル名テーブル（新規）

| レビュー種別 | Skills呼び出し |
|---|---|
| code | `skill="reviewing-code"` |
| architecture | `skill="reviewing-architecture"` |
| security | `skill="reviewing-security"` |

### スキル呼び出し引数フォーマット

```text
skill="reviewing-[type]", args="[レビュー対象] 優先ツール: [codex|claude|gemini]"
```

**フォールバック**: 優先ツール指定はヒントであり強制ではない。スキルが解釈できない場合は `allowed-tools` の順序で自動選択。

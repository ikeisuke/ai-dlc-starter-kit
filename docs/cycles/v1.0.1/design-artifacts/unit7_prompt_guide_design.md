# 統合設計: プロンプト参照ガイド

## 概要
既存のフェーズプロンプトを活用する方法を明確にし、独自プロンプト作成を防ぐ仕組みを確立する。

---

## 1. プロンプト参照ガイドの構成

### ファイルパス（セットアップ時に生成）
`{{AIDLC_ROOT}}/prompts/prompt-reference-guide.md`

**注**: メタ開発のため、セットアップテンプレート（`prompts/setup/common.md`）にテンプレートを追加。
次回セットアップ時に生成される。

### 目次構成

1. **概要**
   - AI-DLCのフェーズ構成図
   - プロンプトファイルの役割説明

2. **クイックスタート**
   - 新規プロジェクト開始時のコマンド
   - 既存プロジェクト継続時のコマンド

3. **各フェーズプロンプトの使い方**
   - Inception Phase
   - Construction Phase
   - Operations Phase

4. **フェーズ間の移行方法**
   - Inception → Construction
   - Construction → Operations
   - Operations → 次サイクル Inception

5. **よくある間違いと対策**
   - 独自プロンプトを作成してしまう
   - 複数のフェーズプロンプトを同時に読み込む
   - コンテキストリセットをしない
   - サイクルディレクトリを間違える

6. **プロンプトのカスタマイズ方法**
   - additional-rules.md の活用
   - プロジェクト固有ルールの追加方法

7. **実行例（コピー＆ペースト可能）**
   - 各フェーズ開始時のプロンプト例

---

## 2. 注意書きの仕様

### 追加位置
各フェーズプロンプトの冒頭（タイトル直後）

### 注意書きの文面

```markdown
> **重要: このプロンプトについて**
> - これはAI-DLC Starter Kitの公式フェーズプロンプトです
> - 独自のプロンプトを作成せず、このファイルを直接読み込んでください
> - プロジェクト固有のルールは `docs/aidlc/prompts/additional-rules.md` に記載してください
> - 詳細は `docs/aidlc/prompt-reference-guide.md` を参照
```

### 対象ファイル（セットアップテンプレート）

**重要**: メタ開発のため、現在使用中のプロンプトではなく、セットアップテンプレートを更新する。
次回セットアップ時に生成されるプロンプトに注意書きが反映される。

| ファイル | 現状 | 更新内容 |
|---------|------|---------|
| `prompts/setup/inception.md` | 注意書きなし | テンプレート内に注意書き追加 |
| `prompts/setup/construction.md` | 注意書きなし | テンプレート内に注意書き追加 |
| `prompts/setup/operations.md` | 注意書きなし | テンプレート内に注意書き追加 |

## 4. 設計判断

1. **シンプルさ優先**: ガイドは簡潔に、実行例をコピペ可能に
2. **防止策は注意書きのみ**: 自動検証ツールは作成しない（Could-have）
3. **既存構造を維持**: プロンプトファイルの構造は変更せず、冒頭に注意書きを追加するのみ

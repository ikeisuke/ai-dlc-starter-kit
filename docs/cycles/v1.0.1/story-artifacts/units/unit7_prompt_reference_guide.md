# Unit: プロンプト参照ガイド

## 概要
既存のフェーズプロンプトを活用する方法を明確にし、独自プロンプト作成を防ぐ仕組みを確立する。

## 含まれるユーザーストーリー
- ストーリー 6.1: プロンプト参照ガイドの作成
- ストーリー 6.2: 独自プロンプト防止の仕組み

## 責務
- プロンプト参照ガイドの作成（`docs/aidlc/prompt-reference-guide.md`）
- 各フェーズプロンプトの使い方の文書化
- フェーズ間の移行方法の説明
- よくある間違いの列挙と対策
- セットアッププロンプトへのガイド参照案内の追加
- 各フェーズプロンプトへの注意書きの追加

## 境界
- プロンプトの自動生成は含まない
- プロンプトの自動検証ツールは含まない（Could-haveのため、実装は任意）
- プロンプトのバージョン管理の詳細は含まない（Unit 2で対応）

## 依存関係

### 依存する Unit
なし

### 外部依存
- 既存のフェーズプロンプト（inception.md、construction.md、operations.md）

## 非機能要件（NFR）
- **パフォーマンス**: 特別な要件なし
- **セキュリティ**: 特別な要件なし
- **スケーラビリティ**: 特別な要件なし
- **可用性**: ガイドが常にアクセス可能

## 技術的考慮事項
- **プロンプト参照ガイドの構成**:
  1. 概要（AI-DLCのフェーズ構成）
  2. 各フェーズプロンプトの使い方
     - Inception Phase: `docs/aidlc/prompts/inception.md` を読み込む
     - Construction Phase: `docs/aidlc/prompts/construction.md` を読み込む
     - Operations Phase: `docs/aidlc/prompts/operations.md` を読み込む
  3. フェーズ間の移行方法（コマンド例）
  4. よくある間違い：
     - 独自プロンプトを作成してしまう
     - 複数のフェーズプロンプトを同時に読み込む
     - コンテキストリセットをしない
  5. プロンプトのカスタマイズ方法（additional-rules.md の活用）
  6. 実行例（コピー＆ペースト可能）

- **独自プロンプト防止の仕組み**:
  - 各フェーズプロンプトの冒頭に注意書きを追加：
    ```markdown
    **重要**: このプロンプトは AI-DLC Starter Kit の公式フェーズプロンプトです。
    独自のプロンプトを作成せず、このファイルを直接読み込んでください。
    プロジェクト固有のルールは `docs/aidlc/prompts/additional-rules.md` に記載してください。
    ```
  - セットアップ完了メッセージにガイド参照を追加
  - プロンプト参照ガイドに防止策を記載

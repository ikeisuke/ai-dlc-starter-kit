# Unit: セットアップバグ修正

## 概要
AI-DLC Starter Kit v1.0.0で発見されたセットアップ時のバグ2件を修正し、セットアップの信頼性を向上させる。

## 含まれるユーザーストーリー
- ストーリー 1.1: セットアップ時のディレクトリ作成バグ修正
- ストーリー 1.2: 日付コマンド展開問題の検証と修正

## 責務
- セットアップ時に `inception/` ディレクトリが作成されるようにする
- 履歴記録時の日付展開が正しく動作することを検証し、必要に応じて修正する
- セットアッププロンプト（`prompts/setup-prompt.md`）の修正
- 各フェーズプロンプトの履歴記録指示の明確化

## 境界
- バージョンアップ時の差分適用には対応しない（Unit 2の範囲）
- 他のディレクトリ構造の変更は行わない
- セットアップ以外のバグ修正は含まない

## 依存関係

### 依存する Unit
なし

### 外部依存
- Bashコマンド（`mkdir -p`, `date`）
- Gitコマンド

## 非機能要件（NFR）
- **パフォーマンス**: セットアップ時間への影響は最小限（数秒以内）
- **セキュリティ**: 特別な要件なし
- **スケーラビリティ**: 特別な要件なし
- **可用性**: セットアップが100%成功すること

## 技術的考慮事項
- `mkdir -p` を使用して、親ディレクトリが存在しない場合でも作成可能にする
- `.gitkeep` ファイルを配置して、空ディレクトリをGit管理対象にする
- 日付取得は heredoc 外で実行し、変数に格納してから使用する方法を推奨
- セットアッププロンプトとフェーズプロンプトの一貫性を保つ

## 実装優先度
High（最優先）

## 見積もり
- 設計: 0.5時間
- 実装: 1時間
- テスト: 0.5時間
- 合計: 2時間

# ドメインモデル: Overconfidence Prevention原則の導入

## 概要

AIの過信防止原則を共通ルール（rules.md）に体系化する。既存の「予想禁止・一問一答質問ルール」を包含し、確信度判定の行動規範を明文化する。

**重要**: このドメインモデル設計では**コードは書かず**、概念構造と責務の定義のみを行います。ファイル配置・章構成は論理設計を参照。

## 概念エンティティ

### Overconfidence Prevention原則

- **責務**: AIエージェントが「いつ・なぜ」質問すべきかの判断基準を提供
- **構成要素**:
  - 基本原則（デフォルト動作の定義）
  - 質問トリガー（質問すべき場面の条件集合）
  - 質問フロー（質問の実行手順）
  - 品質評価基準（レッドフラグ・成功指標）

### 質問と回答の記録（既存・変更なし）

- **責務**: 不明点の記録仕様（[Question]/[Answer]タグ）
- **関係**: Overconfidence Prevention原則の判断に基づいて質問が発生 → この仕組みで記録される

## 判定モデル

AIが各ステップで自己評価する確信度判定の概念モデル:

| 状態 | 確信度 | 次のアクション |
|------|--------|-------------|
| 要件が明確 | 高 | 作業を継続（proceed） |
| 要件に曖昧さがある | 低〜中 | 質問する（ask_clarification） |
| 複数の解釈が可能 | 低 | 質問する（ask_clarification） |
| 矛盾を検出 | 低 | 質問する（ask_clarification） |
| 前提条件が不明 | 低 | 質問する（ask_clarification） |
| 曖昧な回答を受領 | 低 | 追加質問する（ask_clarification） |

**デフォルト動作**: 確信度が判定できない場合は「質問する」（安全側に倒す）

## ユビキタス言語

- **Overconfidence（過信）**: AIが確信度が低い状態で推測・仮定に基づいて行動すること
- **レッドフラグ**: 過信が発生している兆候を示す行動パターン
- **成功指標**: 適切な質問行動が行われている状態を示す指標
- **ハイブリッド方式**: 質問の概要を先に提示し、1問ずつ順番に詳細を質問する方式
- **確信度（Confidence Level）**: AIが現在のコンテキストに対してどの程度明確に理解しているかの自己評価

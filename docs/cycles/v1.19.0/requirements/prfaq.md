# PRFAQ: AI-DLC v1.19.0 - Amazon AIDLCエッセンス取り込み + jjサポート整理

## Press Release（プレスリリース）

**見出し**: AI-DLC v1.19.0 - AIの判断精度向上と開発ワークフロー効率化を同時に実現

**副見出し**: 過信防止原則・成果物詳細度制御・セッション継続性の3つの新概念を導入し、AIエージェントとの協調開発をより信頼性の高いものに

**発表日**: 2026-03

**本文**:

AI-DLCスターターキットは、AIエージェントと開発者の協調開発を体系化するフレームワークです。v1.19.0では、Amazon AIDLCリポジトリ（awslabs/aidlc-workflows）の調査から得られた知見を取り込み、AIの判断品質とワークフロー効率の両面を強化します。

**Overconfidence Prevention原則**は、AIエージェントが不確実な判断を行う際に推測ではなく質問を優先する行動規範を共通ルールとして明文化します。これにより、AIの誤った推測に基づく実装ミスや手戻りを未然に防止します。

**Depth Levels**は、タスクの複雑度に応じて成果物の詳細度をminimal/standard/comprehensiveの3段階で制御する仕組みです。シンプルなバグ修正にフルのInception Phaseを適用する非効率を解消し、複雑な機能開発では十分な検討を確保します。

**Session Continuity**は、セッション中断時の作業状態を自動保存し、再開時にコンテキスト喪失なく作業を継続できる正式サポートです。長時間の開発タスクにおけるセッション中断の影響を最小化します。

加えて、既存コードベースの体系的解析（Reverse Engineering）をInception Phaseに組み込み、brownfieldプロジェクトへの適用をより構造的にサポートします。

コードベース整理として、別リポジトリに移植済みのjjサポート関連処理を本体から削除し、保守負担を軽減しました。

「v1.19.0の目標は、AIエージェントがより信頼性の高い判断を行い、開発者がプロジェクトの規模に応じた適切な成果物を効率的に作成できる環境を提供することです」

## FAQ（よくある質問）

### Q1: Overconfidence Prevention原則を導入すると、AIの応答が遅くなりませんか？
A: 質問の頻度は若干増える可能性がありますが、推測による誤った実装と手戻りの削減により、全体の開発効率は向上すると期待されます。既存の「予想禁止・一問一答質問ルール」を体系化したものであり、新たな制約ではなく既存ルールの明確化です。

### Q2: Depth Levelsのデフォルトは何ですか？既存プロジェクトへの影響はありますか？
A: デフォルトは`standard`であり、未設定時は現行動作と完全に同等です。`aidlc.toml`に`[rules.depth_level]`を追加しない限り、既存のワークフローに変更は生じません。

### Q3: Session Continuityは既存のcompaction対応やコンテキストリセットと何が違いますか？
A: 既存のcompaction対応・コンテキストリセットを発展させた上位機能です。session-state.mdにサイクル番号・フェーズ・ステップ状態等を包括的に記録し、より確実な状態復元を実現します。既存のprogress.mdによる復元はフォールバックとして維持されます。

### Q4: jjサポートが削除されると、jjを使っている環境はどうなりますか？
A: `docs/aidlc.toml`の`[rules.jj]`設定キーは残ります。`enabled = true`の場合は「jjサポートはv1.19.0で削除されました。versioning-with-jjスキルを別途インストールしてください」と警告が表示されます。別リポジトリからスキルをインストールすることで引き続きjjを利用できます。

### Q5: Reverse Engineering機能はgreenfieldプロジェクトでも実行されますか？
A: いいえ。既存のConditional分岐を維持しており、greenfieldプロジェクトではステップ2がスキップされます。brownfieldプロジェクトでのみ、体系的な解析手順が実行されます。

# PRFAQ: AI-DLC v1.26.0 — 設定の柔軟化と品質基盤の強化

## Press Release（プレスリリース）

**見出し**: AI-DLC v1.26.0 — プロジェクトごとの設定柔軟化でAIエージェントの品質と使い勝手を向上

**副見出し**: バックログモード別バリデーション追加、プリフライトチェックとSelf-Healingリトライ回数の設定化により、プロジェクト特性に応じたカスタマイズが可能に。

**発表日**: 2026-03-20

**本文**:

AI-DLCフレームワークを利用する開発者から「バックログモードに応じた成果物形式の制約がなく、AIが誤った形式で保存してしまう」「ghが不要なプロジェクトでもプリフライトチェックが走る」「テストスイートが重いプロジェクトでSelf-Healingのリトライ回数を減らしたい」という声が寄せられていました。

v1.26.0では3つの改善を行いました。まず、バックログモード（git, issue, git-only, issue-only）ごとに許可・禁止される成果物形式を排他的バリデーションルールとして`rules.md`に明文化しました。次に、`aidlc.toml`の`[rules.preflight]`セクションでプリフライトチェック項目のon/offを制御可能にしました。最後に、`rules.construction.max_retry`でSelf-Healingループの最大リトライ回数を設定可能にしました。

「issue-onlyモードでローカルファイルに保存するミスがなくなった。設定も`aidlc.toml`を数行変えるだけで済むので、導入コストが低い」（想定利用者の声）

今後も`aidlc.toml`の設定項目拡充を通じて、プロジェクト特性に応じたカスタマイズ性の向上を継続していきます。

## FAQ（よくある質問）

### Q1: 設定を何も変更しなくても既存の動作は維持されますか？
A: はい。すべての新規設定キーにはデフォルト値が設定されており、未設定時は従来と同じ動作をします。`rules.preflight.checks`未設定時は全項目チェック、`max_retry`未設定時は3回です。

### Q2: プリフライトチェックを完全に無効化できますか？
A: `enabled=false`に設定するとオプションチェック（gh, review-tools, config-validation）はスキップされますが、blockerチェック（git, aidlc.toml）は安全性のため常時実行されます。

### Q3: Self-Healingのリトライ回数を0に設定するとどうなりますか？
A: `max_retry=0`の場合、Self-Healingループをスキップし、即座にユーザー判断フォールバックに遷移します。

### Q4: 変更対象はどのファイルですか？
A: プロンプトファイル（`prompts/package/`配下の`rules.md`, `preflight.md`, `construction.md`）と設定ファイル（`docs/aidlc.toml`）のみです。スクリプトの変更はありません。

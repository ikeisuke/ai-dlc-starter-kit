### ホームディレクトリにユーザー共通設定を配置可能にする
- **元サイクル**: v1.2.1
- **延期日**: 2025-12-13
- **延期理由**: 現時点では単一プロジェクトでの利用が主であり、優先度が低い
- **概要**: `~/.aidlc/` などにユーザー固有の共通設定を配置できるようにする
- **詳細**:
  - 現状: 設定はプロジェクトごとに `docs/aidlc/` 配下に配置
  - 要望: 複数プロジェクトで共通して使いたい個人設定（命名規則、コーディングスタイル等）をホームディレクトリに配置

  - 読み込み優先順位: ホーム設定 → プロジェクト設定（プロジェクト設定で上書き可能）
- **期待効果**: 複数プロジェクトでの設定の一貫性、セットアップ時間の短縮
- **再開条件**: 複数プロジェクトでAI-DLCを使用するユースケースが増えた時点

### 作業中の割り込み対応ルールの明確化
- **発見日**: 2025-12-11
- **発見サイクル**: v1.3.1 Inception Phase
- **概要**: 作業中にユーザーから追加要望があった場合、適切に分類して対応すべき
- **詳細**:
  - 現状: AIが割り込み要望に対して即時実装に寄りがち
  - 問題: 全ての要望を即時実装すると計画が崩れる
  - 要望の分類:
    1. 関係ない案件 → バックログに記録
    2. 関係あるけど今じゃない案件 → バックログに記録
    3. 関係あるのでUnitには追加するけど今すぐ実行しない案件 → Unit定義に追加
    4. 関係あるし前提なのですぐにやる案件 → 即時実装
- **対応案**: Construction Phaseのプロンプトに割り込み対応フローを追加
- **優先度**: 中

### Inception Phase内にセットアップステップを統合する検討
- **発見日**: 2025-12-13
- **発見サイクル**: v1.3.2 Operations Phase
- **概要**: セットアップを経由せずにInception Phaseを直接開始するとブランチ作成ステップがスキップされる問題
- **詳細**:
  - 現状: セットアップ（setup-cycle.md）でブランチ作成を提案するが、v1.3.1で「セットアップスキップ機能」を追加
  - 問題: 直接Inception Phaseを開始すると、ブランチ作成の提案がなく、mainで直接作業してしまう
  - v1.3.2で実際に発生: サイクル全体をmainブランチで作業し、PRなしで直接push
- **対応案**:
  1. Inception Phaseの最初にセットアップ相当のステップ（ブランチ確認・作成）を追加
  2. setup-cycle.mdの内容をInception Phaseの「最初に必ず実行すること」に統合
  3. setup-prompt.mdは初回セットアップ/アップグレード専用にする
- **期待効果**: セットアップ経由でもInception直接開始でも、ブランチ作成が確実に提案される
- **関連**: v1.3.1で追加された「セットアップスキップ機能」との整合性を考慮
- **優先度**: 中

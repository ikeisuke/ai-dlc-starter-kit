## 技術的考慮事項

- prompts/package/prompts/operations.md を編集（591-623行目付近のステップ6.0セクションを削除）
- docs/cycles/rules.md のカスタムワークフローセクションに追加
- rules.mdはプロジェクト固有なので `project.name` の条件判定は不要

### カスタムワークフローの実行コンテキスト

- **実行トリガー**: AIがOperations Phase実行時にrules.mdを読み込み、カスタムワークフローセクションの指示に従って実行する（プロンプト指示ベース）
- **前提コンテキスト**: Operations Phase実行中で `{{CYCLE}}` が確定していること
- **失敗時の扱い**: コマンド実行エラー時はユーザーに表示し、手動対応を促す（標準フローは継続）
- **スキップ対象外の根拠**: operations.mdのステップ0に「rules.mdのカスタムワークフローはスキップ対象外」の注記が存在する

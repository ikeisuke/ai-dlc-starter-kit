# PRFAQ: AI-DLC Starter Kit v1.25.0

## Press Release（プレスリリース）

**見出し**: AI-DLC v1.25.0: 「start express」一言で小規模修正を即開始、プリフライトチェックでセッション中断ゼロへ

**副見出し**: エクスプレスモードのインスタント起動、フェーズ開始時の環境自動検証、PRレビュー自動化の強化により、AI駆動開発のフリクションを大幅削減

**発表日**: 2026年3月

**本文**:

AI-DLCスターターキットを使った開発では、小規模なバグ修正でも aidlc.toml の設定変更が必要でした。また、セッション途中でツール認証が切れて作業が中断される問題や、設定値の読み飛ばしによるワークフロー不整合も発生していました。

v1.25.0では、「start express」と一言指示するだけでエクスプレスモードが起動し、設定ファイルの変更なしにInceptionからConstructionまで一気通貫で小規模修正を完了できます。さらに、各フェーズ開始時にツール存在・認証状態の自動チェックと主要設定値の一覧表示を行うプリフライトチェック機構を導入し、セッション途中の環境起因の中断を未然に防止します。

加えて、Codex PRレビューの絵文字リアクション（👍/👀）による完了判定を自動化し、PRマージ前ゲートの精度を向上させました。スキル基盤も .kiro/skills から .agents/skills に移行し、将来の拡張に備えた整理を実施しています。

「v1.25.0にアップグレードしてから、小規模修正の着手が格段に速くなりました。以前は設定変更→Inception→設定戻しの3ステップが必要でしたが、今は start express の一言で完了します」（想定ユーザーの声）

今後は、プリフライトチェック項目の設定化や、エクスプレスモードのさらなる効率化を検討していきます。

## FAQ（よくある質問）

### Q1: 「start express」を使うと aidlc.toml の設定が変わりますか？
A: いいえ。セッション内限定のオーバーライドとして動作するため、aidlc.toml の設定値は一切変更されません。セッション終了後は通常の設定が維持されます。

### Q2: プリフライトチェックに失敗した場合、フェーズは開始できますか？
A: チェック失敗時はフェーズ開始が中断されます。対処後に再チェックを実行し、成功すればフェーズを開始できます。「中止」を選択してフェーズ開始を取りやめることも可能です。

### Q3: .kiro/skills から .agents/skills への移行は破壊的変更ですか？
A: はい、破壊的変更です。v1.25.0へのアップグレード後、旧 .kiro/skills ディレクトリは手動で削除する必要があります。アップグレード手順はCHANGELOGに記載されます。既存の .agents/skills がある場合はバックアップが自動作成されます。

### Q4: Codex PRレビューの絵文字リアクション検出は必須ですか？
A: 既存のコメントベース判定へのフォールバックが組み込まれているため、リアクション検出に失敗しても従来通りの判定が行われます。リアクション検出はPRマージ前ゲートの精度向上のための追加機能です。

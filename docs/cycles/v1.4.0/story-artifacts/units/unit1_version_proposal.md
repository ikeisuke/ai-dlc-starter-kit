# Unit 1: サイクルバージョン提案改善

## 概要
既存サイクルから次バージョンを自動推測し、バージョン番号の入力ミスを防ぐ

## 含まれるユーザーストーリー
- ストーリー2: サイクルバージョン提案改善

## 責務
- `docs/cycles/` 内の既存サイクルを検出する
- 最新バージョンから次バージョンを推測する（例: v1.3.2 → v1.3.3 または v1.4.0）
- package.json等を見るのは初回AI-DLC導入時のみに限定

## 境界
- バージョン命名規則の変更は扱わない
- セマンティックバージョニングの厳密な適用は扱わない

## 技術的考慮事項
- `setup-prompt.md` と `setup-cycle.md` を編集
- バージョン文字列のパース処理（v1.3.2 → major=1, minor=3, patch=2）
- パッチバージョンのインクリメントをデフォルトとし、メジャー/マイナー変更はユーザー確認

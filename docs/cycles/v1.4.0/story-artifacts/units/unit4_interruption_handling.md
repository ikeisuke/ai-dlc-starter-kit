# Unit 4: 割り込み対応ルール

## 概要
作業中の割り込み要望を適切に分類し、計画を崩さずに追加要望を管理する

## 含まれるユーザーストーリー
- ストーリー5: 割り込み対応ルール

## 責務
- 割り込み要望の分類フローを定義する
- Construction Phaseプロンプトに追加

## 境界
- Inception Phase、Operations Phaseでの割り込み対応は扱わない（将来拡張）
- 複数人での同時作業時の調整は扱わない

## 技術的考慮事項
- `construction.md` を編集
- 分類フロー:
  1. 関係ない案件 → バックログに記録
  2. 関係あるけど今じゃない案件 → バックログに記録
  3. 関係あるのでUnitに追加するが今すぐ実行しない案件 → Unit定義に追加
  4. 関係あるし前提なのですぐにやる案件 → 即時実装

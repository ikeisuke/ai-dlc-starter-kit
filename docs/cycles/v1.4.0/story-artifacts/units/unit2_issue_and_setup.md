# Unit 2: GitHub Issue確認とセットアップ統合

## 概要
Inception Phase開始時にGitHub Issueを確認し、ブランチ作成を確実に提案する

## 含まれるユーザーストーリー
- ストーリー3: GitHub Issue確認
- ストーリー4: セットアップステップ統合

## 責務
- GitHub CLIでオープンなIssueを取得・表示する
- Inception Phaseの「最初に必ず実行すること」にブランチ確認を追加
- 現在のブランチがmain/masterの場合、サイクル用ブランチ作成を提案

## 境界
- Issue自体の管理（作成、更新、クローズ）は扱わない
- setup-cycle.md の廃止は扱わない（重複を整理するのみ）

## 依存関係

### 依存する Unit
なし

### 外部依存
- GitHub CLI（gh コマンド）

## 非機能要件（NFR）
- **パフォーマンス**: GitHub API呼び出しは5秒以内にタイムアウト
- **セキュリティ**: 該当なし
- **スケーラビリティ**: 該当なし
- **可用性**: GitHub CLI未認証時はスキップ

## 技術的考慮事項
- `inception.md` のステップ2.5付近にGitHub Issue確認を追加
- ステップ1（サイクル存在確認）にブランチ確認を追加
- Dependabot PR確認と同様の形式で表示

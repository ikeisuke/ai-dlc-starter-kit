# Intent（開発意図）

## プロジェクト名
AI-DLC Starter Kit v1.4.0

## 開発の目的
バックログに蓄積された低優先度タスク9件を一括対応し、AI-DLCスターターキットの完成度と使いやすさを向上させる。

## ターゲットユーザー
- AI-DLCスターターキットを使用する開発者
- AI-DLCを導入して開発プロセスを改善したいチーム

## ビジネス価値
- 運用上の細かな不便を解消し、開発者体験を向上
- プロンプトの構造改善により、AIの動作精度を向上
- 複数人開発やCI/CD連携など、実運用シナリオへの対応強化

## 成功基準
- 低優先度バックログ9件すべての対応完了
- 各機能が正常に動作することの確認
- ドキュメントの整合性維持

## 期限とマイルストーン
AI-DLCの原則に従い、具体的な期限は設定しない。各Unitを順次完了させる。

## 制約事項
- `docs/aidlc/` は直接編集禁止（`prompts/package/` を編集）
- 既存の動作を壊さないこと（後方互換性の維持）
- メタ開発の意識を常に持つこと

## 対応するバックログ項目

### 低優先度タスク（9件 + 新規1件 = 10件）
1. git worktree を活用したワークフローの提案
2. history.mdの複数人開発時コンフリクト対策
3. サイクルバージョン提案ロジックの改善
4. Operations Phaseでアプリケーションバージョン更新を確認する
5. 別のAI MCPが設定されている場合にレビュー活用を提案する
6. Operations Phase「完了時の必須作業」の構造改善
7. 作業中の割り込み対応ルールの明確化
8. セットアップ時にGitHub Issueもチェックする機能を追加
9. Inception Phase内にセットアップステップを統合する検討
10. backlog.mdの複数人開発時コンフリクト対策（新規追加）

### 中優先度タスク（整理）
- ホームディレクトリにユーザー共通設定を配置可能にする → **延期タスクに移動**
- セットアップ時のファイルコピー判定にハッシュ値を使用 → **対応済み（rsync方式で解消）**

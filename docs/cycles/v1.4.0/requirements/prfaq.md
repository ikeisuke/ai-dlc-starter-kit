# PRFAQ: AI-DLC Starter Kit v1.4.0

## Press Release（プレスリリース）

**見出し**: AI-DLC Starter Kit v1.4.0 - 開発者体験の向上と複数人開発対応

**副見出し**: バックログ消化により、セットアップ体験、フェーズ実行、複数人開発対応が大幅に改善

**発表日**: 2025年12月

**本文**:

AI-DLCスターターキットは、AIを開発プロセスの中心に据えた新しい開発方法論の実践キットです。v1.4.0では、蓄積されたバックログ10件を一括対応し、日々の開発体験を大幅に向上させました。

**改善のハイライト**:

- **セットアップの自動化**: サイクルバージョンの自動推測、GitHub Issue確認、ブランチ作成の確実な提案により、サイクル開始時の手動作業が削減されます。

- **フェーズ実行の改善**: 作業中の割り込み要望を適切に分類するルールを追加。AI MCPが利用可能な場合はレビュー活用も提案されます。

- **Operations Phaseの構造化**: 完了作業がステップ6（リリース準備）として独立し、何をすべきかが明確になりました。

- **複数人開発対応**: history.mdとbacklog.mdのコンフリクト対策を導入し、チーム開発がよりスムーズになります。

「v1.4.0で細かな不便が解消され、AI-DLCの本質である『AI主導の短サイクル開発』に集中できるようになりました」と開発チームはコメントしています。

## FAQ（よくある質問）

### Q1: v1.4.0へのアップグレード方法は？
A: `setup-prompt.md` を読み込んでアップグレードオプションを選択してください。rsyncで差分のみが更新されます。

### Q2: 既存のサイクルに影響はありますか？
A: 既存のサイクルには影響しません。新しいサイクルから改善された機能が利用できます。

### Q3: git worktreeは必須ですか？
A: いいえ、提案されるだけで使用は任意です。通常のブランチ切り替えも引き続き利用できます。

### Q4: AI MCPとは何ですか？
A: Model Context Protocol（MCP）サーバーを通じて別のAIエージェント（例: Codex MCP）を利用する仕組みです。設定されていれば成果物のレビューに活用できますが、必須ではありません。

### Q5: 複数人開発時のコンフリクト対策はどのように機能しますか？
A: history.mdはGit履歴での代替を推奨し、backlog.mdは追記ルールを明確化することでコンフリクトを減らします。詳細はConstruction Phaseで設計されます。

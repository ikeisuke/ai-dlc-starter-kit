### 変更詳細

#### 1. ステップ0: ブランチ確認【推奨】

- 現在のブランチを `git branch --show-current` で確認
- main/master の場合、サイクル用ブランチ（`cycle/{{CYCLE}}`）の作成を提案
- ユーザーが続行を選択した場合は警告を表示

#### 2. ステップ2.7: GitHub Issue確認

- `gh issue list --state open --limit 10` でオープンなIssueを取得
- GitHub CLI未認証時はスキップ
- Issueがある場合、今回のサイクルで対応するか確認
- Dependabot PR確認（ステップ2.5）と同様のパターン

## 設計上の決定

### ブランチ確認の二重化

- `setup-cycle.md` と `inception.md` の両方でブランチ確認を行う
- setup-cycle.md 経由の場合: 両方でチェック（冗長だが安全）
- inception.md 直接の場合: フォールバックとして機能

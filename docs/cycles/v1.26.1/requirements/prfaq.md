# PRFAQ: AI-DLC Starter Kit v1.26.1

## Press Release（プレスリリース）

**見出し**: AI-DLC v1.26.1 - Operations Phaseの品質保証強化とツール基盤の整理

**副見出し**: リリース前の品質チェックを体系化し、設定管理の保守性を向上

**発表日**: 2026-03-21

**本文**:

AI-DLCスターターキット v1.26.1では、Operations Phaseのリリース品質保証を強化し、ツール基盤のデフォルト値管理を整理しました。

これまでOperations Phaseでは、外部レビューツール（Codex GitHub連携等）がrate limitで利用できない場合の代替手段が明確でなく、パーミッション設定の監査も手動で実施していました。また、設定読み込みスクリプト（read-config.sh）では--defaultオプションとdefaults.tomlの二重管理が発生しており、プロンプトの保守性に課題がありました。

v1.26.1では以下の改善を行いました:
- ローカルレビュー手順をoperations-release.mdの標準ステップとして組み込み
- パーミッション設定の分析・監査をカスタムワークフローに追加
- Bash Substitution Checkをリポジトリ固有ルールに移動（他プロジェクトでのエラー防止）
- read-config.shの--defaultオプションを廃止し、デフォルト値をdefaults.tomlに一元化
- プリフライト設定取得をバッチモード1回に集約

これにより、リリース品質の一貫性が向上し、フレームワークの保守が容易になります。

## FAQ（よくある質問）

### Q1: --default廃止による後方互換性への影響は？
A: defaults.tomlに全デフォルト値が集約されているため、動作上の変更はありません。--defaultを使用していたプロンプトからオプションを除去するだけで移行完了です。

### Q2: 他プロジェクトでBash Substitution Checkを使用したい場合は？
A: check-bash-substitution.shをプロジェクトに配置し、rules.mdのカスタムワークフローに追加してください。共通operations-release.mdからは削除されましたが、プロジェクト固有ルールとして設定可能です。

### Q3: パーミッション管理ステップは必須ですか？
A: rules.mdのカスタムワークフローとして定義されるため、AI-DLCスターターキット固有の手順です。他プロジェクトでは不要であれば追加する必要はありません。

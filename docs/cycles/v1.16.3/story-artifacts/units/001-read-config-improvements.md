# Unit: read-config.sh 改善

## 概要
read-config.sh のデフォルト値を全キーに設定し、複数キーの一括読み取り機能を追加する。

## 含まれるユーザーストーリー
- ストーリー 1: read-config.sh のデフォルト値統一 (#221-1)
- ストーリー 2: read-config.sh の一括読み取り (#221-2)

## 責務
- defaults.toml への不足キー追加（`rules.commit.ai_author`, `rules.branch.mode` を含む全キーのデフォルト値定義）
- read-config.sh への --keys オプション実装
- key:value 形式の一括出力対応

## 境界
- TOML パーサー自体の変更やdasel以外への移行は行わない
- read-config.sh の既存の4階層マージロジックは変更しない

## 依存関係

### 依存する Unit
- なし（基盤ツールの改善のため、他のUnitに先行して実施）

### 外部依存
- dasel（既存依存、変更なし）

## 非機能要件（NFR）
- **パフォーマンス**: 一括読み取り時、個別呼び出しN回と同等以下の実行時間
- **セキュリティ**: 該当なし
- **スケーラビリティ**: キー数の増加に対して線形スケール
- **可用性**: 該当なし

## 技術的考慮事項
- defaults.toml は read-config.sh の階層0で既に読み込まれているため、キー追加のみで対応
- --keys オプションは既存の引数パース部分を拡張
- --keys と位置引数の排他チェックを追加

# Unit: コンテキストリセット改善

## 概要
Unit/Phase完了時のコンテキストリセット指示が省略されないよう完了フローに組み込み、リセット告知に次セッション向けのサマリを追加する。

## 含まれるユーザーストーリー
- ストーリー 3: コンテキストリセット指示の省略防止 (#209) 【Must-have、先行実装】
- ストーリー 4: リセット告知にサマリ追加 (#220) 【Should-have、ストーリー3完了後に実装】

## Unit内の実装順序
1. まずストーリー3（リセット指示の省略防止）を完了する
2. その後ストーリー4（サマリ追加）を実装する
3. ストーリー3のみで完了判定可能（ストーリー4は追加価値）

## 責務
- construction.md の Unit完了フローにリセット提示を番号付き必須ステップとして組み込み
- inception.md / operations.md の Phase完了時にリセット提示を番号付き必須ステップとして組み込み
- 完了チェックリストにリセット提示を必須項目として追加
- リセット告知メッセージに3行サマリ（完了作業・リポジトリ状態・次アクション）の生成指示を追加

## 境界
- context-reset.md（ユーザー主導のリセット処理）は変更しない
- コンテキストリセット後のセッション自動再開は実装しない

- サマリ生成はAIが動的に行うため、プロンプトに生成指示とフォーマットを記載
- サマリの必須項目: (1) サイクル番号+完了作業名、(2) ブランチ名+PR状態、(3) 次のアクション指示

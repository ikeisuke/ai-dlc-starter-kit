# PRFAQ: AI-DLC Starter Kit v1.16.3

## Press Release（プレスリリース）

**見出し**: AI-DLC v1.16.3: 設定基盤の堅牢化とセッション管理の改善

**副見出し**: 設定読み取りの安定性向上、セッション間の情報継続性改善、レビューフローの柔軟性向上を実現

**発表日**: 2026-02-22

**本文**:

AI-DLCスターターキットは、AIを開発の中心に据えた開発手法の実践キットです。v1.16.3では、日常的な開発ワークフローで感じていた3つの課題を解決します。

まず、設定読み取りスクリプト（read-config.sh）のデフォルト値を全キーに統一し、一括読み取り機能を追加しました。これにより、プロンプトでの `--default` 指定漏れによるエラーが根絶され、フェーズ開始時の設定読み取りが1コマンドで完了します。

次に、Unit/Phase完了時のコンテキストリセット指示が確実に提示されるよう、完了フローの構造を改善しました。さらに、リセット告知に完了作業・リポジトリ状態・次アクションの3行サマリが追加され、次のセッションでの作業再開がスムーズになります。

また、ブランチ作成方式を設定で固定できるようになり、毎サイクル同じ選択をする手間がなくなりました。レビューフローでは、外部ツール未設定時にセルフレビューが自動実行されるフォールバック機能を追加し、どの環境でもレビュー品質を維持できます。

「サイクルを重ねるたびに感じていた小さなストレスが、v1.16.3で一気に解消されました。特にリセット時のサマリは、次のセッションの立ち上がりが格段に速くなります」と利用者は語ります。

今後も、AI-DLCの開発体験をさらに向上させる改善を継続していきます。

## FAQ（よくある質問）

### Q1: read-config.sh の一括読み取りはどのような形式で出力されますか？
A: env-info.sh と同じ `key:value` 形式で、1行に1キーずつ出力されます。例: `rules.reviewing.mode:required`

### Q2: ブランチ作成方式の設定を変更しても、以前の動作に戻せますか？
A: はい。`[rules.branch].mode = "ask"` に設定するか、設定を削除すれば従来通り毎回質問される動作に戻ります。

### Q3: セルフレビューは外部ツールによるレビューと同じ品質ですか？
A: セルフレビューは外部ツールに比べて品質が劣る可能性があります。そのため、履歴に「セルフレビュー」であることが明記され、外部ツールが利用可能な場合は従来通り外部ツールが優先されます。

### Q4: リセット告知のサマリはどのような内容ですか？
A: 3行で構成されます。(1) サイクル番号と完了作業名、(2) ブランチ名とPR状態、(3) 次のアクション指示。セッション固有の実際の値が含まれます。

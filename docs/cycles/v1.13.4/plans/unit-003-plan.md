# Unit 003 計画: claude-reviewスキルの不安定動作調査・対策

## 概要

claude-reviewスキル（`claude -p`）の不安定動作（レスポンス未返却、指摘の二転三転）について、原因を調査し対策を実装またはワークアラウンドを明文化する。

## 変更対象ファイル

### 調査成果物（新規作成）

- `docs/cycles/v1.13.4/design-artifacts/domain-models/claude-review-stability_domain_model.md`
- `docs/cycles/v1.13.4/design-artifacts/logical-designs/claude-review-stability_logical_design.md`

### 実装対象（変更の可能性あるファイル）

- `prompts/package/skills/claude-review/SKILL.md` - スキル設定の改善・ワークアラウンド追記
- `docs/cycles/v1.13.4/construction/units/claude-review-stability_implementation.md` - 実装記録

## 実装計画

### Phase 1: 設計（調査・分析）

#### ステップ1: ドメインモデル設計（原因調査）

1. **不安定動作の分類と整理**:
   - レスポンス未返却: タイムアウト、プロセスハング、出力バッファの問題など
   - 指摘の二転三転: モデルの非決定性、コンテキスト不足、プロンプト曖昧さなど
2. **再現試行**（最大5回）:
   - `claude -p` でレビュー実行し、不安定動作の再現を試みる
   - `--output-format stream-json` オプションの効果を検証
3. **原因分類**:
   - スキル設定の問題（SKILL.md、パラメータ不足等）
   - Claude Code CLI側の問題（バージョン、オプション等）
   - モデル側の問題（非決定性、温度パラメータ等）

#### ステップ2: 論理設計（対策方針）

- 調査結果に基づき、対策の方向性を決定:
  - スキル設定で改善可能 → SKILL.mdの改善案を設計
  - CLI側の問題 → ワークアラウンドを設計
  - モデル側の問題 → ワークアラウンド（リトライ戦略等）を設計

#### ステップ3: 設計レビュー

- AIレビュー + ユーザー承認

### Phase 2: 実装

#### ステップ4: コード生成

- 設計に基づきSKILL.mdの改善またはワークアラウンドの明文化を実装

#### ステップ5: テスト生成

- 対策の検証（`claude -p` での動作確認）

#### ステップ6: 統合とレビュー

- AIレビュー + ユーザー承認
- 実装記録の作成

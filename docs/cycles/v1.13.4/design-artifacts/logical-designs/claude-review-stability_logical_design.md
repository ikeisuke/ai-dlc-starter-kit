# 論理設計: claude-reviewスキルの不安定動作調査・対策

## 概要

調査結果に基づき、claude-reviewスキル（SKILL.md）の安定性を改善するための変更方針と、モデル非決定性に対するワークアラウンドを定義する。

**重要**: この論理設計では**コードは書かず**、変更方針とインターフェース定義のみを行います。

## アーキテクチャパターン

ドキュメント改善パターン: 既存のSKILL.mdに対策オプションとワークアラウンドを追記し、利用者が安定した動作を得られるようにする。

- **責務**: Claude Code CLIによるAIレビュー実行手順の定義

## インターフェース設計

### 変更1: 実行コマンドの改善（SYMPTOM-001対策）

**現在**:

```bash
claude -p "<レビュー指示>"
```

**変更後**:

```bash
claude -p --output-format stream-json "<レビュー指示>"
```

**変更理由**:
- `--output-format stream-json` により処理途中のイベントが逐次出力される
- 長文ファイルのレビューでも無反応に見える状態を回避
- 再現試行で効果を確認済み（試行2: 37KBファイルが約131秒で正常完了）

**セッション継続コマンドも同様に変更**:

```bash
claude --session-id <uuid> -p --output-format stream-json "<追加指示>"
```

### 変更2: 反復レビュー時のワークアラウンド（SYMPTOM-002対策）

**追記内容**: 「既知の制限事項と対処法」セクションを新設

1. **指摘の非決定性について**:
   - LLMは同一入力に対して毎回異なる出力を生成する性質がある
   - 特に「言い回しの改善」など低重要度の指摘は毎回変わりやすい
2. **推奨対処法**:
   - 構造的な問題や論理的な誤りなど**重要度の高い指摘**を優先的に対応する
   - 言い回しレベルの指摘は、一貫して指摘されない場合は対応不要と判断してよい
   - セッション継続（`--session-id`）を使用し、前回の指摘コンテキストを維持する

### 変更3: stream-json出力の解釈ガイド

**追記内容**: stream-json出力の取り扱い方法

- stream-json出力はJSON Lines形式
- 最終行の `type: "result"` にレビュー結果が含まれる
- 呼び出し元（AI-DLCの反復レビューフロー等）は最終結果を取得して処理する

## 処理フロー概要

### レビュー実行フロー（改善後）

**ステップ**:
1. `claude -p --output-format stream-json "<レビュー指示>"` を実行
2. stream-jsonイベントが逐次出力される（無反応にならない）
3. 処理完了時に `type: "result"` イベントが出力される
4. 結果を確認し、指摘があれば修正
5. 再レビュー時は `--session-id` でセッション継続

**関与するコンポーネント**: SKILL.md、Claude Code CLI

## 非機能要件（NFR）への対応

### パフォーマンス

- **要件**: レビュー実行が安定して結果を返すこと
- **対応策**: `--output-format stream-json` によりストリーミング出力を有効化。処理中も途中イベントが出力されるため、タイムアウト判定が容易になる。

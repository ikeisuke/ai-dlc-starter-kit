# ドメインモデル: claude-reviewスキルの不安定動作調査・対策

## 概要

claude-reviewスキル（`claude -p`）の不安定動作（レスポンス未返却、指摘の二転三転）の原因を分類・文書化し、対策方針を定義する。

**重要**: このドメインモデル設計では**コードは書かず**、問題の構造と対策方針の定義のみを行います。

## エンティティ（Entity）

### 症状: レスポンス未返却

- **ID**: SYMPTOM-001
- **属性**:
  - 発生条件: String - 長文ファイル（37KB以上）のレビュー実行時
  - 現象: String - `claude -p` 実行後、出力が一切表示されず無反応に見える
  - 影響: String - AIレビューフローが中断し、手動停止が必要
- **再現手順**:
  - 実行コマンド: `claude -p "prompts/package/prompts/construction.md をレビューしてください。問題があれば指摘してください。"`
  - 期待する動作: レビュー結果がstdoutに出力される
  - 実際の動作: 3分以上経過しても出力なし、強制停止が必要

### 症状: 指摘の二転三転

- **ID**: SYMPTOM-002
- **属性**:
  - 発生条件: String - 反復レビュー時（修正→再レビューの繰り返し）
  - 現象: String - 修正箇所と無関係な部分の細かい言い回し指摘が毎回変わる
  - 影響: String - レビュー反復が収束せず、完了判定が困難
- **再現手順**（ユーザー報告ベース）:
  - 実行コマンド: 同一のMarkdownファイルに対し `claude -p` でレビューを実行し、指摘を修正後、再度 `claude -p`（または `--session-id` 付き）で再レビュー。これを2〜3回繰り返す
  - 対象ファイル種別: AI-DLCプロンプトファイル（Markdown、数KB〜数十KB）
  - 期待する動作: 修正済み箇所の指摘が消え、残りの指摘が一貫する
  - 実際の動作: 修正と無関係な箇所で新しい言い回し指摘が出現（特に「表現の改善」「文言の統一」等の低重要度指摘）
  - **注意**: 本症状はLLMの非決定性に起因するため、確率的に発生する。特定の入力で100%再現できるものではない

- **不変性**: 調査完了後に確定、以降変更しない
- **等価性**: category + 対象症状で判定

  - exit_code: Integer | null - 終了コード

## 原因分析結果

### SYMPTOM-001: レスポンス未返却

| 項目 | 内容 |
|------|------|
| **原因分類** | CLI + スキル設定 |
| **重大度** | 高 |
| **再現性** | 常時再現（長文ファイル時） |

**詳細分析**:

`claude -p` はデフォルトでは全出力をバッファリングし、処理完了後にまとめて表示する。大規模ファイル（37KB等）のレビューでは処理に2分以上かかるが、その間一切の出力がないため「無反応」に見える。

**再現試行データ**:

| 試行 | コマンドオプション | 対象 | 結果 | 所要時間 |
|------|-------------------|------|------|---------|
| 1 | `claude -p`（オプションなし） | construction.md (37KB) | 無反応→強制停止 | >180秒 |
| 2 | `claude -p --output-format stream-json` | construction.md (37KB) | 正常完了 | 約131秒 |
| 3 | `claude -p --output-format stream-json --include-partial-messages` | SKILL.md (小) | 出力0バイト→強制停止 | >180秒 |

**試行回数に関する注記**: Unit定義の打ち切り基準は「5回で再現不可の場合」に適用される。SYMPTOM-001は試行1で即座に再現でき、試行2で対策（stream-json）の有効性も確認できたため、追加試行は不要と判断した。SYMPTOM-002はユーザー報告に基づく分析であり、LLMの非決定性という既知の特性に起因するため、設計段階では原因の理論的分析を優先した。なお、ステップ5（検証）で取得した観察データにより、非決定性の実在を事後的に確認できた（下記「観察データ」参照）。

**試行3の考察**: `--include-partial-messages` オプション追加により出力量が増大し、ファイルリダイレクトとの組み合わせで問題が発生した可能性。もしくはclaude CLIのプロセス起動自体が不安定なケースの再現。

### SYMPTOM-002: 指摘の二転三転

| 項目 | 内容 |
|------|------|
| **原因分類** | モデル側の問題 |
| **重大度** | 中 |
| **再現性** | 確率的（反復レビュー時） |

**詳細分析**:

LLM（大規模言語モデル）は本質的に非決定的な出力を生成する。同一入力に対しても、温度パラメータや内部サンプリングにより毎回異なる応答が生成される。特に「細かい言い回しの改善提案」のような低重要度の指摘は、モデルが毎回異なる観点から生成するため一貫しない。

**観察データ**（ステップ5検証時に取得）:

同一プロンプト（`claude -p --output-format stream-json "テストレビュー"`）を3回連続実行した結果、出力のresultテキスト長が毎回異なった:

| 試行 | resultテキスト長 | 指摘内容の傾向 |
|------|-----------------|---------------|
| 1 | 629文字 | 指摘あり（stream-json最適性、allowed-toolsパターン等） |
| 2 | 63文字 | 指摘なし（問題なしと評価） |
| 3 | 638文字 | 指摘あり（配布先反映漏れ、他スキルとの一貫性等） |

同一入力に対して「指摘あり/なし」が変動し、指摘内容も試行ごとに異なることが観察された。これはLLMの非決定性を直接示すエビデンスである。

**根本原因**: モデルの非決定性（Claude Code CLIやスキル設定では制御不可）

## ユビキタス言語

- **レスポンス未返却**: `claude -p` 実行後にstdoutへの出力が一定時間（2分以上）得られない状態
- **指摘の二転三転**: 反復レビュー時に修正箇所と無関係な指摘が毎回変化する現象
- **stream-json**: `--output-format stream-json` オプション。処理途中のイベントをJSON形式で逐次出力する
- **非決定性**: LLMが同一入力に対して毎回異なる出力を生成する特性

## 不明点と質問（設計中に記録）

[Question] `claude -p` に温度パラメータを指定するオプションはあるか？
[Answer] Claude Code CLIには温度パラメータの直接指定オプションはない（`claude --help` で確認）。モデル側の非決定性はCLIレベルでは制御不可。

[Question] 試行3で出力が0バイトだった原因の詳細は？
[Answer] `--include-partial-messages` はストリーミングイベントの中間状態も出力するオプション。ファイルリダイレクトとの組み合わせ、またはCLI起動時の環境差異（並行実行数制限等）が原因の可能性がある。試行2との差異は`--include-partial-messages`の有無のみ。ただし実行環境（他のclaude プロセスの同時実行等）も影響した可能性がある。

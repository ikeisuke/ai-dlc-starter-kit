# Unit: トークン数ベースサイズチェック

## 概要
`bin/check-size.sh` にトークン数ベースの評価を追加する。ハイブリッド方式で、デフォルトは近似計算、tiktokenが利用可能な場合は正確な計測に切り替える。

## 含まれるユーザーストーリー
- ストーリー 1: トークン数ベースの閾値チェック（ハイブリッド方式）

## 責務
- `check-size.sh` へのトークン数チェック機能追加（近似計算 + tiktoken検出・呼び出し）
- `aidlc.toml` 設定項目（`max_tokens`、デフォルト: 40000）の追加
- CLI オプション（`--tokens-threshold`）の追加
- 異常系処理（設定値不正、tiktoken import失敗時のフォールバック）

## 境界
- tiktokenのインストール・管理は対象外
- トークナイザーのモデル選択UIは対象外（cl100k_base固定）
- 既存の出力形式（WARNING フォーマット）の変更は対象外

## 依存関係

### 依存する Unit
なし

### 外部依存
- tiktoken（オプション、Python経由）

## 非機能要件（NFR）
- **パフォーマンス**: 近似計算はバイト数計算と同等の速度
- **セキュリティ**: 該当なし
- **スケーラビリティ**: 該当なし
- **可用性**: tiktoken未インストール時でも近似計算で動作

## 技術的考慮事項
- 近似計算: 日本語バイト比率を検出し、言語混在に対応した係数でトークン数を概算
- tiktoken: `python3 -c "import tiktoken"` で利用可否を判定
- デフォルト値: max_tokens=40000（現在の max_bytes=150000 から換算。英語テキスト: ~4 bytes/token、日本語: ~3 bytes/token の加重平均 ~3.75 bytes/token で算出）

# Unit 6: バージョン管理 - 実装計画

## 概要
プロンプトとテンプレートにバージョン情報を埋め込み、更新の要否を判断可能にする

## 対象ファイル

### テンプレート側（prompts/）- 23ファイル
- `prompts/setup-prompt.md`
- `prompts/setup-cycle.md`
- `prompts/setup-init.md`
- `prompts/package/prompts/*.md`（3ファイル）
- `prompts/package/templates/*.md`（17ファイル）

### 生成側（docs/aidlc/）- 21ファイル
- `docs/aidlc/prompts/*.md`（4ファイル）
- `docs/aidlc/templates/*.md`（17ファイル）

## 実装内容

### Phase 1: 設計

#### ステップ1: ドメインモデル設計
- バージョン表記の形式定義
- バージョン比較ロジックの仕様

#### ステップ2: 論理設計
- バージョンコメント形式: `<!-- AI-DLC Toolkit vX.Y.Z -->`
- バージョン管理の仕組み（version.txtとの連携）
- セットアップ時のバージョン比較ユーティリティ

### Phase 2: 実装

#### ステップ4: コード生成
1. **テンプレート側（prompts/）へのバージョンコメント追加**
   - 各ファイルの1行目に `<!-- AI-DLC Toolkit vX.Y.Z -->` を追加

2. **生成側（docs/aidlc/）へのバージョンコメント追加**
   - 各ファイルの1行目に `<!-- AI-DLC Toolkit vX.Y.Z -->` を追加

3. **セットアップ時のバージョン比較ロジック追加**
   - `prompts/setup-prompt.md` に比較手順を追加

#### ステップ5: テスト生成
- バージョンコメントが正しく追加されていることを確認

#### ステップ6: 統合とレビュー
- 全ファイルのバージョンコメント確認
- 実装記録の作成

## 成果物
- 各プロンプト/テンプレートファイルへのバージョンコメント追加
- バージョン比較ユーティリティ（セットアップ時に使用）

## 完了基準
- すべてのプロンプト/テンプレートファイルにバージョンコメントが追加されている
- バージョン番号が version.txt と一致している
- セットアップ時にバージョン比較が可能

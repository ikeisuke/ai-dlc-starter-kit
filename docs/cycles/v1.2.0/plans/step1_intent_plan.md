# ステップ1: Intent明確化 計画

## 目的
v1.2.0 サイクルの開発意図（Intent）を明確化し、intent.md を作成する

## 入力
- バックログ（`docs/cycles/backlog.md`）から抽出した7つの改善項目
- ユーザーとの対話で確認済みの優先順位

## 作成するIntent

### プロジェクト名
AI-DLC Starter Kit v1.2.0

### 開発の目的
AI-DLCスターターキットの安定性と使いやすさを向上させる

### 対応項目（7項目）

| # | 項目 | 優先度 | 概要 |
|---|------|--------|------|
| 1 | プロンプトの分割・短縮化 | 高 | AIがファイルを最後まで読まない問題への対策 |
| 2 | 変数の具体例を削除（テンプレート側） | 高 | `prompts/setup/*.md` のテンプレート修正 |
| 3 | セットアップ処理の分離 | 中 | 初回セットアップとサイクルセットアップの分離 |
| 4 | プロンプト生成方式の改善 | 中 | テンプレートを個別ファイル化 |
| 5 | マージ後のタグ付け自動化 | 中 | GitHub Actions導入 |
| 6 | プロンプト/テンプレートのバージョン管理 | 中 | バージョン埋め込み |
| 7 | construction.md のパス参照不整合 | 低 | パス参照の修正 |

## 成果物
- `docs/cycles/v1.2.0/requirements/intent.md`

## 備考
- このプロジェクトは「AI-DLCスターターキットを使って、AI-DLCスターターキット自体を開発している」というメタ構造を持つ
- 変更は「ツール側（`prompts/`）」と「成果物側（`docs/aidlc/`, `docs/cycles/`）」の両方に影響する可能性がある

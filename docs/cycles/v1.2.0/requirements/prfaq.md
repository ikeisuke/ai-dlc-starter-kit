# PRFAQ: AI-DLC Starter Kit v1.2.0

## Press Release（プレスリリース）

**見出し**: AI-DLC Starter Kit v1.2.0 - プロンプト最適化とCI/CD自動化で開発効率を向上

**副見出し**: AIがプロンプトを確実に読み、リリース作業を自動化する改善を実施

**発表日**: 2025年12月

---

**本文**:

AI-DLC Starter Kit v1.2.0 をリリースしました。本バージョンでは、開発者からのフィードバックに基づき、7つの改善を実施しています。

**背景**

AI-DLCを実際に運用する中で、「AIが長いプロンプトを最後まで読まない」「セットアップ時に変数が誤った値に置換される」「タグ付けを忘れやすい」といった課題が報告されていました。v1.2.0 ではこれらの課題を解決し、より安定した開発体験を提供します。

**主な改善点**

1. **プロンプトの分割・短縮化**: 大きなプロンプトを小さいファイルに分割し、AIが確実に内容を読むようにしました。これにより、Claude Code以外のAIツールでも安定動作します。

2. **変数ハードコード問題の解消**: テンプレート内の具体例を削除し、AIが誤ったデフォルト値を使用する問題を解決しました。

3. **タグ付け自動化**: mainブランチへのマージ時に自動でGitタグを作成するGitHub Actionsを導入しました。

4. **セットアップ処理の分離**: 初回セットアップとサイクルセットアップを分離し、2回目以降のセットアップ時間を短縮しました。

**顧客の声**

「v1.1.0 では長いプロンプトの一部しか読まれないことがありましたが、v1.2.0 では改善されました」- 早期導入者

**今後の展開**

引き続き、AI-DLC手法の安定性と使いやすさを向上させていきます。Issue駆動統合などの機能は、運用が安定した時点で検討予定です。

## FAQ（よくある質問）

### Q1: v1.1.0 から v1.2.0 へのアップデート方法は？

A: セットアッププロンプトを再実行してください。version.txt の比較により、更新が必要なファイルが自動で検出されます。

### Q2: プロンプト分割により、使い方は変わりますか？

A: 基本的な使い方は変わりません。引き続き `docs/aidlc/prompts/inception.md` などを読み込んで開始してください。分割されたファイルは内部的に参照されます。

### Q3: タグ付け自動化を使わない場合は？

A: GitHub Actionsワークフローを削除するか、無効化してください。手動でのタグ付けも引き続き可能です。

### Q4: セットアップ分離のメリットは？

A: 新しいサイクルを開始する際に、共通ファイル（プロンプト・テンプレート）の再生成が不要になります。サイクル固有のディレクトリ作成のみで開始できます。

### Q5: 既存プロジェクトへの影響は？

A: 後方互換性を維持しています。既存の成果物やワークフローはそのまま利用できます。

### Q6: バージョン管理機能とは？

A: プロンプトとテンプレートの1行目にバージョン情報（`<!-- AI-DLC Toolkit vX.Y.Z -->`）が埋め込まれます。これにより、セットアップ時に更新要否を判断できます。

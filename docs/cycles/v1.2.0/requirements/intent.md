# Intent（開発意図）

## プロジェクト名
AI-DLC Starter Kit v1.2.0

## 開発の目的
AI-DLCスターターキットの安定性と使いやすさを向上させる。特に以下の課題を解決する：
- AIがプロンプトを最後まで読まない問題
- セットアップ時の変数ハードコード問題
- セットアップ処理の複雑さ
- リリースフローの手動作業

## ターゲットユーザー
- AI-DLCスターターキットを使用して開発を行う開発者
- AI-DLC手法を導入したいチーム

## ビジネス価値
- プロンプトの分割により、AIの応答品質が向上し、開発効率が上がる
- セットアップの分離により、サイクル開始時の作業が軽減される
- 自動タグ付けにより、リリース作業のミスが減少する
- バージョン管理により、プロンプト更新の判断が容易になる

## 成功基準
- 全7項目の改善が完了すること
- セットアップ処理が「初回セットアップ」と「サイクルセットアップ」に分離されること
- GitHub Actionsによる自動タグ付けが動作すること
- プロンプトにバージョン情報が埋め込まれること

## 期限とマイルストーン
- AI-DLCの短サイクル原則に従い、段階的に実装

## 制約事項
- メタ開発: AI-DLCスターターキット自体をAI-DLCで開発しているため、変更が即時反映されるか次回セットアップ時に反映されるかを常に意識する
- 既存のAI-DLC利用プロジェクトとの後方互換性を考慮する

## 対応項目

| # | 項目 | 優先度 | 概要 |
|---|------|--------|------|
| 1 | プロンプトの分割・短縮化 | 高 | AIがファイルを最後まで読まない問題への対策 |
| 2 | 変数の具体例を削除（テンプレート側） | 高 | `prompts/setup/*.md` のテンプレート修正 |
| 3 | セットアップ処理の分離 | 中 | 初回セットアップとサイクルセットアップの分離 |
| 4 | プロンプト生成方式の改善 | 中 | テンプレートを個別ファイル化 |
| 5 | マージ後のタグ付け自動化 | 中 | GitHub Actions導入 |
| 6 | プロンプト/テンプレートのバージョン管理 | 中 | バージョン埋め込み |
| 7 | construction.md のパス参照不整合 | 低 | パス参照の修正 |

# Unit 6: プロンプト生成方式改善

## 概要
出力用プロンプトを個別ファイルとして管理し、変数置換のみで生成が完了するようにする

## 含まれるユーザーストーリー
- ストーリー 4.2: プロンプト生成方式の改善

## 責務
- テンプレートファイルの分離
- 変数置換処理の簡略化
- 差分確認の容易化

## 境界
- 生成方式の改善のみ
- プロンプト内容の変更は最小限

## 依存関係

### 依存するUnit
- Unit 5: セットアップ分離（セットアップ処理と連携）

### 外部依存
- なし

## 非機能要件（NFR）
- **パフォーマンス**: 生成処理が簡略化
- **セキュリティ**: 該当なし
- **スケーラビリティ**: 該当なし
- **可用性**: 該当なし

## 技術的考慮事項
- 新しいディレクトリ構造案:
  ```
  prompts/
  ├── setup-prompt.md      # メインエントリ
  ├── setup/               # セットアップロジック
  └── templates/           # 出力テンプレート（新規）
      ├── inception.md
      ├── construction.md
      └── operations.md
  ```
- セットアップ処理: テンプレートをコピー → 変数置換のみ
- メンテナンス: テンプレートファイルを直接編集

Medium

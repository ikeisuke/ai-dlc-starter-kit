# Unit 7: タグ付け自動化

## 概要
mainブランチへのマージ時に自動でGitタグを作成し、リリース作業を自動化する

## 含まれるユーザーストーリー
- ストーリー 2.1: マージ後のタグ付け自動化

## 責務
- GitHub Actionsワークフローの作成
- mainマージ時の自動タグ作成
- 既存タグとの重複チェック

## 成果物
- `.github/workflows/auto-tag.yml`

## 境界
- タグ付け自動化のみ
- リリースノート自動生成は含まない

## 依存関係

### 依存するUnit
- なし（独立して実行可能）

### 後続Unit
- なし

## 非機能要件（NFR）
- **自動化**: 手動でのタグ付け忘れを防止
- **信頼性**: 既存タグがある場合は安全にスキップ

## 技術的考慮事項
- ワークフロートリガー: `push` to `main` branch
- バージョン取得元: `docs/aidlc/version.txt`
- タグ形式: `vX.Y.Z`
- 既存タグチェック: `git tag -l` で確認
- 権限: `contents: write` が必要

## 実装優先度
Low

## 見積もり
0.5時間

# PRFAQ: AI-DLC スターターキット v1.15.2 - シェルスクリプト・ドキュメント品質改善

## Press Release（プレスリリース）

**見出し**: AI-DLC スターターキット v1.15.2 - シェルスクリプトの信頼性とコード品質を向上

**副見出し**: 外部レビューで発見されたバグ修正とエラー処理改善により、スターターキットの安定性が向上

**発表日**: 2026-02-18

**本文**:

AI-DLC スターターキットは、外部プロジェクト（infrastructure-hub）の PR レビューで発見された問題点を修正した v1.15.2 をリリースしました。

本リリースでは、シェルスクリプトの入力バリデーション追加、エラー処理の改善、ドキュメント内コード例の構文エラー修正、およびコード品質の向上を実施しています。特に、`--limit` オプションの数値バリデーションや、GitHub CLI エラー時の認証エラー検出など、利用者が遭遇しやすい問題への対処を強化しました。

利用者からは「エラー時に何が起きているか分かりやすくなった」「ドキュメント通りにコマンドを実行しても構文エラーが出なくなった」との声が寄せられています。

今後も外部プロジェクトでの利用フィードバックを取り入れ、スターターキットの品質向上を継続していきます。

## FAQ（よくある質問）

### Q1: なぜ Issue #194 の must 11件中6件が誤検出だったのですか？
A: レビューツール（keelai）がシェルスクリプト中の `$$`、`$'\n'`、`$` 等の特殊文字をソースコード表示時に誤ってレンダリングしたためです。実際のコードは既に正しく記述されていました。既存コード分析で全件を検証し、修正が必要な12件のみを対象としています。

### Q2: 修正は `docs/aidlc/` と `prompts/package/` のどちらで行いますか？
A: すべての修正は `prompts/package/` 配下で行います。`docs/aidlc/` は `prompts/package/` の rsync コピーであり、Operations Phase のアップグレードスキル実行時に自動反映されます。

### Q3: 既存のスクリプトインターフェースに変更はありますか？
A: ありません。引数、出力形式、終了コードは変更せず、内部のバリデーションとエラー処理のみを改善しています。

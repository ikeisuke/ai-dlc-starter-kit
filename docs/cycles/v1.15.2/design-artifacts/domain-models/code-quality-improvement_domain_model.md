# ドメインモデル: コード品質向上

## 概要

既存シェルスクリプト（`aidlc-git-info.sh`、`env-info.sh`）のコード品質リファクタリング。新規エンティティの追加はなく、既存コードの堅牢性・可読性向上のみ。

**重要**: このドメインモデル設計では**コードは書かず**、構造と責務の定義のみを行います。実装はImplementation Phase（コード生成ステップ）で行います。

## 対象コンポーネント

### aidlc-git-info.sh

- **責務**: Git/jj のVCS状態（ブランチ名、ワークツリー状態、直近コミット）を取得・出力するユーティリティスクリプト
- **変更点**: スクリプト冒頭の初期化処理にIFS明示設定を追加
- **影響範囲**: 初期化処理のみ。各関数の入出力インターフェースは変更なし

### env-info.sh

- **責務**: 依存ツール（gh, dasel, jj, git）の状態を一覧で出力するユーティリティスクリプト
- **変更点**: TOML設定値取得における `cat | dasel` パターンを stdin リダイレクトに変更
- **影響範囲**: `get_project_name`、`get_backlog_mode`、`get_starter_kit_version` 関数内部のみ。各関数の入出力インターフェースは変更なし

## 不変条件

- 各スクリプトの出力形式（`key:value` 形式）は変更しない
- 終了コードの契約は変更しない
- 正常系・異常系の既存動作を完全に維持する

## ユビキタス言語

- **IFS（Internal Field Separator）**: bash がワード分割に使用する区切り文字。デフォルトはスペース・タブ・改行
- **UUOC（Useless Use of Cat）**: `cat file | command` のように不要な `cat` を使うアンチパターン。`command < file` で代替可能
- **stdin リダイレクト**: `< file` によるファイル入力。パイプと異なりサブプロセスを生成しない

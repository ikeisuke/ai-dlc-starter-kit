# Unit 003 計画: label-cycle-issues.shバグ修正

## 概要

`label-cycle-issues.sh`のawkパターンが`Closes #数字`形式に対応していないバグを修正する。

## 関連Issue

- Closes #148

## バグ原因

スクリプトの`extract_issue_numbers`関数内のawkパターン:
```awk
in_section && /^- #[0-9]+/ {
```

- **対応している形式**: `- #数字` (例: `- #58: 説明`)
- **非対応の形式**: `Closes #数字`, `Fixes #数字`

v1.13.0のUnit定義ファイルでは`Closes #数字`形式が使われているため、Issue番号が抽出されない。

## 変更対象ファイル

1. `prompts/package/bin/label-cycle-issues.sh` - awkパターンの修正

## 実装計画

### Phase 1: 設計（ドメインモデル・論理設計）

このUnitはバグ修正のため、設計フェーズは省略し直接実装に進む。
（Unit定義の「技術的考慮事項」に基づくシンプルなバグ修正）

### Phase 2: 実装

1. **awkパターンの拡張**
   - 対応形式:
     - `Closes #数字` / `Fixes #数字`（行頭から直接）※GitHubの標準キーワード
     - `- Closes #数字` / `- Fixes #数字`（箇条書き付き）
     - `- #数字`（既存形式、回帰対応）
   - 注: `Resolves`は現在のUnit定義で未使用のため対象外（必要時に追加可能）
   - 先頭空白を許容（`^\s*`相当）
   - 大文字小文字を区別しない（BSD awk互換のため`tolower()`を使用）

   **BSD awk互換の実装方針**:
   ```awk
   # tolower()で小文字変換してからマッチング
   in_section {
       lower_line = tolower($0)
       if (match(lower_line, /^[[:space:]]*(- )?(closes|fixes) #[0-9]+/) ||
           match(lower_line, /^[[:space:]]*- #[0-9]+/)) {
           # 数字部分を抽出: gsub で #より前を削除、数字以降を削除
           line = $0
           gsub(/^.*#/, "", line)
           gsub(/[^0-9].*$/, "", line)
           if (line != "") print line
       }
   }
   ```

2. **テスト実行**
   - `v1.13.0`サイクルでスクリプトを実行し、Issue番号が正しく抽出されることを確認
   - ラベル付与の動作確認
   - **回帰テスト**: 既存形式 `- #数字` が引き続き動作することを確認
   - **大文字小文字混在テスト**: `closes`, `CLOSES`, `Closes` 等の動作確認

3. **ドキュメント更新**
   - スクリプトのコメント（対応形式の説明）を更新

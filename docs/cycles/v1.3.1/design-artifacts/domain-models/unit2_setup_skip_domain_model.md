# ドメインモデル: セットアップスキップ

## 概要

Inception Phase開始時のサイクル存在確認フローを拡張し、サイクルが存在しない場合に自動作成を提案する機能のドメインモデル。

**重要**: このUnitはプロンプトファイル（Markdown）の修正のみで、コード実装は不要。以下はフロー設計の概念整理として記述。

## 概念モデル

### サイクル（Cycle）

AI-DLCにおける開発の単位。バージョン番号で識別される。

- **識別子**: バージョン番号（例: v1.3.1）
- **状態**:
  - 存在しない（未作成）
  - 存在する（作成済み）
- **構成要素**:
  - ディレクトリ構造
  - history.md
  - backlog.md
  - 各フェーズのサブディレクトリ

### バージョン情報（Version）

AI-DLCツールキットのバージョン情報。

- **パッケージバージョン**: `prompts/package/version.txt`
- **インストール済みバージョン**: `docs/aidlc/version.txt`
- **比較結果**:
  - 一致: アップグレード不要
  - 不一致: アップグレード推奨
  - 不明: いずれかのファイルが存在しない

## フロー概念

### サイクル存在確認フロー

```
[Inception Phase開始]
        |
        v
  [サイクル存在確認]
        |
    +---+---+
    |       |
  存在    存在しない
    |       |
    v       v
 継続   [バージョン確認]
            |
        +---+---+
        |       |
     不一致   一致/不明
        |       |
        v       v
  [アップグレード推奨]  [サイクル作成提案]
        |               |
     承認/拒否       承認/拒否
        |               |
        v               v
  アップグレード    [サイクル作成]
  または続行            |
                       v
                   [Inception継続]
```

### サイクル作成フロー

setup-cycle.md のステップ4〜6を移植:

1. **ディレクトリ構造作成**: 必要なサブディレクトリを作成
2. **history.md初期化**: サイクル履歴の初期ファイルを作成
3. **backlog.md作成**: サイクル固有バックログの初期ファイルを作成
4. **Gitコミット（任意）**: 作成したファイルをコミット

## ユビキタス言語

- **サイクル**: AI-DLCにおける開発の単位。バージョン番号で管理
- **セットアップ**: AI-DLCツールキットのインストールまたはアップグレード
- **アップグレード**: ツールキットの新バージョンへの更新
- **サイクル作成**: 新しいサイクルのディレクトリ構造と初期ファイルの生成

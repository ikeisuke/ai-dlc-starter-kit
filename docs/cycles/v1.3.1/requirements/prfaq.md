# PRFAQ: AI-DLC v1.3.1 - Inception Phase効率化

## Press Release（プレスリリース）

**見出し**: AI-DLC v1.3.1 - Inception Phaseの効率化アップデート

**副見出し**: バックログ対応済みチェック、セットアップスキップ、Dependabot PR確認でサイクル開始をスムーズに

**発表日**: 2025-12

**本文**:

AI-DLCスターターキットは、開発サイクルの開始をより効率的にするv1.3.1をリリースしました。

**背景**

AI-DLCを使った開発サイクルでは、Inception Phase開始時に複数の確認作業が必要でした。バックログの重複確認、毎回のセットアップ実行、セキュリティ更新の見落としなど、改善の余地がありました。

**プロダクト**

v1.3.1では3つの改善を導入しました：

1. **バックログ対応済みチェック**: バックログ確認時に、過去のサイクルで対応済みの項目を自動でチェック。重複した作業を防止します。

2. **セットアップスキップ**: アップグレードが不要な場合、セットアップを経由せずにInception Phaseを直接開始可能に。サイクルディレクトリは自動作成されます。

3. **Dependabot PR確認**: Inception Phase開始時にDependabot PRの有無を確認。セキュリティ更新の見落としを防止します。

**顧客の声**

「バックログを何度も確認する手間が省けた」
「セットアップの手順が減って開発開始が早くなった」
「Dependabotの確認を忘れなくなった」

**今後の展開**

引き続きAI-DLCの使いやすさを向上させる改善を継続していきます。

## FAQ（よくある質問）

### Q1: バックログ対応済みチェックはどのように動作しますか？
A: バックログ確認時に `docs/cycles/backlog-completed.md` の内容と照合し、対応済みの項目があればその旨を通知します。自動で項目を移動することはありません。

### Q2: セットアップをスキップしても問題ありませんか？
A: アップグレードが不要な場合は問題ありません。Inception Phase開始時に新しいバージョンが利用可能かどうかを通知しますので、必要に応じてアップグレードを実行してください。

### Q3: Dependabot PRがない場合はどうなりますか？
A: PRがない場合は何も表示されず、通常通りInception Phaseが進行します。

### Q4: 既存のセットアップフローは変わりますか？
A: 既存のsetup-cycle.mdのフローは変更ありません。アップグレード後にサイクルを開始する場合は従来通りsetup-cycle.mdを使用できます。

### Q5: GitHub CLIがインストールされていない場合は？
A: Dependabot PR確認はスキップされ、他の機能は通常通り動作します。

# 監視・ロギング戦略

## 概要
v1.3.1 では監視・ロギングツールを導入しない。

## 理由
- ドキュメント・テンプレートプロジェクトである
- サーバーサイドのアプリケーションではない
- ユーザーがクローンして利用する形態のため、運用監視の対象がない

## 現状の確認手段
- GitHub Insights によるリポジトリトラフィック確認（手動）
- GitHub Issues による問題報告の受付

## 将来検討事項
運用引き継ぎ情報（docs/cycles/operations.md）より：
- GitHub Insights による利用状況のトラッキング
- ユーザーからのフィードバック収集の仕組み

これらは次回以降のサイクルで検討。

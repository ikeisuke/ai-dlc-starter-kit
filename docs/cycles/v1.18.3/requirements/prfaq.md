# PRFAQ: AI-DLC Starter Kit v1.18.3

## Press Release（プレスリリース）

**見出し**: AI-DLC Starter Kit v1.18.3 - セミオートモードの信頼性向上とワークフロー効率化

**副見出し**: 既知のバグ修正、CI品質チェック追加、開発ワークフローの自動化を通じて日常的な摩擦を軽減

**発表日**: 2026-03（予定）

**本文**:

AI-DLCスターターキット v1.18.3 は、v1.18.2で発見されたセミオートモードのレビューサマリ未生成バグの修正をはじめ、upgrade-aidlc.shの堅牢化、CI品質チェックの新規追加、worktree運用の自動化を含むパッチリリースです。

セミオートモード（`automation_mode=semi_auto`）でConstruction Phaseを実行した際に、レビューサマリファイルが生成されない問題が修正されました。これにより、セミオートモードでもレビュー結果の追跡と品質管理が確実に行えるようになります。

upgrade-aidlc.shではdaselを必須依存として明確化し、使用されていない`--config`オプションを廃止することで、コードの簡素化と堅牢性の向上を実現しました。

CI品質チェックとして、プロンプト内Bashコードブロックの`$()`およびバッククォートによるコマンド置換を自動検出する仕組みが導入され、ルール違反の回帰を防止します。

サイクルバージョン決定時にバックログや過去サイクルの状況が表示されるようになり、より適切なバージョン決定が可能になりました。また、SemVer以外のサイクル命名にも対応し、プロジェクトに合った柔軟な命名が行えます。

worktree環境での開発では、PRマージ後のmain同期とブランチ削除が自動化され、手動作業が大幅に削減されました。

## FAQ（よくある質問）

### Q1: セミオートモードでのレビューサマリ未生成はどのような影響がありましたか？
A: v1.18.2のOperations Phaseで5 Unit全てのレビューサマリが未生成であることが発見されました。レビュー結果の追跡ができないため品質管理に支障がありましたが、レビュー自体は正常に実行されていたため品質への直接的な影響はありませんでした。

### Q2: daselを必須にした場合、daselがない環境ではどうなりますか？
A: upgrade-aidlc.shの実行時に明確なエラーメッセージとインストール手順（`brew install dasel` 等）が表示されます。なお、daselはread-config.sh等の他のスクリプトでも使用されており、AI-DLC全体で実質的に必須の依存です。

### Q3: --configオプションの廃止で既存ユーザーに影響はありますか？
A: ありません。全ユーザーがデフォルトパス（`docs/aidlc.toml`）を使用しており、カスタムパスの利用者は確認されていません。リリースノートに廃止を明記します。

### Q4: SemVer以外のサイクル命名とは具体的にどのようなものですか？
A: `feature-auth`、`2026-03`、`sprint-5` のような自由な命名が可能になります。ディレクトリ構造やブランチ作成は既にSemVer以外も受け付けていたため、suggest-version.shとInceptionプロンプトの対応で完全にサポートされます。

### Q5: post-merge-sync.shは全プロジェクトで使えますか？
A: いいえ。このスクリプトはai-dlc-starter-kitリポジトリ固有のworktree運用に特化しています。共通プロンプト（`prompts/package/`）には含まれません。他のプロジェクトでは、worktree運用に合わせたカスタムスクリプトを作成する必要があります。

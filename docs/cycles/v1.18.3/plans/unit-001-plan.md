# Unit 001 計画: レビューサマリ生成修正（セミオートモード）

## 概要

review-flow.md内のレビューサマリ更新指示が、反復レビュー完了ブロック内に埋め込まれているため、AIエージェントが見落とす問題を修正する。レビューサマリ更新を独立ステップとして配置し、セミオートゲート判定の直前で確実に実行されるようにする。

## 変更対象ファイル

- `prompts/package/prompts/common/review-flow.md`

## 根本原因分析

### ステップ5（AIレビューフロー）

- L209 point 5: 「全種別で指摘がゼロになった時点で」のブロック内に以下が含まれる:
  - メッセージ出力 + 履歴記録 + **レビューサマリ更新**（L251）
- L209の説明文に「メッセージを出力し、履歴に記録してからレビュー後コミット→ユーザーへの承認提示ステップへ進む」と書かれており、2アクション（メッセージ+履歴）の後に進む指示になっている
- レビューサマリ更新は3つ目のアクションとして後に記載されているため、見落とされやすい

### ステップ5.5（セルフレビューフロー）

- L739 point 7: 同様の構造で、レビューサマリ更新（L776）が埋め込まれている

## 実装計画

### 変更1: ステップ5のレビューサマリ更新を独立ステップ化

1. L251のレビューサマリ更新を指摘0件ブロック内から**削除**
2. L665（レビュー後コミット）とL667（セミオートゲート判定）の間に、新しい独立ステップとして**レビューサマリ更新**を追加

### 変更2: ステップ5.5のレビューサマリ更新を独立ステップ化

1. L776のレビューサマリ更新をセルフレビュー完了ブロック内から**削除**
2. L821（レビュー後コミット）とL823（セミオートゲート判定）の間に、新しい独立ステップとして**レビューサマリ更新**を追加

### 変更しない箇所

- L374: 千日手検出時のレビューサマリ更新（千日手パスはユーザー承認へ進むため、セミオートゲートを通らない。ここに残すのが正しい）
- L662: 指摘対応判断フローの「全て先送り」時のレビューサマリ更新（ユーザーレビューフローへ進むため、セミオートゲートを通らない。ここに残すのが正しい）

- [ ] ステップ5（AIレビューフロー）とステップ5.5（セルフレビューフロー）の両方で修正
- [ ] 指摘対応判断フロー経由のパスでもサマリが生成されることを保証
- [ ] `automation_mode=manual` の既存フローに影響がないこと

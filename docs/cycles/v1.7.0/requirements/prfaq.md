# PRFAQ: AI-DLC Starter Kit v1.7.0

## Press Release（プレスリリース）

**見出し**: AI-DLC Starter Kit v1.7.0 - AIエージェント連携強化とバックログ管理の柔軟性向上

**副見出し**: 複数AIエージェントの許可リスト対応、GitHub Issue連携、jj実験的サポートを追加

**発表日**: 2026年1月（予定）

**本文**:

AI-DLC（AI-Driven Development Lifecycle）スターターキットの新バージョン v1.7.0 をリリースしました。

このバージョンでは、開発者がAI駆動開発をよりスムーズに行えるよう、主に3つの領域で機能強化を行いました。

**1. AIエージェント連携の強化**
Claude Code、Codex、Kiro-CLIなど、様々なAIエージェントで使用される安全なコマンドの許可リストガイドを提供します。これにより、開発者は確認ダイアログの頻度を減らし、よりスムーズな開発フローを実現できます。

**2. バックログ管理の柔軟性向上**
GitHub Issuesを使ったバックログ管理が可能になりました。従来のgit管理（ローカルファイル）との選択制を採用し、チームの運用スタイルに合わせた管理方法を選べます。どちらのモードでも、参照時は両方のソースを確認するため、情報の見落としを防ぎます。

**3. 次世代バージョン管理への対応**
jj（Jujutsu）の実験的サポートを追加しました。Gitとの共存を前提とした設計で、先進的な開発者が新しいバージョン管理ワークフローを試すことができます。

**4. メンテナンス性の向上**
アップグレード時に使用するセットアッププロンプトのパスを自動記録する機能を追加。また、不要なバックアップ処理を削除してサイクルセットアップをシンプルにしました。

すべての新機能はオプトイン方式で、既存のワークフローに影響を与えません。

「AI-DLCを使い始めてから、AIエージェントの確認ダイアログが気になっていました。v1.7.0の許可リストガイドで、安全なコマンドを一括設定できるようになり、開発のテンポが良くなりました」— 先行利用開発者

今後もAI-DLCは、AI駆動開発の体験向上に向けて継続的に機能拡張を行っていきます。

## FAQ（よくある質問）

### Q1: AIエージェントの許可リストは、どのエージェントに対応していますか？
A: v1.7.0では、Claude Code、Codex、Kiro-CLIの設定方法を記載しています。各エージェントの設定ファイルパスと設定方法の概要を提供しますが、詳細な設定手順や設定ファイルテンプレートは対象外です。

### Q2: バックログ管理で「issue」モードと「git」モードの違いは何ですか？
A: バックログの**保存先**の違いです。`issue`モードはGitHub Issueに保存し、`git`モードは従来通りローカルファイル（`docs/cycles/backlog/`）に保存します。どちらのモードでも、参照時は両方（Issue + ローカルファイル）を確認します。

### Q3: GitHub Issueモードを使うには何が必要ですか？
A: GitHub CLIが認証済みで、リポジトリへの書き込み権限が必要です。未認証の場合は自動的に`git`モードにフォールバックするため、エラーにはなりません。

### Q4: 参照時に両方確認して、不整合があった場合はどうなりますか？
A: 両方のソースを並列で表示し、ユーザーが判断します。自動マージや優先順位の設定は行いません。保存先を一方に統一することで不整合を防ぐことを推奨します。

### Q5: jjサポートは本番環境で使えますか？
A: v1.7.0では実験的機能として位置づけています。Gitとの共存（colocate）を前提とした設計で、基本的なワークフローの文書化を行います。プロンプトへのjj対応コマンド追加は将来検討事項です。

### Q6: 既存の `.github/` ディレクトリがある場合、Issueテンプレートのコピーで上書きされますか？
A: いいえ。既存の `.github/` や `.github/ISSUE_TEMPLATE/` がある場合は上書きせず、マージまたはユーザー確認を行います。

### Q7: セットアッププロンプトパスの記録は何に使いますか？
A: アップグレード時に、以前使用したスターターキットセットアッププロンプトのパスを確認できます。パスはghq前提の環境非依存形式（同一リポジトリ内は相対パス、外部リポジトリは `ghq:` 形式）で記録されるため、チームメンバー全員が同じパスでアクセスできます。

### Q8: 旧形式バックログのバックアップ処理が削除されましたが、安全ですか？
A: はい。gitで履歴管理しているため、バックアップファイルは不要です。万が一必要になった場合はgit履歴から復元できます。

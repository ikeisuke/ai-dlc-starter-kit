# Intent（開発意図）

## プロジェクト名
AI-DLC Starter Kit v1.7.0

## 開発の目的
AI-DLCスターターキットの機能拡張と改善を行い、より柔軟で使いやすい開発環境を提供する。具体的には、バージョン管理システムの選択肢拡大、AIエージェント連携の強化、GitHub Issuesとの統合による進捗管理の可視化を実現する。

## ターゲットユーザー
- AI-DLCスターターキットを使用してAI駆動開発を行う開発者
- 複数のAIエージェント（Claude Code、Cursor、Cline等）を活用する開発者
- jj（Jujutsu）などの新しいバージョン管理システムに興味を持つ先進的な開発者

## ビジネス価値と成功基準

| # | ビジネス価値 | 成功基準 | 成果物 |
|---|------------|---------|--------|
| 1 | **開発体験の向上**: AIエージェントの許可リスト設定により、確認ダイアログの削減とスムーズな開発フローを実現 | 各AIエージェント向けの許可リストドキュメントが作成され、セットアップ時に案内される | `prompts/package/guides/ai-agent-allowlist.md` |
| 2 | **進捗管理の可視化**: GitHub Issuesとの統合により、サイクルの進捗をチーム全体で可視化 | Issueテンプレートが整備され、サイクル・Unit単位での進捗追跡が可能になる | `.github/ISSUE_TEMPLATE/` |
| 3 | **技術選択肢の拡大**: jjサポートにより、最新のバージョン管理ワークフローを採用可能に | jjを使用した基本的なワークフローが文書化され、オプトインで利用可能になる | `prompts/package/guides/jj-support.md` |
| 4 | **メンテナンス性向上（パス記録）**: 設定パス記録によるアップグレード体験の改善 | セットアッププロンプトのパスが記録され、アップグレード時に参照できる | `docs/aidlc.toml` の `[setup].prompt_path` |
| 5 | **メンテナンス性向上（簡素化）**: 不要なバックアップ処理の削除 | 旧形式バックログ移行時のバックアップ処理が削除される | `prompts/package/prompts/setup.md` |

## 期限とマイルストーン
AI-DLCでは具体的な日程は設定せず、サイクル単位で管理

## 制約事項
- **メタ開発**: このプロジェクトはAI-DLCスターターキット自体の開発であり、`prompts/package/` を編集し、Operations Phaseで `docs/aidlc/` に反映する
- **後方互換性**: 既存のワークフローに影響を与えないよう、新機能はオプトイン方式で導入
- **jjサポート**: 実験的機能として位置づけ、Gitとの共存を前提とする
- **GitHub Issues連携の前提**: Issue駆動統合はGitHub CLIが認証済みであること、リポジトリへの書き込み権限があることを前提とする。未認証の場合はスキップ可能とする

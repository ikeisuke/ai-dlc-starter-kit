# Unit 001: バックアップ処理削除 - 実装計画

## 概要

旧形式バックログ移行時の不要なバックアップ処理を削除し、サイクルセットアップをシンプルにする。

## 変更対象ファイル

- `prompts/package/prompts/setup.md`（サイクルセットアップ）

## 変更内容

### 1. バックアップ作成ステップの削除

**現在の処理（削除対象）**:
```bash
BACKUP_TIMESTAMP=$(date '+%Y%m%d_%H%M%S')
cp docs/cycles/backlog.md "docs/cycles/backlog.md.bak.${BACKUP_TIMESTAMP}"
```

**変更後**: このステップを完全に削除

### 2. 移行確認メッセージの修正

**現在のメッセージ**:
```text
移行を実行しますか？（Y/n）
→ 移行後、元ファイルは backlog.md.bak として保存されます
```

**変更後**:
```text
移行を実行しますか？（Y/n）
→ 移行後、元ファイル（docs/cycles/backlog.md）は削除されます
```

### 3. 移行成功時の処理変更

**現在の処理**: バックアップファイルへの案内メッセージを表示

**変更後**:
- 元ファイルを直接削除: `rm docs/cycles/backlog.md`
- サマリメッセージからバックアップ関連の言及を削除

### 4. 移行結果サマリの修正

**現在のメッセージ（削除対象）**:
```text
元ファイルは docs/cycles/backlog.md.bak.[タイムスタンプ] として保存されています。
問題がなければ、後で削除してください。
```

**変更後**:
```text
元ファイル docs/cycles/backlog.md を削除しました。
（gitで履歴が残るため、バックアップは不要です）
```

## 実装ステップ

1. Phase 1: 設計（対話形式）
   - ドメインモデル設計（本Unitでは該当なし - プロンプト修正のため省略可）
   - 論理設計（変更箇所の特定と修正内容の確認）
   - 設計レビュー

2. Phase 2: 実装
   - `prompts/package/prompts/setup.md` の修正
   - 動作確認（構文エラーがないこと）

- [ ] 移行確認メッセージが更新されている
- [ ] 移行成功時に元ファイルが削除される処理に変更されている
- [ ] サマリメッセージからバックアップ関連の言及が削除されている

## 技術的考慮事項

- gitで履歴が残るため、バックアップファイルは不要
- 既存の移行ロジック（項目解析・ファイル作成）への影響は最小限

## 見積もり

小（プロンプトファイルの修正のみ）

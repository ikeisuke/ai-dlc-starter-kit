# 論理設計: 意思決定プロセスの記録

## 概要
意思決定記録テンプレートの作成と、Inception Phase プロンプトへの記録フロー統合。

## コンポーネント構成

### 1. テンプレート層

**新規作成**: `prompts/package/templates/decision_record_template.md`

意思決定記録の標準フォーマットを定義するテンプレートファイル。

### 2. プロンプト層

**変更対象**: `prompts/package/prompts/inception.md`

Inception Phase の完了時の必須作業セクションに、意思決定記録の生成ステップを追加。

### 3. テンプレート索引

**変更対象**: `prompts/package/templates/index.md`（存在する場合）

新規テンプレートの索引追加。

## 統合フロー

### 記録フローの挿入箇所

Inception Phase のステップ完了処理に以下を追加:

```text
### 意思決定記録の作成

各ステップで意思決定（AIが複数の選択肢を提示し、ユーザーが選択した場面）があった場合、
`docs/cycles/{{CYCLE}}/inception/decisions.md` に記録する。

**記録対象**:
- 2つ以上の明確な選択肢からユーザーが選択した場面
- 技術選定、設計方針、スコープ決定などの重要な判断

**記録対象外**:
- Yes/No の単純な承認確認
- 手続き的な選択（ブランチ方式、ファイル名等）

**記録手順**:
1. 意思決定が発生した時点で、テンプレート（`docs/aidlc/templates/decision_record_template.md`）に従い記録を作成
2. `docs/cycles/{{CYCLE}}/inception/decisions.md` に追記
3. 記録IDは連番（DR-001, DR-002, ...）
```

### Inception Phase への挿入位置

`inception.md` の完了時の必須作業セクション（コミット前）に追加。具体的には、履歴記録の後、コミットの前に「意思決定記録の確認」ステップを挿入する。

**フロー**:
1. セッション中に発生した意思決定を振り返る
2. 記録対象に該当するものがあれば `decisions.md` を作成・追記
3. 記録対象がなければスキップ（ファイル未作成で問題なし）

## エラーハンドリング

| シナリオ | 対応 |
|---------|------|
| 意思決定が1つもなかった | `decisions.md` を作成しない（正常動作） |
| `inception/` ディレクトリが未作成 | `mkdir -p` で自動作成 |

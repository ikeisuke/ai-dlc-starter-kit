# ドメインモデル: 意思決定プロセスの記録

## 概要
Inception Phase で複数の選択肢からユーザーが意思決定した際に、選択理由・却下理由を構造的に記録する仕組みの設計。

**重要**: このドメインモデル設計では**コードは書かず**、構造と責務の定義のみを行います。

## 値オブジェクト（Value Object）

### DecisionRecord
- **属性**:
  - id: string - 連番の意思決定ID（例: DR-001）
  - step: string - 発生したステップ名（例: "ステップ3: ユーザーストーリー作成"）
  - context: string - 意思決定の背景・状況
  - options: Option[] - 提示された選択肢のリスト
  - decision: string - 最終的に選択された選択肢
  - rationale: string - 選択理由
- **不変性**: 記録後は変更不可（追記のみ）
- **等価性**: id の一致

### Option
- **属性**:
  - name: string - 選択肢の名前
  - description: string - 選択肢の概要
  - pros: string[] - メリット
  - cons: string[] - デメリット
  - selected: boolean - 選択されたかどうか

## ドメインサービス

### DecisionRecorder
- **責務**: 意思決定の検出と記録
- **操作**:
  - recordDecision(step, context, options, decision, rationale) - decisions.md に追記
- **記録対象の判定基準**:
  - AIが2つ以上の明確な選択肢を提示し、ユーザーが選択した場面
  - 技術選定、設計方針、スコープ決定など、後から「なぜこの判断をしたか」が重要になる決定
- **記録対象外**:
  - Yes/No の単純な確認（例: 「この計画で進めてよいですか？」）
  - 手続き的な選択（例: ブランチ作成方式の選択）

## 保存仕様

### ファイルパス
`docs/cycles/{{CYCLE}}/inception/decisions.md`

### ファイル構造
```markdown
# 意思決定記録 - {{CYCLE}}

## DR-001: [決定事項のタイトル]

- **ステップ**: [発生ステップ]
- **日時**: YYYY-MM-DD

### 背景
[意思決定が必要になった背景・状況]

### 選択肢

| # | 選択肢 | メリット | デメリット |
|---|--------|---------|-----------|
| 1 | [選択肢1] | [メリット] | [デメリット] |
| 2 | [選択肢2] | [メリット] | [デメリット] |

### 決定
**[選択された選択肢]** を採用

### 選択理由
[なぜこの選択肢を選んだか]

---

## DR-002: ...
```

## 記録タイミング

Inception Phase の各ステップで意思決定が発生した時点で即座に記録する。

| ステップ | 想定される意思決定 |
|---------|------------------|
| ステップ1: Intent明確化 | スコープの範囲決定 |
| ステップ3: ユーザーストーリー作成 | ストーリーの優先度・分割方針 |
| ステップ4: Unit定義 | Unit の分割方針、依存関係の設計 |

## ユビキタス言語

- **意思決定記録（Decision Record）**: 複数の選択肢から特定の選択肢を選んだ際の記録
- **選択理由（Rationale）**: なぜその選択肢を選んだかの根拠
- **却下理由**: 選ばれなかった選択肢のデメリットや不適切さの説明

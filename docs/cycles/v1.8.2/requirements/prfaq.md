# PRFAQ: AI-DLC v1.8.2

## Press Release（プレスリリース）

**見出し**: AI-DLCスターターキット、AIスキル統合とjjサポート強化で開発体験を向上

**副見出し**: プロジェクトをcloneしてセットアップするだけでAIスキルが即座に使える環境を実現

**発表日**: 2026-01（予定）

**本文**:

### 背景

AI-DLCスターターキットは、AIを開発プロセスの中心に据えた開発方法論の実践キットとして、多くの開発者に利用されています。しかし、AIスキル（codex/claude/gemini）のセットアップには個人環境への手動インストールが必要であり、プロジェクト間での再利用や新メンバーのオンボーディングに課題がありました。また、jjワークフローの採用が進む中、Gitとの微妙な違いによるミスが報告されていました。

### プロダクト

v1.8.2では、以下の改善を行いました：

1. **AIスキル統合**: スキルファイルをプロジェクト内に配置し、セットアップ時にシンボリックリンクで自動的に利用可能に。プロジェクトをcloneしてセットアップを実行するだけで、codex/claude/geminiスキルが即座に使えます。

2. **AIレビュー設定強化**: `aidlc.toml` でMCPツールの優先順位を設定可能に。環境に応じて最適なAIレビューツールを選択できます。

3. **jjサポート強化**: よくあるミスと対処法を追加し、Gitとの違いによるトラブルを素早く解決できるようになりました。

## FAQ（よくある質問）

### Q1: 既存のプロジェクトにAIスキル統合を適用するには？

A: `prompts/setup-prompt.md` を読み込んでアップグレードを実行してください。セットアップスクリプトが自動的にスキルファイルのコピーとシンボリックリンクの作成を行います。

### Q2: 個人環境の `~/.claude/skills/` にある既存のスキルと競合しますか？

A: Claude Codeではプロジェクトローカルのスキルが優先されます。ただし、同名のスキルがある場合は、Personal（個人環境）がProject（プロジェクト）より優先されるため、異なる動作をする可能性があります。

### Q3: KiroCLIでもスキルを使えますか？

A: はい。KiroCLIの `resources` フィールドでスキルファイルを指定することで利用可能です。詳細はAGENTS.mdのKiroCLI対応セクションを参照してください。

### Q4: `mcp_tools` 設定が空の場合はどうなりますか？

A: 既存の動作（Skills優先 → MCP `mcp__codex__codex` フォールバック）が維持されます。設定を追加しなくても現在の動作に影響はありません。

### Q5: jjを使っていない場合、jjサポート強化は関係ありますか？

A: 関係ありません。jjサポートは `aidlc.toml` の `[rules.jj].enabled = true` を設定している場合のみ有効です。Gitのみを使用している場合は影響ありません。

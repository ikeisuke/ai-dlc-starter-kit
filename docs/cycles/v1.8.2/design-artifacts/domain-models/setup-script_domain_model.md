# ドメインモデル: セットアップスクリプト修正

## 概要

スキルファイルの配布とシンボリックリンク作成を自動化するセットアップスクリプトの拡張。AI-DLCセットアップ時に、プロジェクト固有のAIスキル（codex/claude/gemini）を利用可能にする。

**重要**: このドメインモデル設計では**コードは書かず**、構造と責務の定義のみを行います。

## 概念モデル

このUnitはセットアップスクリプト（Markdown）の修正のため、従来のDDDパターン（エンティティ等）ではなく、概念モデルとして定義する。

### スキルファイル（SkillFile）

AIエージェントが利用するスキル定義ファイル。

- **構成**: `{skill-name}/SKILL.md` 形式
- **配置場所**:
  - ソース: `prompts/package/skills/`
  - 同期先: `docs/aidlc/skills/`
- **属性**:
  - スキル名（codex, claude, gemini）
  - YAML フロントマター（name, description）
  - Markdownコンテンツ

### シンボリックリンク（SymbolicLink）

Claude Codeがスキルファイルを参照するためのリンク。

- **ソース**: `.claude/skills/`
- **ターゲット**: `docs/aidlc/skills/`
- **特性**:
  - 相対パスを使用（環境依存を排除）
  - Gitで追跡可能

## 処理フロー

### rsync同期フロー

```
1. ドライランで削除対象を確認
2. 削除対象があればユーザーに確認
3. 承認後に実際の同期を実行
```

### シンボリックリンク作成フロー

```
1. .claude/skills/ ディレクトリの存在確認
2. 既存リンクの有無を確認
3. 既存リンクがある場合:
   - 同じターゲットなら: スキップ
   - 異なるターゲットなら: ユーザーに上書き確認
4. リンク作成（相対パス使用）
5. 結果を表示
```

## ユビキタス言語

- **スキルファイル**: AIエージェントの能力を拡張する定義ファイル（SKILL.md形式）
- **rsync同期**: スターターキットからプロジェクトへのファイル同期処理
- **シンボリックリンク**: ファイルシステム上の参照リンク

# 論理設計: スキルファイル配置

## 概要

AIスキル（codex/claude/gemini）のファイル配置構造を定義する。ランタイムコードは存在せず、静的ファイルの配置のみ。

**重要**: この論理設計では**コードは書かず**、コンポーネント構成とインターフェース定義のみを行います。

## アーキテクチャパターン

ファイルシステムベースの静的アセット管理。コードの実行は不要。

## コンポーネント構成

### ディレクトリ構造

```text
prompts/package/skills/
├── codex/
│   └── SKILL.md
├── claude/
│   └── SKILL.md
└── gemini/
    └── SKILL.md
```

### コンポーネント詳細

#### codex/SKILL.md

- **責務**: OpenAI Codex CLIを使用したコードレビュー・分析の実行方法を定義
- **依存**: なし
- **公開インターフェース**: YAMLフロントマター（name, description）、Markdownコンテンツ

#### claude/SKILL.md

- **責務**: Anthropic Claude CLI (Claude Code) を使用したコードレビュー・分析の実行方法を定義
- **依存**: なし
- **公開インターフェース**: YAMLフロントマター（name, description）、Markdownコンテンツ

#### gemini/SKILL.md

- **責務**: Google Gemini CLIを使用したコードレビュー・分析の実行方法を定義
- **依存**: なし
- **公開インターフェース**: YAMLフロントマター（name, description）、Markdownコンテンツ

## ファイル形式

- **形式**: Markdown + YAMLフロントマター
- **主要フィールド**:
  - name: String - スキル識別子（必須）
  - description: String - スキルの説明（必須）
  - 本文: Markdown - 実行コマンド、パラメータ、使用例

## 処理フロー概要

### スキルファイル配置の処理フロー

**ステップ**:

1. `prompts/package/skills/` ディレクトリを作成
2. 各スキル用サブディレクトリ（codex, claude, gemini）を作成
3. `~/.claude/skills/` から各 SKILL.md をコピー

**関与するコンポーネント**: ファイルシステムのみ

## 非機能要件（NFR）への対応

### セキュリティ

- **要件**: スキルファイルに機密情報を含めない
- **対応策**: 認証情報やAPIキーは含めず、コマンド実行例のみを記載

### スケーラビリティ

- **要件**: 将来のスキル追加を考慮したディレクトリ構造
- **対応策**: `{skill-name}/SKILL.md` 形式により、新しいスキルはディレクトリ追加で対応可能

## 技術選定

- **言語**: なし（静的ファイルのみ）
- **フレームワーク**: なし
- **ライブラリ**: なし

## 実装上の注意事項

- YAMLフロントマターの `---` デリミタを正しく維持すること
- Markdownのフォーマットは既存ファイルを尊重すること

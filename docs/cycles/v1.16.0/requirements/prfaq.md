# PRFAQ: AI-DLC Starter Kit v1.16.0

## Press Release（プレスリリース）

**見出し**: AI-DLCフェーズをスキルとして独立化 - リポジトリ単位のアップデート不要に

**副見出し**: AI-DLCの3フェーズ（Inception / Construction / Operations）を Claude Code スキルとして再実装し、グローバルインストールで全リポジトリから即座に利用可能に

**発表日**: 2026-02-19

**本文**:

AI-DLC Starter Kit v1.16.0 では、開発ライフサイクルの3フェーズをClaude Codeのスキルシステムに統合しました。

これまで、AI-DLCを利用するには各リポジトリに `docs/aidlc/prompts/` をデプロイし、バージョンアップのたびに `upgrading-aidlc` を実行する必要がありました。v1.16.0では、フェーズプロンプトを `.claude/skills/` 形式で提供することで、`~/.claude/skills/` に一度配置すれば全リポジトリで最新版のAI-DLCフローを利用できるようになります。

「新しいプロジェクトを始めるたびにセットアップが必要で面倒でした。スキル化により、"start inception" と言うだけでどのリポジトリでもAI-DLCを開始できるようになりました」

また、本バージョンでは git worktree環境でのVCS検出バグ修正、バージョン提案のSemVerバリデーション強化、PRマージ前のリモート同期確認ステップ追加も含まれています。

## FAQ（よくある質問）

### Q1: 既存の `docs/aidlc/prompts/` を使った方法は引き続き使えますか？
A: はい。既存のプロンプト読み込み方式は後方互換として維持されます。スキル化はオプションの新しい利用方法です。

### Q2: `docs/aidlc/bin/` のスクリプトはまだ必要ですか？
A: はい。スキルはフェーズの実行フローを定義しますが、サイクルディレクトリ作成や履歴記録などのユーティリティスクリプトは引き続きリポジトリ内に必要です。初回は `upgrading-aidlc` スキルでデプロイしてください。

### Q3: `~/.claude/skills/` に配置するとリポジトリ側のスキルと競合しませんか？
A: Claude Codeはリポジトリの `.claude/skills/` を優先するため、プロジェクト固有のカスタマイズがある場合はそちらが使用されます。

### Q4: PRマージ前のリモート同期確認はどのように動作しますか？
A: Operations PhaseのPRマージステップ前に自動的に `git fetch` と `git log` で未pushコミットを検出し、ある場合はpushを促してマージをブロックします。

# PRFAQ: AI-DLC スターターキット v1.7.3

## Press Release（プレスリリース）

**見出し**: AI-DLC スターターキット v1.7.3 - ドキュメント品質とDXを向上

**副見出し**: ドキュメント整合性の改善とdasel/jjサポートの強化により、開発者体験を向上

**発表日**: 2026-01

**本文**:

AI-DLCスターターキット v1.7.3 をリリースしました。本バージョンでは、バックログに蓄積されていたドキュメントの整合性問題を解消し、開発者体験（DX）を向上させる改善を実施しています。

主な改善点：
- **ドキュメント整合性**: プロンプトのステップ番号統一、CIとチェックリストの整合性確保、設定ファイルの整理
- **daselサポート強化**: TOML設定読み込みにdaselを活用し、コードの可読性と堅牢性を向上
- **Markdownlint最適化**: 過去サイクルを除外し、効率的なlint実行を実現
- **jjサポート改善**: 作業フローの明確化とガイドの追加

これらの改善により、AI-DLCを使用する開発者はより快適に開発サイクルを進められるようになります。

## FAQ（よくある質問）

### Q1: v1.7.3ではどのような問題が修正されましたか？
A: 主にドキュメントの整合性問題が修正されました。setup.mdのステップ番号の統一、deployment_checklist.mdとCIの対象範囲の一致、aidlc.tomlの冗長なコメント削除などが含まれます。

### Q2: daselは必須ですか？
A: いいえ、daselは必須ではありません。daselがインストールされていない環境でも、従来のgrep/sedを使用したフォールバック処理が動作します。

### Q3: jjサポートの改善で何が変わりましたか？
A: jjの作業フローがより明確になり、作業開始・終了時のガイドが追加されました。また、推奨設定も文書化されています。

### Q4: 破壊的変更はありますか？
A: いいえ、v1.7.3には破壊的変更は含まれていません。既存のワークフローはそのまま動作します。

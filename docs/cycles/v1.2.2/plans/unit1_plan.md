# Unit 1: 気づき記録フロー定義 - 実装計画

## 概要
Unit作業中に別Unitに関する気づきがあった場合の対応フローを定義する

## 対象ファイル
- `docs/aidlc/prompts/construction.md`

## 実装内容

### 追加するセクション
「開発ルール」セクション内に「**気づき記録フロー【重要】**」を追加

### 追加内容
1. **気づきの定義**: Unit作業中に発見した、現在のUnit以外に関連する改善点・問題点・アイデア
2. **記録先**: `docs/cycles/{{CYCLE}}/backlog.md` に追記
3. **記録フォーマット**: 日時、関連Unit/新規、概要、詳細
4. **作業継続ルール**: 現在のUnit作業を中断せず、記録のみ行い作業を継続
5. **読み込みタイミング**: 後続Unitまたは次サイクルのInception Phaseで確認

### 挿入位置
- 「コンテキストリセット対応【重要】」セクションの直前（開発ルールセクション内）

## 完了基準
- construction.mdに気づき記録フローが追加されている
- 記録方法が明確に記載されている
- 作業を中断しないルールが明記されている

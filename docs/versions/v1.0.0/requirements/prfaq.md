# PRFAQ: AI-DLC Starter Kit v1.0.0

## Press Release（プレスリリース）

**見出し**: AI-DLC Starter Kit v1.0.0 リリース - プロンプトの共通化でメンテナンス負荷を大幅削減

**副見出し**: プロンプトとバージョンごとの成果物を分離し、スターターキット改善が即座に全バージョンに反映される新構造を実現

**発表日**: 2025-11-24

**本文**:

### 背景
AI-DLC (AI-Driven Development Lifecycle) を活用した開発が広がる中、AI-DLC Starter Kit は多くの開発者に利用されてきました。しかし、v0.1.0 では、プロンプトファイルがバージョンごとに生成される構造になっており、スターターキットを改善する度にすべてのバージョンのプロンプトを手動で更新する必要がありました。これはメンテナンス負荷が高く、スターターキットの継続的改善を阻む大きな課題となっていました。

### プロダクト
AI-DLC Starter Kit v1.0.0 では、プロンプトファイルとバージョンごとの成果物を完全に分離する新しいディレクトリ構造を導入しました。

**主な変更点**:
- **プロンプトの一元管理**: すべてのバージョンで共通のプロンプト (`docs/aidlc/prompts/`) を使用
- **テンプレートの共通化**: すべてのバージョンで共通のテンプレート (`docs/aidlc/templates/`) を使用
- **成果物の分離**: バージョン固有の成果物 (`docs/versions/{VERSION}/`) は独立して管理
- **シンプルなUX**: 各フェーズで1つのプロンプトファイルを読むだけで開始可能（common.md の内容を各フェーズプロンプトに含めることで実現）
- **バージョン管理**: セットアップ時に使用したスターターキットのバージョンを記録し、将来的なアップデート管理を可能に

**使い方**:
既存のv0.1.0ユーザーは、`prompts/setup-prompt.md` を使用して新しいバージョンをセットアップするだけで、v1.0.0の恩恵を即座に受けられます。v1.0.0以降に作成されたバージョンは、スターターキットの改善が自動的に反映されます。

### 今後の展開
v1.0.0の導入により、スターターキットの継続的改善が可能になりました。今後は、コミュニティからのフィードバックを受けて、プロンプトやテンプレートの改善を継続的に行い、すべてのユーザーに即座に価値を提供していきます。

また、v2.0.0では、アップデート管理機能の強化や、さらなるUX改善を予定しています。

## FAQ（よくある質問）

### Q1: v0.1.0からv1.0.0へのマイグレーションは必要ですか？
A: いいえ、v0.1.0で作成された成果物はそのまま使用できます。v1.0.0の恩恵を受けるには、次の新バージョン（例: v1.1.0）を v1.0.0 の `prompts/setup-prompt.md` を使用してセットアップしてください。既存のバージョンはv0.1.0の構造のまま維持されます。

### Q2: v1.0.0で既存の機能は失われますか？
A: いいえ、すべての既存機能（対話形式、進捗管理、Unit依存関係、JIT生成等）は維持されます。変更されたのはディレクトリ構造のみです。

### Q3: docs/example/ は削除されるのですか？
A: はい、v1.0.0では新しいディレクトリ構造 (`docs/aidlc/` + `docs/versions/`) に統一するため、`docs/example/` は削除されます。ただし、Git履歴には残るため、必要に応じて過去のバージョンから参照可能です。

### Q4: 各フェーズで読むプロンプトファイルは何が変わりますか？
A: v0.1.0では `common.md` と フェーズプロンプト（例: `inception.md`）の2つを読む必要がありましたが、v1.0.0では各フェーズプロンプトの先頭に `common.md` の内容が含まれているため、1つのファイルだけを読めば開始できます。

例:
- **v0.1.0**: `docs/example/v1/prompts/common.md` + `docs/example/v1/prompts/inception.md` を読む
- **v1.0.0**: `docs/aidlc/prompts/inception.md` だけを読む

### Q5: スターターキットのバージョン管理はどうなりますか？
A: v1.0.0では、セットアップ完了時に `docs/aidlc/version.txt` が作成され、使用したスターターキットのバージョンが記録されます。将来的には、このファイルを使用してアップデート通知や自動マイグレーション機能を提供予定です。

### Q6: 既存のv0.1.0プロジェクトはどうなりますか？
A: v0.1.0で作成されたプロジェクトはそのまま動作します。v1.0.0は新しいバージョンをセットアップする際に適用されるため、既存のプロジェクトに影響はありません。

### Q7: プロンプトのカスタマイズはどうなりますか？
A: プロジェクト固有のカスタマイズは、引き続き `additional-rules.md` で行えます。v1.0.0では、共通の `docs/aidlc/prompts/additional-rules.md` とバージョン固有の `docs/versions/{VERSION}/additional-rules.md` の両方をサポートしています。

### Q8: v1.0.0の主な利点は何ですか？
A: 以下の3つが主な利点です：
1. **メンテナンス負荷の削減**: プロンプト更新が1箇所で完結
2. **イニシャルコストの削減**: 新バージョン開始時のセットアップが高速化
3. **一貫性の向上**: すべてのバージョンで同じプロンプトを使用するため、動作が一貫

### Q9: JIT（Just-In-Time）テンプレート生成は維持されますか？
A: はい、JIT生成は維持されます。変更されたのは生成先が `docs/aidlc/templates/` になった点のみです。

### Q10: 後方互換性はありますか？
A: setup-prompt.md では、既存の `DOCS_ROOT` と `VERSION` 変数も使用可能です。新しい変数（`AIDLC_ROOT`、`VERSIONS_ROOT`）は追加されたものであり、既存の変数を置き換えるものではありません。

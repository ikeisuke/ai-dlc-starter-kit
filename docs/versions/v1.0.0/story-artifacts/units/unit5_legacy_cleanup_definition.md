# Unit: 旧構造の削除とバージョン管理

## 概要
docs/example/ を削除し、VERSION ファイルと Git タグを更新する

## 含まれるユーザーストーリー
- ストーリー 10: docs/example/ の削除
- ストーリー 11: VERSION ファイルの更新
- ストーリー 12: Git タグの作成

## 責務
- `docs/example/` ディレクトリの削除
- `VERSION` ファイルを 1.0.0 に更新
- Git タグ `v1.0.0` の作成
- 削除に関する README.md の説明追加

## 境界
- v0.1.0 の他のドキュメント（docs/translations/ 等）は削除しない

## 依存関係

### 依存する Unit
- Unit1: setup-prompt.md のリファクタリング（依存理由: 新しい仕組みが動作することを確認してから削除）
- Unit2: 各フェーズプロンプトのパス参照更新（依存理由: 新しいプロンプトが正常に動作することを確認してから削除）
- Unit3: 新しいディレクトリ構造の作成（依存理由: 新構造が完成してから削除）
- Unit4: README.md の更新（依存理由: README が新構造を説明してから削除）

### 外部依存
- Git

## 非機能要件（NFR）
- **パフォーマンス**: 特に制約なし
- **セキュリティ**: 誤って重要なファイルを削除しないこと
- **スケーラビリティ**: 特に制約なし
- **可用性**: 削除前のバックアップ確認

## 技術的考慮事項
- Gitコミット前に削除を実施（コミット履歴に残る）
- Operations Phase で実施

## 実装優先度
Medium

## 見積もり
1〜2時間

# Unit: setup-prompt.md のリファクタリング

## 概要
setup-prompt.md を新しいディレクトリ構造に対応させ、共通プロンプト・テンプレートと バージョン固有成果物を適切に分離できるようにする

## 含まれるユーザーストーリー
- ストーリー 5: 新しい変数の追加
- ストーリー 6: プロンプトファイル生成先の変更
- ストーリー 7: テンプレートファイル生成先の変更

## 責務
- `AIDLC_ROOT`、`VERSIONS_ROOT` 変数の追加
- プロンプトファイル生成先を `{{AIDLC_ROOT}}/prompts/` に変更
- テンプレートファイル生成先を `{{AIDLC_ROOT}}/templates/` に変更
- 各フェーズプロンプトに common.md の内容を含める処理の実装
- `docs/aidlc/version.txt` の自動生成
- additional-rules.md を共通とバージョン固有の両方に配置

## 境界
- 既存の機能（JIT生成、MODE切り替え等）の変更は含まない
- README.md の更新は含まない
- 各フェーズプロンプトの内容変更は含まない（パス参照の置換のみ）

## 依存関係

### 依存する Unit
- なし

### 外部依存
- Bash
- Git

## 非機能要件（NFR）
- **パフォーマンス**: セットアップ完了時間は v0.1.0 と同等
- **セキュリティ**: ファイル操作の安全性確保
- **スケーラビリティ**: 新しいバージョン追加時の作業量削減
- **可用性**: エラーハンドリングの実装

## 技術的考慮事項
- Bashスクリプトによるディレクトリ作成とファイル生成
- heredoc による複数行テキストの生成
- 変数置換によるパスのカスタマイズ
- 後方互換性の維持（既存の `DOCS_ROOT`、`VERSION` 変数も使用可能に）

# Gemini

Gemini CLIを使用してコードレビュー・分析を実行するスキル。

## 実行コマンド

```bash
gemini -p "<request>" --sandbox
```

## パラメータ

| パラメータ | 説明 |
|-----------|------|
| `-p "<request>"` | 依頼内容（日本語可） |
| `--sandbox` | サンドボックスモード（安全な分析用） |
| `--yolo` | 自動承認モード（確認なしで実行） |
| `-r, --resume` | セッションの再開（`latest` または インデックス番号） |
| `--list-sessions` | 利用可能なセッション一覧を表示 |

## 使用例

### コードレビュー

```bash
gemini -p "このプロジェクトのコードをレビューして、改善点を指摘してください" --sandbox
```

### バグ調査

```bash
gemini -p "認証処理でエラーが発生する原因を調査してください" --sandbox
```

### 自動実行モード

```bash
gemini -p "テストを実行して結果を報告してください" --yolo
```

### セッション引き継ぎ（同じコンテキストで話を続ける）

意図しないセッションの混同を防ぐため、一覧から内容と実行時期を確認して番号を指定することを推奨します。

1. セッション一覧を確認

   ```bash
   gemini --list-sessions
   ```

2. 番号を指定して再開（最新が一番下の番号になる傾向があります）

   ```bash
   gemini --resume <session_index> -p "先ほど指摘した修正を行いました。確認してください。"
   ```

## 実行手順

1. ユーザーから依頼内容を受け取る
2. 対象プロジェクトのディレクトリに移動（必要に応じてcdで移動）
3. 上記コマンド形式でGeminiを実行
4. 結果をユーザーに報告

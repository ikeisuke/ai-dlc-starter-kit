# 終了コード規約

AI-DLC シェルスクリプトの終了コード規約を定義する。

## 終了コード定義

| コード | 意味 | 用途 |
|--------|------|------|
| 0 | 成功 | 正常完了 |
| 1 | バリデーションエラー | 引数不正、入力値不正、前提条件不成立 |
| 2 | システムエラー・警告 | 環境エラー、外部コマンド失敗、警告付き完了 |

## 使い分け基準

### exit 0: 成功

- 処理が正常に完了した場合
- ヘルプ表示（`--help`）

### exit 1: バリデーションエラー

引数や入力値が不正で、処理を開始できない場合:

- 必須引数の欠落（`--cycle is required`）
- 引数値の形式不正（`--unit must be a 3-digit number`）
- 排他引数の同時指定（`--message and --message-file are mutually exclusive`）
- 不明なオプション（`unknown option`）
- 入力ファイルの不存在・空ファイル
- 前提条件の不成立（`--from is not an ancestor of --to`）

### exit 2: システムエラー・警告

環境や外部コマンドの問題、または警告付きの完了:

- 外部コマンドの実行失敗（git, curl 等）
- 設定ファイルの読み取りエラー
- 環境未構成（必要なツール未インストール）
- 警告付き完了（処理は完了したが一部で問題が発生）

## 呼び出し元でのハンドリングパターン

```text
終了コード 0 → 正常完了、次の処理に進む
終了コード 1 → ユーザーに入力修正を求める
終了コード 2 → エラー内容を表示し、ユーザーに対応を求める（または警告表示して続行）
```

プロンプト（`rules.md` 等）での記述例:

```text
**終了コード**:
- 0: 値あり
- 1: キー不在
- 2: エラー
```

## ゴールドスタンダード

`bin/migrate-config.sh` を規約準拠の参考実装とする:

- `_has_warnings` フラグで警告状態を追跡
- 処理中に警告が発生したら `_has_warnings=true` を設定
- スクリプト末尾で `_has_warnings=true` なら `exit 2`、そうでなければ `exit 0`
